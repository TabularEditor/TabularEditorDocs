Acción de código `DR005` (Legibilidad) **Añadir prefijo a una columna temporal**

## Descripción

Se recomienda usar un prefijo coherente para las columnas temporales, para distinguirlas más fácilmente de las columnas base o las medidas. El prefijo predeterminado es `@`, pero puedes configurar qué prefijo se debe usar para que se ajuste al estilo que prefieras, en **Herramientas > Preferencias > Editor de DAX > Acciones de código**.

### Ejemplo

Cambiar:

```dax
ADDCOLUMNS(
    'Sales',
    "Sales With Tax", 'Sales'[Sales Amount] * 1.25
)
```

A:

```dax
ADDCOLUMNS(
    'Sales',
    "@Sales With Tax", 'Sales'[Sales Amount] * 1.25
)
```

## ¿Por qué Tabular Editor sugiere esto?

Esta acción de código está diseñada para mejorar la legibilidad de tu código DAX. Al usar una convención de nomenclatura coherente para tus columnas temporales, es más fácil entender el propósito de cada columna y distinguir entre columnas base, medidas y columnas de extensión.

## Observaciones

Esta acción de código tiene una variante **(Todas las apariciones)**, que se mostrará cuando se puedan mejorar varias secciones del código. Esta variante aplicará la acción de código a todas las secciones relevantes del documento de una sola vez.
