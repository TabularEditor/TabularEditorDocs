Acción de código `DR004` (Legibilidad) **Añadir prefijo a la variable**

## Descripción

Se recomienda usar un prefijo coherente para las variables para distinguirlas más fácilmente de las referencias a tablas. El prefijo predeterminado es `_`, pero puedes configurar qué prefijo se usa para que se ajuste a tu estilo preferido, en **Herramientas > Preferencias > Editor de DAX > Acciones de código**.

### Ejemplo

Cambiar:

```dax
VAR sales = SUM('Internet Sales'[Sales Amount])
RETURN sales * 1,25;
```

A:

```dax
VAR _sales = SUM('Internet Sales'[Sales Amount])
RETURN _sales * 1,25;
```

## ¿Por qué Tabular Editor sugiere esto?

Esta acción de código está diseñada para mejorar la legibilidad de tu código DAX. Al usar una convención de nombres coherente para tus variables, es más fácil entender el propósito de cada variable y distinguir entre variables y tablas.

Además, usar un carácter especial (como un guion bajo) como prefijo para las variables puede ayudar a evitar conflictos de nombres con los nombres de las tablas, por si en el futuro se añade al modelo una tabla con el mismo nombre que la variable.

## Observaciones

Esta acción de código tiene una variante **(Todas las apariciones)**, que aparecerá cuando se puedan mejorar varias secciones de código. Esta variante aplicará la acción de código a todas las secciones pertinentes del documento de una sola vez.
