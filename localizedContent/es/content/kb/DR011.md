Acción de código `DR011` (Legibilidad) **Reescribir con ISBLANK**

## Descripción

En lugar de comparar una expresión con el valor devuelto por [`BLANK()`](https://dax.guide/BLANK), utiliza la función [`ISBLANK`](https://dax.guide/ISBLANK).

### Ejemplo

Cambiar:

```dax
IF(
    Document[Type] == BLANK(), 
    [Sales Amount], 
    [Sales Amount] * 1.25
)
```

A:

```dax
IF(
    ISBLANK(Document[Type]), 
    [Sales Amount], 
    [Sales Amount] * 1.25
)
```

## ¿Por qué Tabular Editor sugiere esto?

La función `ISBLANK` es una forma más concisa y fácil de leer para comprobar si una expresión devuelve un valor en blanco. Al usar `ISBLANK`, el código se vuelve más legible y la intención de la expresión queda más clara.

## Notas

Esta acción de código solo se aplica a la [comparación de igualdad estricta (==)](https://dax.guide/op/strictly-equal-to/) con `BLANK()`. La comparación de igualdad estándar `Document[Type] = BLANK()` no produce el mismo resultado que `ISBLANK(Document[Type])` si `[Type]` es una cadena vacía o cero (en cuyo caso `ISBLANK(Document[Type])` devolvería `FALSE`, mientras que `Document[Type] = BLANK()` devolvería `TRUE`).
