## Problemas conocidos

- El Asistente de importación de tablas no ha llegado a esta versión debido a algunos errores de alta prioridad que había que corregir. ¡Estará disponible lo antes posible! Como solución alternativa, crea una nueva tabla y especifica la consulta de la partición (SQL o M). Luego, haz clic con el botón derecho en la tabla y selecciona "Actualizar esquema de tabla..." para importar las columnas.
- Consulta la [hoja de ruta](https://github.com/TabularEditor/TabularEditor3/issues/12) para obtener más información.

## Mejoras en 3.0.5:

- La vista de diagrama ahora se ve mucho mejor (tanto en el tema claro como en el oscuro), y las rutas de las relaciones se actualizan mientras arrastras las tablas por el lienzo.

## Correcciones de errores en 3.0.5:

- El editor de expresiones, si está oculto, ya no aparece automáticamente al seleccionar un elemento del diagrama o un elemento del Explorador TOM. En su lugar, haz doble clic en el elemento en el Explorador TOM o vuelve a mostrarlo desde el menú Ver (consulta el problema [#43](https://github.com/TabularEditor/TabularEditor3/issues/43).
- El analizador de DAX ya no debería Report dependencias circulares cuando no las hay (consulta los problemas [#63](https://github.com/TabularEditor/TabularEditor3/issues/63) y [#64](https://github.com/TabularEditor/TabularEditor3/issues/64))
- Se corrigió un problema con la determinación, por parte del analizador de DAX, de los valores de SourceColumn en CalculatedTableColumns, que podía provocar que se eliminaran columnas (y, por extensión, relaciones) al cargar el modelo.
- El analizador de DAX ahora muestra un error si se usan determinadas palabras clave como variables de DAX (lo cual no está permitido). Consulta el problema [#58](https://github.com/TabularEditor/TabularEditor3/issues/58).
- Al pulsar ':' al iniciar una búsqueda, ahora cambia inmediatamente a "Modelo completo" y al modo de LINQ dinámico.
- Dynamic LINQ ahora también busca particiones y elementos de grupos de cálculo; consulta el problema [#70](https://github.com/TabularEditor/TabularEditor3/issues/70).

## Vienes de Tabular Editor 2.x?

Mira [este vídeo](https://www.youtube.com/watch?v=pt3DdcjfImY) para hacerte una idea de las nuevas funciones de Tabular Editor 3.

**Resumen de las principales características de Tabular Editor 3:**

- IDE totalmente personalizable, con soporte para varios monitores, Hi-DPI y temas
- Nuevo y potente editor de código DAX con autocompletado, comprobación de sintaxis, plegado de código y mucho, mucho más
- \*Modo del área de trabajo, que te permite guardar tus cambios en el disco y sincronizar simultáneamente los metadatos del modelo con Analysis Services
- \*Previsualiza los datos de la tabla con desplazamiento infinito, crea PivotGrids o escribe consultas DAX para explorar el modelo o probar la lógica de cálculo
- \*Actualiza los esquemas de tabla tanto en Data source de proveedor como en origen de datos estructurado (sí, ¡incluso para consultas M!)
- \*Programa las actualizaciones de datos
- Crea diagramas del Data model
- Crea scripts DAX que te permiten editar varias medidas u otros objetos calculados en un único documento
- Graba C# Scripts y guárdalos como macros (antes conocidas como "Custom Actions")
- Integración con Analizador VertiPaq

\*=Solo cuando estés conectado a una instancia de Analysis Services o Power BI
