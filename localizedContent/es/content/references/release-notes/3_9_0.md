### Requisitos previos

Tabular Editor 3.3.0 (y versiones posteriores) usa .NET 6. Es posible que tengas que instalar el [.NET 6,0 Runtime for Desktop](https://dotnet.microsoft.com/en-us/download/dotnet/6.0/runtime) antes de poder iniciar Tabular Editor 3.3.0 (o una versión posterior). A partir de la versión 3.3.1, el instalador de Tabular Editor 3 descargará e instalará estos requisitos previos si no están instalados.

## Novedades en la versión 3.9.0

Consulta nuestro [blog de lanzamiento](https://blog.tabulareditor.com/2023/07/21/tabular-editor-3-july-2023-release/) para ver un resumen de las actualizaciones más importantes de esta versión.

- Los clientes que usan alguna de las nubes de Azure for Government ahora pueden especificar la URL base que Tabular Editor 3 debe usar al conectarse a la API REST de Power BI, por ejemplo, al importar tablas desde un Dataflow de Power BI. Esta configuración se encuentra en **Herramientas > Preferencias > Power BI > Servicio de Power BI**.
- Al seleccionar más de una celda en el resultado de una consulta DAX o en un Pivot Grid, ahora mostramos un resumen de los valores seleccionados en la barra de estado de la aplicación (suma, mínimo, máximo, media, etc.). Mantén pulsadas las teclas Mayús/Ctrl para seleccionar rangos como lo haces habitualmente. Al hacer Mayús+clic en el encabezado de una columna se seleccionarán todas las celdas de esa columna. Consulta [#475](https://github.com/TabularEditor/TabularEditor3/issues/475).

## Mejoras en la versión 3.9.0

- Hemos reorganizado algunas páginas y secciones del cuadro de diálogo de preferencias para facilitar la navegación.
- Ahora usamos la versión más reciente de TOM/AMO ([19.65.7.2](https://www.nuget.org/packages/Microsoft.AnalysisServices.NetCore.retail.amd64)) y [TMDL preview-3](https://www.nuget.org/packages/Microsoft.AnalysisServices.Tabular.Tmdl.NetCore.retail.amd64/19.65.7.2-TmdlPreview).
- También hemos actualizado AdomdClient a la versión [19.65.7.2](https://www.nuget.org/packages/Microsoft.AnalysisServices.AdomdClient.NetCore.retail.amd64/), lo que debería mejorar el rendimiento de nuestro componente Pivot Grid.
- El Analizador VertiPaq se ha actualizado a la versión [1.2.18](https://www.nuget.org/packages/Dax.Metadata).
- Cuando una tabla calculada se deja con una expresión en blanco, ahora mostramos el icono de error inmediatamente en el Explorador TOM.
- Ahora puedes agregar objetos a Pivot Grid o a Diagramas, incluso cuando esos documentos no tienen el foco (siempre que no tengas abiertos más de uno de esos documentos). Además, ahora puedes crear un Pivot Grid haciendo clic con el botón derecho en una medida, lo que te ahorra algunos clics cuando solo necesitas probar el valor de una medida.
- Ahora mostramos un mensaje de error si intentas asignar a una columna calculada una expresión que devuelve un valor de tipo 'Variant' (no se permite).
- Hemos agregado una opción independiente para guardar copias de seguridad comprimidas de model.bim en **Herramientas > Preferencias > Implementación del modelo**. Ahora puedes elegir que las copias de seguridad se guarden en cada implementación, en cada guardado (mientras estés conectado a una instancia de Analysis Services) o en ambos casos. Nota: Esta función no está disponible en la Edición de escritorio de TE3. Esto también corrigió el problema [#796](https://github.com/TabularEditor/TabularEditor3/issues/796).
- Hemos agregado columnas de información adicionales al Analizador VertiPaq para que coincidan con las disponibles en [DAX Studio](https://daxstudio.org).

> [!NOTE]
> TMDL sigue en versión preliminar y, por lo tanto, esta función también debe considerarse en versión preliminar en Tabular Editor 3. Asegúrate de mantener una copia de seguridad de Model.bim / Database.json de los metadatos de tu modelo para evitar perder trabajo.

## Correcciones de errores en 3.9.0

- Se corrigieron falsos errores de "dependencia circular" que se producían tanto en el Editor de DAX como en el Explorador TOM / vista de mensajes. Consulta las incidencias [#920](https://tabulareditor.zendesk.com/agent/tickets/920) y [#931](https://github.com/TabularEditor/TabularEditor3/issues/931).
- Se corrigió un problema que impedía que el cuadro de diálogo Buscar y reemplazar buscara en ventanas de documentos flotantes (desacopladas).
- Varias mejoras de estabilidad (deberían corregir [#937](https://github.com/TabularEditor/TabularEditor3/issues/937), entre otras cosas).
- Se corrigió un cierre inesperado que se producía al intentar dar formato a código DAX que contenía funciones de ventana.
- Se corrigió un error que hacía que Tabular Editor usara el proxy del sistema al seleccionar el tipo de proxy "Ninguno" en **Herramientas > Preferencias > Configuración de proxy**, y viceversa.
- El Analizador semántico ahora funciona correctamente con la función [`DETAILROWS`](https://dax.guide/detailrows); consulta [#938](https://github.com/TabularEditor/TabularEditor3/issues/938).
- Se corrigió un problema por el cual cambiar el nombre de una macro no actualizaba la estructura de carpetas de las macros hasta que se reiniciaba la aplicación.
- Se corrigió un error que hacía que determinadas propiedades de columna y jerarquía fueran de solo lectura, lo que impedía a los usuarios configurar correctamente tablas calculadas, relaciones, etc., al conectarse a Power BI Desktop. Consulta [#929](https://github.com/TabularEditor/TabularEditor3/issues/929) y [#930](https://github.com/TabularEditor/TabularEditor3/issues/930).
- Se corrigió un bloqueo en el Editor de DAX al usar la función [`WINDOW`](https://dax.guide/window) con determinados parámetros.
- Se corrigió un problema en el analizador de M, que no interpretaba correctamente `[]`, lo que impedía que el comparador de esquemas sin conexión dedujera metadatos de un Dataflow a partir de una expresión M como `PowerPlatform.Dataflows([])`. Consulta [#936](https://github.com/TabularEditor/TabularEditor3/issues/936).
- Se corrigió un problema por el cual PivotGrid se bloqueaba si a una medida se le asignaba una cadena de formato mal formada.
- Se corrigió un problema por el cual el Analizador semántico asignaba "Variant" como tipo de datos de una expresión resultante del uso de [`SWITCH`](https://dax.guide/switch) con una o más ramas que devolvían [`BLANK`](https://dax.guide/blank).
- Se corrigieron algunos errores relacionados con los filtros de columna en la Vista previa de tabla. Al obtener la lista de valores únicos de una columna, ahora leemos los primeros 5000 valores (en orden ascendente). Antes, solo se leían 500 valores, sin ningún orden especificado, lo que provocaba un comportamiento inconsistente. Consulta [#924](https://github.com/TabularEditor/TabularEditor3/issues/924).

## Vienes de Tabular Editor 2.x?

Mira [este vídeo](https://www.youtube.com/watch?v=pt3DdcjfImY) para hacerte una idea de las nuevas funciones de Tabular Editor 3. Además, asegúrate de consultar nuestra [guía de incorporación](https://docs.tabulareditor.com/onboarding/index.html).

**Resumen de las principales funciones de Tabular Editor 3:**

- IDE totalmente personalizable, con soporte para varios monitores, HiDPI y temas
- Nuevo y potente editor de código DAX con autocompletado, comprobación de sintaxis, plegado de código y mucho, mucho más
- \*Modo del área de trabajo, que te permite guardar tus cambios en el disco y sincronizar simultáneamente los metadatos del modelo con Analysis Services
- \*Previsualiza los datos de las tablas con desplazamiento infinito; crea PivotGrids o escribe consultas DAX para explorar el modelo o probar la lógica de cálculo
- \*Programa actualizaciones de datos
- Actualiza los esquemas de tablas tanto en los Data source del proveedor como en los orígenes de datos estructurados (sí, ¡incluso para consultas M!)
- Crea diagramas del Data model
- Crea scripts DAX que te permiten editar varias medidas u otros objetos calculados en un único documento
- Graba C# Scripts y guárdalos como macros (antes conocidas como "Custom Actions")
- Integración con el Analizador VertiPaq
- Depurador de DAX

\*=Solo mientras estés conectado a una instancia de Analysis Services o Power BI
