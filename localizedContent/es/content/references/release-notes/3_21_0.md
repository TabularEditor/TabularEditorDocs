## Novedades en 3.21.0

Echa un vistazo a nuestro [blog de la versión](https://tabulareditor.com/blog/tabular-editor-3-april-2025-release) para obtener una breve panorámica de las actualizaciones más importantes de esta versión.

## Mejoras en 3.21.0

- En esta versión, hemos hecho aún más ajustes y mejoras visuales en nuestra **Vista de diagrama**:
  - Los iconos indican visualmente los tipos de datos de las columnas.
  - Flechas dobles cuando una relación utiliza filtrado cruzado bidireccional
  - Un botón de chevrón en la esquina superior derecha de la tabla te permite alternar rápidamente entre distintas formas de mostrar las columnas: todas las columnas, solo las columnas clave o ninguna.
  - Varios ajustes de tamaño de fuente y otros pequeños retoques
- Se actualizó AMO/TOM a la versión más reciente [19.94.1.1](https://www.nuget.org/packages/Microsoft.AnalysisServices/19.94.1.1).
- Hemos añadido una nueva propiedad en el objeto `Model`, `MetadataSource`, que incluye información sobre de dónde se cargaron los metadatos del modelo. Esto es útil para C# Script; por ejemplo, si un script necesita iterar archivos en la misma estructura de carpetas desde la que se cargaron los metadatos del modelo. Consulta la [documentación de la API](https://docs.tabulareditor.com/api/TabularEditor.TOMWrapper.Model.html#TabularEditor_TOMWrapper_Model_MetadataSource) para obtener más información.
- Los usuarios del Optimizador de DAX ahora también pueden ver los resultados de las expresiones de RLS y de elementos de cálculo mediante nuestra **Integración del Optimizador de DAX**.
- Ahora puedes generar scripts de objetos individuales como [TMDL (Tabular Model Definition Language)](https://learn.microsoft.com/en-us/analysis-services/tmdl/tmdl-overview?view=asallproducts-allversions) desde el **Explorador TOM**. Esto resulta útil cuando quieres copiar scripts de TMDL desde Tabular Editor a Power BI Desktop. Haz clic con el botón derecho en un objeto y elige **Exportar script > TMDL**. Tendrás la opción de exportar el script a un archivo o directamente al portapapeles.

## Correcciones de errores en 3.21.0

- La ordenación por "Progress" o "Duration" en la vista **Data Refresh** ahora debería funcionar correctamente. Además, para determinados objetos, "Duration" ya no se restablece al final de la operación de actualización.
- Varias correcciones de errores relacionadas con la vista de eventos de progreso en la vista **Data Refresh**. Por ejemplo, ahora los eventos de progreso deberían llegar incluso si una sesión anterior ha expirado (por ejemplo, cuando Tabular Editor ha estado inactivo durante 60 minutos o más).
- Cuando la implementación de particiones está desactivada, o está activada pero se omiten las particiones de actualización incremental, ahora también dejamos intacta la configuración de la política de actualización en la tabla (es decir, **Source Expression**, **Polling Expression**, etc.)
- El Analizador semántico ya no debería reportar un error cuando las tablas están conectadas solo mediante relaciones inactivas y se activa una relación mediante la función [`USERELATIONSHIP`](https://dax.guide/userelationship/).
- Se corrigió un error por el cual el Analizador semántico determinaba un tipo de datos resultante incorrecto al usar la función [`GENERATESERIES`](https://dax.guide/generateseries/).
- Se corrigió un error por el cual el Analizador semántico reportaba falsos errores de "no encontrado" al usar [`DEFINE`ing](https://dax.guide/DEFINE) y hacer referencia a tablas y columnas dentro de un script/consulta DAX.
- Ya no es posible asignar un nombre vacío a un objeto desde el **Explorador TOM**.
- Ciertas palabras que antes se trataban como palabras clave reservadas ya no producirán un error semántico cuando se usen como referencias de tabla/variable sin comillas.
- Se corrigió un error por el cual el Depurador de DAX no atenuaba correctamente la rama inactiva al usar IF.EAGER o IF con solo 2 parámetros.
- Al abrir un modelo con una conexión a un Workspace (es decir. La característica del modo del área de trabajo de Tabular Editor 3), ahora mostramos correctamente el aviso de MFA de Microsoft Entra, si este fue el método de autenticación utilizado cuando se estableció originalmente la conexión con la base de datos del Workspace.
- Nuestros editores de código ahora admiten por completo caracteres anchos y emojis Unicode (nota: esto no supone una aprobación de los emojis en DAX, pero ahora es posible usarlos sin que el editor se rompa :)).
- Si el archivo .tmuo contiene JSON no válido, Tabular Editor ahora mostrará un mensaje de error y continuará cargando el modelo sin las opciones especificadas en el archivo, en lugar de provocar una excepción no controlada.
- Se corrigió un error por el que las opciones de política de actualización no aparecían en la cuadrícula de **Propiedades** de una tabla en un modelo cargado desde disco, si la tabla no tenía ninguna partición especificada en los metadatos cargados.
- Los atajos `Ctrl+U` (Quitar comentario del código) y `Ctrl+/` (Alternar comentarios) no funcionaban correctamente debido a conflictos con otros atajos de teclado del editor. Hemos eliminado los atajos predeterminados de los comandos en conflicto. Si necesitas asignaciones de teclado para los comandos _Editor.Lowercase_ y _Editor.Uppercase_ (anteriormente `Ctrl+U` y `Ctrl+Shift+U`, respectivamente), puedes configurarlas en **Herramientas > Preferencias > Teclado**.

## Vienes de Tabular Editor 2.x?

Mira [este vídeo](https://youtu.be/O4ATwdzCvWc) para hacer un recorrido rápido por las principales características de Tabular Editor 3. Además, asegúrate de consultar nuestra [guía de incorporación](https://docs.tabulareditor.com/onboarding/index.html).

**Resumen de las principales características de Tabular Editor 3:**

- IDE totalmente personalizable, con compatibilidad con varios monitores, Hi-DPI y temas
- Nuevo y potente editor de código DAX con autocompletado, comprobación de sintaxis, plegado de código y mucho, mucho más
- \*Modo del área de trabajo, que te permite guardar tus cambios en disco y sincronizar los metadatos del modelo con Analysis Services al mismo tiempo
- \*Previsualizar los datos de la tabla con desplazamiento infinito, crear PivotGrids o escribir consultas DAX para explorar el modelo o probar la lógica de cálculo
- \*Programar actualizaciones de datos
- Actualizar esquemas de tablas tanto en Data sources de proveedor como en orígenes de datos estructurados (sí, ¡incluso para consultas M!)
- Crear diagramas del Data model
- Crear scripts DAX que te permitan editar varias medidas u otros objetos calculados en un solo documento
- Grabar C# Scripts y guardarlos como macros (antes conocidas como "Acciones personalizadas")
- Integración con Analizador VertiPaq
- Depurador de DAX
- Integración con Optimizador de DAX
- Acciones de código para refactorizar fácilmente tu código DAX.

\*=Solo cuando estés conectado a una instancia de Analysis Services o Power BI
