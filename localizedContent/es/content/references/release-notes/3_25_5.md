## Mejoras en 3.25.5

- Vista de diagrama mejorada con selección bidireccional de tablas y columnas entre el Explorador TOM y la Vista de diagrama.
- Se añadió compatibilidad con las columnas "Ordenar por" como columnas asociadas implícitas en el Editor de calendario. Cuando una columna de unidad de tiempo principal tiene definida una columna "Ordenar por", la columna "Ordenar por" ahora se trata automáticamente como columna asociada, lo que evita errores de asignación duplicada y alinea el comportamiento con Analysis Services.
- Dependencias de terceros actualizadas:
  - [Microsoft.AnalysisServices (AMO/TOM)](https://www.nuget.org/packages/Microsoft.AnalysisServices/) actualizado a 19.112.0, añadiendo compatibilidad con la propiedad [ExpressionContext](https://learn.microsoft.com/en-us/dotnet/api/microsoft.analysisservices.tabular.calculatedcolumn.expressioncontext?view=analysisservices-dotnet) de la columna calculada (CL 1705).
  - [Microsoft.PowerBI.Api](https://www.nuget.org/packages/Microsoft.PowerBI.Api) actualizado a 5.0.0 y [Microsoft.Fabric.Api](https://www.nuget.org/packages/microsoft.fabric.api) actualizado a 2.1.0
  - [Microsoft.CodeAnalysis (Roslyn)](https://www.nuget.org/packages/microsoft.codeanalysis) actualizado a 5.0.0.
  - ...y actualizaciones de versión menores para otros paquetes.

## Correcciones de errores en 3.25.5

- Se corrigió un error que impedía que los elementos de cálculo se serializaran en archivos individuales mediante la opción de Tabular Editor de guardar en carpeta basada en JSON.
- Se corrigió un error en la acción de código [Replace IFERROR with DIVIDE](https://docs.tabulareditor.com/kb/DI014.html), que a veces se activaba con funciones no relacionadas.
- Se corrigió un problema por el que el nombre del modelo en archivos de Power BI Desktop se truncaba incorrectamente al usar guiones. Consulta [este problema](https://github.com/TabularEditor/TabularEditor3/issues/1526)
- Se corrigió un error menor por el que el tirador de arrastre de la ventana emergente de autocompletado de C# quedaba oculto por los botones de conmutación en ciertos ajustes de escalado de DPI.
- Se corrigió un problema en las particiones SQL heredadas por el que el atajo de comentarios y el botón de la barra de herramientas insertaban comentarios de estilo DAX (//) en lugar de comentarios SQL (--)
- Se corrigió una excepción en el cuadro de diálogo Buscar y reemplazar. La pestaña Reemplazar ahora solo se mostrará cuando haya documentos abiertos en los que se pueda reemplazar texto. Consulta [este problema](https://github.com/TabularEditor/TabularEditor3/issues/1524)
- Se corrigió el aspecto del menú contextual del clic derecho en los editores de código C#, M y SQL.
- Se corrigió una condición de carrera en el Servicio de procesamiento de modelos que podía provocar bloqueos intermitentes durante las operaciones de actualización de datos, sobre todo al actualizar modelos grandes o modelos que generan advertencias.
- Se corrigió un error al cambiar la configuración regional o las traducciones en modelos que contienen funciones definidas por el usuario.
- Se corrigió un problema por el que los objetos ocultos no estaban disponibles en el Editor de calendario. Las columnas y tablas ocultas ahora aparecen correctamente en los menús desplegables de tablas y columnas, y en los paneles Columnas asociadas y Columnas relacionadas con el tiempo.
- Se corrigió una NullReferenceException al actualizar el esquema en orígenes de datos ODBC/OleDb
- Se corrigió un problema por el que "Reemplazar todo" con el ámbito establecido en "Selección" reemplazaba texto incorrectamente en todo el documento en lugar de hacerlo solo dentro del rango de texto seleccionado. Esto afectaba a las Consultas DAX, los scripts DAX y los editores de C# Script.
- Se corrigió un problema por el que el **Explorador TOM** se volvía inestable tras usar la opción "Revertir" del menú Archivo.
- Se corrigió un problema intermitente por el que el **Explorador TOM** no mostraba los indicadores de error/advertencia.
- Se corrigió un problema por el que los nombres de las medidas se mostraban incorrectamente en la vista **Mensajes**.
- Se corrigió un problema por el que el Editor de calendario permanecía deshabilitado después de pegar una tabla de fechas con calendarios en un modelo recién creado. La vista ahora se actualiza correctamente y se habilita cuando se agregan calendarios mediante operaciones de pegado.
- Se corrigió un error en el Editor de calendario por el que, al marcar una columna en el panel Columnas asociadas mientras había un filtro de búsqueda activo, se registraba la columna incorrecta.
- Se corrigió un error en el Editor de calendario por el que, al eliminar un calendario, se borraban las etiquetas de Columnas relacionadas con el tiempo de todos los calendarios restantes hasta que se aceptaban los cambios.
- Se corrigió un problema por el que un calendario creado mediante el Explorador TOM no aparecía en el Editor de calendario hasta que posteriormente se le cambiaba el nombre.
- Se corrigió un bloqueo causado por una `NullReferenceException` no controlada al deshacer la eliminación de un grupo de columnas relacionadas con el tiempo en el Editor de colecciones del Editor de calendario.
- Se corrigió un problema en nuestro Analizador de M, lo que garantiza que ahora se admiten funciones de M como `Table.ColumnNames`, `Table.ExpandTableColumn`, `Table.ReplaceValues` y muchas de las funciones `Text.*` para la detección de esquemas sin conexión (es decir, al realizar **Actualizar esquema de tabla** con **Usar Analysis Services para la detección de cambios** _deshabilitado_).

## Novedades en 3.25.0

Consulta nuestro [blog de la versión](https://tabulareditor.com/blog/tabular-editor-3-january-2026-release) para obtener una visión general rápida de las actualizaciones más importantes de esta versión.

- Tabular Editor ya está disponible en **chino y español (versión preliminar)**. Todas las traducciones se han realizado inicialmente con IA, con revisión humana. El japonés, el alemán y el francés también están disponibles como idiomas experimentales sin revisión.
- Tabular Editor ahora incluye un conjunto de **reglas integradas del Best Practice Analyzer (BPA)** que cubren las mejores prácticas clave en el formato, los metadatos, el diseño del modelo, las expresiones DAX y las traducciones.
- Tabular Editor ahora incluye la funcionalidad **Semantic Bridge**, que permite crear fácilmente modelos semánticos a partir de las [vistas de métricas](https://docs.databricks.com/aws/en/metric-views/) de Databricks.\*
- Se añadió una opción para guardar modelos semánticos con **archivos auxiliares para Fabric**. Al habilitarla, Tabular Editor puede crear y actualizar los archivos .platform y definition.pbism junto con los metadatos del modelo para ajustarse a la estructura del repositorio de Fabric.\*\*
- Esta versión también incorpora un nuevo Editor de calendario (disponible en la Edición Business y la Edición Enterprise), que facilita la administración de los objetos de calendario usados para la inteligencia temporal basada en calendario.
- Por último, hemos añadido un cuadro de diálogo de actualización avanzada, desde el que puedes configurar opciones avanzadas de actualización como el paralelismo y la fecha de vigencia para la actualización incremental. También puedes especificar perfiles de sobrescritura de actualización.\*\*

_Estas características solo están disponibles en Tabular Editor _Edición Enterprise__

\*\*Estas funciones solo están disponibles en Tabular Editor, en las ediciones _Edición Business_ y _Edición Enterprise_

## Mejoras en 3.25.0

- Se ha añadido compatibilidad para establecer la propiedad CustomData de la cadena de conexión directamente desde el cuadro de diálogo de suplantación.
- La vista de DAX Package Manager ahora descarga y muestra los iconos de los paquetes de DaxLib.
- DAX Package Manager ahora almacena en caché los metadatos de los paquetes, lo que permite ver la información de los paquetes sin necesidad de conexión a Internet.
- Hemos actualizado el paquete de la biblioteca de cliente de AS a la versión más reciente [19.108.5](https://www.nuget.org/packages/Microsoft.AnalysisServices/19.108.5). Esta actualización permite controlar la nueva propiedad [UnicodeCharacterBehavior](https://learn.microsoft.com/en-us/dotnet/api/microsoft.analysisservices.tabular.database.unicodecharacterbehavior?view=analysisservices-dotnet#microsoft-analysisservices-tabular-database-unicodecharacterbehavior) en el objeto Database.
- Se han añadido las categorías de datos de columna específicas de Power BI: Longitud, Latitud y Lugar.
- Ahora mostramos un mensaje de error si intentas establecer el nivel de compatibilidad de un modelo en un valor que el servidor no admite. Esto evita errores inesperados más adelante al guardar los cambios en el modelo. Si necesitas establecer un nivel de compatibilidad no admitido con fines de migración, guarda los metadatos del modelo en disco (en formato .bim o TMDL) y, a continuación, abre los metadatos sin ninguna base de datos de Workspace asociada.
- Los archivos de Opciones de usuario (.tmuo) ahora también se guardan cuando abres un modelo directamente desde un servidor o un punto de conexión XMLA. Los archivos se almacenan en la carpeta %LocalAppData%\TabularEditor3\UserOptions.
- Se ha añadido la opción "Actualizar completo (ignorar la política)" al menú contextual de actualización de datos para que puedas actualizar fácilmente los datos en particiones existentes sin desencadenar operaciones de agregar, eliminar o combinar particiones en el servidor cuando una tabla se rige por una política de actualización incremental.
- Se han reorganizado las opciones del menú contextual de actualización de datos para llevar las opciones más utilizadas a la parte superior.
- Se ha añadido la hora de inicio al panel de actualización, lo que permite ordenar las actualizaciones por su hora de inicio; por ejemplo, para ver primero la más reciente.
- Se ha añadido un panel de búsqueda al cuadro de diálogo de Preferencias para buscar rápidamente preferencias individuales.
- Se han añadido "Funciones definidas por el usuario" y "Calendarios" a la lista del editor de reglas de mejores prácticas.
- Se ha añadido la propiedad `UsedInCalendars` a las columnas, lo que facilita comprobar en un C# Script si una columna se usa en algún objeto de calendario.
- La acción del menú "Archivo > Cerrar" ha pasado a llamarse "Cerrar documento" y ahora permite cerrar cualquier panel o documento del área principal de documentos. **Ctrl + W** se ha asignado como el método abreviado predeterminado.
- Se ha añadido compatibilidad del Analizador semántico para la nueva palabra clave `ENDALIGNED` del parámetro <extension> de [`DATEADD`](https://dax.guide/DATEADD). [Más información](https://learn.microsoft.com/en-us/dax/dateadd-function-dax#extension-parameter-for-small--large-period-shifts).
- Al guardar en un archivo model.bim o en una estructura de carpetas TMDL con la opción de serialización **Ignore incremental refresh partitions** habilitada, ahora añadimos una partición ficticia para sustituir las particiones PolicyRange. Esto garantiza que el modelo todavía pueda cargarse en Power BI Desktop, ya que no se guarda ninguna tabla sin particiones.

## Corrección de errores en 3.25.0

- Se corrigió un problema crítico que impedía a algunos clientes conectarse a la API REST de Power BI o a Databricks mediante la autenticación de Azure AD.
- Se corrigió una excepción no controlada que se producía al eliminar una columna que también se usaba en un objeto de calendario.
- Se corrigió la lista desplegable de columnas del editor de relaciones para que muestre automáticamente los nombres completos de las columnas.
- Al duplicar una tabla que contiene uno o varios calendarios, ahora los calendarios se renombran automáticamente para garantizar nombres únicos en todo el modelo.
- Varios errores/advertencias detectados durante una operación de actualización de datos ya no deberían provocar una excepción no controlada.
- Se corrigió un problema por el que la lista de paquetes DAX mostraba información desactualizada después de ejecutar "Actualizar todo".
- Se corrigió un error por el que no se podía deshacer el cambio en la propiedad `SchemaName` de las particiones de entidad.
- La acción de código [DI005 - Reescribir filtro de tabla como predicado escalar](https://docs.tabulareditor.com/kb/DI005.html) ya no se sugiere cuando se produce la [expansión de tablas](https://www.sqlbi.com/articles/expanded-tables-in-dax/), ya que reescribir ese tipo de filtro como un filtro de columna no necesariamente produciría el mismo resultado.
- Al hacer clic en el botón de puntos suspensivos junto a la propiedad Cadena de conexión, ya no debería producirse una excepción no controlada cuando la cadena de conexión especifica un proveedor OLE DB que no está instalado en el sistema actual.
- Las exportaciones TMDL ahora excluyen correctamente las particiones de actualización incremental cuando la preferencia correspondiente está habilitada, en línea con el comportamiento de JSON.
- Se corrigió un problema por el que, al hacer clic en el botón "Iniciar sesión..." al intentar actualizar o importar desde Dataflows, se mostraba el siguiente mensaje de error: `This page is not normally shown and could be a sign of a phishing attempt.`

## ¿Vienes de Tabular Editor 2.x?

Mira [este vídeo](https://youtu.be/O4ATwdzCvWc) para ver un recorrido rápido por las principales características de Tabular Editor 3. Además, asegúrate de consultar nuestra [guía de introducción](https://docs.tabulareditor.com/onboarding/index.html).

**Resumen de las principales características de Tabular Editor 3:**

- IDE totalmente personalizable, con compatibilidad con varios monitores, Hi-DPI y temas
- Nuevo y potente editor de código DAX con autocompletado, comprobación de sintaxis, plegado de código y mucho, mucho más
- \*Modo del área de trabajo, que te permite guardar tus cambios en el disco y sincronizar los metadatos del modelo con Analysis Services al mismo tiempo
- \*Previsualiza los datos de la tabla con desplazamiento infinito, crea PivotGrids o escribe consultas DAX para explorar el modelo o probar la lógica de cálculo
- \*Programa actualizaciones de datos
- Actualiza los esquemas de tablas tanto en el proveedor como en orígenes de datos estructurados (sí, incluso para consultas M!)
- Crea diagramas del Data model
- Crea scripts DAX que te permitan editar varias medidas u otros objetos calculados en un único documento
- Graba C# Scripts y guárdalos como macros (antes conocidas como "Custom Actions")
- Integración del Analizador VertiPaq
- Depurador de DAX
- Integración del Optimizador de DAX
- Acciones de código para refactorizar fácilmente tu código DAX.

\*=Solo mientras esté conectado a una instancia de Analysis Services o Power BI
