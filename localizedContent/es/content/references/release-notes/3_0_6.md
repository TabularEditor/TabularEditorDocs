## Problemas conocidos

- El asistente para importar tablas no llegó a esta versión debido a algunos errores de alta prioridad que había que corregir. ¡Estará disponible lo antes posible! Como solución alternativa, crea una tabla nueva y especifica la consulta de la partición (SQL o M). Luego, haz clic con el botón derecho en la tabla y elige "Actualizar esquema de tabla..." para importar columnas.
- Consulta la [hoja de ruta](https://github.com/TabularEditor/TabularEditor3/issues/12) para obtener más información.

## Mejoras en 3.0.6:

- Compatibilidad con licencias de consultoría. [Más detalles](https://tabulareditor.com/license-terms).
- Las consultas DAX ahora muestran el recuento de filas y los tiempos de ejecución (consulta el problema [#10](https://github.com/TabularEditor/TabularEditor3/issues/10))
- Las consultas DAX ahora están limitadas a 1000 filas de forma predeterminada para evitar tiempos de espera o problemas de memoria. Este límite se puede cambiar en Herramientas > Preferencias > Exploración de datos > Consulta DAX.
- Ahora se pueden ordenar y filtrar las columnas en los resultados de la consulta DAX (solo ordenación y filtrado locales). Consulta el problema [#60](https://github.com/TabularEditor/TabularEditor3/issues/60)
- Los datos de las rejillas (resultados de consultas DAX, Vista previa de tabla, etc.) ahora se pueden copiar al portapapeles
- Al copiar una expresión DAX, el resaltado de sintaxis ahora se conserva en el portapapeles. Consulta el problema [#80](https://github.com/TabularEditor/TabularEditor3/issues/80).
- Al agregar una tabla calculada mediante una expresión de C# Script, el analizador semántico ahora añade inmediatamente las columnas de la tabla calculada en función de la expresión DAX (en lugar de hacerlo tras ejecutar el script).
- Se actualizó la biblioteca cliente de TOM a la versión 19.22.0.1
- Ya están disponibles las versiones portátiles (.zip). Recuerda que tienes que [registrar manualmente Tabular Editor 3 en Power BI Desktop](https://docs.microsoft.com/en-us/power-bi/transform-model/desktop-external-tools#registering-external-tools) si usas la versión portátil.

## Correcciones de errores en 3.0.6:

- Ahora puedes usar los métodos `EvaluateDax`, `ExecuteCommand`, `SelectTable`, `SelectMeasure`, etc. en scripts de C# Script (similar a Tabular Editor 2.x); consulta [#44](https://github.com/TabularEditor/TabularEditor3/issues/44).
- El método `CustomAction` en scripts de C# Script ahora funciona correctamente; consulta [#79](https://github.com/TabularEditor/TabularEditor3/issues/79).
- Mejor restablecimiento de las conexiones con AS (deberías ver menos cuelgues por "conexión perdida")
- Ahora se puede crear una instancia de Best Practice Analyzer mediante un C# Script: `new TabularEditor.Shared.BPA.Analyzer();` - esto es diferente de Tabular Editor 2.x, en el que la clase Analyzer residía en un espacio de nombres distinto. Consulta el problema [#45](https://github.com/TabularEditor/TabularEditor3/issues/45)
- Ahora se admiten operaciones de copia cuando hay carpetas seleccionadas; consulta [#84](https://github.com/TabularEditor/TabularEditor3/issues/84)
- Filtrado mejorado de columnas de fecha en la Vista previa de tabla; consulta [#77](https://github.com/TabularEditor/TabularEditor3/issues/77)
- Se corrigió un error por el que las sobrescrituras de características del motor semántico no surtían efecto; consulta [#62](https://github.com/TabularEditor/TabularEditor3/issues/62).
