## Mejoras en 3.2.1

- El depurador se ha mejorado de varias formas. Por ejemplo, ahora es más fácil administrar expresiones de seguimiento mediante un menú contextual al hacer clic con el botón derecho. También puedes añadir rápidamente una medida, una referencia de variable o una subexpresión a la vista de expresiones de seguimiento haciendo clic con el botón derecho en el editor. Consulta [este artículo](https://docs.tabulareditor.com/te3/features/dax-debugger.html) para obtener más información.
- Hay disponible una nueva opción en las opciones de serialización, que garantiza que las matrices JSON de roles/perspectivas se ordenen por nombre al guardarse en disco; consulta [#341](https://github.com/TabularEditor/TabularEditor3/issues/341). Puedes encontrar la nueva opción en **Herramientas > Preferencias > Formatos de archivo > Ordenar matrices por nombre** (o, para cambiar la configuración de un modelo existente, ve a **Modelo > Opciones de serialización...**).

## Correcciones de errores en 3.2.1

- El depurador de DAX ahora admite consultas DAX que usan la función ROLLUPGROUP.
- Se corrigió un error por el que se enviaba un filtro inverso al Depurador de DAX al iniciar desde una Pivot Grid con un filtro en una columna que contenía dos valores únicos.
- Mejoras generales de estabilidad del Depurador de DAX; consulta [#369]((https://github.com/TabularEditor/TabularEditor3/issues/369).
- Los accesos directos Step Over (F10) y Step In (F11) en el Depurador de DAX ahora se comportan de forma más intuitiva.
- Se corrigió un bloqueo al cargar el modelo cuando el Explorador TOM aún no se había inicializado; consulta la incidencia [#357](https://github.com/TabularEditor/TabularEditor3/issues/357).
- Se corrigió un problema por el que el diseño del Workspace no se restauraba correctamente al reiniciar la aplicación o al cambiar entre diseños.
- Asegura que el contenido de los paneles de ocultación automática se cargue cuando el panel se esté expandiendo
- El analizador semántico ya no debería reportar un error cuando se omite el tercer argumento de las funciones [SEARCH](https://dax.guide/search) o [FIND](https://dax.guide/find).
- El analizador semántico ya no debería reportar errores al usar las funciones DAX [ROLLUPGROUP](https://dax.guide/rollupgroup) y [ROLLUP](https://dax.guide/rollup) (a veces presentes en consultas generadas por PBI Desktop).
- Varias mejoras del formateador DAX sin conexión para que sea más coherente con https://daxformatter.com.
- Se corrigió un problema por el que se agregaban marcos de comentarios a la expresión anterior en un script DAX; consulta [#331](https://github.com/TabularEditor/TabularEditor3/issues/331).
- Se corrigió el cierre inesperado de la aplicación al iniciarse; consulta el problema [#328](https://github.com/TabularEditor/TabularEditor3/issues/328).
- Ahora es posible establecer las propiedades traducidas de varios objetos a la vez; consulta el problema [#359](https://github.com/TabularEditor/TabularEditor3/issues/359).
- Se corrigió el indicador de progreso de actualización que se quedaba "bloqueado"; consulta el problema [#360](https://github.com/TabularEditor/TabularEditor3/issues/360).
- Se corrigió el escalado de fuentes del diagrama; consulta el problema [#351](https://github.com/TabularEditor/TabularEditor3/issues/351).
- Se corrigió el renderizado del árbol de llamadas en high-dpi; consulta el problema [#355](https://github.com/TabularEditor/TabularEditor3/issues/355).
- Se asegura que los scripts DAX se analicen inmediatamente al cargarse desde un archivo; consulta el problema [#352](https://github.com/TabularEditor/TabularEditor3/issues/352).
- Se corrigió un error que hacía que TE3 dejara de sincronizar los metadatos desde AS tras el primer cambio; consulta [#353](https://github.com/TabularEditor/TabularEditor3/issues/353).
- Se corrigió un problema por el que el árbol de dependencias no se reconstruía correctamente en algunos casos; consulta [#364](https://github.com/TabularEditor/TabularEditor3/issues/364).
- Reemplazar una tabla existente ya no debería provocar un cierre inesperado; consulta el problema [#368](https://github.com/TabularEditor/TabularEditor3/issues/368).
- Se corrigió un problema por el que el BPA no se ejecutaba hasta que se mostraba por primera vez la vista de BPA; consulta el problema [#367](https://github.com/TabularEditor/TabularEditor3/issues/367).
- Se usa una sola partición para la detección de cambios de AS cuando se conecta al nivel estándar de AS (que no permite tablas con varias particiones). Consulta [#336](https://github.com/TabularEditor/TabularEditor3/discussions/336).
- Las columnas agregadas mediante la comparación de esquemas heredan la pertenencia a la perspectiva. Consulta el problema [#342](https://github.com/TabularEditor/TabularEditor3/discussions/342).
- Usa las credenciales de red predeterminadas para el proxy web (debería resolver el problema de los errores 407 al conectarse a AS a través de un proxy).
- El analizador semántico ya no debería reportar un error cuando las referencias a variables están entre comillas; consulta el problema [#302](https://github.com/TabularEditor/TabularEditor3/discussions/302).
- Se mejoró la visualización de los mensajes de error de Pivot Grids (ahora aparecen en la **vista de mensajes**).

## ¿Vienes de Tabular Editor 2.x?

Mira [este vídeo](https://www.youtube.com/watch?v=pt3DdcjfImY) para hacerte una idea de las nuevas características de Tabular Editor 3. Además, asegúrate de consultar nuestra [guía de incorporación](https://docs.tabulareditor.com/onboarding/index.html).

**Resumen de las principales características de Tabular Editor 3:**

- IDE totalmente personalizable, con compatibilidad con varios monitores, Hi-DPI y temas
- Nuevo y potente editor de código DAX con autocompletado, comprobación de sintaxis, plegado de código y mucho, mucho más
- \*Modo del área de trabajo, que te permite guardar tus cambios en el disco y sincronizar simultáneamente los metadatos del modelo con Analysis Services
- \*Previsualiza los datos de tablas con desplazamiento infinito, crea PivotGrids o escribe consultas DAX para explorar el modelo o poner a prueba la lógica de cálculo
- \*Programa actualizaciones de datos
- Actualiza los esquemas de las tablas tanto en el Data source del proveedor como en los orígenes de datos estructurados (¡sí, incluso para consultas M!)
- Crea diagramas del Data model
- Crea scripts DAX que te permitan editar varias medidas u otros objetos calculados en un solo documento
- Graba C# Scripts y guárdalos como macros (antes conocidas como "Acciones personalizadas")
- Integración con el Analizador VertiPaq
- \*Depurador de DAX

\*=Solo cuando estés conectado a una instancia de Analysis Services o Power BI
