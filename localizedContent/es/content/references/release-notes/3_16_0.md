## Requisitos previos

Tabular Editor 3 usa .NET 6. Si estás usando la versión portátil, es posible que tengas que instalar manualmente el [.NET 6.0 Runtime for Desktop](https://dotnet.microsoft.com/en-us/download/dotnet/6.0/runtime) antes de poder iniciar Tabular Editor 3.

## Novedades en 3.16.0

Consulta nuestro [blog de versiones](https://blog.tabulareditor.com/2024/06/13/tabular-editor-3-july-2024-release/) para ver un breve resumen de las actualizaciones más importantes de esta versión.

En esta versión, hemos renovado a fondo la forma en que funcionan los **Pivot Grid**. Algunos aspectos destacados:

- ¡Ahora puedes guardar diseños de Pivot Grid en archivos y cargarlos desde ellos! Se utiliza un nuevo formato de archivo, `.te3pivot`, para almacenar una representación JSON del diseño de Pivot Grid.
- Los objetos ocultos (medidas, columnas, jerarquías) ahora pueden usarse en un Pivot Grid, y cambiar el nombre de los objetos ya no debería provocar que el Pivot Grid pierda su diseño ni que se borren los filtros.
- Una **Lista de campos** del Pivot Grid nueva y mejorada, que ahora se muestra de forma predeterminada y ofrece una experiencia más familiar al agregar campos al Pivot Grid.

Consulta la [documentación de Pivot Grid](https://docs.tabulareditor.com/features/pivot-grid.html) para obtener más información sobre estos cambios.

## Mejoras en 3.16.0

- Hemos reducido el número de mensajes de error redundantes que se muestran cuando las expresiones DAX contienen errores de sintaxis o semántica.
- Los Scripts DAX ahora admiten expresiones no especificadas o vacías. Esto te permite especificar un Script DAX que no modifique las expresiones existentes de los objetos, o un Script DAX que establezca explícitamente una expresión para que quede vacía.
- Ahora puedes exportar los resultados de las consultas DAX a CSV/Excel. Busca las opciones **Exportar datos** y **Exportar datos (todos los resultados)** en el menú **Consulta**.
- Al exportar estadísticas del Analizador VertiPaq, ahora sugerimos un nombre predeterminado para el archivo VPAX en función del nombre de la base de datos.
- Cambiamos los accesos directos de teclado predeterminados de `Editor.Dax.CollapseAllFolds`, `Editor.Dax.ExpandAllFolds` y `Editor.Dax.ToggleAllFolds` de **Ctrl+Alt+(key)** a **Ctrl+Shift+(key)**. Esto evita conflictos con ciertos caracteres que requieren la tecla **AltGr** (equivalente a **Ctrl+Alt**) para escribirse en teclados no estadounidenses. Anteriormente, `Editor.Dax.CollapseFold`, `Editor.Dax.ExpandFold` y `Editor.Dax.ToggleFold` usaban los accesos directos **Ctrl+Shift+(key)**. Sus valores predeterminados se han cambiado a **Ctrl+(key)**.
- Actualizado [AMO/TOM a 19.80.0](https://www.nuget.org/packages/Microsoft.AnalysisServices.NetCore.retail.amd64).
- Ahora, los menús y las barras de herramientas están bloqueados en su sitio de forma predeterminada, lo que evita que se reposicionen por accidente. Para desbloquearlos, ve a **Herramientas > Personalizar... > Opciones** y desmarca la opción **Bloquear menús y barras de herramientas**.
- Nuestro analizador semántico de DAX ahora admite la sintaxis `WITH VISUAL SHAPE` que se usa en consultas DAX que contienen [cálculos de Visual](https://learn.microsoft.com/en-us/power-bi/transform-model/desktop-visual-calculations-overview). Ten en cuenta que la depuración de este tipo de consultas todavía no es compatible (ya que solo se pueden depurar medidas, y los cálculos de Visual no son medidas).
- Muestra una advertencia al depurar el valor de una celda de Pivot Grid o una consulta DAX que incluya elementos de cálculo en el contexto de filtro (algo que el depurador de TE3 no admite actualmente).

## Correcciones de errores en 3.16.0

- Al pulsar combinaciones de teclas con Control a las que no se les haya asignado ningún atajo, ya no deberían insertarse caracteres de control especiales en los editores de código.
- Las macros que se hayan modificado y guardado ya no deberían ejecutarse varias veces al invocarlas desde la interfaz de usuario.
- El método `.Output()` ahora debería mostrar el cuadro de diálogo de salida en ejecuciones posteriores de la macro, incluso si en una ejecución anterior se marcó la opción **No mostrar más salidas**.
- Las referencias a nombres de tabla sin comillas que, además, coincidan con palabras clave de DAX (como `Currency`) ya no se tratarán como palabras clave durante el formateo automático (es decir, no se escribirán en mayúsculas). Además, ahora algunas propiedades de objetos en los Script DAX también pueden estar vacías. Consulta la [documentación actualizada de Script DAX](https://docs.tabulareditor.com/features/dax-scripts.html) para obtener más información.
- Ahora, los errores durante la ejecución de Script DAX se gestionan correctamente, y los cambios realizados por el script antes de que se produjera el error se revierten automáticamente.
- Se corrigió un error por el que los botones de macros asignados a barras de herramientas personalizadas no se conservaban tras reiniciar la aplicación.
- Se corrigió un error que provocaba que Tabular Editor se cerrara inesperadamente al usar el método de extensión `ForEach<T>(IEnumerable<T>, Action<T>)` en un C# Script.
- Se corrigieron errores relacionados con la eliminación/restablecimiento de atajos de teclado, que no funcionaban correctamente para algunos comandos.
- Se corrigió un error por el que el botón Guardar permanecía deshabilitado después de que finalizara una operación de actualización.
- Se corrigió un error por el que determinadas acciones podían hacer que el **Explorador TOM** se bloqueara y no funcionara correctamente hasta que se reiniciara la aplicación.
- Se corrigió un error por el que, al eliminar un objeto en el **Explorador TOM**, la selección saltaba al nodo **Modelo**, en lugar de al nodo padre más cercano del objeto eliminado.
- Se corrigieron varios errores relacionados con Pivot Grids.
- El analizador semántico ya no debería producir falsos errores al comparar con `BLANK()` en expresiones como `"string value" IN { BLANK() }`.
- Se corrigió un error por el que los cambios en la configuración de serialización no se conservaban cuando se cargaba un modelo desde una carpeta TMDL. Consulta [#1285](https://github.com/TabularEditor/TabularEditor3/issues/1285).
- Al ordenar una columna en la Vista previa de tabla, cuando la columna tiene `IsAvailableInMDX = false` (es decir, no se han creado jerarquías de atributos para la columna), ahora usaremos consultas `TOPN` para previsualizar los datos, ya que `TOPNSKIP` no admite ordenar por columnas sin jerarquías de atributos.

## Vienes de Tabular Editor 2.x?

Mira [este vídeo](https://www.youtube.com/watch?v=pt3DdcjfImY) para hacerte una idea de las nuevas funcionalidades de Tabular Editor 3. Además, no dejes de consultar nuestra [guía de introducción](https://docs.tabulareditor.com/onboarding/index.html).

**Descripción general de las principales características de Tabular Editor 3:**

- IDE totalmente personalizable, con compatibilidad con varios monitores y Hi-DPI, y con temas
- Nuevo y potente editor de código DAX con autocompletado, comprobación de sintaxis, plegado de código y mucho, mucho más
- \*Modo del área de trabajo, que te permite guardar tus cambios en disco y sincronizar los metadatos del modelo con Analysis Services al mismo tiempo
- \*Previsualiza los datos de las tablas con desplazamiento infinito, crea PivotGrids o escribe consultas DAX para explorar el modelo o probar la lógica de cálculo
- \*Programa actualizaciones de datos
- Actualiza los esquemas de las tablas tanto en Data sources de tipo Provider como en orígenes de datos estructurados (¡sí, incluso para consultas M!)
- Crea diagramas del Data model
- Crea Scripts DAX que te permitan editar varias medidas u otros objetos calculados en un solo documento
- Graba C# Scripts y guárdalos como macros (antes conocidas como "Acciones personalizadas")
- Integración del Analizador VertiPaq
- Depurador de DAX

\*=Solo cuando estés conectado a una instancia de Analysis Services o Power BI
