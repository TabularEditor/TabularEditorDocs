## Vienes de Tabular Editor 2.x?

Mira [este vídeo](https://www.youtube.com/watch?v=pt3DdcjfImY) para hacerte una idea de las nuevas características de Tabular Editor 3.

**Resumen de las principales características de Tabular Editor 3:**

- IDE totalmente personalizable, con soporte para varios monitores, Hi-DPI y temas
- Nuevo y potente editor de código DAX con autocompletado, comprobación de sintaxis, plegado de código y mucho, mucho más
- \*Modo del área de trabajo, que te permite guardar los cambios en disco y sincronizar los metadatos del modelo con Analysis Services al mismo tiempo
- \*Previsualizar los datos de las tablas con desplazamiento infinito, crear PivotGrids o escribir consultas DAX para explorar el modelo o probar la lógica de cálculo
- \*Actualizar los esquemas de tabla tanto en Data sources de proveedor como en orígenes de datos estructurados (sí, ¡incluso para consultas M!)
- \*Programar actualizaciones de datos
- Crear diagramas del Data model
- Crear scripts DAX que te permitan editar varias medidas u otros objetos calculados en un único documento
- Grabar C# Scripts y guardarlos como macros (antes conocidas como "Custom Actions")
- Integración con el Analizador VertiPaq

\*=Solo cuando estés conectado a una instancia de Analysis Services o Power BI

## Problemas conocidos

- El Asistente de importación de tablas no se incluyó en esta versión. Estará disponible en la versión 3.0.3, que saldrá muy pronto. Como solución alternativa, crea una tabla nueva y especifica la consulta de partición (SQL o M). Después, haz clic con el botón derecho en la tabla y elige "Update Table Schema..." para importar columnas.
- Consulta la [hoja de ruta](https://github.com/TabularEditor/TabularEditor3/issues/12) para obtener más información.

## Novedades de la versión 3.0.2:

- Ahora puedes cambiar tu clave de licencia desde el cuadro de diálogo Ayuda > Acerca de Tabular Editor (opción disponible solo cuando no hay ningún modelo cargado).
- Se añadió un botón "Revert" en la barra de herramientas principal y en el menú Archivo (quizá tengas que restaurar el Workspace predeterminado para verlo). Al hacer clic, este botón vuelve a cargar el TOM desde el origen conectado (modelo de Power BI o base de datos de Analysis Services). Puedes usarlo para sincronizar manualmente los cambios entre ambos si la opción automática está desactivada en Herramientas > Preferencias.

## Correcciones de errores en la versión 3.0.2:

- Compatibilidad del Editor de DAX con las palabras clave TEXT / INTEGER en las funciones DAX PATHITEM / PATHITEMREVERSE.
- Las preferencias del Editor de DAX ahora se conservan entre sesiones (consulta el [#23](https://github.com/TabularEditor/TabularEditor3/issues/23))
- Diversas mejoras en el autocompletado y en los calltips del Editor de DAX.
- Corregido el problema [#14](https://github.com/TabularEditor/TabularEditor3/issues/14) (los errores de objeto no se mostraban en el Explorador TOM)
- Compatibilidad con la función de extensión DAX KMeansClustering.
- Corregido un error por el que el analizador de DAX generaba un Report de errores al comparar un valor con BLANK() en la función SWITCH.
- Algunas actualizaciones menores de Visual
- Corregido un bloqueo que se producía al intentar ejecutar un script DAX que edita o añade objetos que no se admiten en un archivo de Power BI Desktop (por ejemplo, [todavía no se pueden añadir tablas calculadas ni columnas calculadas a través de herramientas externas](https://docs.microsoft.com/en-us/power-bi/transform-model/desktop-external-tools))
- Ahora se pueden editar las DetailRowsExpressions en las medidas al conectarse a un archivo de Power BI Desktop.
- Corregido un error con SciLexer.dll que, en algunos casos, provocaba un bloqueo justo al iniciar.
- Los nombres de columna en la consulta DAX ahora se muestran tal cual (sin añadir espacios antes de las letras mayúsculas
