## ¡Importante!

**NOTA**: Tabular Editor 3.3.0 (y versiones posteriores) usa .NET 6. Es posible que debas instalar el [.NET 6.0 Runtime for Desktop](https://dotnet.microsoft.com/en-us/download/dotnet/6.0/runtime) antes de poder iniciar Tabular Editor 3.3.0 (o una versión posterior). A partir de la versión 3.3.1, el instalador de Tabular Editor 3 descargará e instalará estos requisitos previos si faltan.

## Mejoras en 3.3.6

- Se actualizó TOM a [19.52.2.2](https://www.nuget.org/packages/Microsoft.AnalysisServices.NetCore.retail.amd64).
- Se agregó compatibilidad con expresiones de cadena de formato dinámicas para medidas (nivel de compatibilidad 1601 o superior).
- Las consultas M `SourceExpression` y `PollingExpression` de las tablas ahora se pueden editar desde el Editor de expresiones principal.
- Al hacer clic con el botón derecho en una propiedad de expresión (como `SourceExpression`, `PollingExpression`, `FormatStringExpression`, etc.) en la cuadrícula de propiedades, ahora aparece un acceso directo para editar el valor en el Editor de expresiones principal.
- El valor de la propiedad `SourceColumn` de una columna de datos ahora aparece en la columna "Expression" del Explorador TOM
- Se agregaron descripciones emergentes en los encabezados de columna del Explorador TOM y del Analizador VertiPaq.
- Mejoras en el analizador de M y compatibilidad con funciones adicionales de M al realizar actualizaciones de esquema sin conexión, incluida la compatibilidad con CommandTimeout y otras opciones de conexión.
- Tabular Editor ahora muestra una notificación emergente cuando una operación de actualización de datos finaliza en segundo plano. Esto se puede desactivar en **Herramientas > Preferencias > Características > Notificaciones**.
- Se agregó compatibilidad con nuevas funciones DAX ([OFFSET](https://dax.guide/offset), [TOJSON](https://dax.guide/tojson), [TOCSV](https://dax.guide/tocsv) y más).
- Las macros ahora se compilan en un subproceso en segundo plano, lo que mejora el tiempo de inicio de la aplicación y también corrige el problema [#708](https://github.com/TabularEditor/TabularEditor3/issues/708).
- Se actualizó para usar la versión más reciente de [Analizador VertiPaq](https://github.com/sql-bi/VertiPaq-Analyzer).
- Se agregó el nivel de compatibilidad 1600 como nuevo valor predeterminado para los modelos de SQL Server 2022.

## Correcciones de errores en 3.3.6

- El depurador de DAX ahora conserva correctamente la expresión de consulta original durante la depuración, manteniendo así el contexto original de filtro sombra y cualquier modificación de auto-exist aplicada a los filtros de tabla. Entre otras cosas, esto significa que las expresiones que contienen `ALLSELECTED` ahora se pueden depurar correctamente; consulta [#669](https://github.com/TabularEditor/TabularEditor3/issues/669).
- Se mejoró la conectividad asíncrona en el Asistente para importar tablas y durante la Actualización de esquema. También se mejoró el control de errores; por ejemplo, cuando falla la conexión SQL debido a la falta de un certificado.
- Se corrigió un problema que hacía que el botón Eliminar no funcionara en la Vista de macros.
- Se corrigió un problema por el que el analizador de M no admitía registros cuando el nombre del campo contiene un espacio, como `[Order Date]`. Esto debería permitir que la función de Actualización de esquema funcione sin conexión cuando una expresión M contenga una referencia de registro de este tipo.
- Se corrigió un error por el que el archivo .tmuo no se actualizaba al usar el modo del área de trabajo con una estructura de carpetas, lo que provocaba que las credenciales no se guardaran entre sesiones.
- Se corrigió un error que impedía buscar en todas las propiedades de las expresiones; consulta [#663](https://github.com/TabularEditor/TabularEditor3/issues/663).
- Se corrigió un problema por el que el script del depurador no se cerraba al cerrar un modelo.
- Se eliminó la posibilidad de agregar particiones a tablas calculadas.
- Se corrigió un problema por el que la aplicación a veces se bloqueaba al usar el cuadro de diálogo Buscar/Reemplazar.
- Se corrigió la información sobre herramientas del botón Nuevo Script DAX, que era incorrecta; consulta [#678](https://github.com/TabularEditor/TabularEditor3/issues/678).
- Se corrigió un problema por el que, al cerrar documentos flotantes, no se solicitaba guardar; consulta [#673](https://github.com/TabularEditor/TabularEditor3/issues/673).
- Se corrigió un problema por el que el filtro de columna en una vista previa de datos a veces hacía que la aplicación se cerrara inesperadamente al alternar los valores (All) o (Blank).
- Se corrigió un problema con la inferencia de tipos de datos de Snowflake al usar Actualización de esquema o el Asistente para importar tablas (las columnas NUMBER se importaban como Double cuando deberían importarse como Int64 o Decimal).
- El analizador semántico ya no debería informar de un error cuando se usan columnas de extensión en [SUMMARIZE](https://dax.guide/summarize).
- El analizador semántico ahora informa de un error cuando se usa 'ID' como referencia de tabla sin comillas.

## ¿Vienes de Tabular Editor 2.x?

Mira [este vídeo](https://www.youtube.com/watch?v=pt3DdcjfImY) para hacerte una idea de las nuevas características de Tabular Editor 3. Además, asegúrate de consultar nuestra [guía de incorporación](https://docs.tabulareditor.com/onboarding/index.html).

**Descripción general de las principales características de Tabular Editor 3:**

- IDE totalmente personalizable, con soporte para varios monitores, Hi-DPI y temas
- Nuevo y potente editor de código DAX con autocompletado, comprobación de sintaxis, plegado de código y mucho, mucho más
- \*Modo del área de trabajo, que te permite guardar los cambios en el disco y sincronizar los metadatos del modelo con Analysis Services al mismo tiempo
- \*Previsualizar los datos de una tabla con desplazamiento infinito, crear PivotGrids o escribir consultas DAX para explorar el modelo o probar la lógica de cálculo
- \*Programar actualizaciones de datos
- Actualizar esquemas de tablas tanto en Data sources de proveedor como en orígenes de datos estructurados (sí, incluso para consultas M)
- Crear diagramas del Data model
- Crea scripts DAX que te permitan editar varias medidas u otros objetos calculados en un único documento
- Graba C# Scripts y guárdalos como macros (antes conocidas como "Acciones personalizadas")
- Integración con el Analizador VertiPaq
- \*Depurador de DAX

\*=Solo al estar conectado a una instancia de Analysis Services o Power BI
