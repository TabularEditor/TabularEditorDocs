## Mejoras en 3.1.7

- Cuando una consulta DAX hace que Analysis Services devuelva mensajes de error, esos mensajes ahora se muestran en lugar de la cuadrícula de resultados anterior.
- ¡Ahora puedes hacer zoom en las cuadrículas (Ctrl+rueda del ratón)!
- Varias mejoras en el autocompletado y en la sangría automática.
- Un nuevo menú al hacer clic con el botón derecho facilita añadir permisos de tabla a los roles (antes, tenías que añadir un permiso de tabla asignando una expresión de RLS o especificar una configuración de OLS). Consulta la incidencia [#257](https://github.com/TabularEditor/TabularEditor3/issues/257). Puedes añadir varios permisos de tabla en varios roles a la vez.
- El cuadro de diálogo "Seleccionar base de datos" ahora recuerda el orden de clasificación de las columnas. Consulta [#321](https://github.com/TabularEditor/TabularEditor3/issues/321).

## Correcciones de errores en 3.1.7

- Se ha corregido un problema por el que SaveModelMetadataBackup no funcionaba con los ID de la base de datos (cuando los ID y los nombres son diferentes).
- Se corrigió un bloqueo al restablecer las preferencias de formato automático
- Se ha corregido un problema por el que no se exportaba TOM en VPAX. Se ha corregido el cuadro de diálogo de mensajes cuando VPAX contiene TOM.
- Se ha mejorado el formato DAX (sin conexión); consulta la incidencia [#306](https://github.com/TabularEditor/TabularEditor3/issues/306).
- Se ha corregido un problema con las funciones auxiliares `FormatDax(...)` y `CallDaxFormatter()` al usar el formateador predeterminado (sin conexión); consulta [#307](https://github.com/TabularEditor/TabularEditor3/issues/307).
- Se ha corregido un problema por el que el autocompletado a veces se quedaba "atascado".
- Debería haber menos ventanas/cuadros de diálogo con problemas de escalado cuando se cambie la resolución o el escalado de pantalla de Windows. Consulta [#308](https://github.com/TabularEditor/TabularEditor3/issues/308) y [#81](https://github.com/TabularEditor/TabularEditor3/issues/81).
- El autoformateador ya no debería repetir bloques de comentarios de varias líneas. Consulta [#319](https://github.com/TabularEditor/TabularEditor3/issues/319].
- Se corrigió un problema por el que el análisis semántico en segundo plano generaba un Report de errores de DAX no válidos; consulta [#311](https://github.com/TabularEditor/TabularEditor3/issues/311).
- Abrir el administrador de BPA desde la barra de menús ya no debería provocar un bloqueo; consulta [#312](https://github.com/TabularEditor/TabularEditor3/issues/312).
- Guardar un modelo en modo del área de trabajo, Workspace, cargado desde un archivo Database.json, ya no debería provocar el error "Save failed!", consulta el problema [#322](https://github.com/TabularEditor/TabularEditor3/issues/322).
- Los comentarios ahora se conservan correctamente al aplicar un script DAX; consulta el problema [#323](https://github.com/TabularEditor/TabularEditor3/issues/323).

## Vienes de Tabular Editor 2.x?

Mira [este vídeo](https://www.youtube.com/watch?v=pt3DdcjfImY) para hacerte una idea de las nuevas características de Tabular Editor 3.

**Descripción general de las principales características de Tabular Editor 3:**

- IDE totalmente personalizable, con compatibilidad con varios monitores, Hi-DPI y temas
- Nuevo y potente editor de código DAX con autocompletado, comprobación de sintaxis, plegado de código y mucho, mucho más
- \*Modo del área de trabajo, Workspace, que permite guardar los cambios en disco y sincronizar los metadatos del modelo con Analysis Services al mismo tiempo
- \*Obtén una vista previa de los datos de tabla con desplazamiento infinito, crea PivotGrids o escribe consultas DAX para explorar el modelo o probar la lógica de cálculo
- \*Programar actualizaciones de datos
- Actualizar esquemas de tablas tanto en proveedores como en orígenes de datos estructurados (sí, incluso para consultas M!)
- Crear diagramas del Data model
- Crear scripts DAX que te permitan editar varias medidas u otros objetos calculados en un solo documento
- Grabar C# Scripts y guardarlos como macros (anteriormente conocidos como "Custom Actions")
- Integración del Analizador VertiPaq

\*=Solo mientras estés conectado a una instancia de Analysis Services o Power BI
