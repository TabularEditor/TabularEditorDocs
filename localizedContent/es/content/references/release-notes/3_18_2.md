## Correcciones de errores en 3.18.2

- Se corrigió una regresión relacionada con la configuración regional del sistema operativo, que provocaba un bloqueo al abrir por primera vez el cuadro de diálogo de preferencias.
- Se corrigió el parpadeo en la vista de actualización de datos y se muestra el icono de advertencia correcto para los mensajes de advertencia en la salida

## Mejoras en 3.18.1

- El Analizador semántico ya no muestra un error al usar el nombre de una tabla como nombre de variable. **Nota:** en el momento de escribir esto, la posibilidad de usar nombres de tabla para variables es compatible con Power BI Desktop (octubre de 2024) y con el servicio Power BI. Puedes controlar el comportamiento ante errores en **Herramientas > Preferencia > Editor de DAX > Configuración de DAX > Variables con nombre de tabla**.
- Hemos actualizado las bibliotecas cliente AMO/TOM a la versión más reciente [19.86.6](https://www.nuget.org/packages/Microsoft.AnalysisServices/19.86.6). Esto debería solucionar el problema **Can't obtain account information for '(e-mail)' while trying to refresh the token.** que algunos usuarios han experimentado al conectarse al punto de conexión XMLA de Power BI; consulta [#1387](https://github.com/TabularEditor/TabularEditor3/issues/1387).
- Una nueva opción en **Herramientas > Preferencia > Power BI** te permite especificar el modo de autenticación predeterminado que se usará al conectarte a los Workspaces de Power BI / Fabric o a Azure Analysis Services. Esto te permite, por ejemplo, establecer `Microsoft Entra MFA` como valor predeterminado, forzando la aparición del selector de cuenta o de la ventana emergente de MFA al conectarte.
- Para las licencias multiusuario, ahora mostramos una opción para cambiar la clave de licencia si se ha superado el número de usuarios de la licencia. De este modo, ya no tendrás que restablecer manualmente la clave de licencia [a través del Registro de Windows](https://docs.tabulareditor.com/te3/getting-started.html#changing-a-license-key-through-the-registry).

## Correcciones de errores en 3.18.1

- Al intentar conectarte al servicio del Optimizador de DAX con una cuenta de Tabular Tools que no se ha configurado por completo, ahora mostramos un mensaje de error más descriptivo, en lugar de generar una excepción no controlada.
- Se corrigió un error por el que los archivos VPAX creados en Tabular Editor 3 se marcaban como corruptos al abrirlos en el servicio del Optimizador de DAX.
- Se corrigió un error por el que se cargaba un archivo VPAX vacío en el servicio Optimizador de DAX si la recopilación de estadísticas de VPAX fallaba durante el proceso de carga.
- Se corrigió un error por el que las características del motor semántico no siempre se actualizaban para el **Editor de expresiones** al abrir un modelo nuevo (lo que podía hacer que aparecieran errores falsos en el editor).
- El Analizador semántico ya no debería provocar la excepción "An item with the same key has already been added." en modelos con expresiones DAX complejas y anidadas; consulta [#1390](https://github.com/TabularEditor/TabularEditor3/issues/1390) y [#1393](https://github.com/TabularEditor/TabularEditor3/issues/1393).
- El Analizador semántico ahora infiere correctamente el tipo de datos de una expresión como `SELECTEDVALUE('MyTable'[String Column], BLANK())` (en este caso, el tipo de datos resultante debería ser **String**, pero antes se infería como **Variant**).
- Se corrigió un error por el que las acciones de código **Simplificar variable** generaban código que no era semánticamente equivalente al código original. Además, esta acción ya no provocará un "bucle" de acciones de código aplicándose una y otra vez; consulta [#1386](https://github.com/TabularEditor/TabularEditor3/issues/1386).

## Novedades en 3.18.0

Consulta la [entrada de nuestro blog sobre el lanzamiento](https://blog.tabulareditor.com/2024/10/31/tabular-editor-3-october-2024-release) para obtener un breve resumen de las actualizaciones más importantes de esta versión.

- Nuestra **integración con Optimizador de DAX** ya no está en versión preliminar, y además hemos realizado algunas actualizaciones para que ahora sea compatible con las [nuevas métricas de evaluación que formaban parte de Optimizador de DAX 1.2](https://www.tabulartools.com/blog/introducing-new-evaluation-metrics-in-dax-optimizer-1-2/). Ah, y por cierto: todos los usuarios de Tabular Editor 3 de la **Edición Enterprise** ahora disponen de 4 ejecuciones gratuitas de Optimizador de DAX al día. Descarga Tabular Editor 3.18.0 desde uno de los enlaces anteriores y obtén tu código de canje de Optimizador de DAX en la opción de menú **Ayuda > Acerca de Tabular Editor**.
- Nos alegra presentar una nueva funcionalidad en todos los editores de DAX de Tabular Editor 3: **Acciones de código**. Esta función de productividad ofrece discretamente sugerencias para mejorar tu código DAX y te permite aplicar la sugerencia con un solo clic. Por ejemplo, la función puede usarse para eliminar variables no utilizadas, seguir las prácticas recomendadas para las referencias de columnas y medidas, reescribir filtros de tabla como [filtros de columna más eficientes](https://www.sqlbi.com/articles/filter-columns-not-tables-in-dax/) y [mucho, mucho más](xref:code-actions). Tenemos previsto añadir más acciones en futuras versiones, ¡así que mantente al tanto!
- Después de actualizar a la [última versión de la biblioteca cliente de Microsoft AMO/TOM](https://www.nuget.org/packages/Microsoft.AnalysisServices/), cambió el comportamiento de la opción de autenticación **Integrada**, de modo que el selector de cuentas de Entra ID/ventana emergente de MFA ya no se muestra si el sistema operativo Windows ya está conectado a una cuenta de Entra ID. Por ello, hemos añadido una nueva opción de autenticación, **Microsoft Entra MFA**, que hará que aparezca el selector de cuentas/ventana emergente de MFA, lo que te permitirá elegir una cuenta distinta de la que tienes actualmente iniciada en Windows, si lo necesitas.

## Mejoras en 3.18.0

- Las preferencias de la interfaz de usuario se han ampliado para permitir ocultar los detalles del origen del modelo en la barra de título, lo que ayuda a proteger información confidencial durante grabaciones y presentaciones.
- Las preferencias de la interfaz de usuario ahora incluyen una nueva sección "Elementos recientes". Esta sección permite personalizar el número de archivos y modelos recientes que se muestran en el menú Archivo, así como los servidores que se muestran en conexiones recientes. Las nuevas preferencias también permiten borrar estos elementos.
- El Analizador semántico ahora muestra una advertencia si una variable se usa directamente como primer argumento de [`CALCULATE`](https://dax.guide/CALCULATE) o [`CALCULATETABLE`](https://dax.guide/CALCULATETABLE). Dado que las variables no se ven afectadas por la transición de contexto ni por contextos de filtro modificados, usar una variable de este modo probablemente no sea intencionado.
- Nuestro editor SQL ahora usa un esquema de colores mejorado, lo que facilita distinguir las palabras clave SQL de los identificadores.
- Hemos actualizado AMO/TOM y otras dependencias a sus versiones más recientes.
- Ahora usamos el nivel de compatibilidad 1606 para los nuevos modelos semánticos de Power BI / Fabric. En este nivel, hay disponible una nueva propiedad del modelo, [ValueFilterBehavior](https://learn.microsoft.com/en-us/dotnet/api/microsoft.analysisservices.tabular.valuefilterbehaviortype?view=analysisservices-dotnet), que puede usarse para controlar el comportamiento _auto-exist_ de [`SUMMARIZECOLUMNS`](https://dax.guide/SUMMARIZECOLUMNS).
- Hemos añadido compatibilidad con la colección TOM [ChangedProperties](https://learn.microsoft.com/en-us/dotnet/api/microsoft.analysisservices.tabular.changedproperty?view=analysisservices-dotnet) (en tablas, medidas, columnas y algunos otros objetos). Esta colección se usa en modelos compuestos o Direct Lake para indicar que una propiedad (como el **Name** de una columna) ha cambiado y ya no debe sincronizarse desde el origen subyacente.
- Hemos actualizado el Analizador VertiPaq a la versión más reciente [(1.7.2)](https://github.com/sql-bi/VertiPaq-Analyzer/releases/tag/v1.7.2), que corrige un problema relacionado con la extracción de estadísticas de un modelo que contiene Field Parameters / Group By Columns.
- Hemos actualizado nuestra versión del conector .NET de Snowflake a la versión 4.1.0. Esto corrige las siguientes vulnerabilidades: [CVE-2023-34230](https://nvd.nist.gov/vuln/detail/CVE-2023-34230) y [CVE-2023-51662](https://nvd.nist.gov/vuln/detail/CVE-2023-51662).
- El componente de diagramas ahora debería ser considerablemente más rápido al trabajar con diagramas grandes con muchas relaciones.
- Ahora se pueden ocultar columnas en la Vista previa de datos haciendo clic con el botón derecho en el encabezado de la columna y seleccionando "Ocultar esta columna". Para mostrar las columnas ocultas, haz clic con el botón derecho en un encabezado de columna y selecciona "Selector de columnas". Esto abrirá una ventana desde la que podrás acceder a las columnas ocultas anteriormente.

## Correcciones de errores en 3.18.0

- Se corrigió un error por el que el menú desplegable "Server" del cuadro de diálogo **Cargar modelo semántico desde la base de datos** no mostraba los servidores usados recientemente.
- La configuración de la vista previa de datos (orden de clasificación, filtros, etc.) ahora se conserva tras actualizar la vista previa, lo que corrige un error anterior por el que no se mantenía la configuración.
- Se corrigió un cierre inesperado que podía producirse al deshacer (Ctrl+Z) durante o inmediatamente después de usar la refactorización "Rename" en el Editor de DAX.
- Ahora la interfaz principal se bloquea correctamente para evitar interacciones del usuario mientras se muestra el cuadro de diálogo "Please wait", por ejemplo, al realizar una actualización del esquema de una tabla.
- El Analizador semántico ya no debería mostrar una advertencia al hacer referencia a la medida interna "FormatString" en una expresión DAX. Como consecuencia, ahora es posible depurar consultas que contienen esas referencias (aunque actualmente no se puede depurar la propia expresión FormatString); consulta el [#1377](https://github.com/TabularEditor/TabularEditor3/issues/1377).

## Problemas conocidos en 3.18.0

- Si te aparece el mensaje de error **Can't obtain account information for '(e-mail)' while trying to refresh the token.**, normalmente ayuda volver a realizar la operación. Se trata de un problema en las versiones más recientes de las bibliotecas cliente AMO/TOM. También deberías poder evitar este problema eligiendo la opción de autenticación **Microsoft Entra MFA**, en lugar de la opción **Integrada**, al conectarte por primera vez.

## Vienes de Tabular Editor 2.x?

Mira [este vídeo](https://youtu.be/O4ATwdzCvWc) para hacer un recorrido rápido por las principales características de Tabular Editor 3. Además, no dejes de consultar nuestra [guía de incorporación](https://docs.tabulareditor.com/onboarding/index.html).

**Descripción general de las principales características de Tabular Editor 3:**

- IDE totalmente personalizable, con compatibilidad con varios monitores, pantallas HiDPI y temas
- Nuevo y potente editor de código DAX con autocompletado, comprobación de sintaxis, plegado de código y mucho, mucho más
- \*Modo del área de trabajo, que te permite guardar los cambios en disco y sincronizar los metadatos del modelo con Analysis Services al mismo tiempo
- \*Previsualizar datos de tablas con desplazamiento infinito, crear PivotGrids o escribir consultas DAX para explorar el modelo o probar la lógica de cálculo
- \*Programar actualizaciones de datos
- Actualizar esquemas de tablas tanto en Data sources de proveedor como en orígenes de datos estructurados (sí, ¡incluso para consultas M!)
- Crear diagramas del Data model
- Crear scripts DAX que te permiten editar varias medidas u otros objetos calculados en un único documento
- Grabar C# Scripts y guardarlos como macros (antes conocidas como "Custom Actions")
- Integración con el Analizador VertiPaq
- Depurador de DAX
- Integración con el Optimizador de DAX
- Acciones de código para refactorizar tu DAX fácilmente.

\*=Solo al conectarte a una instancia de Analysis Services o Power BI
