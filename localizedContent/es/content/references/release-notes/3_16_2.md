## Requisitos previos

Tabular Editor 3 usa .NET 6. Si estás usando la versión portátil, es posible que tengas que instalar manualmente el runtime de .NET 6.0 para escritorio ([.NET 6.0 Runtime for Desktop](https://dotnet.microsoft.com/en-us/download/dotnet/6.0/runtime)) antes de poder iniciar Tabular Editor 3.

***

Consulta nuestro [blog de lanzamientos](https://blog.tabulareditor.com/) para obtener un breve resumen de las actualizaciones más importantes de la versión 3,16.

## Mejoras en 3.16.2

- Se revirtió [AMO/TOM a la versión 19.79.1.1](https://www.nuget.org/packages/Microsoft.AnalysisServices.NetCore.retail.amd64/19.79.1.1), ya que la versión 19.80.0 introdujo varios problemas relacionados con la autenticación MFA y otros problemas de conectividad con XMLA. Consulta [#1317](https://github.com/TabularEditor/TabularEditor3/issues/1317) y [#1322](https://github.com/TabularEditor/TabularEditor3/issues/1322).

## Correcciones de errores en 3.16.1

- Se corrigió un error que impedía que los archivos de diagrama guardados (.te3diag) se abrieran correctamente; consulta [#1313](https://github.com/TabularEditor/TabularEditor3/issues/1313)

## Mejoras en 3.16.1

- Los resultados de las consultas DAX ahora muestran el "formato de hora larga" para los valores de fecha y hora, en lugar del "formato de hora corta". De este modo, también se muestran los segundos en la cuadrícula de resultados.

## Novedades en 3.16.0

En esta versión, hemos renovado a fondo el funcionamiento de las **Pivot Grid**. Algunos puntos destacados:

- ¡Ahora puedes guardar y cargar diseños de Pivot Grid en/desde archivos! Se utiliza un nuevo formato de archivo, `.te3pivot`, para almacenar una representación JSON del diseño de Pivot Grid.
- Los objetos ocultos (medidas, columnas, jerarquías) ahora se pueden usar en un Pivot Grid, y los cambios de nombre de objetos ya no deberían hacer que el Pivot Grid pierda su diseño ni que se borre ningún filtro.
- Una **Lista de campos** de Pivot Grid nueva y mejorada, que ahora se muestra de forma predeterminada y ofrece una experiencia más familiar al agregar campos al Pivot Grid.

Consulta la [documentación de Pivot Grid](https://docs.tabulareditor.com/te3/features/pivot-grid.html) para obtener más información sobre estos cambios.

## Mejoras en 3.16.0

- Se redujo el número de mensajes de error inútiles o redundantes que se muestran cuando las expresiones DAX contienen errores de sintaxis o semántica.
- Los Scripts DAX ahora admiten expresiones no especificadas o vacías. Esto te permite especificar un Script DAX que no modifique las expresiones existentes en los objetos, o un script que deje explícitamente una expresión vacía.
- Ahora puedes exportar los resultados de las consultas DAX a CSV/Excel. Busca las opciones **Exportar datos** y **Exportar datos (todos los resultados)** en el menú **Consulta**.
- Al exportar las estadísticas del Analizador VertiPaq, ahora sugerimos un nombre predeterminado para el archivo VPAX en función del nombre de la base de datos.
- Hemos cambiado los atajos de teclado predeterminados de `Editor.Dax.CollapseAllFolds`, `Editor.Dax.ExpandAllFolds` y `Editor.Dax.ToggleAllFolds` de **Ctrl+Alt+(tecla)** a **Ctrl+Mayús+(tecla)**. Esto evita conflictos con ciertos caracteres que requieren la tecla **AltGr** (equivalente a **Ctrl+Alt**) para poder escribirse en teclados que no son de EE. UU. Los atajos **Ctrl+Mayús+(tecla)** se usaban anteriormente para `Editor.Dax.CollapseFold`, `Editor.Dax.ExpandFold` y `Editor.Dax.ToggleFold`. Sus valores predeterminados se han cambiado a **Ctrl+(tecla)**.
- Actualizado [AMO/TOM a 19.80.0](https://www.nuget.org/packages/Microsoft.AnalysisServices.NetCore.retail.amd64).
- Los menús y las barras de herramientas ahora se bloquean en su sitio de forma predeterminada, evitando que se reubiquen accidentalmente. Para desbloquearlos, ve a **Herramientas > Personalizar... > Opciones** y desmarca la opción **Bloquear menús y barras de herramientas**.
- Nuestro analizador semántico de DAX ahora admite la sintaxis `WITH VISUAL SHAPE` usada en consultas DAX que contienen [cálculos de Visual](https://learn.microsoft.com/en-us/power-bi/transform-model/desktop-visual-calculations-overview). Ten en cuenta que la depuración de estas consultas todavía no es compatible (solo se pueden depurar medidas, y los cálculos de Visual no son medidas).
- Muestra una advertencia al depurar un valor de celda de Pivot Grid o una consulta DAX que incluya elementos de cálculo en el contexto de filtro (algo que el depurador TE3 no admite actualmente).

## Correcciones de errores en 3.16.0

- Al pulsar combinaciones con la tecla Control que no tengan ningún atajo asignado, ya no deberían insertarse caracteres de control especiales en los editores de código.
- Las macros que se hayan modificado y guardado ya no deberían ejecutarse varias veces al invocarlas desde la interfaz de usuario.
- El método `.Output()` ahora debería mostrar el cuadro de diálogo de salida en ejecuciones posteriores de la macro, incluso cuando en una ejecución anterior se hubiera marcado la opción **No mostrar más salidas**.
- Las referencias a nombres de tabla sin comillas que, además, sean palabras clave de DAX (como `Currency`) ya no se tratarán como palabras clave durante el formateo automático (es decir, no se pondrán en mayúsculas). Además, ciertas propiedades de objetos en los Scripts DAX ahora también pueden estar vacías. Consulta la [documentación actualizada de Scripts DAX](https://docs.tabulareditor.com/te3/features/dax-scripts.html) para obtener más información.
- Los errores durante la ejecución de un Script DAX ahora se gestionan de forma controlada, y los cambios que el script haya realizado antes de que se produjera el error se revierten automáticamente.
- Corregido un error por el que los botones de macro asignados a barras de herramientas personalizadas no se conservaban después de reiniciar la aplicación.
- Corregido un error que hacía que Tabular Editor se cerrara inesperadamente al usar el método de extensión `ForEach<T>(IEnumerable<T>, Action<T>)` en un C# Script.
- Se corrigieron errores relacionados con la eliminación o el restablecimiento de atajos de teclado, que no funcionaban correctamente en algunos comandos.
- Se corrigió un error por el que el botón Guardar permanecía deshabilitado tras finalizar una operación de actualización.
- Se corrigió un error por el que determinadas acciones podían hacer que el **Explorador TOM** se quedara bloqueado y no se comportara correctamente hasta reiniciar la aplicación.
- Se corrigió un error por el que, al eliminar un objeto en el **Explorador TOM**, la selección saltaba al nodo **Modelo** en lugar de al nodo padre más cercano del objeto eliminado.
- Se corrigieron varios errores relacionados con las Pivot Grid.
- El analizador semántico ya no debería generar falsos errores al comparar con `BLANK()` en expresiones como `"string value" IN { BLANK() }`.
- Se corrigió un error por el que los cambios en la configuración de serialización no se conservaban cuando se cargaba un modelo desde una carpeta TMDL. Ver [#1285](https://github.com/TabularEditor/TabularEditor3/issues/1285).
- Al ordenar una columna en la Vista previa de tabla, cuando esta tiene `IsAvailableInMDX = false` (es decir, no se han creado jerarquías de atributos para la columna), no recurriremos a consultas `TOPN`, ya que `TOPNSKIP` no admite ordenar por columnas sin jerarquías de atributos.

## ¿Vienes de Tabular Editor 2.x?

Mira [este vídeo](https://www.youtube.com/watch?v=pt3DdcjfImY) para hacerte una idea de las nuevas funcionalidades de Tabular Editor 3. Además, asegúrate de consultar nuestra [guía de inicio](https://docs.tabulareditor.com/onboarding/index.html).

**Descripción general de las principales funciones de Tabular Editor 3:**

- IDE totalmente personalizable, con compatibilidad con varios monitores, Hi-DPI y temas
- Nuevo y potente editor de código DAX con autocompletado, comprobación de sintaxis, plegado de código y mucho, mucho más
- \*Modo del área de trabajo, que te permite guardar los cambios en disco y sincronizar simultáneamente los metadatos del modelo con Analysis Services
- \*Previsualizar datos de tablas con desplazamiento infinito, crear PivotGrids o escribir consultas DAX para explorar el modelo o probar la lógica de cálculo
- \*Programar actualizaciones de datos
- Actualizar los esquemas de tablas tanto en Data source de proveedor como en origen de datos estructurado (sí, incluso para consultas M)
- Crear diagramas del Data model
- Crear archivos Script DAX que te permitan editar varias medidas u otros objetos calculados en un único documento
- Grabar un C# Script y guardarlo como macro (antes conocida como "Acciones personalizadas")
- Integración del Analizador VertiPaq
- Depurador de DAX

\*=Solo disponible cuando estés conectado a una instancia de Analysis Services o Power BI
