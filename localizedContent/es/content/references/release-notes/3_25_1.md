## Novedades en 3.25.0

Echa un vistazo a nuestro [blog de la versión](https://tabulareditor.com/blog/tabular-editor-3-january-2026-release) para ver un resumen rápido de las actualizaciones más importantes de esta versión.

- Tabular Editor ahora está disponible en [**chino y español (versión preliminar)**](xref:references-application-language). Todas las traducciones se han realizado inicialmente con IA, con revisión por parte de personas. El japonés, el alemán y el francés también están disponibles como idiomas experimentales, aunque aún no se han revisado.
- Tabular Editor ahora incluye un conjunto de [**reglas integradas de Best Practice Analyzer (BPA)**](xref:built-in-bpa-rules) que cubren las principales prácticas recomendadas en formato, metadatos, diseño del modelo, expresiones DAX y traducciones.
- Tabular Editor ahora incluye la función **Semantic Bridge**, que permite crear fácilmente modelos semánticos a partir de [Metric Views](https://docs.databricks.com/aws/en/metric-views/) de Databricks (_solo disponible en Tabular Editor 3 Edición Enterprise_).
- Se ha agregado una opción para guardar modelos semánticos con [**archivos complementarios para Fabric**](xref:save-with-supporting-files). Cuando está habilitado, Tabular Editor puede crear y actualizar los archivos .platform y definition.pbism junto con los metadatos del modelo para que coincidan con la estructura del repositorio de Fabric.
- Esta versión también incorpora un nuevo **Editor de calendario**, que facilita la administración de los objetos de calendario usados para la inteligencia temporal basada en calendario.
- Por último, pero no menos importante, hemos agregado un cuadro de diálogo de [**Actualización avanzada**](xref:advanced-refresh), mediante el cual puedes configurar opciones avanzadas de actualización, como el paralelismo y la fecha efectiva para la actualización incremental. También puedes especificar [perfiles de anulación de actualización](xref:refresh-overrides) (_solo en Tabular Editor 3 Edición Business y Edición Enterprise_).

## Mejoras en 3.25.0

- Se agregó compatibilidad para establecer la propiedad CustomData de la cadena de conexión directamente desde el cuadro de diálogo de suplantación.
- La vista de DAX Package Manager ahora descarga y muestra los iconos de los paquetes de DaxLib.
- DAX Package Manager ahora almacena en caché los metadatos de los paquetes, lo que te permite ver la información de los paquetes sin necesidad de conexión a Internet.
- Hemos actualizado el paquete de la biblioteca cliente de AS a la versión más reciente [19.108.5](https://www.nuget.org/packages/Microsoft.AnalysisServices/19.108.5). Esta actualización te permite controlar la nueva propiedad [UnicodeCharacterBehavior](https://learn.microsoft.com/en-us/dotnet/api/microsoft.analysisservices.tabular.database.unicodecharacterbehavior?view=analysisservices-dotnet#microsoft-analysisservices-tabular-database-unicodecharacterbehavior) en el objeto Database.
- Se agregaron las categorías de datos de columna específicas de Power BI: Longitud, Latitud y Lugar.
- Ahora mostramos mensajes de error si intentas establecer el nivel de compatibilidad de un modelo a un valor que no admite el servidor. Esto es para evitar errores inesperados más adelante al guardar los cambios en el modelo. Si necesitas establecer un nivel de compatibilidad no admitido con fines de migración, guarda los metadatos del modelo en disco (en formato .bim o TMDL) y luego abre los metadatos sin ninguna base de datos de Workspace asociada.
- Los archivos de opciones de usuario (.tmuo) ahora también se guardan cuando abres un modelo directamente desde un servidor o un punto de conexión XMLA. Los archivos se almacenan en la carpeta %LocalAppData%\TabularEditor3\UserOptions.
- Se agregó la opción "Actualizar completo (ignorar política)" al menú contextual de actualización de datos, para que puedas actualizar fácilmente los datos en las particiones existentes, sin desencadenar operaciones de agregar/eliminar/combinar particiones en el servidor, cuando una tabla está regida por una política de actualización incremental.
- Se reorganizaron las opciones del menú contextual de actualización de datos para llevar las opciones más usadas a la parte superior.
- Se agregó Hora de inicio al panel de actualización, lo que permite ordenar las actualizaciones según cuándo se iniciaron; por ejemplo, ver primero la actualización más reciente.
- Se agregó un panel de búsqueda al cuadro de diálogo de Preferencias para buscar rápidamente preferencias individuales.
- Se agregaron "Funciones definidas por el usuario" y "Calendarios" a la lista del editor de reglas de prácticas recomendadas.
- Se agregó la propiedad `UsedInCalendars` a las columnas, lo que facilita comprobar dentro de un C# Script si una columna se usa en algún objeto de calendario.
- La acción del menú "Archivo > Cerrar" pasó a llamarse "Cerrar documento" y ahora permite cerrar cualquier panel o documento en el área principal de documentos. Se asignó **Ctrl + W** como acceso directo predeterminado.
- Se agregó compatibilidad del analizador semántico con la nueva palabra clave `ENDALIGNED` del parámetro <extension> de [`DATEADD`](https://dax.guide/DATEADD). [Más información](https://learn.microsoft.com/en-us/dax/dateadd-function-dax#extension-parameter-for-small--large-period-shifts).
- Al guardar en un archivo model.bim o en una estructura de carpetas TMDL, con la opción de las opciones de serialización **Ignorar particiones de actualización incremental** habilitada, ahora se añade una partición ficticia para reemplazar las particiones PolicyRange. Esto garantiza que el modelo se pueda seguir cargando en Power BI Desktop, ya que no se guarda ninguna tabla sin particiones.

## Correcciones de errores en 3.25.0

- Se corrigió un problema crítico que impedía a algunos clientes conectarse a la API REST de Power BI o a Databricks mediante autenticación de Azure AD.
- Se corrigió una excepción no controlada que se producía al eliminar una columna que también se usaba en un objeto de calendario.
- Se corrigió la lista desplegable de columnas del editor de relaciones para que muestre automáticamente los nombres completos de las columnas.
- Cuando se duplica una tabla que contiene uno o varios calendarios, ahora los calendarios se renombran automáticamente para garantizar nombres únicos en todo el modelo.
- Varios errores/advertencias detectados durante una operación de actualización de datos ya no deberían provocar una excepción no controlada.
- Se corrigió un problema por el que la lista de paquetes DAX mostraba información desactualizada después de ejecutar "Update All".
- Se corrigió un error por el que la propiedad `SchemaName` de las particiones de entidad no se podía deshacer.
- La acción de código [DI005 - Reescribir el filtro de tabla como predicado escalar](https://docs.tabulareditor.com/kb/DI005.html) ya no se sugiere cuando se produce la [expansión de tablas](https://www.sqlbi.com/articles/expanded-tables-in-dax/), ya que reescribir ese tipo de filtro como un filtro de columna no necesariamente produciría el mismo resultado.
- Al hacer clic en el botón de puntos suspensivos junto a la propiedad Cadena de conexión, ya no debería producirse una excepción no controlada cuando la cadena de conexión especifique un proveedor OLE DB que no esté instalado en el sistema actual.
- Las exportaciones de TMDL ahora excluyen correctamente las particiones de actualización incremental cuando la preferencia correspondiente está habilitada, igual que en JSON.
- Se corrigió un problema por el que, al hacer clic en el botón "Iniciar sesión..." al intentar actualizar o importar desde flujos de datos, se mostraba el siguiente mensaje de error: `This page is not normally shown and could be a sign of a phishing attempt.`

## Correcciones de errores en 3.25.1

- Se corrigieron algunas cadenas que indicaban incorrectamente que las reglas BPA integradas solo estaban disponibles para la Edición Enterprise.

## Vienes de Tabular Editor 2.x?

Mira [este vídeo](https://youtu.be/O4ATwdzCvWc) para hacer un recorrido rápido por las principales características de Tabular Editor 3. Además, no dejes de consultar nuestra [guía de incorporación](https://docs.tabulareditor.com/onboarding/index.html).

**Descripción general de las principales características de Tabular Editor 3:**

- IDE totalmente personalizable, con compatibilidad con varios monitores y Hi-DPI, y con temas
- Nuevo y potente editor de código DAX con autocompletado, comprobación de sintaxis, plegado de código y mucho, mucho más
- \*Modo del área de trabajo, que te permite guardar los cambios en disco y sincronizar simultáneamente los metadatos del modelo con Analysis Services
- \*Previsualiza los datos de la tabla con desplazamiento infinito; crea PivotGrids o escribe consultas DAX para explorar el modelo o probar la lógica de cálculo
- \*Programar actualizaciones de datos
- Actualizar los esquemas de las tablas tanto en los Data source del proveedor como en los orígenes de datos estructurados (sí, incluso para consultas M!)
- Crear diagramas del Data model
- Crear scripts de DAX que te permiten editar varias medidas u otros objetos calculados en un único documento
- Grabar C# Script y guardarlos como macros (antes conocidas como "Custom Actions")
- Integración con el Analizador VertiPaq
- Depurador de DAX
- Integración con el Optimizador de DAX
- Acciones de código para refactorizar fácilmente tu código DAX.

\*=Solo cuando estés conectado a una instancia de Analysis Services o Power BI
