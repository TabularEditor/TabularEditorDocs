## Problemas conocidos

- El Asistente de importación de tablas no llegó a esta versión debido a algunos errores de máxima prioridad que hubo que corregir. ¡Estará disponible lo antes posible! Como solución alternativa, simplemente crea una tabla nueva y especifica la consulta de partición (SQL o M). Luego, haz clic con el botón derecho en la tabla y elige "Actualizar esquema de tabla..." para importar columnas.
- Consulta la [hoja de ruta](https://github.com/TabularEditor/TabularEditor3/issues/12) para obtener más información.

## Correcciones de errores en 3.0.4:

- Al actualizar el Best Practice Analyzer ya no se produce un bloqueo

## Nuevas funciones en 3.0.3:

- Al hacer clic con el botón derecho en un elemento de la Vista de procesamiento, ahora puedes poner de nuevo ese elemento en cola o, si el elemento representa una sola tabla, abrir una vista previa de los datos de esa tabla.

## Correcciones de errores en 3.0.3:

- Cuando el analizador semántico detecta un error en una columna calculada o una tabla calculada, ya no modifica las columnas de esa tabla (antes esto hacía que, en algunas circunstancias, se perdieran relaciones)
- Se corrigió un problema con el estado de habilitación de ciertos controles en el cuadro de diálogo Buscar/Reemplazar
- Varias correcciones basadas en la telemetría y en los Reports de errores, como:
  - Ahora funciona copiar y pegar columnas con etiquetas de linaje
  - Pegar una tabla sobre otra existente cuando la tabla se renombra ya no provoca un bloqueo
  - El botón "Revertir" ahora vuelve a conectarse a AS antes de actualizar el TOM local, en lugar de provocar un bloqueo
  - Al hacer clic con el botón derecho en la cuadrícula de propiedades cuando está vacía, ya no se produce un cierre inesperado
  - Y muchos, muchos más...
- Se corrigió un problema por el que los MRU no se guardaban correctamente en la lista desplegable "Servidor"
- Se corrigieron varios problemas por los que el modo del área de trabajo no cargaba correctamente, desde el archivo de opciones de usuario, los valores de anulación del Data source. Consulta [#28](https://github.com/TabularEditor/TabularEditor3/issues/28).
- Se corrigieron y mejoraron varios errores semánticos "fantasma" de DAX (compatibilidad con ROLLUPADDISSUBTOTAL, COUNTROWS con 0 argumentos, etc.)
- Se mejoraron el autocompletado / calltips para varias funciones DAX (TOPN, TOPNPERLEVEL, parámetro de intervalo para DATEADD/DATESINPERIOD/PARALLELPERIOD), tras la palabra clave RETURN, etc.

## ¿Vienes de Tabular Editor 2.x?

Mira [este vídeo](https://www.youtube.com/watch?v=pt3DdcjfImY) para hacerte una idea de las nuevas funciones de Tabular Editor 3.

**Descripción general de las características principales de Tabular Editor 3:**

- IDE totalmente personalizable, con compatibilidad con varios monitores, Hi-DPI y temas
- Nuevo y potente editor de código DAX con autocompletado, comprobación de sintaxis, plegado de código y mucho, mucho más
- \*Modo del área de trabajo, que te permite guardar los cambios en disco y sincronizar los metadatos del modelo con Analysis Services al mismo tiempo
- \*Previsualiza los datos de las tablas con desplazamiento infinito, crea PivotGrids o escribe consultas DAX para explorar el modelo o probar la lógica de cálculo
- \*Actualiza los esquemas de las tablas tanto en Data sources de proveedor como en orígenes de datos estructurados (sí, incluso para consultas M!)
- \*Programa actualizaciones de datos
- Crea diagramas de Data model
- Crea scripts DAX que te permiten editar varias medidas u otros objetos calculados en un único documento
- Graba C# Scripts y guárdalos como macros (antes conocidos como "Acciones personalizadas")
- Integración con el Analizador VertiPaq

\*=Solo cuando estés conectado a una instancia de Analysis Services o Power BI
