## Mejoras en 3.18.1

- El analizador semántico ya no muestra un error al usar el nombre de una tabla como nombre de variable. **Nota:** en el momento de redactar esto, la posibilidad de usar nombres de tabla para variables está disponible en Power BI Desktop (octubre de 2024) y en el Power BI Service. Puedes controlar el comportamiento de los errores en **Herramientas > Preferencias > Editor de DAX > Configuración de DAX > Variables con nombre de tabla**.
- Hemos actualizado las bibliotecas cliente AMO/TOM a la versión más reciente [19.86.6](https://www.nuget.org/packages/Microsoft.AnalysisServices/19.86.6). Esto debería corregir el problema **Can't obtain account information for '(e-mail)' while trying to refresh the token.** que algunos usuarios han experimentado al conectarse al punto de conexión XMLA de Power BI; consulta [#1387](https://github.com/TabularEditor/TabularEditor3/issues/1387).
- Una nueva opción en **Herramientas > Preferencias > Power BI** te permite especificar el modo de autenticación predeterminado que se usará al conectarte a Workspaces de Power BI/Fabric o a Azure Analysis Services. Así puedes, por ejemplo, establecer `Microsoft Entra MFA` como valor predeterminado, haciendo que aparezca el selector de cuentas/ventana emergente de MFA al conectarte.
- Para las licencias multiusuario, ahora mostramos una opción para cambiar la clave de licencia si se ha superado el número de usuarios incluidos en la licencia. De este modo, ya no tienes que restablecer manualmente la clave de licencia [a través del Registro de Windows](https://docs.tabulareditor.com/te3/getting-started.html#changing-a-license-key-through-the-registry).

## Correcciones de errores en 3.18.1

- Al intentar conectarte al servicio del Optimizador de DAX con una cuenta de Tabular Tools que aún no se ha configurado por completo, ahora mostramos un mensaje de error más descriptivo, en lugar de generar una excepción no controlada.
- Se corrigió un error por el que los archivos VPAX creados en Tabular Editor 3 se reportaban como dañados al abrirlos en el servicio del Optimizador de DAX.
- Se corrigió un error por el que se cargaba un archivo VPAX vacío en el servicio Optimizador de DAX si la recopilación de estadísticas de VPAX fallaba durante el proceso de carga.
- Se corrigió un error por el que las características del motor semántico no siempre se actualizaban para el **Editor de expresiones** al abrir un modelo nuevo (lo que podía hacer que aparecieran errores falsos en el editor).
- El Analizador semántico ya no debería provocar la excepción "An item with the same key has already been added." en modelos con expresiones DAX complejas y anidadas; consulta [#1390](https://github.com/TabularEditor/TabularEditor3/issues/1390) y [#1393](https://github.com/TabularEditor/TabularEditor3/issues/1393).
- El Analizador semántico ahora infiere correctamente el tipo de datos de una expresión como `SELECTEDVALUE('MyTable'[String Column], BLANK())` (en este caso, el tipo de datos resultante debería ser **String**, pero antes se infería como **Variant**).
- Se corrigió un error por el que las acciones de código **Simplificar variable** generaban código que no era semánticamente equivalente al original. Además, esta acción ya no provocará un "bucle" de acciones de código aplicadas; consulta [#1386](https://github.com/TabularEditor/TabularEditor3/issues/1386).

## Novedades de 3.18.0

Consulta nuestro [blog de la versión](https://blog.tabulareditor.com/2024/10/31/tabular-editor-3-october-2024-release) para ver un resumen rápido de las actualizaciones más importantes de esta versión.

- Nuestra **integración del Optimizador de DAX** ya no está en versión preliminar y, además, hemos realizado algunas actualizaciones para que ahora sea compatible con las [nuevas métricas de evaluación](https://www.tabulartools.com/blog/introducing-new-evaluation-metrics-in-dax-optimizer-1-2/) incluidas en el Optimizador de DAX 1.2. Y, por cierto: ahora todos los usuarios de Tabular Editor 3 de la **Edición Enterprise** reciben 4 ejecuciones gratuitas del Optimizador de DAX al día. Descarga Tabular Editor 3.18.0 desde uno de los enlaces anteriores y obtén tu código de canje del Optimizador de DAX a través del elemento de menú **Ayuda > Acerca de Tabular Editor**.
- Nos alegra presentar una nueva característica en todos los Editores de DAX de Tabular Editor 3: **Acciones de código**. Esta característica de productividad ofrece de forma discreta sugerencias para mejorar tu código DAX y te permite aplicar la sugerencia con un solo clic. Por ejemplo, puedes usar esta característica para quitar variables sin usar, seguir prácticas recomendadas para referencias de columnas y de medida, reescribir filtros de tabla como [filtros de columna más eficientes](https://www.sqlbi.com/articles/filter-columns-not-tables-in-dax/) y [mucho, mucho más](xref:code-actions). Planeamos añadir acciones adicionales en próximas versiones, ¡así que no te lo pierdas!
- Después de actualizar a la [versión más reciente de la biblioteca cliente Microsoft AMO/TOM](https://www.nuget.org/packages/Microsoft.AnalysisServices/), el comportamiento de la opción de autenticación **Integrada** cambió, de modo que ya no se muestra la ventana emergente del selector de cuentas/MFA de Entra ID si el sistema Windows ya está conectado a una cuenta de Entra ID. Por ello, hemos añadido una nueva opción de autenticación, **Microsoft Entra MFA**, que forzará la aparición de la ventana emergente del selector de cuentas/MFA, lo que te permitirá elegir una cuenta distinta de la que tienes iniciada en Windows, si lo necesitas.

## Mejoras en 3.18.0

- Las preferencias de la interfaz de usuario se han ampliado para permitir ocultar los detalles del origen del modelo en la barra de título, lo que ayuda a proteger información confidencial durante grabaciones y presentaciones.
- Las preferencias de la interfaz de usuario ahora incluyen una nueva sección de «Elementos recientes». Esta sección permite personalizar el número de archivos y modelos recientes que se muestran en el menú Archivo, así como los servidores que se muestran en las conexiones recientes. Las nuevas preferencias también permiten borrar estos elementos.
- El Analizador semántico ahora muestra una advertencia si una variable se usa directamente como primer argumento de [`CALCULATE`](https://dax.guide/CALCULATE) o [`CALCULATETABLE`](https://dax.guide/CALCULATETABLE). Como las variables no se ven afectadas por la transición de contexto ni por contextos de filtro modificados, lo más probable es que usar una variable de esta forma sea involuntario.
- Nuestro editor SQL ahora usa un esquema de colores mejorado, lo que facilita distinguir las palabras clave de SQL de los identificadores.
- Se actualizaron AMO/TOM y otras dependencias a sus versiones más recientes.
- Ahora usamos el nivel de compatibilidad 1606 para los nuevos modelos semánticos de Power BI / Fabric. En este nivel, está disponible una nueva propiedad del modelo, [ValueFilterBehavior](https://learn.microsoft.com/en-us/dotnet/api/microsoft.analysisservices.tabular.valuefilterbehaviortype?view=analysisservices-dotnet), que se puede usar para controlar el comportamiento de _auto-exist_ de [`SUMMARIZECOLUMNS`](https://dax.guide/SUMMARIZECOLUMNS).
- Hemos añadido compatibilidad con la colección TOM [ChangedProperties](https://learn.microsoft.com/en-us/dotnet/api/microsoft.analysisservices.tabular.changedproperty?view=analysisservices-dotnet) (en tablas, medidas, columnas y algunos otros objetos). Esta colección se usa en modelos compuestos o Direct Lake para indicar que una propiedad (como el **Nombre** de una columna) ha cambiado y ya no debe sincronizarse desde el origen subyacente.
- Actualizamos el Analizador VertiPaq a la versión más reciente [(1.7.2)](https://github.com/sql-bi/VertiPaq-Analyzer/releases/tag/v1.7.2), que corrige un problema al extraer estadísticas de un modelo que contiene parámetros de campo / columnas de «Agrupar por».
- Actualizamos nuestra versión del conector .NET de Snowflake a la v4.1.0. Esto corrige las siguientes vulnerabilidades: [CVE-2023-34230](https://nvd.nist.gov/vuln/detail/CVE-2023-34230) y [CVE-2023-51662](https://nvd.nist.gov/vuln/detail/CVE-2023-51662).
- El componente de diagrama ahora debería ser considerablemente más rápido al trabajar con diagramas grandes con muchas relaciones.
- Ahora se pueden ocultar columnas en la vista previa de datos haciendo clic con el botón derecho en el encabezado de la columna y seleccionando «Ocultar esta columna». Para mostrar las columnas ocultas, haz clic con el botón derecho en el encabezado de una columna y selecciona «Selector de columnas». Se abrirá una ventana desde la que podrás acceder a las columnas ocultas anteriormente.

## Correcciones de errores en 3.18.0

- Corregimos un error por el que la lista desplegable «Servidor» del cuadro de diálogo **Cargar modelo semántico desde la base de datos** no mostraba los servidores usados recientemente.
- Configuración de la vista previa de datos (ordenación, filtros, etc.) ahora se conservan al actualizar la vista previa, lo que soluciona un error anterior por el que no se mantenían las opciones.
- Se corrigió un bloqueo que podía producirse al deshacer (Ctrl+Z) durante o justo después de usar la refactorización "Rename" en el Editor de DAX.
- La interfaz de usuario principal ahora queda correctamente bloqueada para evitar interacciones del usuario mientras se muestra el cuadro de diálogo "Please wait", por ejemplo, al realizar una actualización del esquema de una tabla.
- El Analizador semántico ya no debería mostrar una advertencia al hacer referencia a la medida interna "FormatString" en una expresión DAX. Como consecuencia, ahora es posible depurar consultas que contengan este tipo de referencias (aunque, por el momento, la propia expresión FormatString no se puede depurar); consulta [#1377](https://github.com/TabularEditor/TabularEditor3/issues/1377).

## Problemas conocidos en 3.18.0

- Si en Mensajes te aparece el mensaje de error **Can't obtain account information for '(e-mail)' while trying to refresh the token.** al intentar actualizar el token, suele ayudar repetir la operación. Es un problema de las versiones más recientes de las bibliotecas cliente AMO/TOM. También puedes evitar este problema eligiendo la opción de autenticación **Microsoft Entra MFA**, en lugar de la opción **Integrada**, al conectarte por primera vez.

## ¿Vienes de Tabular Editor 2.x?

Mira [este vídeo](https://youtu.be/O4ATwdzCvWc) para ver un recorrido rápido por las principales características de Tabular Editor 3. Además, no dejes de consultar nuestra [guía de incorporación](https://docs.tabulareditor.com/onboarding/index.html).

**Resumen de las principales características de Tabular Editor 3:**

- IDE totalmente personalizable, con compatibilidad con varios monitores, Hi-DPI y temas
- Nuevo y potente editor de código DAX con autocompletado, comprobación de sintaxis, plegado de código y mucho, mucho más
- \*Modo del área de trabajo, que te permite guardar los cambios en disco y sincronizar los metadatos del modelo con Analysis Services al mismo tiempo
- \*Previsualiza los datos de las tablas con desplazamiento infinito, crea PivotGrids o escribe consultas DAX para explorar el modelo o probar la lógica de cálculo
- \*Programa actualizaciones de datos
- Actualiza los esquemas de tablas tanto en Data sources de proveedor como en orígenes de datos estructurados (sí, ¡incluso para consultas M!)
- Crea diagramas del Data model
- Crea scripts DAX que te permitan editar varias medidas u otros objetos calculados en un único documento
- Graba C# Scripts y guárdalos como macros (antes conocidas como "Custom Actions")
- Integración con el Analizador VertiPaq
- Depurador de DAX
- Integración con el Optimizador de DAX
- Acciones de código para refactorizar fácilmente tu código DAX.

\*=Solo mientras esté conectado a una instancia de Analysis Services o Power BI
