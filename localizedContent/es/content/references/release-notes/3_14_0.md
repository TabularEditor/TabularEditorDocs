### Requisitos previos

Tabular Editor 3.3.0 (y versiones posteriores) usa .NET 6. Es posible que tengas que instalar el [.NET 6.0 Runtime para escritorio](https://dotnet.microsoft.com/en-us/download/dotnet/6.0/runtime) antes de poder iniciar Tabular Editor 3.3.0 (o una versión posterior). A partir de la versión 3.3.1, el instalador de Tabular Editor 3 descargará e instalará estos requisitos previos si faltan.

## Novedades en 3.14.0

Consulta nuestro [blog de lanzamiento](https://blog.tabulareditor.com/2024/02/20/tabular-editor-3-february-2024-release/) para obtener un breve resumen de las actualizaciones más importantes de esta versión.

## Mejoras en 3.14.0

- Las columnas calculadas ahora muestran el glifo "_fx_" en la Vista previa de tabla, por lo que es más fácil distinguirlas de las columnas no calculadas.
- El limitador de filas de consultas DAX ahora se deshabilitará si una consulta DAX usa [`TOPN`](https://dax.guide/TOPN) o [`START AT`](https://dax.guide/st/start-at/), para evitar que interfiera con los resultados devueltos.
- La función de autocompletado de DAX ahora admite cualquier expresión escalar en la cláusula [`ORDER BY`](https://dax.guide/st/order-by/) de una consulta DAX, y además ya no sugerirá columnas que ya se hayan especificado antes en la cláusula [`ORDER BY`](https://dax.guide/st/order-by/).
- Al importar/actualizar tablas en un modelo Direct Lake, ahora añadimos a tablas y columnas las mismas anotaciones que si el modelo se hubiera creado mediante el modelado web de Power BI, lo que garantiza, entre otras cosas, que los modelos entre Workspaces puedan actualizarse sin problemas.
- Compatibilidad con las [funciones `INFO` de DAX](https://powerbi.microsoft.com/en-us/blog/dax-query-view-introduces-new-info-dax-functions/) en el autocompletado del editor. Estas funciones solo se sugieren en una consulta DAX, ya que no se pueden usar en elementos calculados (por ejemplo, columnas o tablas) ni en medidas de un modelo.
- TOM actualizado a [19.76.0](https://www.nuget.org/packages/Microsoft.AnalysisServices.NetCore.retail.amd64), que incluye la versión más reciente de TMDL (Preview-9).
- Con TMDL Preview-9, ahora contamos con un conjunto de nuevas opciones para configurar cómo se serializa el TMDL. Puedes encontrar estas opciones en **Herramientas > Preferencia > Guardar en carpeta** (para los modelos que se hayan guardado previamente como TMDL, puedes cambiar estos ajustes en **Modelo > Opciones de serialización**):

> [!IMPORTANT]
> TMDL sigue siendo una característica en versión preliminar, lo que significa que podrían introducirse cambios incompatibles entre versiones. Si tienes problemas al deserializar un modelo después de actualizar Tabular Editor, revierte a la versión anterior más reciente de Tabular Editor y, a continuación, guarda temporalmente el modelo como un archivo .bim normal antes de volver a serializarlo a TMDL con la versión más reciente de Tabular Editor.

![Configuración de serialización de TMDL](https://docs.tabulareditor.com/images/tmdl-options.png)

> [!NOTE]
> Actualmente no disponemos de opciones para personalizar qué objetos se serializan como archivos individuales al usar TMDL. Si te gustaría ver esta funcionalidad, [háznoslo saber](https://github.com/TabularEditor/TabularEditor3/discussions/1198#discussioncomment-8382137)!

## Correcciones de errores en 3.14.0

- Se corrigió un problema por el que Buscar/Reemplazar buscaba o realizaba reemplazos inesperadamente fuera de la selección actual.
- Se corrigió un problema por el que el autocompletado de DAX no sugería una columna después de un operador binario cuando esa columna ya se había usado previamente en la misma expresión de filtro; consulta [#1156](https://github.com/TabularEditor/TabularEditor3/issues/1156).
- Se corrigió un problema por el que desanclar la **vista de propiedades** provocaba que se restablecieran todos los ajustes de diseño al volver a abrir TE3.
- El Analizador semántico de DAX ahora gestiona correctamente las expresiones [`ORDER BY`](https://dax.guide/st/order-by/) y [`START AT`](https://dax.guide/st/start-at/) en consultas DAX. Además, el autocompletado funciona correctamente dentro de estas cláusulas, y el limitador automático de filas ya no genera DAX no válido cuando está presente [`START AT`](https://dax.guide/st/start-at/). Consulta [#1182](https://github.com/TabularEditor/TabularEditor3/issues/1182). Por último, el formateador de DAX integrado ya no debería bloquearse al dar formato a una consulta que contenga [`START AT`](https://dax.guide/st/start-at/).
- Se corrigió un error por el que la cuadrícula de vista previa de datos parecía admitir la ordenación por más de una columna. En realidad, solo es efectiva la primera columna seleccionada para ordenar, ya que la función [`TOPNSKIP`](https://dax.guide/topnskip) que se usa para consultar la tabla no admite ordenar por más de una columna.
- Se corrigió un error que podía provocar que los documentos se quedaran "bloqueados" al restaurar un diseño desde el menú **Ventana**.
- Se corrigió un error que impedía que se mostraran ciertos tipos de errores en el **Explorador TOM** y en la vista de **mensajes**; consulta [1217](https://github.com/TabularEditor/TabularEditor3/issues/1217).
- Se corrigió un error que podía hacer que la aplicación se quedara colgada durante la activación.
- Se corrigió un problema por el que las propiedades extendidas en las perspectivas no se serializaban al usar la opción (heredada) "Guardar en carpeta"; consulta [#1165](https://github.com/TabularEditor/TabularEditor/issues/1165).
- Al introducir un '.' (punto), ya no debería cerrarse la ventana emergente de autocompletado de DAX al filtrar funciones DAX que tienen un punto en su nombre.

## ¿Vienes de Tabular Editor 2.x?

Mira [este vídeo](https://www.youtube.com/watch?v=pt3DdcjfImY) para hacerte una idea de las nuevas funcionalidades de Tabular Editor 3. Además, no dejes de consultar nuestra [guía de incorporación](https://docs.tabulareditor.com/onboarding/index.html).

**Descripción general de las principales características de Tabular Editor 3:**

- IDE totalmente personalizable, con compatibilidad con varios monitores, Hi-DPI y diferentes temas
- Nuevo y potente editor de código DAX con autocompletado, comprobación de sintaxis, plegado de código y mucho, mucho más
- \*Modo del área de trabajo, que te permite guardar los cambios en disco y sincronizar simultáneamente los metadatos del modelo con Analysis Services
- \*Previsualiza los datos de las tablas con desplazamiento infinito; crea PivotGrids o escribe consultas DAX para explorar el modelo o probar la lógica de cálculo
- \*Programa actualizaciones de datos
- Actualiza los esquemas de tablas tanto en Data sources de proveedor como en orígenes de datos estructurados (sí, incluso para consultas M)
- Crea diagramas del Data model
- Crea scripts DAX que te permitan editar varias medidas u otros objetos calculados en un solo documento
- Graba C# Scripts y guárdalos como macros (antes conocidas como "Acciones personalizadas")
- Integración con el Analizador VertiPaq
- Depurador de DAX

\*=Solo al conectarse a una instancia de Analysis Services o Power BI
