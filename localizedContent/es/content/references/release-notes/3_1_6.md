## Mejoras en 3.1.6

- Mejoras en el Editor de DAX:
  - **Ya está disponible el formato DAX sin conexión**. Los botones y atajos (F6) de "Format DAX" ahora usan el formateador de DAX integrado en lugar de www.daxformatter.com. El formateador integrado incluye algunas opciones de configuración adicionales y también corrige referencias de columnas y medidas que no siguen las prácticas recomendadas, entre otras mejoras. Como el formateador integrado no realiza una solicitud web, también es considerablemente más rápido. Si tienes algún problema con el formateador integrado, puedes desactivarlo habilitando **"Use daxformatter.com instead of built-in formatter"** en **Herramientas > Preferencias > Editor de DAX**.
  - Ahora puedes configurar palabras clave y funciones para que se formateen con distintos usos de mayúsculas y minúsculas, según tus preferencias. Puedes encontrar la configuración en **Herramientas > Preferencias > Editor de DAX > Formato automático**; consulta [#58](https://github.com/TabularEditor/TabularEditor3/issues/58). [Más detalles](xref:personalizing-te3#dax-settings).
  - Se agregó una opción de formato de "Debug comma" para consultas DAX (coloca todas las comas al principio de la línea, para comentar y descomentar subexpresiones fácilmente); consulta [#96](https://github.com/TabularEditor/TabularEditor3/issues/96).
  - Ahora puedes seleccionar una referencia completa a un objeto (medida, columna, etc.) haciendo doble clic en ella (antes, solo se seleccionaba la palabra bajo el cursor).
  - Cuando se selecciona una subexpresión y se introduce un delimitador de apertura (paréntesis o llave), la selección se envuelve automáticamente con el delimitador de cierre correspondiente; consulta [#48](https://github.com/TabularEditor/TabularEditor3/issues/48).
  - Code Assist ya no interfiere al escribir comentarios.
- TE3 ahora puede importar y actualizar esquemas de tablas desde Snowflake (solo para Data model de Power BI).
- TOM actualizado a 19.27.2.1
- Ahora está disponible el método `SelectObjects` de C# Script, que solicita al usuario que seleccione uno o varios objetos de una lista.
- Nueva opción: **Herramientas > Preferencias > Explorador TOM > Mostrar rama completa**. Cuando activas esta opción, al filtrar el Explorador TOM también se mostrarán todos los objetos secundarios, incluso aunque no coincidan con la cadena de filtro. Consulta [#276](https://github.com/TabularEditor/TabularEditor3/issues/276).
- Nueva opción: **Herramientas > Preferencias > Explorador TOM > Mostrar siempre advertencias de eliminación**. Cuando activas esta opción, todas las operaciones de eliminación mostrarán un cuadro de diálogo de confirmación. Cuando la desactivas, solo las eliminaciones con selección múltiple y la eliminación de objetos a los que hacen referencia otros objetos mostrarán el cuadro de diálogo de confirmación (recuerda que todas las operaciones se pueden deshacer en Tabular Editor). Consulta [#10](https://github.com/TabularEditor/TabularEditor3/issues/10).
- Se ha agregado la opción de menú "Aplicar política de actualización" en las tablas, cuando corresponde. Consulta [Configurar la actualización incremental con Tabular Editor](https://www.youtube.com/watch?v=icCGFG6KpIA) para más información; consulta también [#291](https://github.com/TabularEditor/TabularEditor3/issues/291).
- Se ha agregado la vista de dependencias de DAX al menú **Ver**, y una casilla que permite que la vista de dependencias siga la selección actual en el Explorador TOM. Consulta [#10](https://github.com/TabularEditor/TabularEditor3/issues/10).

## Correcciones de errores en 3.1.6

- Los cuadros de diálogo Preferencias, Seleccionar elementos y Guardar/Editar macro ahora se pueden cambiar de tamaño. Consulta [#51](https://github.com/TabularEditor/TabularEditor3/issues/51).
- Cuando se realiza una operación de "Actualizar esquema de tabla" en un modelo de Power BI Desktop que usa consultas nativas para la importación de datos, se traerá la instancia de Power BI Desktop al frente, ya que en Desktop aparece un cuadro de diálogo que debe aceptarse antes de que la operación de "Actualizar esquema de tabla" pueda completarse. Consulta la incidencia [#55](https://github.com/TabularEditor/TabularEditor3/issues/55).
- El icono de la barra de tareas ya no debería dejar de funcionar al actualizar. Consulta [#163](https://github.com/TabularEditor/TabularEditor3/issues/163).
- Al actualizar esquemas de tabla, ahora verás un cuadro de diálogo de conexión que te permite cambiar el modo de autenticación. Consulta [#262](https://github.com/TabularEditor/TabularEditor3/issues/262).
- La propiedad de miembro de rol `MemberId` ahora se elimina automáticamente solo al implementar un modelo en Azure AS o Power BI. Consulta [#263](https://github.com/TabularEditor/TabularEditor3/issues/263).
- Se corrigió un problema que bloqueaba la interfaz de usuario al poner en cola varias operaciones de actualización. Consulta [#281](https://github.com/TabularEditor/TabularEditor3/issues/281).
- El selector de objetos de AAD ahora debería generar un nombre de objeto bien formado. Consulta [#286](https://github.com/TabularEditor/TabularEditor3/issues/286).
- Al ejecutar un script DAX que crea un grupo de cálculo, ahora los ordinales se asignan correctamente. Consulta [#296](https://github.com/TabularEditor/TabularEditor3/issues/296).
- Ahora puedes usar CTRL + flechas para expandir/contraer todos los nodos en el Explorador TOM, al igual que en TE 2.x. Consulta [#297](https://github.com/TabularEditor/TabularEditor3/issues/297).
- Al ejecutar un script DAX que crea un grupo de cálculo, ahora se asigna correctamente el nombre de la columna; consulta [#298](https://github.com/TabularEditor/TabularEditor3/issues/298).
- Se corrigió un error por el que las configuraciones regionales de los scripts DAX no se actualizaban correctamente al alternar entre el modo EE. UU. y el modo no EE. UU.
- Se corrigió un bloqueo que se producía al intentar conectarse a un servidor de AS que contiene bases de datos sin nivel de compatibilidad (ocurre al conectarse a una instancia de AS tabular que comparte su directorio de datos con una instancia de AS multidimensional).
- Se corrigió un error que, a veces, hacía que la función "Actualizar esquema de tabla" generara un Report en el que se indicaba que faltaban todas las columnas en el origen.
- Se corrigió un problema con la función "Vista previa de tabla", que provocaba que las columnas de la cuadrícula se configuraran incorrectamente al usar la suplantación de identidad.
- Se corrigió un problema por el que la barra de notificación de cambios semánticos no desaparecía al guardar el modelo.
- Se corrigió un problema por el que las ediciones en la cuadrícula de propiedades no se confirmaban automáticamente al guardar el modelo.

## Vienes de Tabular Editor 2.x?

Mira [este vídeo](https://www.youtube.com/watch?v=pt3DdcjfImY) para hacerte una idea de las nuevas funcionalidades de Tabular Editor 3.

**Descripción general de las principales características de Tabular Editor 3:**

- IDE totalmente personalizable, con compatibilidad con varios monitores, Hi-DPI y temas
- Nuevo y potente editor de código DAX con autocompletado, comprobación de sintaxis, plegado de código y mucho, mucho más
- \*Modo del área de trabajo, que te permite guardar los cambios en el disco y sincronizar los metadatos del modelo con Analysis Services al mismo tiempo
- \*Previsualiza los datos de la tabla con desplazamiento infinito, crea PivotGrids o escribe consultas DAX para explorar el modelo o probar la lógica de cálculo
- \*Programa actualizaciones de datos
- Actualiza los esquemas de tabla tanto en Data sources de proveedor como en orígenes de datos estructurados (sí, ¡incluso para consultas M!)
- Crea diagramas del Data model
- Crea scripts DAX que te permitan editar varias medidas u otros objetos calculados en un solo documento
- Graba C# Scripts y guárdalos como macros (antes conocidas como "Custom Actions")
- Integración con el Analizador VertiPaq

\*=Solo al estar conectado a una instancia de Analysis Services o Power BI
