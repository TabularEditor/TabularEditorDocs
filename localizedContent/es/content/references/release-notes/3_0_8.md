## Problemas conocidos

- El Asistente para importar tablas no se incluyó en esta versión debido a algunos errores de alta prioridad que había que corregir. ¡Estará disponible lo antes posible! Como solución alternativa, crea una nueva tabla y especifica la consulta de la partición (SQL o M). Después, haz clic con el botón derecho en la tabla y elige "Actualizar esquema de tabla..." para importar las columnas.
- Consulta la [hoja de ruta](https://github.com/TabularEditor/TabularEditor3/issues/12) para obtener más información.

## Mejoras en 3.0.8

- Se agregó una opción para ejecutar solo las instrucciones EVALUATE seleccionadas en consultas DAX (atajo: Mayús+F5). Consulta el problema [#47](https://github.com/TabularEditor/TabularEditor3/issues/47).
- Ahora puedes usar el atajo Ctrl+/ para comentar o quitar comentarios del código. Consulta el problema [#46](https://github.com/TabularEditor/TabularEditor3/issues/46). También puedes cambiar el estilo de comentario (-- o //) en Herramientas > Preferencias > Editor de DAX > General.
- Script DAX ahora también incluirá en el script los KPI de las medidas, las columnas calculadas y las tablas calculadas de la selección actual. También agregamos comentarios para que el código sea más legible (se puede desactivar en Herramientas > Preferencias > Script DAX).
- Ahora hay una opción para agregar automáticamente un carácter de salto de línea al comienzo de todas las expresiones DAX al trabajar con un modelo de Power BI. Esto hace que el código se vea mejor en la barra de fórmulas de Power BI Desktop. Se puede desactivar en Herramientas > Preferencias >; consulta el problema [#26](https://github.com/TabularEditor/TabularEditor3/issues/26).
- Al agregar miembros del rol de AD de Windows, ahora aparece el cuadro de diálogo Selector de objetos de directorio. Consulta el problema [#152](https://github.com/TabularEditor/TabularEditor3/issues/152).
- La opción "Agregar metadatos de implementación" ahora está disponible en Herramientas > Preferencias > Implementación del modelo.
- Mejoras en Pivot Grid:
  - Ahora puedes arrastrar objetos del Explorador TOM a todas las áreas del Pivot Grid; consulta [#49](https://github.com/TabularEditor/TabularEditor3/issues/49).
  - Se añadió una opción en el menú contextual del Explorador TOM para agregar medidas, columnas y jerarquías directamente al Pivot Grid.
  - Se añadió el botón "Mostrar campo" en Pivot Grid para mostrar todos los campos disponibles en la base de datos conectada.
  - Se añadió una opción del menú contextual "Ir al objeto" en el Pivot Grid.
- Errores de relación (rutas ambiguas, estado no válido, etc.) ahora se detectan y se indican en el Explorador TOM

## Correcciones de errores en 3.0.8

- Se cambió el color del signo de exclamación de "Ejecutar consulta" de rojo a verde.
- Se corrigió un bloqueo al invertir relaciones desde la Vista de diagrama
- Se corrigió un problema en el Editor de relaciones por el que los cambios no se guardaban correctamente
- Se corrigió un bloqueo que se producía al iniciar TE3 desde PBI Desktop mientras se ejecutaba una instancia de SSAS en la máquina local.
- Se corrigió un error que impedía crear nuevos modelos con una base de datos del Workspace en el punto de conexión XMLA de Power BI.
- Se corrigió un error por el que la selección de código se borraba tras usar la acción "Comentar".
- El Asistente de implementación ahora recuerda el último servidor utilizado.
- Las expresiones de columnas calculadas del Script DAX ya no deberían mostrar mensajes de error semántico falsos sobre la falta de contexto de fila.
- Se corrigió un error que impedía a los titulares de licencias de Edición Business usar el servidor de Workspace Integrada de Visual Studio.
- Se corrigió la errata en "successful".
- Se corrigió un problema por el que el formateador automático a veces duplicaba código al introducir un paréntesis de cierre.
- Se añadió la opción "Guardar copia de seguridad de Model.bim" en la interfaz de usuario; consulta el problema [#37](https://github.com/TabularEditor/TabularEditor3/issues/37).
- La selección de perspectiva ya no debería quedarse "atascada"; consulta el problema [#52](https://github.com/TabularEditor/TabularEditor3/issues/52).
- La cuadrícula de propiedades ahora muestra correctamente ejemplos de formato de fecha; consulta el problema [#86](https://github.com/TabularEditor/TabularEditor3/issues/86).
- Se corrigió el color de resaltado al buscar; consulta el problema [#98](https://github.com/TabularEditor/TabularEditor3/issues/98).
- "Buscar todo" ahora funciona como se esperaba; consulta el problema [#107](https://github.com/TabularEditor/TabularEditor3/issues/107).
- Ahora se detecta correctamente el elemento de cálculo que hacía que los tipos de datos de las medidas se convirtieran en Variant; consulta la incidencia [#111](https://github.com/TabularEditor/TabularEditor3/issues/111).
- Se corrigió un falso error semántico al usar EARLIER en una función FILTER; consulta la incidencia [#118](https://github.com/TabularEditor/TabularEditor3/issues/118).
- Se corrigió un falso error semántico al usar KEEPFILTERS en una función FILTER; consulta la incidencia [#122](https://github.com/TabularEditor/TabularEditor3/issues/122).
- Una expresión de filas de detalle en blanco (o solo con comentarios) ya no provoca una indicación de error; consulta la incidencia [#127](https://github.com/TabularEditor/TabularEditor3/issues/127).
- Se corrigieron los avisos repetidos de credenciales; consulta la incidencia [#131](https://github.com/TabularEditor/TabularEditor3/issues/131).
- Se corrigió un problema al importar vpax; consulta la incidencia [#133](https://github.com/TabularEditor/TabularEditor3/issues/133).
- Se corrigió el comportamiento de la cuadrícula de propiedades al seleccionar una carpeta de visualización; consulta la incidencia [#134](https://github.com/TabularEditor/TabularEditor3/issues/134).
- El diagrama ya no elimina la tabla al pulsar "Supr" (ahora solo la quita del diagrama); consulta la incidencia [#139 ](https://github.com/TabularEditor/TabularEditor3/issues/139).
- Pivot Grid ahora gestiona correctamente los campos de fecha; consulta la incidencia [#142](https://github.com/TabularEditor/TabularEditor3/issues/142).
- Al copiar un grupo de cálculo, ya no se crearán medidas con nombres duplicados y el número de precedencia también se modificará para garantizar su unicidad; consulta las incidencias [#155](https://github.com/TabularEditor/TabularEditor3/issues/155) y [#156](https://github.com/TabularEditor/TabularEditor3/issues/156).
- El botón "Descomentar código" ahora funciona con líneas que tienen sangría; consulta la incidencia [#159](https://github.com/TabularEditor/TabularEditor3/issues/159).
