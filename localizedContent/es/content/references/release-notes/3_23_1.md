## Novedades de 3.23.0

Echa un vistazo a nuestro [blog de lanzamientos](https://blog.tabulareditor.com/) para ver un breve resumen de las actualizaciones más importantes de esta versión.

- Tabular Editor 3 ahora admite **funciones DAX definidas por el usuario (UDF)**. Se trata de una funcionalidad importante, y no podemos hacerle justicia en unas pocas viñetas. Consulta la [sección específica de la documentación](xref:udfs) para obtener más información.
- Por si una gran novedad no fuera suficiente, en esta versión también hemos añadido compatibilidad con **calendarios**, incluido el soporte en el analizador y el editor de DAX para 8 nuevas funciones de DAX relacionadas con la inteligencia temporal basada en semanas. De nuevo, consulta la [sección específica de la documentación](xref:calendars) para obtener más información.

## Mejoras en 3.23.1

- Hemos añadido el nivel de compatibilidad 1700 (SQL Server 2025) al menú desplegable del cuadro de diálogo **Nuevo modelo**. Además, el nivel de compatibilidad predeterminado para los modelos de Power BI se ha actualizado a 1702 (lo cual es necesario para las UDF y los calendarios).

## Correcciones de errores en 3.23.1

- Esta actualización corrige varios errores relacionados con UDF y calendarios, en comparación con 3.23.0. La mayoría están relacionados con nuestro analizador DAX integrado, pero también hay varios ajustes y mejoras de estabilidad para cosas como el autocompletado, el formato, etc.
- Hemos bajado la versión de la biblioteca MSAL (Microsoft.Identity.Client) a [4.72.1](https://www.nuget.org/packages/Microsoft.Identity.Client/4.72.1), ya que las versiones más recientes de esta biblioteca (usadas en la versión anterior de Tabular Editor 3) no funcionan bien con la biblioteca cliente de AS, [provocando que la aplicación se quede colgada al usar la opción de autenticación de Microsoft Entra ID](https://github.com/TabularEditor/TabularEditor3/issues/1485).
- Hemos eliminado las acciones **Duplicar** y **Cortar/Copiar/Pegar/Eliminar** para las columnas de tablas calculadas, ya que estos objetos no se pueden añadir ni quitar manualmente de las tablas, y hacerlo provocaría problemas al guardar los cambios en Analysis Services / Power BI. En su lugar, modifica el DAX de la tabla calculada.
- Los catálogos reflejados de Azure Databricks ahora también se muestran en la IU del Asistente para importar tablas al elegir la opción **Base de datos reflejada de Fabric**.
- Hemos corregido un problema por el que, al abrir un modelo creado con versiones antiguas de Power BI Desktop, aparecía el error "The given key 'Extended Properties' was not present in the dictionary".
- Hemos corregido un error que impedía las referencias adelantadas en las propiedades de los objetos de script DAX. Es decir. ya no aparece un error si la propiedad `FormatString` de una medida hace referencia a una medida definida más adelante en el script.

## ¿Vienes de Tabular Editor 2.x?

Mira [este vídeo](https://youtu.be/O4ATwdzCvWc) para hacerte un recorrido rápido por las principales funciones de Tabular Editor 3. Además, no dejes de consultar nuestra [guía de incorporación](https://docs.tabulareditor.com/onboarding/index.html).

**Resumen de las principales características de Tabular Editor 3:**

- IDE totalmente personalizable, con compatibilidad con varios monitores, Hi-DPI y temas
- Nuevo y potente editor de código DAX con autocompletado, comprobación de sintaxis, plegado de código y mucho, mucho más
- \*Modo del área de trabajo, que te permite guardar los cambios en disco y sincronizar simultáneamente los metadatos del modelo con Analysis Services
- \*Previsualiza los datos de las tablas con desplazamiento infinito, crea PivotGrids o escribe consultas DAX para explorar el modelo o probar la lógica de cálculo
- \*Programa actualizaciones de datos
- Actualiza los esquemas de las tablas tanto en Data sources de proveedor como en orígenes de datos estructurados (sí, incluso para consultas M!)
- Crea diagramas del modelo de datos
- Crea scripts de DAX que te permitan editar varias medidas u otros objetos calculados en un solo documento
- Graba C# Scripts y guárdalos como macros (antes conocidas como "Custom Actions")
- Integración con el Analizador VertiPaq
- Depurador de DAX
- Integración con el Optimizador de DAX
- Acciones de código para refactorizar fácilmente tu código DAX.

\*=Solo mientras estés conectado a una instancia de Analysis Services o Power BI
