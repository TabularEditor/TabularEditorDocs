## ¡Importante!

Tabular Editor 3 ahora usa [.NET 6](https://docs.microsoft.com/en-us/dotnet/core/whats-new/dotnet-6) (hasta ahora veníamos usando .NET Framework 4.8.2). Con .NET 6, Microsoft ha proporcionado una plataforma unificada y moderna para el desarrollo de aplicaciones, que incluye varias mejoras de rendimiento y actualizaciones de seguridad. Como usuario de Tabular Editor 3, no deberías notar ninguna diferencia, salvo un tamaño de descarga ligeramente mayor y posibles mejoras de rendimiento.

**NOTA**: Es posible que tengas que instalar el [runtime de .NET 6,0 para escritorio](https://dotnet.microsoft.com/en-us/download/dotnet/6.0/runtime) (versión 6,0) antes de poder iniciar Tabular Editor 3.3.0. Asegúrate de descargar el runtime de **escritorio** que coincida con tu versión de Tabular Editor 3:

- [.NET Desktop Runtime x64](https://dotnet.microsoft.com/en-us/download/dotnet/thank-you/runtime-desktop-6.0.6-windows-x64-installer)
- [.NET Desktop Runtime x86](https://dotnet.microsoft.com/en-us/download/dotnet/thank-you/runtime-desktop-6.0.6-windows-x86-installer)

## Mejoras en 3.3.0

- TOM actualizado a [19.39.2.2](https://www.nuget.org/packages/Microsoft.AnalysisServices.NetCore.retail.amd64/)
- Tabular Editor 3 ahora usa [Roslyn](https://github.com/dotnet/roslyn) para compilar C# Scripts. Esto significa que tus scripts de C# ahora pueden usar todas las fantásticas nuevas características del lenguaje de [C# 10,0](https://docs.microsoft.com/en-us/dotnet/csharp/whats-new/csharp-10). Además, puedes definir clases y otros tipos directamente en tus scripts. Y por si fuera poco, por fin tenemos **autocompletado de código** e **información de parámetros** con tecnología de Roslyn para C# Scripts (ver captura).
  ![IntelliSense de C#](~/content/assets/images/csharp-intellisense.png)
- En la misma línea, asegúrate de echarle un vistazo a nuestro nuevo sitio de documentación de la [API de scripting](xref:api-index), donde encontrarás documentación actualizada de todos los objetos y miembros a los que puedes acceder a través de la característica de C# Script de Tabular Editor 3.
- También hemos mejorado mucho la función de autocompletado en nuestros editores de DAX, que ahora mostrará descripciones emergentes con más información sobre los objetos de la lista de autocompletado. La información sobre herramientas mostrará la `Description` de medidas, columnas, etc. Si no se especifica ninguna descripción, mostraremos las primeras 10 líneas de código DAX del objeto:
  ![Dax Intellisense Improv](~/content/assets/images/dax-intellisense-improv.png)
- Al abrir un modelo desde Analysis Services o el punto de conexión XMLA de Power BI, ahora ofrecemos opciones para ayudarte a evitar hacer cambios en el modelo por accidente, lo cual es útil cuando tienes varias instancias de Tabular Editor 3 abiertas. La lista desplegable **Modo** (consulta la captura de pantalla a continuación) te permite abrir un modelo en modo de solo lectura o de solo lectura/solo actualización. La lista desplegable **Color de la barra de estado** te permite establecer un color para la barra de estado de Tabular Editor 3, de modo que puedas distinguir fácilmente, por ejemplo, modelos de producción, prueba y desarrollo que estén abiertos a la vez en distintas instancias de Tabular Editor. Consulta [#558](https://github.com/TabularEditor/TabularEditor3/issues/558).
  ![New Open Fromdb Options](~/content/assets/images/new-open-fromdb-options.png)
- Por fin hemos añadido la opción del menú contextual "Duplicar objeto" (Ctrl+D) en el Explorador TOM; consulta [#501](https://github.com/TabularEditor/TabularEditor3/issues/501).
- Se añadió compatibilidad con OAuth de Snowflake y la autenticación ExternalBrowser; consulta [#546](https://github.com/TabularEditor/TabularEditor3/issues/546).
- La cuadrícula de propiedades ahora tiene una barra de búsqueda.
- Se añadieron más opciones de configuración de proxy en **Herramientas** > **Preferencias** > **Configuración del proxy**.
- Se actualizó la biblioteca SQL Native Client a la versión más reciente. Las conexiones a SQL Server ahora requieren cifrado de forma predeterminada (salvo al conectarse a localhost).
- Se añadió una opción **Borrar al actualizar** en los niveles de modelo, tabla y partición; consulta [#511](https://github.com/TabularEditor/TabularEditor3/issues/511).
- Se añadió una casilla para abrir un Data model sin la base de datos del Workspace; consulta [#523](https://github.com/TabularEditor/TabularEditor3/issues/523).

## Correcciones de errores en 3.3.0

- Se corrigió un problema con la forma en que el analizador semántico gestiona la función DAX `NAMEOF`, que a veces podía provocar un bloqueo; consulta [#538](https://github.com/TabularEditor/TabularEditor3/issues/538).
- Las cuadrículas de propiedades y los editores de colecciones ahora usan componentes de DevExpress y se muestran con el tema aplicado y se escalan correctamente en monitores HiDPI; consulta [#548](https://github.com/TabularEditor/TabularEditor3/issues/548).
- Se corrigió un error por el que el submenú **Actualizar modelo** a veces desaparecía del menú **Modelo**.
- "**Actualizar esquema de tabla...**" ahora gestionará correctamente las columnas cuyo nombre solo haya cambiado en mayúsculas/minúsculas.
- TE3 Edición Business ahora impide añadir múltiples particiones o perspectivas a modelos de Analysis Services (lo que evitará que TE3 vuelva a cargar el mismo modelo más adelante debido a las restricciones de la edición).
- Se corrigió un problema por el que las acciones del editor a veces se deshabilitaban; consulta [#519](https://github.com/TabularEditor/TabularEditor3/issues/519).
- Se corrigió un problema por el que los botones **Cancelar** y **Cancelar todo** no se habilitaban después de iniciar una operación de actualización.
- Se corrigió un problema por el que no se mostraba el progreso de actualización al actualizar un modelo de Power BI a través del punto de conexión XMLA.
- Se corrigió un error tipográfico; consulta [#553](https://github.com/TabularEditor/TabularEditor3/issues/553).
- Se corrigió un problema por el que seguían apareciendo las calltips a pesar de haber desmarcado la opción "Mostrar automáticamente la información de parámetros"; consulta el [#545](https://github.com/TabularEditor/TabularEditor3/issues/545).
- Se corrigió un error al comentar/descomentar; consulta el [#536](https://github.com/TabularEditor/TabularEditor3/issues/536).
- Se corrigió un problema con el tipo de retorno incorrecto en CUSTOMDATA(); consulta el [#527](https://github.com/TabularEditor/TabularEditor3/issues/527).
- En la vista previa de datos, al filtrar una columna de texto por "(En blanco)", ahora incluimos tanto BLANK como las cadenas ""; consulta el [#524](https://github.com/TabularEditor/TabularEditor3/issues/524).
- Se corrigió el error "Tabular Editor no pudo abrir el archivo especificado" al intentar abrir un archivo Database.json mediante el cuadro de diálogo genérico para abrir archivos, con "Todos los archivos" seleccionado en la lista desplegable. Consulta también el [#563](https://github.com/TabularEditor/TabularEditor3/issues/563).
- Se corrigió un error por el que la función Guardar no respetaba la opción seleccionada en la casilla de configuración de serialización.
- Se corrigió el color del enlace en la barra de estado. Consulta el [#521](https://github.com/TabularEditor/TabularEditor3/issues/521).
- Se corrigió un bloqueo al intentar borrar trazas huérfanas en el cuadro de diálogo de preferencias.
- Se corrigió un falso positivo de error semántico al comparar con variantes; consulta el [#516](https://github.com/TabularEditor/TabularEditor3/issues/516).
- Se corrigió un problema con la generación de DAX al depurar una asignación de variable que contiene llamadas a las funciones IF/SWITCH; consulta el [#513](https://github.com/TabularEditor/TabularEditor3/issues/513).
- Ya no se permite usar WEEK como parámetro de intervalo para DATEADD. Consulta el [#508](https://github.com/TabularEditor/TabularEditor3/issues/508).
- Se muestra el indicador de error del elemento de cálculo en la tabla del grupo de cálculo; consulta el [#506](https://github.com/TabularEditor/TabularEditor3/issues/506).
- Se corrigió un problema por el que el método `ImportProperties` del C# Script no resolvía correctamente las rutas de los objetos.
- Se agregó un enlace al portal de autoservicio cuando se superó el número de instalaciones. Se actualizaron varias URL y enlaces
- Mejor generación de código DAX al depurar USERELATIONSHIP.
- Ahora puedes desmarcar la preferencia de guardar una copia de seguridad de los metadatos del modelo. Mejoras en los mensajes de error cuando falla la copia de seguridad de los metadatos del modelo. Ahora puedes cancelar la copia de seguridad de los metadatos del modelo.
- Se corrigió un problema con REMOVEFILTERS y FILTER. Consulta el [#562](https://github.com/TabularEditor/TabularEditor3/issues/562).

## Vienes de Tabular Editor 2.x?

Mira [este vídeo](https://www.youtube.com/watch?v=pt3DdcjfImY) para hacerte una idea de las nuevas funciones de Tabular Editor 3. Además, asegúrate de consultar nuestra [guía de primeros pasos](https://docs.tabulareditor.com/onboarding/index.html).

**Descripción general de las principales funciones de Tabular Editor 3:**

- IDE totalmente personalizable, con soporte para varios monitores, Hi-DPI y temas
- Nuevo y potente editor de código DAX con autocompletado, comprobación de sintaxis, plegado de código y mucho, mucho más
- \*Modo del área de trabajo, que te permite guardar los cambios en disco y sincronizar a la vez los metadatos del modelo con Analysis Services
- \*Previsualiza los datos de las tablas con desplazamiento infinito, crea PivotGrids o escribe consultas DAX para explorar el modelo o probar la lógica de cálculo
- \*Programa actualizaciones de datos
- Actualiza los esquemas de tablas tanto en los Data sources de proveedor como en los orígenes de datos estructurados (sí, ¡incluso en consultas M!)
- Crea diagramas del Data model
- Crea scripts DAX que te permiten editar varias medidas u otros objetos calculados en un único documento
- Graba C# Scripts y guárdalos como macros (antes conocidas como "Acciones personalizadas")
- Integración con el Analizador VertiPaq
- \*Depurador de DAX

\*=Solo disponible cuando estés conectado a una instancia de Analysis Services o Power BI
