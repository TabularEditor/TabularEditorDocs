### Requisitos previos

Tabular Editor 3.3.0 (y versiones posteriores) usa .NET 6. Es posible que tenga que instalar el [.NET 6.0 Runtime for Desktop](https://dotnet.microsoft.com/en-us/download/dotnet/6.0/runtime) antes de poder iniciar Tabular Editor 3.3.0 (o una versión posterior). A partir de la versión 3.3.1, el instalador de Tabular Editor 3 descargará e instalará estos requisitos previos si faltan.

## Novedades de 3.4.1

- TOM actualizado a [19.55.3](https://www.nuget.org/packages/Microsoft.AnalysisServices.NetCore.retail.amd64).
- Nueva vista: [Editor de perspectivas](xref:perspective-editor).

## Mejoras menores en 3.4.1

- El Depurador de DAX ahora admite funciones de ventana ([`OFFSET`](https://dax.guide/offset), [`INDEX`](https://dax.guide/index) y [`WINDOW`](https://dax.guide/window)).
- El Depurador de DAX ahora permite depurar una medida a la que se hace referencia en un constructor de tablas, como `EVALUATE { [Internet Sales] }`, o en la función [`ROW`](https://dax.guide/row).
- La vista **Contexto de evaluación** ahora muestra la expresión de las variables de tabla usadas como filtros.
- Restablecer el diseño de la ventana desde el menú **Ventana** ya no afecta a los botones de la barra de herramientas (pueden modificarse o restablecerse desde el cuadro de diálogo **Herramientas > Personalizar**).

## Correcciones de errores en 3.4.1

- Se corrigió un problema con el cuadro de búsqueda situado encima de la lista de comandos de teclado en el cuadro de diálogo de preferencias.
- Los editores de DAX ahora gestionan correctamente los comentarios anidados; consulta [#681](https://github.com/TabularEditor/TabularEditor3/issues/681).
- Al intentar depurar medidas con ámbito de consulta ya no se produce un cierre inesperado.
- Se corrigió un problema que hacía que las líneas de relación en la vista de diagrama se quedaran "atascadas"; consulta [#787](https://github.com/TabularEditor/TabularEditor3/issues/787).
- Al seguir la selección del Explorador TOM en la vista de dependencias, la aplicación ya no debería cerrarse inesperadamente al seleccionar relaciones; consulta [#782](https://github.com/TabularEditor/TabularEditor3/issues/782).
- Se corrigió el error "DataSourceConverter cannot convert from TabularEditor.TOMWrapper.ProviderDataSource" al intentar cambiar o restablecer la propiedad DataSource en una partición.
- Tabular Editor 3 ya no debería reemplazar los espacios en blanco al final de los nombres de archivo por `%20` al usar "Guardar en carpeta".
- El Editor de expresiones ya no debería aparecer vacío hasta que se haga clic en su encabezado; consulta [#739](https://github.com/TabularEditor/TabularEditor3/issues/739).
- El Depurador de DAX ahora simula correctamente el comportamiento de auto-existencia de [`SUMMARIZECOLUMNS`](https://dax.guide/summarizecolumns).
- Se corrigió un problema por el cual la presencia de `ALL()`, `ALLSELECTED()` y otras funciones para quitar filtros no hacía que los filtros externos aparecieran tachados (lo que indica que se habían sobrescrito) en la vista de **Contexto de evaluación**.
- Se corrigió la falta de la opción del menú contextual para actualizar una tabla calculada o un grupo de cálculo.
- Se corrigió un problema que a veces provocaba un cierre inesperado al intentar realizar una actualización de esquema.
- Ahora ya no es posible depurar una medida que se haya quitado del modelo (lo cual, de lo contrario, provocaría un cierre inesperado).
- Mejor gestión de excepciones al pegar objetos en el Explorador TOM.
- Se corrigió un problema que podía provocar un cierre inesperado cuando se eliminaba un objeto mientras se mostraba en la vista de **Resultados de búsqueda**.
- Se corrigió un problema que podía provocar un cierre inesperado en la **Vista previa de tabla** cuando se eliminaban columnas o tablas.
- El Analizador semántico ya no emite un Report de error cuando el parámetro `<Relation>` de una función de ventana se deja en blanco o cuando contiene columnas de extensión. Consulta [#807](https://github.com/TabularEditor/TabularEditor3/issues/807).
- El botón "Restablecer" del cuadro de diálogo Personalizar ya no debería eliminar todos los botones de una barra de herramientas.
- El editor de C# Script ya no debería insertar una tabulación o un carácter de nueva línea adicional al realizar un autocompletado pulsando una de estas teclas.
- Se corrigió un problema al llamar al método `DependsOn.Deep()` mediante un C# Script.
- El editor DAX ya no debería cerrarse inesperadamente al intentar aplicar formato automático al código.

## ¿Vienes de Tabular Editor 2.x?

Mira [este vídeo](https://www.youtube.com/watch?v=pt3DdcjfImY) para hacerte una idea de las nuevas funcionalidades de Tabular Editor 3. Además, asegúrate de consultar nuestra [guía de primeros pasos](https://docs.tabulareditor.com/onboarding/index.html).

**Resumen de las principales características de Tabular Editor 3:**

- IDE totalmente personalizable, con soporte para varios monitores, Hi-DPI y temas
- Nuevo y potente editor de código DAX con autocompletado, comprobación de sintaxis, plegado de código y mucho, mucho más
- \*Modo del área de trabajo, que te permite guardar los cambios en disco y sincronizar simultáneamente los metadatos del modelo con Analysis Services
- \*Previsualiza los datos de la tabla con desplazamiento infinito, crea PivotGrids o escribe consultas DAX para explorar el modelo o probar la lógica de cálculo
- \*Programa actualizaciones de datos
- Actualiza los esquemas de tablas tanto en Data sources de proveedor como en orígenes de datos estructurados (sí, ¡incluso para consultas M!)
- Crea diagramas del Data model
- Crea scripts DAX que te permitan editar varias medidas u otros objetos calculados en un solo documento
- Graba C# Scripts y guárdalos como macros (antes conocidas como "Acciones personalizadas")
- Integración con el Analizador VertiPaq
- \*Depurador de DAX

\*=Solo al estar conectado a una instancia de Analysis Services o Power BI
