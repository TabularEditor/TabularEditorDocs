## Problemas conocidos

- El Asistente para importar tablas no se incluyó en esta versión debido a algunos errores de alta prioridad que había que corregir. ¡Estará disponible lo antes posible! Como solución alternativa, simplemente crea una tabla nueva y especifica la consulta de la partición (SQL o M). Después, haz clic con el botón derecho en la tabla y elige "Actualizar esquema de tabla..." para importar las columnas.
- Consulta la [hoja de ruta](https://github.com/TabularEditor/TabularEditor3/issues/12) para obtener más información.

## Mejoras en 3.0.10

- AMO/TOM actualizado a la versión [19.22.5](https://www.nuget.org/packages/Microsoft.AnalysisServices.retail.amd64/).
- Se permite la búsqueda incremental en los cuadros de diálogo "Seleccionar elemento", por ejemplo, al añadir tablas a un diagrama; consulta el problema [#167](https://github.com/TabularEditor/TabularEditor3/issues/167).

## Correcciones de errores en 3.0.10

- Se corrigió un problema por el que el Explorador TOM perdía la selección actual al intentar arrastrar un objeto a un editor de texto
- Se corrigió un problema por el que el analizador semántico generaba falsos errores "No se puede determinar un valor único para la columna ... no se puede determinar", cuando una expresión DAX en algún lugar del modelo usaba ALLEXCEPT. Consulta [#140](https://github.com/TabularEditor/TabularEditor3/issues/140).
- El grabador de macros ahora funciona con expresiones de elementos de cálculo, expresiones de cadena de formato dinámica, expresiones de filas de detalle de medida y mucho más. Consulta [#143](https://github.com/TabularEditor/TabularEditor3/issues/143).
- Se añadió el método `Model.AddCalculationGroupTable(<name>)` para su uso en C# Script. Esto también corrige el problema [#144](https://github.com/TabularEditor/TabularEditor3/issues/144).
- El Script DAX ya no debería añadir sangría adicional. Consulta [#165](https://github.com/TabularEditor/TabularEditor3/issues/165).
- Se corrigieron problemas de conectividad del Pivot Grid. Consulta [#166](https://github.com/TabularEditor/TabularEditor3/issues/166).
- Se corrigió un problema con los tipos de datos del Analizador VertiPaq; consulta [#169](https://github.com/TabularEditor/TabularEditor3/issues/169).

## ¿Vienes de Tabular Editor 2.x?

Mira [este vídeo](https://www.youtube.com/watch?v=pt3DdcjfImY) para hacerte una idea de las nuevas funciones de Tabular Editor 3.

**Resumen de las principales funciones de Tabular Editor 3:**

- IDE totalmente personalizable, con compatibilidad con varios monitores y Hi-DPI, además de temas
- Nuevo y potente editor de código DAX con autocompletado, comprobación de sintaxis, plegado de código y mucho, mucho más
- \*Modo del área de trabajo, que te permite guardar los cambios en disco y sincronizar simultáneamente los metadatos del modelo con Analysis Services
- \*Previsualiza los datos de las tablas con desplazamiento infinito, crea PivotGrids o escribe consultas DAX para explorar el modelo o probar la lógica de cálculo
- \*Actualiza los esquemas de tablas tanto en el proveedor como en los orígenes de datos estructurados (¡sí, incluso para consultas M!)
- \*Programa actualizaciones de datos
- Crea diagramas del Data model
- Crea Scripts DAX que te permiten editar varias medidas u otros objetos calculados en un solo documento
- Graba C# Scripts y guárdalos como macros (antes conocidos como "Custom Actions")
- Integración del Analizador VertiPaq

\*=Solo cuando estés conectado a una instancia de Analysis Services o Power BI
