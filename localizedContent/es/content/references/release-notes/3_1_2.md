## Correcciones de errores en 3.1.2

- Se corrigió un problema por el que "Actualizar esquema de tabla" no funcionaba con Data sources de SQL DW / Synapse SQL Pool; consulta [#231](https://github.com/TabularEditor/TabularEditor3/issues/231).
- Se corrigió un problema por el que el Best Practice Analyzer seguía teniendo en cuenta reglas ignoradas; consulta [#203](https://github.com/TabularEditor/TabularEditor3/issues/203).
- Se corrigió un problema por el que no se podían guardar modelos en el disco (archivo o carpeta) debido a un error de serialización de JSON; consulta [#232](https://github.com/TabularEditor/TabularEditor3/issues/232).
- Se corrigió un problema que provocaba un cierre inesperado al inspeccionar los valores de las propiedades de un origen de datos estructurado.

## Mejoras en 3.1.1

- Las expresiones compartidas (expresiones de Power Query / M) ahora se pueden serializar como archivos individuales. Puedes controlar la serialización en **Herramientas > Preferencia > Guardar en carpeta** o, para un modelo existente, en **Modelo > Opciones de serialización...** ([#218](https://github.com/TabularEditor/TabularEditor3/issues/218))
- Nuevas opciones para controlar cómo se aplican las asignaciones de perspectiva a los objetos nuevos en **Herramientas > Preferencia > Operaciones de modelado** (consulta el problema [#216](https://github.com/TabularEditor/TabularEditor3/issues/216)).
- La propiedad `PrivacySetting` ahora se puede excluir de la serialización para evitar problemas con Microsoft.AnalysisServices.Deployment. Esto se controla en **Modelo > Opciones de serialización...** (consulta el problema [#210](https://github.com/TabularEditor/TabularEditor3/issues/210)).
- Ahora puedes definir enums, structs y clases directamente dentro de un script de C#, lo que facilita estructurar y reutilizar el código C#.
- El Analizador VertiPaq ahora incluye una opción para recopilar estadísticas de los datos (**Herramientas > Preferencia > Analizador VertiPaq**), lo que mejora la calidad de las estadísticas; por ejemplo, cuando las columnas tienen IsAvailableInMDX = false. Consulta el problema [#190](https://github.com/TabularEditor/TabularEditor3/issues/290).
- Cuando se crea un modelo nuevo, Tabular Editor 3 ahora pregunta si el modelo debe guardarse como archivo o como carpeta la primera vez que se guarda. También puedes establecer el modo predeterminado en **Herramientas > Preferencia > General > Formato de guardado predeterminado para modelos nuevos**. Consulta el problema [#50](https://github.com/TabularEditor/TabularEditor3/issues/50).
- Mejora del rendimiento de inicio (algunos usuarios han informado de mejoras adicionales en el tiempo de inicio al añadir el conmutador `-multicorejit` al acceso directo de TabularEditor3.exe).
- Se reestructuró el menú "Modelo" para que opciones como "Nuevo grupo de cálculo" estén más fácilmente accesibles.
- Se actualizaron las bibliotecas cliente de AMO/TOM a la versión [19.26.1.7](https://www.nuget.org/packages/Microsoft.AnalysisServices.retail.amd64/).
- Ahora se pueden recorrer individualmente los miembros de los roles del modelo en el Best Practice Analyzer
- "Workspace" del menú "Ventana" se ha cambiado a "Layout" para evitar confusiones con el [modo del área de trabajo](https://docs.tabulareditor.com/te3/tutorials/workspace-mode.html).
- Se añadió Ctrl+Shift+D como acceso directo para "Implementar"
- Compatibilidad con nuevas funciones DAX ([`HASH`](https://dax.guide/hash), [`NAMEOF`](https://dax.guide/nameof), [`COLUMNSTATISTICS`](https://dax.guide/columnstatistics), [`BITAND`](https://dax.guide/bitand), [`BITLSHIFT`](https://dax.guide/bitlshift), [`BITOR`](https://dax.guide/bitor), [`BITRSHIFT`](https://dax.guide/bitrshift), [`BITXOR`](https://dax.guide/bitxor)) y compatibilidad con el parámetro opcional de nombre de columna en [`SELECTCOLUMNS`](https://dax.guide/selectcolumns).

## Correcciones de errores en 3.1.1

- El Explorador TOM ya no debería bloquearse ni cerrarse inesperadamente al eliminar objetos (consulta las incidencias [#223](https://github.com/TabularEditor/TabularEditor3/issues/223), [#209](https://github.com/TabularEditor/TabularEditor3/issues/209)).
- Se corrigió un problema de color con temas oscuros en los editores ([#207](https://github.com/TabularEditor/TabularEditor3/issues/207)) y en los iconos ([#215](https://github.com/TabularEditor/TabularEditor3/issues/215))
- Las tablas ahora se ordenan alfabéticamente en el cuadro de diálogo "Crear/Editar relación" (consulta la incidencia [#214](https://github.com/TabularEditor/TabularEditor3/issues/214)
- Se corrigió un problema con los C# Script que impedía guardarlos como macros (consulta la incidencia [#208](https://github.com/TabularEditor/TabularEditor3/issues/208)).
- Se corrigió un problema por el que las contraseñas o credenciales del Data source no se aplicaban correctamente a la base de datos del Workspace (consulta la incidencia [#205](https://github.com/TabularEditor/TabularEditor3/issues/205)).
- La casilla "Seleccionar todas las columnas" ya no debería provocar un cierre inesperado (consulta la incidencia [#204](https://github.com/TabularEditor/TabularEditor3/issues/204)).
- Tabular Editor ahora siempre busca en el directorio de instalación cuando un script hace referencia a DLL externas (mediante la directiva del precompilador `#r`); consulta la incidencia [#192](https://github.com/TabularEditor/TabularEditor3/issues/192).
- Se corrigió un error con el esquema lingüístico que hacía que Q&A dejara de funcionar en algunos casos
- Filtrar por una columna de fecha en la vista previa de datos ya no debería provocar un cierre inesperado
- Al seleccionar todas las perspectivas en el desplegable de perspectivas del Explorador TOM, ya no se muestran los objetos que no están incluidos en ninguna perspectiva. También se corrigió un problema por el que el cuadro desplegable no actualizaba los nombres de las perspectivas.
- Se permite usar determinadas palabras clave de DAX como variables (`WEEK`, `DAY`, `INTEGER`, etc.) sin mostrar ningún error.
- Se corrigió un problema por el que las opciones de serialización no se almacenaban la primera vez que se guardaba un modelo.
- Se corrigió un problema con el analizador de DAX que podía provocar un cierre inesperado al cargar un modelo (consulta [#229](https://github.com/TabularEditor/TabularEditor3/issues/229)).

## Vienes de Tabular Editor 2.x?

Mira [este vídeo](https://www.youtube.com/watch?v=pt3DdcjfImY) para hacerte una idea de las nuevas funcionalidades de Tabular Editor 3.

**Descripción general de las principales características de Tabular Editor 3:**

- IDE totalmente personalizable, con soporte para varios monitores, Hi-DPI y temas
- Nuevo y potente editor de código DAX con autocompletado, comprobación de sintaxis, plegado de código y mucho, mucho más
- \*Modo del área de trabajo, que te permite guardar los cambios en el disco y sincronizar simultáneamente los metadatos del modelo con Analysis Services
- \*Previsualiza los datos de las tablas con desplazamiento infinito, crea PivotGrids o escribe consultas DAX para explorar el modelo o probar la lógica de cálculo
- \*Programa actualizaciones de datos
- Actualiza los esquemas de las tablas tanto en Data sources de proveedor como en orígenes de datos estructurados (sí, ¡incluso para consultas M!)
- Crea diagramas de Data model
- Crea scripts DAX que te permitan editar varias medidas u otros objetos calculados en un único documento
- Graba C# Scripts y guárdalos como macros (antes conocidas como "Custom Actions")
- Integración con el Analizador VertiPaq

\*=Solo cuando estás conectado a una instancia de Analysis Services o Power BI
