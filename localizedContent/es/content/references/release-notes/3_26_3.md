## Correcciones de errores en la versión 3.26.3

- Se corrigió un problema por el cual un Archivo > Revertir fallido (por ejemplo, cuando el TMDL en disco contiene una referencia que no se puede resolver) podía desactivar de forma permanente el seguimiento de cambios sin guardar durante el resto de la sesión. Antes, las ediciones posteriores dejaban de mostrar el asterisco de cambios sin guardar, y Revertir o Cerrar podían descartarlos sin mostrar ninguna advertencia. Ahora el seguimiento de cambios sigue activo después de una reversión fallida.
- En los modelos serializados en carpetas (TMDL), Archivo > Revertir ya no hace que el modelo aparezca como si tuviera cambios sin guardar después de una reversión correcta. Antes, el asterisco de la barra de título y la confirmación "¿Actualizar metadatos desde el origen?" seguían apareciendo en cada reversión posterior hasta que se guardaba el modelo.
- Se corrigió el autocompletado de DAX, que sugería funciones definidas por el usuario (UDF) que no eran válidas en el contexto actual del argumento y omitía otras que sí eran válidas. Ahora las UDF se filtran según su tipo de retorno inferido: cuando se espera un escalar, solo se muestran las UDF que devuelven un escalar, una tabla de una sola columna o un parámetro de entrada sin tipo; cuando se espera una tabla, solo se muestran las UDF que devuelven tablas (o passthrough); y los argumentos de filtro (como los de CALCULATE / CALCULATETABLE) ahora también incluyen sugerencias de UDF, que antes se omitían por completo. Las UDF cuya definición contiene un error semántico (por ejemplo, uno que requiere contexto de fila o usa MATCHBY de forma incorrecta) ya no se ofrecen como sugerencias, ya que no pueden invocarse de forma válida.
- El formateador de DAX ya no sustituye el `=` del valor predeterminado de un parámetro de una función DAX por `:`. Antes, al dar formato a una función como `(p = [m]) => 1`, se generaba la expresión no válida `(p: [m]) => 1`, porque el formateador asumía que cualquier token posterior al nombre de un parámetro era una anotación de tipo `: type`. Ahora las anotaciones de tipo, los valores predeterminados y la combinación de ambos (`p: INT64 = 1`) se formatean correctamente.
- Se corrigió un problema por el que marcar un problema del Optimizador de DAX como ignorado no tenía efecto después de cargar una nueva versión de un modelo ofuscado: el problema ignorado volvía a aparecer como abierto. Ahora los problemas ignorados se conservan entre versiones de los modelos ofuscados.
- Se corrigió un cierre inesperado intermitente ("Collection was modified") que podía producirse en la vista del Optimizador de DAX justo después de cargar un modelo o una nueva versión. Ahora la vista se actualiza mientras todavía se muestra el spinner de progreso, en lugar de actualizarse en segundo plano mientras se redibujaba la ventana.

## Vienes de Tabular Editor 2.x?

Mira [este vídeo](https://youtu.be/O4ATwdzCvWc) para hacer un recorrido rápido por las principales funcionalidades de Tabular Editor 3. Además, asegúrate de consultar nuestra [guía de inicio](https://docs.tabulareditor.com/onboarding/index.html).

**Resumen de las principales funcionalidades de Tabular Editor 3:**

- IDE totalmente personalizable, con compatibilidad con varios monitores, Hi-DPI y temas
- Nuevo y potente editor de código DAX con autocompletado, comprobación de sintaxis, plegado de código y mucho, mucho más
- \*Modo del área de trabajo, que te permite guardar los cambios en disco y sincronizar los metadatos del modelo con Analysis Services simultáneamente
- \*Previsualiza los datos de la tabla con desplazamiento infinito, crea PivotGrids o escribe consultas DAX para explorar el modelo o probar la lógica de cálculo
- \*Programar actualizaciones de datos
- Actualizar los esquemas de las tablas tanto en los Data source del proveedor como en los orígenes de datos estructurados (sí, ¡incluso para consultas M!)
- Crear diagramas del Data model
- Crear scripts DAX que te permitan editar varias medidas u otros objetos calculados en un solo documento
- Grabar C# Scripts y guardarlos como macros (antes conocidas como "Custom Actions")
- Integración con el Analizador VertiPaq
- Depurador de DAX
- Integración del Optimizador de DAX
- Acciones de código para refactorizar fácilmente tus expresiones DAX.

\*=Solo mientras estés conectado a una instancia de Analysis Services o Power BI
