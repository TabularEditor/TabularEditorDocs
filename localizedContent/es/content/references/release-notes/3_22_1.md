## Novedades en 3.22.0

Consulta nuestro [blog de lanzamientos](https://blog.tabulareditor.com/) para obtener una breve visión general de las actualizaciones más importantes de esta versión.

(**Solo para la Edición Enterprise**) Tabular Editor 3 ahora admite plenamente Direct Lake sobre OneLake y Direct Lake sobre SQL. Esto significa:

- El asistente de importación de tablas tiene nuevas opciones de importación para los elementos Warehouse y Lakehouse de Fabric. Ya no necesitas conocer el punto de conexión de análisis SQL del Warehouse o Lakehouse al que quieres conectarte. En su lugar, te mostramos una lista de todos los Warehouses/Lakehouses de tu Workspace, para que puedas elegir el que quieras usar.
- En la última página del Asistente para importar tablas, puedes elegir si quieres usar Direct Lake en OneLake o en modo SQL, o si prefieres configurar la tabla en el tradicional modo "import".
- Cuando el modelo contiene tablas en modo Direct Lake sobre OneLake, ahora también puedes agregar tablas en Import mode para crear modelos híbridos o en modo mixto.
- Al previsualizar datos en tablas Direct Lake, ya no limitamos el número de filas devueltas (para que puedas desplazarte libremente por todos los datos de la tabla, igual que en Import mode). Ten en cuenta que, al realizar una vista previa de datos, todas las columnas se cargan en el modelo semántico en la capacidad de Fabric. Considera usar una Consulta DAX si te preocupa el consumo de memoria en tu capacidad de Fabric.

> [!NOTE]
> Ya no configuramos los modelos Direct Lake para que usen una intercalación que distinga mayúsculas y minúsculas. Si planeas usar Direct Lake sobre SQL con un Warehouse de Fabric que usa una intercalación que distingue entre mayúsculas y minúsculas, debes completar manualmente la propiedad **Collation** de tu modelo semántico _antes_ de agregar cualquier tabla u otros objetos.

## Mejoras en 3.22.0

- Al guardar un modelo como un solo archivo, el cuadro de diálogo **Guardar archivo** ahora sugiere "model.bim" como nombre de archivo predeterminado.
- Se actualizaron varias dependencias a sus versiones más recientes, incluidas [AMO/TOM a 19.98.0.3](https://www.nuget.org/packages/Microsoft.AnalysisServices/). Esta actualización corrige problemas de autenticación persistentes con cuentas profesionales o educativas y solicitudes repetidas de inicio de sesión.
- El editor de Consulta DAX ahora es compatible con la palabra clave [`MPARAMETER`](https://dax.guide/st/mparameter/); consulta [#1467](https://github.com/TabularEditor/TabularEditor3/issues/1467).
- Nuestros editores de DAX ahora admiten términos de búsqueda de autocompletado **basados en palabras**. Es decir, si escribes `sales ytd` en el editor, el autocompletado ahora sugerirá medidas como `[Sales Margin YTD]`, `[Sales Revenue YTD]`, etc.
- Con la actualización de junio de 2025 de Power BI Desktop, [las herramientas externas ahora pueden realizar cualquier operación de escritura en el modelo semántico](https://powerbi.microsoft.com/en-us/blog/power-bi-june-2025-feature-summary/#post-30307-_Toc269410729). Por ello, ya no restringimos las operaciones de modelado disponibles en Tabular Editor al conectarse a un modelo en esta o en versiones más recientes de Power BI Desktop.

## Correcciones de errores en 3.22.1

- Se corrigió un error por el que la vista **Actualización de datos** quedaba en blanco y no mostraba operaciones de actualización adicionales hasta que se reiniciaba la aplicación. Consulta [#1461](https://github.com/TabularEditor/TabularEditor3/issues/1461).

## Correcciones de errores en 3.22.0

- Se agregó el manejo de excepciones al encontrar excepciones IOException durante las operaciones de guardado en carpetas (en las que a veces necesitamos eliminar carpetas de la estructura, lo que puede verse bloqueado por bloqueos del sistema de archivos impuestos por sistemas de control de versiones).
- Se corrigió un error por el que ciertos atajos de teclado (tanto los predeterminados como los personalizados) no funcionaban cuando la página "Novedades" tenía el foco.
- Al asignar un atajo de teclado personalizado que ya está asignado a otro comando, ahora mostramos un mensaje de advertencia y quitamos el atajo del otro comando, para evitar ambigüedades.
- Se corrigió un error por el que el Analizador semántico mostraba un mensaje de error falso al usar referencias a columnas sin calificar en una función de ventana DAX. Consulta [#1460](https://github.com/TabularEditor/TabularEditor3/issues/1460).
- Se corrigió un error que en ocasiones podía hacer que la vista **Data Refresh** se quedara congelada, lo que obligaba a reiniciar la aplicación para solucionarlo. Consulta [#1461](https://github.com/TabularEditor/TabularEditor3/issues/1461).
- Se corrigieron varios problemas menores del editor DAX relacionados con el autocompletado, el formato automático y el resaltado de sintaxis de las palabras clave.
- Ahora es posible modificar los nombres de los Query Group (es decir, las "carpetas de visualización" de las expresiones M compartidas y las particiones M). Además, si pegas un objeto con un Query Group en un modelo que no contiene ese Query Group, se creará automáticamente para evitar los mensajes de error en posteriores operaciones de guardado o implementación del modelo.
- Se corrigió un error por el que, en ocasiones, no era posible deshacer cambios anteriores después de guardar los metadatos del modelo en el servidor.

## Vienes de Tabular Editor 2.x?

Mira [este vídeo](https://youtu.be/O4ATwdzCvWc) para hacer un recorrido rápido por las principales funciones de Tabular Editor 3. Además, asegúrate de consultar nuestra [guía de inicio](https://docs.tabulareditor.com/onboarding/index.html).

**Descripción general de las principales características de Tabular Editor 3:**

- IDE totalmente personalizable, con compatibilidad con varios monitores, Hi-DPI y temas
- Nuevo y potente editor de código DAX con autocompletado, comprobación de sintaxis, plegado de código y mucho, mucho más
- \*Modo del área de trabajo, que te permite guardar los cambios en disco y sincronizar los metadatos del modelo con Analysis Services a la vez
- \*Previsualiza los datos de la tabla con desplazamiento infinito, crea PivotGrids o escribe Consultas DAX para explorar el modelo o probar la lógica de cálculo
- \*Programa actualizaciones de datos
- Actualiza los esquemas de las tablas tanto en Data sources del proveedor como en el origen de datos estructurado (sí, incluso para consultas M!)
- Crear diagramas del Data model
- Crear scripts DAX que te permitan editar varias medidas u otros objetos calculados en un único documento
- Graba C# Scripts y guárdalos como macros (antes conocidos como "Custom Actions")
- Integración del Analizador VertiPaq
- Depurador de DAX
- Integración del Optimizador de DAX
- Acciones de código para refactorizar fácilmente tu código DAX.

\*=Solo cuando estés conectado a una instancia de Analysis Services o Power BI
