## Novedades en 3.25.0

Echa un vistazo a nuestro [blog de lanzamientos](https://tabulareditor.com/blog/tabular-editor-3-january-2026-release) para ver un breve resumen de las actualizaciones más importantes de esta versión.

- Tabular Editor ahora está disponible en [**chino y español (vista previa)**](xref:references-application-language). Todas las traducciones se realizaron inicialmente con IA y con revisión humana. El japonés, el alemán y el francés también están disponibles como idiomas experimentales, pero todavía no se han revisado.
- Tabular Editor ahora incluye un conjunto de [**reglas integradas de Best Practice Analyzer (BPA)**](xref:built-in-bpa-rules) que cubren prácticas recomendadas clave en formato, metadatos, diseño del modelo, expresiones DAX y traducciones.
- Tabular Editor ahora incluye la función **Semantic Bridge**, que permite crear fácilmente modelos semánticos a partir de las [Metric Views](https://docs.databricks.com/aws/en/metric-views/) de Databricks (_solo en la Edición Enterprise de Tabular Editor 3_).
- Se añadió una opción para guardar modelos semánticos con [**archivos auxiliares para Fabric**](xref:save-with-supporting-files). Cuando está habilitado, Tabular Editor puede crear y actualizar los archivos .platform y definition.pbism junto con los metadatos del modelo para que coincidan con la estructura del repositorio de Fabric.
- Esta versión también agrega un nuevo **Editor de calendario**, que facilita la administración de los objetos de calendario usados para la inteligencia temporal basada en calendario.
- Por último, pero no menos importante, hemos agregado un cuadro de diálogo de [**Actualización avanzada**](xref:advanced-refresh), desde el cual puedes configurar opciones avanzadas de actualización como el paralelismo y la fecha efectiva para la actualización incremental. También puedes especificar [perfiles de anulación de actualización](xref:refresh-overrides) (_solo para Tabular Editor 3 Edición Business y Edición Enterprise_).

## Mejoras en 3.25.0

- Se agregó compatibilidad para establecer la propiedad CustomData de la cadena de conexión directamente desde el cuadro de diálogo de suplantación.
- La vista del Administrador de paquetes DAX ahora descarga y muestra los iconos de los paquetes de DaxLib.
- El Administrador de paquetes DAX ahora almacena en caché los metadatos de los paquetes, lo que permite mostrar información del paquete sin necesidad de conexión a Internet.
- Actualizamos el paquete de la biblioteca cliente de AS a la versión más reciente [19.108.5](https://www.nuget.org/packages/Microsoft.AnalysisServices/19.108.5). Esta actualización te permite controlar la nueva propiedad [UnicodeCharacterBehavior](https://learn.microsoft.com/en-us/dotnet/api/microsoft.analysisservices.tabular.database.unicodecharacterbehavior?view=analysisservices-dotnet#microsoft-analysisservices-tabular-database-unicodecharacterbehavior) en el objeto Database.
- Se agregaron las categorías de datos de columna específicas de Power BI: Longitud, Latitud y Lugar.
- Ahora mostramos un mensaje de error si intentas establecer el nivel de compatibilidad de un modelo en un valor que el servidor no admite. Esto ayuda a evitar errores inesperados más adelante al guardar cambios en el modelo. Si necesitas establecer un nivel de compatibilidad no admitido por motivos de migración, guarda los metadatos del modelo en disco (en formato .bim o TMDL) y, luego, abre los metadatos sin ninguna base de datos de Workspace asociada.
- Los archivos de opciones de usuario (.tmuo) ahora también se guardan cuando abres un modelo directamente desde un servidor o un punto de conexión XMLA. Los archivos se almacenan en la carpeta %LocalAppData%\TabularEditor3\UserOptions.
- Se agregó la opción "Actualizar completo (ignorar política de actualización)" al menú contextual de actualización de datos, de modo que puedas actualizar fácilmente los datos en particiones existentes sin desencadenar operaciones de agregar/eliminar/combinar particiones en el servidor cuando una tabla esté regida por una política de actualización incremental.
- Se reorganizaron las opciones del menú contextual de actualización de datos para colocar las opciones más usadas en la parte superior.
- Se agregó "Hora de inicio" al panel de actualización, lo que permite ordenar las actualizaciones por el momento en que se iniciaron; por ejemplo, ver primero la actualización más reciente.
- Se agregó un panel de búsqueda al cuadro de diálogo de Preferencias para buscar rápidamente preferencias individuales.
- Se agregaron "Funciones definidas por el usuario" y "Calendarios" a la lista del editor de reglas de prácticas recomendadas.
- Se agregó una propiedad `UsedInCalendars` a las columnas, lo que facilita comprobar, dentro de un C# Script, si una columna se usa en algún objeto de calendario.
- La acción de menú "Archivo > Cerrar" pasó a llamarse "Cerrar documento" y ahora permite cerrar cualquier panel o documento del área principal de documentos. Se asignó **Ctrl + W** como atajo predeterminado.
- Se agregó compatibilidad del Analizador semántico con la nueva palabra clave `ENDALIGNED` del parámetro <extension> de [`DATEADD`](https://dax.guide/DATEADD). [Más información](https://learn.microsoft.com/en-us/dax/dateadd-function-dax#extension-parameter-for-small--large-period-shifts).
- Al guardar en un archivo model.bim o en una estructura de carpetas TMDL, con las **opciones de serialización** **Ignorar particiones de actualización incremental** habilitadas, ahora se agrega una partición ficticia para reemplazar las particiones PolicyRange. Esto garantiza que el modelo aún se pueda cargar en Power BI Desktop, ya que no se guardan tablas sin particiones.

## Correcciones de errores en 3.25.0

- Se corrigió un problema crítico que impedía a algunos clientes conectarse a la API REST de Power BI o a Databricks mediante autenticación de Azure AD.
- Se corrigió una excepción no controlada que se producía al eliminar una columna que también se usaba en un objeto de calendario.
- Se corrigió el desplegable de columnas del editor de relaciones para que muestre automáticamente los nombres completos de las columnas.
- Al duplicar una tabla que contiene uno o varios calendarios, ahora los calendarios se renombran automáticamente para garantizar nombres únicos en todo el modelo.
- Varios errores/advertencias detectados durante una operación de actualización de datos ya no deberían provocar una excepción no controlada.
- Se corrigió la lista de paquetes de DAX, que mostraba información desactualizada después de ejecutar "Actualizar todo".
- Se corrigió un error por el que no se podía deshacer el cambio en la propiedad `SchemaName` de las particiones de entidad.
- La acción de código [DI005 - Reescribir filtro de tabla como predicado escalar](https://docs.tabulareditor.com/kb/DI005.html) ya no se sugiere cuando se produce la [expansión de tablas](https://www.sqlbi.com/articles/expanded-tables-in-dax/), ya que reescribir ese tipo de filtro como un filtro de columna no necesariamente produce el mismo resultado.
- Al hacer clic en el botón de puntos suspensivos junto a la propiedad Connection String, ya no debería producirse una excepción no controlada cuando la cadena de conexión especifica un proveedor OLE DB que no está instalado en el sistema actual.
- Las exportaciones a TMDL ahora excluyen correctamente las particiones de actualización incremental cuando la preferencia correspondiente está habilitada, en línea con el comportamiento de JSON.
- Se corrigió un problema por el que, al hacer clic en el botón "Iniciar sesión..." al intentar actualizar o importar desde Dataflows, se mostraban los siguientes **mensajes** de error: `This page is not normally shown and could be a sign of a phishing attempt.`

## Correcciones de errores en 3.25.1

- Se corrigieron algunas cadenas que indicaban incorrectamente que las reglas de BPA integradas solo estaban disponibles en la Edición Enterprise.

## Correcciones de errores en 3.25.2

- Se eliminó la dependencia de PowerShell del instalador .exe (lo que impedía la instalación de Tabular Editor 3 en algunos casos)

## Vienes de Tabular Editor 2.x?

Mira [este vídeo](https://youtu.be/O4ATwdzCvWc) para hacer un recorrido rápido por las principales características de Tabular Editor 3. Además, asegúrate de consultar nuestra [guía de incorporación](https://docs.tabulareditor.com/onboarding/index.html).

**Resumen de las principales características de Tabular Editor 3:**

- IDE totalmente personalizable, con compatibilidad con varios monitores, HiDPI y temas
- Nuevo y potente editor de código DAX con autocompletado, comprobación de sintaxis, plegado de código y mucho, mucho más
- \*Modo del área de trabajo, que te permite guardar tus cambios en disco y sincronizar al mismo tiempo los metadatos del modelo con Analysis Services
- \*Obtén una vista previa de los datos de las tablas con desplazamiento infinito, crea PivotGrids o escribe consultas DAX para explorar el modelo o probar la lógica de cálculo
- \*Programa actualizaciones de datos
- Actualiza los esquemas de las tablas tanto en Data sources de proveedor como en orígenes de datos estructurados (sí, incluso para consultas M)
- Crea diagramas del Data model
- Crea scripts DAX que te permitan editar varias medidas u otros objetos calculados en un único documento
- Graba C# Scripts y guárdalos como macros (antes conocidas como "Custom Actions")
- Integración con el Analizador VertiPaq
- Depurador de DAX
- Integración con el Optimizador de DAX
- Acciones de código para refactorizar fácilmente tu código DAX.

\*=Solo mientras estés conectado a una instancia de Analysis Services o Power BI
