## ¡Importante!

**NOTA**: Tabular Editor 3.3.0 (y versiones posteriores) usa .NET 6. Es posible que tengas que instalar el [Runtime de .NET 6.0 para escritorio](https://dotnet.microsoft.com/en-us/download/dotnet/6.0/runtime) antes de poder iniciar Tabular Editor 3.3.0 (o posterior). A partir de la versión 3.3.1, el instalador de Tabular Editor 3 descargará e instalará estos requisitos previos si faltan.

## Nuevas funciones en 3.4.0

- TOM actualizado a la versión [19.54.1](https://www.nuget.org/packages/Microsoft.AnalysisServices.NetCore.retail.amd64).
- Tabular Editor 3 ahora es compatible con las [nuevas funciones de ventana de DAX](https://pbidax.wordpress.com/2022/12/15/introducing-dax-window-functions-part-1/) anunciadas en la actualización de diciembre de 2022 de Power BI Desktop.
- ¡Ya están aquí los atajos de teclado personalizables! ¿Quieres usar siempre CTRL+S para guardar el modelo, independientemente del contexto? ¿Quieres asignar una tecla de acceso rápido para realizar una actualización del modelo? ¿Qué tal un atajo del editor para [seleccionar todas las apariciones de la selección actual](https://github.com/TabularEditor/TabularEditor3/issues/630)? Encontrarás esta funcionalidad en **Herramientas > Preferencias > Teclado**. Más información [aquí](xref:preferences#tabular-editor--keyboard).
- Todos los editores de código permiten ahora realizar selecciones múltiples (por ejemplo, mantén pulsada la tecla CTRL y arrastra con el ratón para crear una nueva selección), además de selecciones rectangulares (mantén pulsada ALT mientras arrastras o usa SHIFT + las flechas).

## Mejoras menores en 3.4.0

- Tabular Editor 3 ahora muestra el icono correcto en la cinta de Herramientas externas de Power BI.
- Edición unificada de las propiedades de tipo columna en la cuadrícula de propiedades. Ahora puedes especificar una columna escribiendo su nombre en la cuadrícula de propiedades. Cuando hay más de 7 columnas, aparece un cuadro de diálogo de "Seleccionar columna" en lugar de una lista desplegable. Esto también corrige [#738](https://github.com/TabularEditor/TabularEditor3/issues/738).
- Se han eliminado los corchetes de los nombres de columna en la Vista previa de tabla
- Ahora es posible conectarse y seleccionar otro Workspace de Power BI en el Asistente para importar tablas.
- La lista de campos de Pivot Grid ahora es una ventana acoplable; consulta [#742](https://github.com/TabularEditor/TabularEditor3/issues/742).
- Los calltips de DAX ahora se formatean correctamente cuando las descripciones de origen contienen etiquetas HTML.
- Ahora todos los editores de código cuentan con selecciones rectangulares muy mejoradas al realizar selecciones con el teclado. Además, copiar/pegar ahora funciona de forma más intuitiva con selecciones rectangulares.
- Se añadió una opción en el cuadro de diálogo Herramientas > Personalizar (en la pestaña "Opciones") para bloquear los menús y la barra de herramientas y evitar que se arrastren accidentalmente.
- La selección múltiple de Scintilla, añadir siguiente/añadir cada, ahora respeta las opciones de búsqueda especificadas en el cuadro de diálogo Buscar.
- Ahora el atajo File.SaveModel funciona independientemente del contexto; consulta [#762](https://github.com/TabularEditor/TabularEditor3/issues/762).
- Ahora es posible cancelar una operación de guardado de larga duración, por ejemplo, cuando la base de datos está bloqueada por una operación de actualización. Consulta [#730](https://github.com/TabularEditor/TabularEditor3/issues/730).
- El cuadro de diálogo para crear/editar una relación ahora se puede cambiar de tamaño; consulta [#732](https://github.com/TabularEditor/TabularEditor3/issues/732).
- Ahora es posible expandir/contraer con las teclas de flecha todas las listas en árbol (incluidos BPA, la Vista de dependencias y la Vista de actualización de datos); consulta [#775](https://github.com/TabularEditor/TabularEditor3/issues/775).
- ¡Hemos escuchado tus comentarios! "Mostrar relaciones de la tabla" vuelve a estar en la Vista de dependencias; consulta [#731](https://github.com/TabularEditor/TabularEditor3/issues/731).

## Correcciones de errores en 3.4.0

- El cuadro de diálogo de conexión de AAD ahora debería mostrarse como un cuadro de diálogo modal en todos los casos.
- Crear una transacción de deshacer cuando se editan varios objetos en lote
- Ahora puedes restablecer una propiedad de objeto (como SortByColumn) haciendo clic con el botón derecho y eligiendo "Restablecer".
- Accesos directos mediante clic derecho para editar las propiedades de expresiones de tabla (SourceExpression, PollingExpression, etc.) ya están disponibles; consulta [#721](https://github.com/TabularEditor/TabularEditor3/issues/721).
- Pivot Grid ya no debería bloquearse cuando se arrastran campos sobre ella; consulta [#676](https://github.com/TabularEditor/TabularEditor3/issues/676).
- Se corrigió el problema [#723](https://github.com/TabularEditor/TabularEditor3/issues/723).
- Se corrigió un problema que impedía las actualizaciones del esquema de la tabla y provocaba que el Asistente para importar tablas se cerrara al intentar importar desde un Dataflow de Power BI existente, especificado en el archivo .tmuo.
- Se corrigió un problema por el cual el Depurador de DAX generaba consultas DAX no válidas para las expresiones de seguimiento; consulta [#770](https://github.com/TabularEditor/TabularEditor3/issues/770).
- Se corrigió un problema que, a veces, provocaba que las ventanas del depurador se cerraran (mostrando cruces rojas sobre un fondo blanco).
- Se corrigió un problema por el cual el Asistente para importar tablas a veces mostraba Dataflows sin nombre.
- Ahora el botón "Revertir" debería estar siempre habilitado. Además, el aviso de confirmación solo se muestra cuando se han realizado cambios. Consulta [#729](https://github.com/TabularEditor/TabularEditor3/issues/729).
- Al mostrar la lista de campos, ya no se borra la Pivot Grid; consulta [#741](https://github.com/TabularEditor/TabularEditor3/issues/741).
- Se corrigió un problema por el que las columnas del Explorador TOM a veces aparecían "recortadas" al iniciar la aplicación.
- El Explorador TOM ahora solo permite agregar EntityPartitions en Datasets de Power BI
- Se corrigió un bloqueo en la vista de dependencias; consulta [#758](https://github.com/TabularEditor/TabularEditor3/issues/758)
- Se corrigió un problema por el que las ventanas flotantes podían aparecer fuera del área visible de la pantalla al reiniciar la aplicación; consulta [#652](https://github.com/TabularEditor/TabularEditor3/issues/652).
- Se corrigió un problema por el que el servidor reciente no se guardaba.
- Se corrigieron algunos errores relacionados con cómo aparecen las acciones en el menú "Modelo" y/o "Contexto". Por ejemplo, «Script DAX» en el menú Modelo solo generaba scripts de los objetos dentro de la selección actual, cuando lo previsto era generar scripts de todos los objetos del modelo.
- Se corrigió el comportamiento de la barra de desplazamiento horizontal de los editores de texto.
- El cuadro de diálogo de conexión SQL ahora guarda los cambios en la cadena de conexión; consulta [#755](https://github.com/TabularEditor/TabularEditor3/issues/755).
- Al seleccionar una base de datos después de filtrar la lista en el cuadro de diálogo "Elegir base de datos", ahora se garantiza que se cargue la base de datos correcta. Consulta [#761](https://github.com/TabularEditor/TabularEditor3/issues/761).
- Se corrigió un problema al desplazarse por las tablas (consulta [#733](https://github.com/TabularEditor/TabularEditor3/issues/733), y otro por el que el recuento de filas no se actualizaba al aplicar criterios de filtro.

## ¿Vienes de Tabular Editor 2.x?

Mira [este vídeo](https://www.youtube.com/watch?v=pt3DdcjfImY) para hacerte una idea de las nuevas características de Tabular Editor 3. Además, asegúrate de consultar nuestra [guía de incorporación](https://docs.tabulareditor.com/onboarding/index.html).

**Resumen de las principales características de Tabular Editor 3:**

- IDE totalmente personalizable, con compatibilidad con varios monitores, Hi-DPI y temas
- Nuevo y potente editor de código DAX con autocompletado, comprobación de sintaxis, plegado de código y mucho, mucho más
- \*Modo del área de trabajo, que te permite guardar los cambios en disco y sincronizar los metadatos del modelo con Analysis Services al mismo tiempo
- \*Previsualizar los datos de la tabla con desplazamiento infinito, crear PivotGrids o escribir consultas DAX para explorar el modelo o probar la lógica de cálculo
- \*Programar actualizaciones de datos
- Actualizar los esquemas de tabla tanto en el Data source del proveedor como en el origen de datos estructurado (sí, ¡incluso para consultas M!)
- Crear diagramas del Data model
- Crear scripts DAX que te permiten editar varias medidas u otros objetos calculados en un solo documento
- Grabar C# Scripts y guardarlos como macros (antes conocidas como "Acciones personalizadas")
- Integración con el Analizador VertiPaq
- \*Depurador de DAX

\*=Solo al estar conectado a una instancia de Analysis Services o Power BI
