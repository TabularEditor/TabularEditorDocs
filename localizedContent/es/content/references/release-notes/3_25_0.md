## Novedades en 3.25.0

Consulta nuestro [blog de lanzamiento](https://tabulareditor.com/blog/tabular-editor-3-january-2026-release) para ver un breve resumen de las actualizaciones más importantes de esta versión.

- Tabular Editor ahora está disponible en [**chino y español (versión preliminar)**](xref:references-application-language). Todas las traducciones se han realizado inicialmente con IA, con revisión humana. El japonés, el alemán y el francés también están disponibles como idiomas experimentales y aún no se han revisado.
- Tabular Editor ahora incluye un conjunto de [**reglas integradas de Best Practice Analyzer (BPA)**](xref:built-in-bpa-rules) que abarcan las mejores prácticas clave en cuanto a formato, metadatos, diseño del modelo, expresiones DAX y traducciones.
- Tabular Editor ahora incluye la funcionalidad **Semantic Bridge**, que permite crear fácilmente modelos semánticos a partir de las [Metric Views](https://docs.databricks.com/aws/en/metric-views/) de Databricks (_solo para la Edición Enterprise de Tabular Editor 3_).
- Se ha agregado una opción para guardar modelos semánticos con [**archivos auxiliares para Fabric**](xref:save-with-supporting-files). Al habilitar esta opción, Tabular Editor puede crear y actualizar los archivos .platform y definition.pbism junto con los metadatos del modelo para que coincidan con la estructura del repositorio de Fabric.
- Esta versión también agrega un nuevo **Editor de calendario**, que facilita la administración de los objetos de calendario utilizados para la inteligencia temporal basada en calendario.
- Por último, pero no por ello menos importante, hemos añadido un cuadro de diálogo de [**Actualización avanzada**](xref:advanced-refresh), desde el que puedes configurar opciones avanzadas de actualización, como el paralelismo y la fecha efectiva, para la actualización incremental. También puedes especificar [perfiles de anulación de actualización](xref:refresh-overrides) (_solo disponible en Tabular Editor 3 Edición Business y Edición Enterprise_).

## Mejoras en 3.25.0

- Se ha añadido compatibilidad para establecer directamente la propiedad CustomData de la cadena de conexión desde el diálogo de suplantación.
- La vista de DAX Package Manager ahora descarga y muestra los iconos de los paquetes de DaxLib.
- DAX Package Manager ahora almacena en caché los metadatos de los paquetes, lo que te permite ver información de los paquetes sin necesidad de conexión a Internet.
- Hemos actualizado el paquete de la biblioteca cliente de AS a la versión más reciente [19.108.5](https://www.nuget.org/packages/Microsoft.AnalysisServices/19.108.5). Esta actualización te permite controlar la nueva propiedad [UnicodeCharacterBehavior](https://learn.microsoft.com/en-us/dotnet/api/microsoft.analysisservices.tabular.database.unicodecharacterbehavior?view=analysisservices-dotnet#microsoft-analysisservices-tabular-database-unicodecharacterbehavior) en el objeto Database.
- Se han añadido las categorías de datos de columna específicas de Power BI: Longitud, Latitud y Lugar.
- Ahora mostramos un mensaje de error si intentas establecer el nivel de compatibilidad de un modelo en un valor que el servidor no admite. Esto es para evitar errores inesperados más adelante al guardar los cambios en el modelo. Si necesitas establecer un nivel de compatibilidad no admitido por motivos de migración, guarda los metadatos del modelo en disco (en formato .bim o TMDL) y, a continuación, abre los metadatos sin ninguna base de datos del Workspace asociada.
- Los archivos de opciones de usuario (.tmuo) ahora también se guardan cuando abres un modelo directamente desde un servidor o un punto de conexión XMLA. Los archivos se almacenan en la carpeta %LocalAppData%\TabularEditor3\UserOptions.
- Se ha añadido la opción "Actualizar por completo (ignorar política)" al menú contextual de actualización de datos, para que puedas actualizar fácilmente los datos en particiones existentes sin desencadenar operaciones de adición/eliminación/combinación de particiones en el servidor cuando una tabla esté regida por una política de actualización incremental.
- Se reorganizaron las opciones del menú contextual de actualización de datos para llevar las opciones más usadas a la parte superior.
- Se ha añadido Hora de inicio al panel de actualización, lo que permite ordenar las actualizaciones según cuándo comenzaron; por ejemplo, ver primero la actualización más reciente.
- Se ha añadido un panel de búsqueda al diálogo de Preferencias para buscar rápidamente preferencias individuales.
- Se han añadido "Funciones definidas por el usuario" y "Calendarios" a la lista del editor de reglas de buenas prácticas.
- Se ha añadido la propiedad `UsedInCalendars` a las columnas, lo que facilita comprobar en un C# Script si una columna se usa en algún objeto de calendario.
- La acción de menú "Archivo > Cerrar" se ha renombrado a "Cerrar documento" y ahora permite cerrar cualquier panel o documento del área principal de documentos. Se asigna **Ctrl + W** como acceso directo predeterminado.
- Se ha añadido compatibilidad del Analizador semántico con la nueva palabra clave `ENDALIGNED` del parámetro <extension> de [`DATEADD`](https://dax.guide/DATEADD). [Más información](https://learn.microsoft.com/en-us/dax/dateadd-function-dax#extension-parameter-for-small--large-period-shifts).
- Al guardar en un archivo model.bim o en una estructura de carpetas TMDL con la opción de serialización **Ignorar particiones de actualización incremental** habilitada, ahora añadimos una partición ficticia para sustituir las particiones PolicyRange. Esto garantiza que el modelo aún se pueda cargar en Power BI Desktop, ya que no se guarda ninguna tabla sin sus particiones.

## Correcciones de errores en 3.25.0

- Se corrigió un problema crítico que impedía a algunos clientes conectarse a la API REST de Power BI o a Databricks usando autenticación de Azure AD.
- Se corrigió una excepción no controlada que se producía al eliminar una columna que también se usaba en un objeto de calendario.
- Se corrigió la lista desplegable de columnas del editor de relaciones para que muestre automáticamente los nombres completos de las columnas.
- Al duplicar una tabla que contiene uno o más calendarios, ahora los calendarios se renombran automáticamente para garantizar nombres únicos en todo el modelo.
- Varios errores o advertencias detectados durante una operación de actualización de datos ya no deberían provocar una excepción no controlada.
- Se corrigió un problema por el que la lista de paquetes DAX mostraba información desactualizada después de ejecutar "Update All".
- Se corrigió un error por el que la propiedad `SchemaName` de las particiones de entidades no se podía deshacer.
- La acción de código [DI005 - Reescribir el filtro de tabla como predicado escalar](https://docs.tabulareditor.com/kb/DI005.html) ya no se sugiere cuando se produce la [expansión de tabla](https://www.sqlbi.com/articles/expanded-tables-in-dax/), ya que reescribir ese tipo de filtro como filtro de columna no necesariamente produciría el mismo resultado.
- Al hacer clic en el botón de puntos suspensivos junto a la propiedad Cadena de conexión, ya no debería provocarse una excepción no controlada cuando la cadena de conexión especifica un proveedor OLE DB que no está instalado en el sistema actual.
- Las exportaciones de TMDL ahora excluyen correctamente las particiones de actualización incremental cuando la preferencia correspondiente está habilitada, en consonancia con el comportamiento de JSON.
- Se corrigió un problema por el que, al hacer clic en el botón "Sign in..." al intentar actualizar o importar desde Dataflows, se mostraba el siguiente mensaje de error: `This page is not normally shown and could be a sign of a phishing attempt.`

## Vienes de Tabular Editor 2.x?

Mira [este vídeo](https://youtu.be/O4ATwdzCvWc) para hacer un recorrido rápido por las principales características de Tabular Editor 3. Además, no dejes de consultar nuestra [guía de incorporación](https://docs.tabulareditor.com/onboarding/index.html).

**Descripción general de las principales características de Tabular Editor 3:**

- IDE totalmente personalizable, con compatibilidad con varios monitores, Hi-DPI y temas
- Nuevo y potente editor de código DAX con autocompletado, comprobación de sintaxis, plegado de código y mucho, mucho más
- \*Modo del área de trabajo, que te permite guardar los cambios en disco y sincronizar los metadatos del modelo con Analysis Services al mismo tiempo
- \*Previsualiza los datos de la tabla con desplazamiento infinito, crea PivotGrids o escribe consultas DAX para explorar el modelo o probar la lógica de cálculo
- \*Programa actualizaciones de datos
- Actualiza los esquemas de tablas tanto en Data sources basados en proveedor como en orígenes de datos estructurados (sí, ¡incluso para consultas M!)
- Crea diagramas del Data model
- Crea scripts DAX que te permiten editar varias medidas u otros objetos calculados en un único documento
- Graba scripts de C# y guárdalos como macros (antes conocidas como "Acciones personalizadas")
- Integración con el Analizador VertiPaq
- Depurador de DAX
- Integración con el Optimizador de DAX
- Acciones de código para refactorizar fácilmente tus expresiones DAX.

\*=Solo cuando estés conectado a una instancia de Analysis Services o Power BI
