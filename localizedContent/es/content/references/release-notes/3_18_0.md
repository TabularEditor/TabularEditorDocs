## Novedades en 3.18.0

Consulta nuestra [entrada del blog de la versión](https://blog.tabulareditor.com/2024/10/31/tabular-editor-3-october-2024-release) para obtener un breve resumen de las actualizaciones más importantes de este lanzamiento.

- Nuestra **integración con el Optimizador de DAX** ya ha salido de la vista previa y, además, hemos realizado algunas actualizaciones, de modo que ahora es compatible con las [nuevas métricas de evaluación que formaban parte del Optimizador de DAX 1,2](https://www.tabulartools.com/blog/introducing-new-evaluation-metrics-in-dax-optimizer-1-2/). Y, por cierto: todos los usuarios de la **Edición Enterprise** de Tabular Editor 3 ahora disponen de 4 ejecuciones gratuitas del Optimizador de DAX al día. Descarga Tabular Editor 3.18.0 desde uno de los enlaces anteriores y obtén tu código de canje del Optimizador de DAX a través de la opción de menú **Ayuda > Acerca de Tabular Editor**.
- Nos alegra presentar una nueva funcionalidad en todos los editores de DAX de Tabular Editor 3: **Acciones de código**. Esta funcionalidad de productividad ofrece de forma discreta sugerencias para mejorar tu código DAX y te permite aplicarlas con un solo clic. Por ejemplo, la funcionalidad puede usarse para eliminar variables sin usar, seguir las mejores prácticas para las referencias a columnas y medidas, reescribir los filtros de tabla como [filtros de columnas más eficientes](https://www.sqlbi.com/articles/filter-columns-not-tables-in-dax/), y [mucho, mucho más](xref:code-actions). ¡Tenemos previsto añadir más acciones en próximas versiones, así que permanece atento!
- Tras actualizar a la [versión más reciente de la biblioteca cliente de Microsoft AMO/TOM](https://www.nuget.org/packages/Microsoft.AnalysisServices/), cambió el comportamiento de la opción de autenticación **Integrada**: si el sistema operativo Windows ya está conectado a una cuenta de Entra ID, ya no se muestra el selector de cuenta de Entra ID ni la ventana emergente de MFA. Por ello, hemos añadido una nueva opción de autenticación, **Microsoft Entra MFA**, que hará que aparezca el selector de cuenta/ventana emergente de MFA, para que puedas elegir una cuenta distinta a la que estás usando en Windows, si lo necesitas.

## Mejoras en 3.18.0

- Las preferencias de la interfaz de usuario se han ampliado para permitir ocultar en la barra de título los detalles del origen del modelo, lo que ayuda a proteger información confidencial durante grabaciones y presentaciones.
- Las preferencias de la interfaz de usuario ahora incluyen una nueva sección de "Elementos recientes". Esta sección permite personalizar el número de archivos y modelos recientes que se muestran en el menú Archivo, así como los servidores que se muestran en las conexiones recientes. Las nuevas preferencias también permiten borrar estos elementos.
- El Analizador semántico ahora muestra una advertencia si una variable se usa directamente como primer argumento de [`CALCULATE`](https://dax.guide/CALCULATE) o [`CALCULATETABLE`](https://dax.guide/CALCULATETABLE). Dado que las variables no se ven afectadas por transiciones de contexto ni por contextos de filtro modificados, es muy probable que usar una variable de esta forma no sea intencional.
- Nuestro editor SQL ahora utiliza un esquema de colores mejorado, lo que facilita distinguir las palabras clave de SQL de los identificadores.
- Hemos actualizado AMO/TOM y otras dependencias a sus versiones más recientes.
- Ahora usamos el nivel de compatibilidad 1606 para los nuevos modelos semánticos de Power BI / Fabric. En este nivel, está disponible una nueva propiedad del modelo, [ValueFilterBehavior](https://learn.microsoft.com/en-us/dotnet/api/microsoft.analysisservices.tabular.valuefilterbehaviortype?view=analysisservices-dotnet), que puedes usar para controlar el comportamiento de _auto-exist_ de [`SUMMARIZECOLUMNS`](https://dax.guide/SUMMARIZECOLUMNS).
- Hemos añadido compatibilidad con la colección TOM [ChangedProperties](https://learn.microsoft.com/en-us/dotnet/api/microsoft.analysisservices.tabular.changedproperty?view=analysisservices-dotnet) (en tablas, medidas, columnas y algunos otros objetos). Esta colección se utiliza en modelos compuestos o Direct Lake para indicar que una propiedad (como el **Name** de una columna) ha cambiado y ya no debe sincronizarse desde el origen subyacente.
- Hemos actualizado el Analizador VertiPaq a la versión más reciente [(1.7.2)](https://github.com/sql-bi/VertiPaq-Analyzer/releases/tag/v1.7.2), que corrige un problema relacionado con la extracción de estadísticas en un modelo que contiene parámetros de campo/columnas de "Agrupar por".
- Hemos actualizado nuestra versión del conector .NET de Snowflake a la versión 4.1.0. Esto corrige las siguientes vulnerabilidades: [CVE-2023-34230](https://nvd.nist.gov/vuln/detail/CVE-2023-34230) y [CVE-2023-51662](https://nvd.nist.gov/vuln/detail/CVE-2023-51662).
- El componente de diagramas ahora debería ser considerablemente más rápido al trabajar con diagramas de gran tamaño con muchas relaciones.
- Ahora se pueden ocultar columnas en la Vista previa de datos haciendo clic con el botón derecho en el encabezado de la columna y seleccionando "Ocultar esta columna". Para mostrar las columnas ocultas, haz clic con el botón derecho en un encabezado de columna y selecciona "Selector de columnas". Esto abrirá una ventana desde la que podrás acceder a las columnas ocultas anteriormente.

## Correcciones de errores en 3.18.0

- Se corrigió un error por el que la lista desplegable "Servidor" del cuadro de diálogo **Cargar modelo semántico desde la base de datos** no mostraba los servidores utilizados recientemente.
- La configuración de la Vista previa de datos (orden de clasificación, filtros, etc.) ahora se conserva tras actualizar la Vista previa, lo que corrige un error anterior por el que estos ajustes no se mantenían.
- Se corrigió un bloqueo que podía producirse al deshacer (Ctrl+Z) durante o inmediatamente después de usar la refactorización "Cambiar nombre" en un editor de DAX.
- La interfaz principal ahora se bloquea correctamente para evitar interacciones del usuario mientras se muestra el cuadro de diálogo "Espera, por favor", por ejemplo, al realizar una actualización del esquema de una tabla.
- El Analizador semántico ya no debería mostrar una advertencia al hacer referencia a la medida interna "FormatString" en una expresión DAX. Como consecuencia, ahora es posible depurar consultas que contengan esas referencias (aunque actualmente no se puede depurar la propia expresión FormatString); consulta [#1377](https://github.com/TabularEditor/TabularEditor3/issues/1377).

## Problemas conocidos en 3.18.0

- Si te aparece el mensaje de error **Can't obtain account information for '(e-mail)' while trying to refresh the token.** normalmente ayuda volver a realizar la operación. Este es un problema de las versiones más recientes de las bibliotecas cliente AMO/TOM. También deberías poder evitar este problema eligiendo la opción de autenticación **Microsoft Entra MFA**, en lugar de la opción **Integrada**, al conectarte por primera vez.

## ¿Vienes de Tabular Editor 2.x?

Mira [este vídeo](https://youtu.be/O4ATwdzCvWc) para obtener un recorrido rápido por las principales características de Tabular Editor 3. Además, asegúrate de consultar nuestra [guía de primeros pasos](https://docs.tabulareditor.com/onboarding/index.html).

**Descripción general de las principales características de Tabular Editor 3:**

- IDE totalmente personalizable, con compatibilidad con varios monitores, Hi-DPI y temas
- Nuevo y potente editor de código DAX con autocompletado, comprobación de sintaxis, plegado de código y mucho, mucho más
- \*Modo del área de trabajo, que te permite guardar los cambios en el disco y sincronizar simultáneamente los metadatos del modelo con Analysis Services
- \*Previsualiza los datos de las tablas con desplazamiento infinito, crea PivotGrids o escribe consultas DAX para explorar el modelo o probar la lógica de cálculo
- \*Programa actualizaciones de datos
- Actualizar los esquemas de las tablas tanto en el Data source del proveedor como en el origen de datos estructurado (sí, ¡incluso para consultas M!)
- Crear diagramas del Data model
- Crear scripts DAX que permitan editar varias medidas u otros objetos calculados en un único documento
- Grabar C# Scripts y guardarlos como macros (antes conocidas como "Custom Actions")
- Integración del Analizador VertiPaq
- Depurador de DAX
- Integración del Optimizador de DAX
- Acciones de código para refactorizar DAX fácilmente.

\*=Solo cuando esté conectado a una instancia de Analysis Services o Power BI
