## Mejoras en 3.1.1

- Las expresiones compartidas (expresiones de Power Query/M) ahora pueden serializarse como archivos individuales. Puedes controlar la serialización en **Herramientas > Preferencias > Guardar en carpeta** o, para un modelo existente, en **Modelo > Opciones de serialización...** ([#218](https://github.com/TabularEditor/TabularEditor3/issues/218))
- Nuevas opciones para controlar cómo se aplican las pertenencias a perspectivas a los nuevos objetos en **Herramientas > Preferencias > Operaciones de modelado** (consulta el problema [#216](https://github.com/TabularEditor/TabularEditor3/issues/216)).
- La propiedad `PrivacySetting` ahora puede excluirse de la serialización, para evitar problemas con Microsoft.AnalysisServices.Deployment. Esto se controla en **Modelo > Opciones de serialización...** (consulta el problema [#210](https://github.com/TabularEditor/TabularEditor3/issues/210)).
- Ahora puedes definir enumeraciones, estructuras y clases directamente dentro de un C# Script, lo que facilita estructurar y reutilizar el código C#.
- El Analizador VertiPaq ahora incluye una opción para recopilar estadísticas de los datos (**Herramientas > Preferencias > Analizador VertiPaq**), lo que mejora la calidad de las estadísticas; por ejemplo, cuando las columnas tienen IsAvailableInMDX = false. Consulta el problema [#190](https://github.com/TabularEditor/TabularEditor3/issues/290).
- Cuando se crea un nuevo modelo, Tabular Editor 3 ahora pregunta si el modelo debe guardarse como archivo o como carpeta la primera vez que se guarda. También puedes establecer el modo predeterminado en **Herramientas > Preferencias > General > Formato de guardado predeterminado para modelos nuevos**. Consulta el problema [#50](https://github.com/TabularEditor/TabularEditor3/issues/50).
- Mejor rendimiento de inicio (algunos usuarios han indicado mejoras adicionales en el tiempo de inicio al añadir el modificador `-multicorejit` al acceso directo a TabularEditor3.exe).
- Se reestructuró el menú "Modelo" para que opciones como "Nuevo grupo de cálculo" estén más a mano.
- Se actualizaron las bibliotecas cliente AMO/TOM a [19.26.1.7](https://www.nuget.org/packages/Microsoft.AnalysisServices.retail.amd64/).
- Los miembros del rol del modelo ahora pueden iterarse individualmente en el Best Practice Analyzer
- Se ha cambiado el nombre de "Workspace" en el menú "Ventana" a "Layout" para evitar confusiones con el [modo del área de trabajo](https://docs.tabulareditor.com/te3/tutorials/workspace-mode.html).
- Se añadió Ctrl+Mayús+D como atajo de teclado para "Deploy"
- Compatibilidad con nuevas funciones DAX ([`HASH`](https://dax.guide/hash), [`NAMEOF`](https://dax.guide/nameof), [`COLUMNSTATISTICS`](https://dax.guide/columnstatistics), [`BITAND`](https://dax.guide/bitand), [`BITLSHIFT`](https://dax.guide/bitlshift), [`BITOR`](https://dax.guide/bitor), [`BITRSHIFT`](https://dax.guide/bitrshift), [`BITXOR`](https://dax.guide/bitxor)) y con el parámetro opcional de nombre de columna en [`SELECTCOLUMNS`](https://dax.guide/selectcolumns).

## Correcciones de errores en 3.1.1

- El Explorador TOM ya no debería bloquearse ni cerrarse inesperadamente al eliminar objetos (consulta los problemas [#223](https://github.com/TabularEditor/TabularEditor3/issues/223), [#209](https://github.com/TabularEditor/TabularEditor3/issues/209)).
- Se corrigió un problema de color con los temas oscuros en los editores ([#207](https://github.com/TabularEditor/TabularEditor3/issues/207)) y en los iconos ([#215](https://github.com/TabularEditor/TabularEditor3/issues/215))
- Ahora las tablas se ordenan alfabéticamente en el cuadro de diálogo "Crear/Editar relación" (consulta el problema [#214](https://github.com/TabularEditor/TabularEditor3/issues/214)
- Se corrigió un problema con los C# Scripts que impedía guardarlos como macros (consulta el problema [#208](https://github.com/TabularEditor/TabularEditor3/issues/208)).
- Se corrigió un problema por el que las contraseñas/credenciales del Data source no se aplicaban correctamente a la base de datos del Workspace (consulta el problema [#205](https://github.com/TabularEditor/TabularEditor3/issues/205)).
- La casilla "Seleccionar todas las columnas" ya no debería provocar un cierre inesperado (consulta el problema [#204](https://github.com/TabularEditor/TabularEditor3/issues/204)).
- Ahora Tabular Editor siempre busca en el directorio de instalación cuando un script hace referencia a DLL externas (mediante la directiva de precompilación `#r`); consulta el problema [#192](https://github.com/TabularEditor/TabularEditor3/issues/192).
- Se corrigió un error en el esquema lingüístico que hacía que Q&A dejara de funcionar en algunos casos
- Al filtrar una columna de fecha en la Vista previa de datos ya no debería producirse un cierre inesperado
- Al seleccionar todas las perspectivas en el menú desplegable de perspectivas del Explorador TOM, ya no se muestran los objetos que no están incluidos en ninguna perspectiva. También se corrigió un problema por el que el cuadro desplegable no actualizaba los nombres de las perspectivas.
- Ahora puedes usar ciertas palabras clave de DAX como variables (`WEEK`, `DAY`, `INTEGER`, etc.) sin mostrar ningún error.
- Se corrigió un problema por el que las opciones de serialización no se guardaban la primera vez que se guardaba un modelo.
- Se corrigió un problema con el analizador de DAX que provocaba un cierre inesperado al cargar un modelo (consulta [#229](https://github.com/TabularEditor/TabularEditor3/issues/229)).

## Vienes de Tabular Editor 2.x?

Mira [este vídeo](https://www.youtube.com/watch?v=pt3DdcjfImY) para hacerte una idea de las nuevas funciones de Tabular Editor 3.

**Descripción general de las principales funciones de Tabular Editor 3:**

- IDE totalmente personalizable, con compatibilidad con varios monitores, Hi-DPI y temas
- Nuevo y potente editor de código DAX con autocompletado, comprobación de sintaxis, plegado de código y mucho, mucho más
- \*Modo del área de trabajo, que te permite guardar tus cambios en el disco y sincronizar simultáneamente los metadatos del modelo con Analysis Services
- \*Previsualiza los datos de las tablas con desplazamiento infinito; crea PivotGrids o escribe consultas DAX para explorar el modelo o probar la lógica de cálculo
- \*Programar actualizaciones de datos
- Actualizar los esquemas de las tablas tanto en el Data source del proveedor como en los orígenes de datos estructurados (¡sí, incluso para consultas M!)
- Crear diagramas del Data model
- Crear scripts DAX que te permitan editar varias medidas u otros objetos calculados en un único documento
- Grabar C# Scripts y guardarlos como macros (antes conocidas como "Acciones personalizadas")
- Integración con el Analizador VertiPaq

\*=Solo mientras estés conectado a una instancia de Analysis Services o Power BI
