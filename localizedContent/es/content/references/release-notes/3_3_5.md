## ¡Importante!

**NOTA**: Tabular Editor 3.3.0 (y versiones posteriores) usa .NET 6. Es posible que deba instalar el [.NET 6.0 Runtime for Desktop](https://dotnet.microsoft.com/en-us/download/dotnet/6.0/runtime) antes de poder iniciar Tabular Editor 3.3.0 (o una versión posterior). A partir de la versión 3.3.1, el instalador de Tabular Editor 3 descargará e instalará estos requisitos previos si no están presentes.

## Mejoras en 3.3.5

- Nueva pantalla de inicio e icono, en línea con la nueva marca de nuestro sitio web.
- TOM actualizado a [19.48.0](https://www.nuget.org/packages/Microsoft.AnalysisServices.NetCore.retail.amd64).
- Tabular Editor 3 ahora incluye un conjunto de [directivas configurables](https://docs.tabulareditor.com/common/policies.html), que permiten a los equipos de TI controlar determinadas funciones de Tabular Editor 3.
- Ahora se pueden bloquear los anchos de columna en las vistas mensajes, macros y BPA.
- Se mejoró la Vista de macros con un menú contextual y botones en la barra de herramientas para abrir y editar macros.
- Se añadió compatibilidad del analizador semántico y autocompletado para las funciones DAX [`EVALUATEANDLOG`](https://dax.guide/evaluateandlog) y [`SAMPLEAXISWITHLOCALMINMAX`](https://dax.guide/sampleaxiswithlocalminmax).
- La opción "Revertir" del menú Archivo ahora también está disponible cuando se cargó un modelo desde el disco.
- El Depurador de DAX ahora admite más tipos de consultas DAX; en concreto, ahora es posible depurar valores de celda cuando la consulta usa [`ADDCOLUMNS`](https://dax.guide/addcolumns), [`CALCULATETABLE`](https://dax.guide/calculatetable), [`SAMPLE`](https://dax.guide/sample), [`SAMPLEAXISWITHLOCALMINMAX`](https://dax.guide/sampleaxiswithlocalminmax), [`SUBSTITUTEWITHINDEX`](https://dax.guide/substitutewithindex) y [`SUMMARIZE`](https://dax.guide/summarize). Esto debería garantizar que la mayoría de los tipos de consultas generadas por PBI Desktop puedan usarse con el depurador de TE3. Tenga en cuenta que solo se pueden depurar valores de celda que provengan de medidas explícitas.
- Se añadió la opción "Entrar en la selección" en el Depurador de DAX para facilitar la navegación. Use el atajo de teclado (Ctrl+B) o haga clic con el botón derecho en cualquier parte del código y elija "Entrar en la selección" para navegar.
- Mejor visualización de objetos con errores/advertencias en el Explorador TOM. Mejores iconos en la pestaña de Actualización de datos.

## Correcciones de errores en 3.3.5

- Detección más fiable del nivel Developer (D1) de Azure AS, que a veces impedía a los clientes de la Edición Business acceder a la instancia de AAS.
- Permite cambiar el nombre de los roles en el modo restringido de PBI Desktop; consulte el [#624](https://github.com/TabularEditor/TabularEditor3/issues/624).
- Corrección: no se podía establecer SortByColumn; consulte el [#625](https://github.com/TabularEditor/TabularEditor3/issues/625).
- Se corrigió un problema que impedía que Tabular Editor se conectara a Data sources ODBC cuando las propiedades de la cadena de conexión se especificaban como "Extended Properties" en la cadena de conexión.
- Usar la cláusula TOP para limitar filas en Snowflake ODBC de forma predeterminada.
- Se corrigió un problema por el que el indicador de espera se superponía al cuadro de diálogo "SelectItem" al ejecutarse como script o macro. Ver [#622](https://github.com/TabularEditor/TabularEditor3/issues/622).
- Se corrigió un problema con la forma en que el depurador gestiona (blank) y los valores binarios en los filtros.
- Cambio automático a `Protocol Format=xml` si la conexión falla debido a un XML binario no válido. Ver [#637](https://github.com/TabularEditor/TabularEditor3/issues/637).
- Se corrigió un problema por el que "Batch rename" no estaba disponible en modelos de Power BI Desktop.
- Se añadió compatibilidad del analizador de M con caracteres especiales; consulta [#638](https://github.com/TabularEditor/TabularEditor3/issues/638).
- Tratar 'version' como una palabra reservada de DAX (es decir, ponerla siempre entre comillas y mostrar un error si se usa sin comillas o como nombre de variable).

## Vienes de Tabular Editor 2.x?

Mira [este vídeo](https://www.youtube.com/watch?v=pt3DdcjfImY) para hacerte una idea de las nuevas funciones de Tabular Editor 3. Además, no dejes de consultar nuestra [guía de incorporación](https://docs.tabulareditor.com/onboarding/index.html).

**Descripción general de las principales funcionalidades de Tabular Editor 3:**

- IDE totalmente personalizable, con compatibilidad con varios monitores, Hi-DPI y temas
- Nuevo y potente editor de código DAX con autocompletado, comprobación de sintaxis, plegado de código y mucho, mucho más
- \*Modo del área de trabajo, que te permite guardar tus cambios en disco y sincronizar los metadatos del modelo con Analysis Services al mismo tiempo
- \*Previsualiza los datos de las tablas con desplazamiento infinito, crea PivotGrids o escribe consultas DAX para explorar el modelo o probar la lógica de cálculo
- \*Programar actualizaciones de datos
- Actualizar los esquemas de las tablas tanto en los Data sources del proveedor como en los orígenes de datos estructurados (sí, incluso para consultas M)
- Crear diagramas del Data model
- Crear scripts DAX que te permiten editar varias medidas u otros objetos calculados en un único documento
- Grabar C# Scripts y guardarlos como macros (antes conocidos como "Custom Actions")
- Integración con el Analizador VertiPaq
- \*Depurador de DAX

\*=Solo cuando estés conectado a una instancia de Analysis Services o Power BI
