## Mejoras en 3.2.3

- TOM actualizado a [19.36.0](https://www.nuget.org/packages/Microsoft.AnalysisServices.retail.amd64/)
- Usa el nivel de compatibilidad 1569 para los nuevos modelos de PBI
- Permitir la serialización de CalculationGroups y CalculationItems mediante ExportProperties. Consulta [#976](https://github.com/TabularEditor/TabularEditor/issues/976)
- Permitir especificar MemberID en .AddExternalMember y .AddWindowsMember; consulta [#484](https://github.com/TabularEditor/TabularEditor3/issues/484) y [#483](https://github.com/TabularEditor/TabularEditor3/issues/483)
- Permitir especificar XMLA sin procesar en llamadas a ExecuteCommand, como se comenta en [#495](https://github.com/TabularEditor/TabularEditor3/issues/495)
- Permitir copiar la consulta MDX de Pivot Grid al portapapeles al hacer clic con el botón derecho en cualquier parte de la Pivot Grid; consulta [#494](https://github.com/TabularEditor/TabularEditor3/issues/494).
- Mejora de la vista de BPA (muestra gravedad y categoría), así como de la información sobre herramientas. Consulta [#489](https://github.com/TabularEditor/TabularEditor3/issues/489).
- Cambiar el valor predeterminado de la configuración "Read statistics from data" del Analizador VertiPaq a "false" para alinearla con DAX Studio / Bravo.
- Se añadió una nueva opción para que las ventanas emergentes de autocompletado sean menos intrusivas (solo aparecen tras introducir una letra); consulta [#459](https://github.com/TabularEditor/TabularEditor3/issues/459/). Encontrarás esta configuración en **Herramientas > Preferencias > Editor de DAX > Code Assist**.

## Correcciones de errores en 3.2.3

- Se corrigió un problema por el que los permisos de tabla no se deserializaban correctamente. Consulta [#446](https://github.com/TabularEditor/TabularEditor3/issues/446).
- Se corrigió un cierre inesperado durante el inicio con ciertos diseños; consulta la incidencia [#455](https://github.com/TabularEditor/TabularEditor3/issues/455).
- Se corrigió un problema que podía provocar que la aplicación se cerrara inesperadamente cuando las medidas u otros objetos tenían llaves en sus nombres.
- Quita las propiedades marcadas con asterisco (claves, contraseñas) cuando "Incluir información confidencial" esté configurado como false. Ver [#479](https://github.com/TabularEditor/TabularEditor3/issues/479).
- Se corrigió la serialización al guardar en una carpeta al usar nombres de archivo/carpeta no válidos como nombres de objeto, o cuando un nombre de objeto contiene un espacio al final. Similar a [#952](https://github.com/TabularEditor/TabularEditor/issues/952) y [#978](https://github.com/TabularEditor/TabularEditor/issues/978).
- Se muestran todas las opciones de actualización si se selecciona al menos 1 tabla que no sea una tabla calculada; consulta el problema [#487](https://github.com/TabularEditor/TabularEditor3/issues/487)
- Se corrigió un problema de linaje de columnas del analizador semántico con UNION; ver [#482](https://github.com/TabularEditor/TabularEditor3/issues/482).
- Se corrigió un problema por el que los atajos no funcionaban en ventanas flotantes; ver [#480](https://github.com/TabularEditor/TabularEditor3/issues/480).
- No mostrar la propiedad "EnableRefreshPolicy" para tablas calculadas, grupos calculados o tablas DirectQuery; ver [#470](https://github.com/TabularEditor/TabularEditor3/issues/470).
- Se corrigió un problema del Asistente para importar tablas al crear nuevos orígenes de datos estructurados en algunos escenarios (por ejemplo, Data sources de Oracle).
- Se corrigió un cierre inesperado al intentar agregar un Data source ODBC estructurado mediante el asistente de importación.
- Se corrigió un problema de selección del diagrama al hacer clic en una columna de otra tabla.
- Se corrigió un problema por el que el Explorador TOM no se actualizaba para mostrar el tipo de objeto correcto de una partición cuando se cambiaba su modo
- Se corrigió el formato de las medidas mostradas en Pivot Grid; ver [#492](https://github.com/TabularEditor/TabularEditor3/issues/492)
- El autocompletado no debería insertar paréntesis si ya hay un paréntesis de apertura junto al cursor; ver [#459](https://github.com/TabularEditor/TabularEditor3/issues/459/)
- Se corrigió el autocompletado del segundo parámetro de CONVERT (tipo de datos); ver [#474](https://github.com/TabularEditor/TabularEditor3/issues/474).
- Al iniciar una nueva operación de cambio de nombre, debería finalizarse cualquier operación en curso; ver [#461](https://github.com/TabularEditor/TabularEditor3/issues/461/).
- Se corrigió un problema por el que varias instrucciones EVALUATE no respetaban ORDER BY al ejecutar solo la selección. También se corrigió un problema por el que ORDER BY no funcionaba en tablas con nombres que también son funciones DAX. Ver [#472](https://github.com/TabularEditor/TabularEditor3/issues/472)

## ¿Vienes de Tabular Editor 2.x?

Mira [este vídeo](https://www.youtube.com/watch?v=pt3DdcjfImY) para hacerte una idea de las nuevas características de Tabular Editor 3. Además, asegúrate de consultar nuestra [guía de introducción](https://docs.tabulareditor.com/onboarding/index.html).

**Descripción general de las principales características de Tabular Editor 3:**

- IDE totalmente personalizable, con compatibilidad con varios monitores, soporte para Hi-DPI y temas
- Nuevo y potente editor de código DAX con autocompletado, comprobación de sintaxis, plegado de código y mucho, mucho más
- \*Modo del área de trabajo, que te permite guardar los cambios en el disco y sincronizar al mismo tiempo los metadatos del modelo con Analysis Services
- \*Previsualiza los datos de la tabla con desplazamiento infinito, crea PivotGrids o escribe consultas DAX para explorar el modelo o probar la lógica de cálculo
- \*Programa las actualizaciones de datos
- Actualiza los esquemas de las tablas tanto en los Data sources del proveedor como en los orígenes de datos estructurados (sí, ¡incluso para consultas M!)
- Crea diagramas del Data model
- Crea scripts DAX que te permiten editar varias medidas u otros objetos calculados en un único documento
- Graba C# Scripts y guárdalos como macros (antes conocidas como "Custom Actions")
- Integración con el Analizador VertiPaq
- \*Depurador de DAX

\*=Solo disponible cuando estás conectado a una instancia de Analysis Services o de Power BI
