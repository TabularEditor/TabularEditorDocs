## Novedades en 3.17.0

Echa un vistazo a nuestro [blog de lanzamiento](https://blog.tabulareditor.com/2024/08/14/tabular-editor-3-august-2024-release) para ver un breve resumen de las actualizaciones más importantes de esta versión.

- Nos alegra decirte que Tabular Editor 3 ya está disponible para [.NET 8](https://learn.microsoft.com/en-us/dotnet/core/whats-new/dotnet-8/overview). Seguiremos dando soporte a .NET 6 hasta finales de 2024; aun así, te recomendamos cambiar a la versión de .NET 8 lo antes posible. En nuestras pruebas, la versión de .NET 8 se inicia aproximadamente un 20 % más rápido y, además, consume menos memoria.
- Las versiones portátiles de Tabular Editor 3 ahora incluyen el runtime de .NET correspondiente, por lo que ya no necesitas instalar por separado el runtime de .NET 6/8 para Windows Desktop.

> [!IMPORTANT]
> .NET 6 dejará de recibir soporte en [noviembre de 2024](https://dotnet.microsoft.com/en-us/platform/support/policy/dotnet-core). Si ahora no puedes instalar el [runtime de escritorio de .NET 8, versión 8.0](https://dotnet.microsoft.com/en-us/download/dotnet/8.0/runtime), ponte en contacto con tu departamento de TI para planificarlo con antelación, ya que después de diciembre de 2024 dejaremos de ofrecer compilaciones de Tabular Editor 3 para .NET 6. Como alternativa, usa nuestras compilaciones portátiles, que incluyen el runtime de .NET necesario.

## Mejoras en 3.17.0

- Los resultados de la Consulta DAX y la Vista previa de tabla ahora muestran el "formato de hora larga" para los valores de fecha y hora, en lugar del "formato de hora corta". De este modo, los segundos también se muestran en la cuadrícula de resultados.
- Las expresiones compartidas ahora pueden excluirse de la implementación mediante el Asistente de implementación. Esta mejora te permite conservar tal cual las expresiones compartidas existentes en el servidor de destino al implementar un modelo, incluso si en el modelo que se está implementando se han creado, eliminado o modificado expresiones compartidas.
- La ventana **Locals** del Depurador de DAX ahora te permite "inspeccionar" resultados, de forma similar a la ventana **Watch**. De forma predeterminada, los resultados del inspector se muestran en un cuadro de diálogo emergente en lugar de como una nueva Consulta DAX, pero puedes controlar este comportamiento en **Tools > Preferencias > Depurador de DAX**. Haz clic en el icono de la lupa junto al valor en la vista **Locals** o **Watch** para inspeccionar el valor. Consulta la [documentación actualizada del Depurador de DAX](xref:dax-debugger#locals) para obtener más información.
- Al depurar un predicado escalar en un argumento de filtro de [`CALCULATE`](https://dax.guide/calculate) o [`CALCULATETABLE`](https://dax.guide/calculatetable), la vista **Locals** ahora muestra detalles adicionales sobre el predicado escalar y la expresión de filtro resultante. Consulta la [documentación actualizada del Depurador de DAX](xref:dax-debugger#scalar-predicates) para obtener más información.
- Los scripts DAX ahora se pueden aplicar aunque el script contenga errores semánticos. Se mostrará una advertencia si hay errores semánticos, para que el usuario final pueda decidir si quiere continuar con los errores actuales o cancelar y corregirlos. Los errores de sintaxis, en cambio, seguirán impidiendo que se aplique el script.
- Se añadió compatibilidad de autocompletado para algunas funciones DAX `INFO.*` adicionales.
- La característica de Integración del Optimizador de DAX (versión preliminar) ahora te permite cambiar entre cuentas personales o de grupo en escenarios de licencias de grupo. Puedes elegir qué cuenta quieres usar mediante el menú desplegable **Options**. Consulta también [#1288](https://github.com/TabularEditor/TabularEditor3/issues/1288). Además, ahora puedes cambiar el estado _Fixed_ o _Ignored_ de un problema directamente desde la vista de resultados del Optimizador de DAX.
- El Asistente de importación de tablas y la función Actualizar esquema de tabla ahora admiten **Databricks Multicloud**, además de **Azure Databricks**. Para obtener más información, consulta el debate en [#1347](https://github.com/TabularEditor/TabularEditor3/discussions/1347).
- Se ampliaron las preferencias del Analizador VertiPaq para permitir configurar el tamaño del lote de columnas. Esta preferencia controla el número de columnas que se procesan a la vez durante el análisis de estadísticas de columnas y puede encontrarse en **Herramientas > Preferencias > Analizador VertiPaq**. El valor predeterminado de esta preferencia es 50.
- El formato personalizado aplicado a los Pivot Grid (iconos, barras de datos, celdas resaltadas, etc.) ahora se conserva cuando el diseño de Pivot Grid se guarda en un archivo y posteriormente se vuelve a cargar.
- Los Pivot Grid ahora también conservan cualquier ordenación aplicada a los campos cuando se actualiza el Pivot Grid o cuando el diseño del Pivot Grid se guarda en un archivo y posteriormente se vuelve a cargar.

## Correcciones de errores en 3.17.0

- Se corrigió un error introducido en 3.16.0, que provocaba que la aplicación se quedara congelada durante varios segundos tras cargar un archivo de diagrama (.te3diag)
- Se corrigió un problema que impedía guardar las opciones del cuadro de diálogo "Buscar/Reemplazar" entre sesiones al usar el botón "Buscar todo". Ahora, las últimas preferencias de búsqueda se conservarán en la siguiente sesión.
- Se corrigió un problema que impedía implementar la eliminación de Data sources, roles y miembros de rol en un modelo existente. Anteriormente, las casillas correspondientes del Asistente de implementación se deshabilitaban cuando no había Data sources ni roles en el modelo. Ahora, estas casillas permanecen habilitadas al implementar cambios en un modelo existente, lo que permite implementar la eliminación de estos objetos.
- Se corrigió un problema que impedía abrir archivos con la extensión .ovpax como archivo o como modelo. Ahora se admite abrir archivos y modelos .ovpax.
- Se corrigió un error por el que el cuadro de diálogo de archivo podía sugerir un tipo de archivo incorrecto al exportar archivos ofuscados desde el Analizador VertiPaq. Ahora, al exportar un archivo ofuscado, el cuadro de diálogo para guardar sugerirá correctamente usar la extensión ".ovpax".
- Se corrigió un error por el que los botones de la barra de herramientas seguían deshabilitados cuando el Editor de expresiones se acoplaba como una ventana de herramienta en lugar de como una pestaña de documento.
- Se corrigió un error que impedía abrir modelos semánticos guardados como ".tmdl" con [archivos de Proyecto de Power BI (PBIP)](https://learn.microsoft.com/en-us/power-bi/developer/projects/projects-overview).
- Se corrigió un error que provocaba que la aplicación se quedara bloqueada durante la activación manual.
- El grabador de macros ya no debería generar código duplicado. Además, ahora también debería generar código cuando se eliminan objetos.
- La configuración de **formato automático** de los **Editores de DAX**, incluida la **Consulta DAX**, el **Script DAX** y las expresiones DAX en el Editor de expresiones, ahora se aplica al instante, sin necesidad de reiniciar la aplicación.
- Se corrigió un problema por el que, al desmarcar la opción **Cifrar conexión**, no se deshabilitaba el cifrado en la cadena de conexión para Data sources de Microsoft SQL Server.
- Se corrigió un error por el que el nombre de usuario y la contraseña de ODBC no se añadían a la cadena de conexión al importar tablas o actualizar el esquema de la tabla. Este problema impediría a los usuarios importar tablas o actualizar el esquema de la tabla al usar conexiones ODBC con algunos Data sources, como Snowflake u Oracle.
- Se corrigió un error por el que no era posible enumerar los DSN de ODBC al importar tablas si no había DSN de usuario en el equipo.
- Al consultar información del esquema desde un origen ODBC, ahora usamos varias alternativas, lo que debería mejorar la compatibilidad con más controladores ODBC, como el controlador "Oracle in OraClient12Home1".
- Se corrigió un error por el que los Data sources referenciados en consultas M aparecían como Data sources implícitos en el Asistente para importar tablas.
- Se corrigió un error por el que algunos botones de la barra de herramientas de Pivot Grid seguían deshabilitados después de recargar un Pivot Grid desde un archivo.
- Se corrigió un error por el que las constantes en expresiones M (como `Order.Ascending`) provocaban el error **Unknown identifier** al usar la función **Update Table Schema**. Consulta [#1362](https://github.com/TabularEditor/TabularEditor3/issues/1362).
- Al usar la función DAX `DIVIDE` con un tercer argumento no numérico, el analizador de DAX ahora determina correctamente que el tipo de datos resultante es Variant.
- Se corrigió un error que impedía que Tabular Editor cargara los metadatos del modelo desde una carpeta TMDL sin el archivo database.tmdl.

> [!NOTE]
> Microsoft ha anunciado que [TMDL está en disponibilidad general](https://powerbi.microsoft.com/en-us/blog/announcing-general-availability-of-tabular-model-definition-language-tmdl/) desde el dos de agosto de 2024. Ten en cuenta que Tabular Editor 3 está usando una versión de la [biblioteca cliente AMO/TOM](https://www.nuget.org/packages/Microsoft.AnalysisServices.NetCore.retail.amd64/19.79.1.1), que se publicó antes de que se anunciara que TMDL estaba en GA. Esta versión debería seguir siendo totalmente compatible con TMDL; sin embargo, recomendamos mantener una copia de seguridad de tus archivos TMSL (.bim), por si acaso.

## ¿Vienes de Tabular Editor 2.x?

Mira [este vídeo](https://www.youtube.com/watch?v=pt3DdcjfImY) para hacerte una idea de las nuevas funciones de Tabular Editor 3. Además, no dejes de consultar nuestra [guía de incorporación](https://docs.tabulareditor.com/onboarding/index.html).

**Descripción general de las principales características de Tabular Editor 3:**

- IDE totalmente personalizable, con compatibilidad con varios monitores, Hi-DPI y temas
- Nuevo y potente editor de código DAX con autocompletado, comprobación de sintaxis, plegado de código y mucho, mucho más
- \*Modo del área de trabajo, que te permite guardar los cambios en disco y sincronizar los metadatos del modelo con Analysis Services a la vez
- \*Previsualiza los datos de las tablas con desplazamiento infinito, crea PivotGrids o escribe Consultas DAX para explorar el modelo o probar la lógica de cálculo
- \*Programa actualizaciones de datos
- Actualiza los esquemas de tabla tanto en Data sources de proveedor como en orígenes de datos estructurados (sí, incluso en consultas M)
- Crea diagramas del Data model
- Crea Scripts DAX que te permiten editar varias medidas u otros objetos calculados en un único documento
- Graba C# Scripts y guárdalos como macros (antes conocidas como "Custom Actions")
- Integración del Analizador VertiPaq
- Depurador de DAX

\*=Solo cuando esté conectado a una instancia de Analysis Services o Power BI
