## Problemas conocidos

- El Asistente para importar tablas no se incluyó en esta versión debido a algunos errores de alta prioridad que era necesario corregir. ¡Estará disponible lo antes posible! Como solución alternativa, simplemente crea una tabla nueva y especifica la consulta de la partición (SQL o M). Después, haz clic con el botón derecho en la tabla y elige "Actualizar esquema de la tabla..." para importar las columnas.
- Consulta la [hoja de ruta](https://github.com/TabularEditor/TabularEditor3/issues/12) para obtener más información.

## Novedades de la versión 3.0.3:

- Al hacer clic con el botón derecho en un elemento de la Vista de procesamiento, ahora puedes volver a ponerlo en cola o, si el elemento representa una sola tabla, abrir una vista previa de los datos de esa tabla.

## Correcciones de errores en la versión 3.0.3:

- Cuando el analizador semántico detecta un error en una columna calculada o en una tabla calculada, ya no modifica las columnas de esa tabla (antes, en algunas circunstancias, esto provocaba que se eliminaran las relaciones)
- Se corrigió un problema con el estado de habilitación de ciertos controles en el cuadro de diálogo Buscar/Reemplazar
- Varias correcciones basadas en la telemetría y en los Report de error, por ejemplo:
  - Ahora funciona copiar y pegar columnas con etiquetas de linaje
  - Pegar una tabla sobre otra existente cuando se cambia el nombre de la tabla ya no provoca un bloqueo
  - El botón "Revertir" ahora se vuelve a conectar a AS antes de actualizar el TOM local, en lugar de provocar un bloqueo
  - Hacer clic con el botón derecho en la cuadrícula de propiedades cuando está vacía ya no provoca un bloqueo
  - Y muchos, muchísimos más...
- Se corrigió un problema por el que los MRU no se guardaban correctamente en el menú desplegable "Server"
- Se corrigieron varios problemas por los que el modo del área de trabajo no cargaba correctamente desde el archivo de opciones de usuario los valores de anulación del Data source. Consulta [#28](https://github.com/TabularEditor/TabularEditor3/issues/28).
- Se corrigieron y mejoraron varios errores semánticos de DAX "fantasma" (compatibilidad con ROLLUPADDISSUBTOTAL, COUNTROWS con 0 argumentos, etc.)
- Se mejoraron el autocompletado y las ayudas emergentes para varias funciones de DAX (TOPN, TOPNPERLEVEL, parámetro de intervalo para DATEADD/DATESINPERIOD/PARALLELPERIOD), después de la palabra clave RETURN, etc.

## Vienes de Tabular Editor 2.x?

Mira [este vídeo](https://www.youtube.com/watch?v=pt3DdcjfImY) para hacerte una idea de las nuevas funcionalidades de Tabular Editor 3.

**Descripción general de las principales características de Tabular Editor 3:**

- IDE totalmente personalizable, con compatibilidad con varios monitores, HiDPI y temas
- Nuevo y potente editor de código DAX con autocompletado, comprobación de sintaxis, plegado de código y mucho, mucho más
- \*modo del área de trabajo, que te permite guardar los cambios en disco y sincronizar simultáneamente los metadatos del modelo con Analysis Services
- \*Previsualiza los datos de las tablas con desplazamiento infinito, crea PivotGrids o escribe consultas DAX para explorar el modelo o probar la lógica de cálculo
- \*Actualiza los esquemas de tablas tanto en Data source de proveedor como en origen de datos estructurado (sí, ¡incluso para consultas M!)
- \*Programa actualizaciones de datos
- Crea diagramas del Data model
- Crea scripts DAX que te permitan editar varias medidas u otros objetos calculados en un único documento
- Graba C# Script y guárdalos como macros (antes conocidas como "Acciones personalizadas")
- Integración con el Analizador VertiPaq

\*=Solo cuando estés conectado a una instancia de Analysis Services o Power BI
