## ¡Importante!

**NOTA**: Tabular Editor 3.3.0 (y posteriores) usa .NET 6. Puede que tengas que instalar el [entorno de ejecución .NET 6.0 para escritorio](https://dotnet.microsoft.com/en-us/download/dotnet/6.0/runtime) antes de poder iniciar Tabular Editor 3.3.0 (o una versión posterior). ## Mejoras en 3.3.4

- TOM actualizado a la versión [19.46.0](https://www.nuget.org/packages/Microsoft.AnalysisServices.NetCore.retail.amd64).
- Microsoft.Data.SqlClient actualizado a la versión [5.0.0](https://www.nuget.org/packages/Microsoft.Data.SqlClient).
- SciLexer.dll actualizado a v. 4.4.6, que no se ve afectado por [este CVE](https://cve.mitre.org/cgi-bin/cvename.cgi?name=2019-16294).
- Varios ajustes y mejoras menores.
- Mejoras en el cuadro de diálogo Nuevo/Editar relación: las columnas se ordenan alfabéticamente y por tipo de coincidencia. Ahora también sugerimos la "columna de destino" en función del nombre de la "columna de origen".
- Se cambió la CL predeterminada de los nuevos Datasets de Power BI a 1571 y se añadió compatibilidad con la propiedad [ObjectTranslation.Altered](https://docs.microsoft.com/en-us/dotnet/api/microsoft.analysisservices.tabular.objecttranslation.altered?view=analysisservices-dotnet), que debe establecerse en "true" para evitar que las traducciones de metadatos se sobrescriban desde el origen al crear DirectQuery sobre Datasets de AS.
- Se agregó un nuevo atajo, **Ctrl+Alt+S**, para guardar el modelo actual. Consulta [#595](https://github.com/TabularEditor/TabularEditor3/issues/595), [#561](https://github.com/TabularEditor/TabularEditor3/issues/561).
- Se agregó el atajo **Ctrl+Shift+F6** para el formateo DAX en modo Debug Comma. Consulta [#595](https://github.com/TabularEditor/TabularEditor3/issues/595).
- La detección de esquemas sin conexión ahora debería funcionar cuando parámetros como el nombre del servidor y el nombre de la base de datos se almacenan en expresiones M compartidas.

## Correcciones de errores en 3.3.4

- Se corrigió un problema por el que la opción Actualizar esquema de tabla generaba un Report indicando que se habían quitado todas las columnas; consulta [#612](https://github.com/TabularEditor/TabularEditor3/issues/612).
- Se corrigieron falsos errores del analizador semántico al usar expresiones de filtro anidadas.
- Se corrigieron falsos errores del analizador semántico al usar `NATURALINNERJOIN` / `NATURALLEFTOUTERJOIN` en tablas que están unidas mediante relaciones en lugar de por columnas comunes.
- Se corrigió un problema al intentar mostrar el cuadro de diálogo de conexión para los conectores ODBC; consulta [#448](https://github.com/TabularEditor/TabularEditor3/issues/448)
- La autenticación de AAD con Microsoft.Data.SqlClient ahora muestra un cuadro de diálogo modal (en lugar de abrir el navegador del sistema operativo)
- Se corrigió un problema en el que el botón "Avanzado" del cuadro de diálogo de conexión de SQL Server a veces provocaba un bloqueo.
- El Asistente para importar tablas ahora permite previsualizar datos en Azure Synapse (ya no se añade la sugerencia NOLOCK a la consulta de vista previa de datos)
- El instalador de 32 bits ahora debería comprobar correctamente el .NET Desktop Runtime de 32 bits
- El autocompletado de DAX ahora también puede sugerir variables escalares de tipo Variant; consulta [#615](https://github.com/TabularEditor/TabularEditor3/issues/615)
- Se corrigió un problema por el que el Depurador de DAX no podía mostrar expresiones locales o expresiones de seguimiento con ramas condicionales anidadas.
- Se incluyó SciLexer.dll en la carpeta de la aplicación en lugar de extraerlo o cargarlo en %LocalAppData%\Temp. Esto debería evitar el error "Exception has been thrown by the target of an invocation" que a veces aparece cuando una directiva de grupo impide cargar archivos DLL desde esta ubicación.
- Se corrigió un problema por el que la Cuadrícula de propiedades no podía establecer propiedades anidadas cuando había varios elementos seleccionados. Consulta [#614](https://github.com/TabularEditor/TabularEditor3/issues/614).
- Se corrigió un problema por el que la Cuadrícula de propiedades no se actualizaba para mostrar propiedades agregadas o quitadas dinámicamente; por ejemplo, al activar o desactivar la propiedad "Política de actualización incremental".
- Se corrigió un problema por el que una instancia de AAS de nivel Developer se consideraba una instancia de nivel Standard, lo que impedía la conectividad con la Edición Business de TE3
- Se corrigió un problema por el que el autocompletado de NAMEOF no mostraba todas las referencias a columnas/medidas en algunas situaciones; consulta [#620](https://github.com/TabularEditor/TabularEditor3/issues/620).
- Se corrigió un problema por el que ciertas palabras clave de DAX (START, ORDER, RETURN, etc.) no se trataban como un error cuando se usaban como referencias de tabla sin comillas. Este error también impedía el formato correcto del código que contenía una referencia entrecomillada a esas tablas.
- Se corrigió el error "4096 (0x1000) is an invalid culture identifier" — identificador de configuración regional no válido; consulta [#576](https://github.com/TabularEditor/TabularEditor3/issues/576).
- Se evitó un bloqueo en la búsqueda de Buscar/Reemplazar; consulta [#611](https://github.com/TabularEditor/TabularEditor3/issues/611).
- Se muestran todas las propiedades en modo de solo lectura; consulta [#596](https://github.com/TabularEditor/TabularEditor3/issues/596).
- Se eliminó la animación de las ventanas ancladas y desancladas; consulta [#595](https://github.com/TabularEditor/TabularEditor3/issues/595).
- No insertes automáticamente llaves en los comentarios de DAX; consulta [#597](https://github.com/TabularEditor/TabularEditor3/issues/597).
- Se corrigió un problema por el que el valor de CL predeterminado provocaba un bloqueo en el cuadro de diálogo de Preferencias cuando se establecía en un valor no estándar.

## Vienes de Tabular Editor 2.x?

Mira [este vídeo](https://www.youtube.com/watch?v=pt3DdcjfImY) para hacerte una idea de las nuevas funciones de Tabular Editor 3. Además, asegúrate de consultar nuestra [guía de introducción](https://docs.tabulareditor.com/onboarding/index.html).

**Descripción general de las principales funcionalidades de Tabular Editor 3:**

- IDE totalmente personalizable, con soporte para varios monitores, Hi-DPI y temas
- Nuevo y potente editor de código DAX con autocompletado, comprobación de sintaxis, plegado de código y mucho, mucho más
- \*Modo del área de trabajo, que te permite guardar los cambios en el disco y sincronizar los metadatos del modelo con Analysis Services al mismo tiempo
- \*Previsualiza datos de tablas con desplazamiento infinito, crea PivotGrids o escribe consultas DAX para explorar el modelo o probar la lógica de cálculo
- \*Programa actualizaciones de datos
- Actualiza los esquemas de las tablas tanto en Data sources de proveedor como en orígenes de datos estructurados (¡sí, incluso para consultas M!)
- Crea diagramas del Data model
- Crea scripts DAX que te permitan editar varias medidas u otros objetos calculados en un solo documento
- Graba C# Scripts y guárdalos como macros (antes conocidos como "Acciones personalizadas")
- Integración del Analizador VertiPaq
- \*Depurador de DAX

\*=Solo mientras estés conectado a una instancia de Analysis Services o Power BI
