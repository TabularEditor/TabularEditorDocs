## Novedades de la versión 3.1.0

- ¡Por fin ha llegado el Asistente para importar tablas! Y es mejor que nunca. Lee [este artículo](https://docs.tabulareditor.com/te3/importing-tables.html) para más información. Actualmente funciona con SQL, ODBC y OLE DB, pero háznoslo saber si necesitas compatibilidad con otros tipos de Data sources.
- La detección sin conexión (es decir, sin estar conectado a Analysis Services) de cambios en el esquema de las tablas ya funciona, incluso para particiones M y Data sources implícitos.
- Ahora se pueden mostrar u ocultar objetos en las perspectivas desde el menú contextual de TOM, igual que en TE2.
- La opción "Importar/Exportar traducciones" ya está disponible, igual que en TE2.
- Se ha añadido el cuadro de diálogo de cambio de nombre por lotes, al estilo de TE2 (aparece al seleccionar varios objetos y pulsar "F2" para cambiar el nombre).
- Se ha añadido el cuadro de diálogo Marcar como tabla de fechas, al estilo de TE2.

## Mejoras de la versión 3.1.0

- Al copiar una sola celda del resultado de una consulta DAX, ahora se da formato al valor para que se pueda pegar fácilmente en una expresión DAX; consulta [#200](https://github.com/TabularEditor/TabularEditor3/issues/200).
- Se ha añadido la opción "Eliminar relación" al menú contextual del diagrama; consulta [#195](https://github.com/TabularEditor/TabularEditor3/issues/195).
- Se ha añadido la opción "Quitar campo" al menú contextual de Pivot Grid.
- Las acciones a nivel de modelo, como Crear tabla, Actualizar, Importar tablas, etc., ahora están disponibles en el menú "Modelo".
- La propiedad AuthenticationKind de los orígenes de datos estructurados ahora incluye un menú desplegable. La propiedad Options ahora también se puede editar, y se ha expuesto la propiedad Query.
- Los elementos del menú contextual del Explorador TOM se han reorganizado para alinearse mejor con TE2.
- Ahora se pueden editar los miembros del rol directamente haciendo clic con el botón derecho en un rol.
- Ahora se pueden copiar las dependencias desde la Vista de dependencias a formato de texto/JSON, de forma similar a TE2.
- Nuevas opciones de paleta para los skins "Basic" y "Bezier" (que están basados en vectores y, por tanto, se ven mucho mejor en Hi-DPI que los de estilo VS). Encuentra las nuevas paletas en la parte inferior del menú "Window".

## Correcciones de errores en 3.1.0

- Se corrigió un problema en el cuadro de diálogo "Seleccionar base de datos" al ordenar las bases de datos; consulta [#201](https://github.com/TabularEditor/TabularEditor3/issues/201).
- Se corrigió un cierre inesperado en el cuadro de diálogo "Seleccionar carpeta de visualización"; consulta [#199](https://github.com/TabularEditor/TabularEditor3/issues/199).
- Se corrigió un cierre inesperado al cambiar desde el Workspace clásico; consulta [#198](https://github.com/TabularEditor/TabularEditor3/issues/198).
- Ahora se conservan las traducciones y la información de perspectiva al arrastrar y soltar columnas calculadas o medidas entre tablas; consulta [#194](https://github.com/TabularEditor/TabularEditor3/issues/194).
- Se corrigió un problema con saltos de línea no deseados en código M; consulta [#193](https://github.com/TabularEditor/TabularEditor3/issues/193).
- Se corrigió un problema de foco en el cuadro de diálogo Agregar tabla al diagrama; consulta [#183](https://github.com/TabularEditor/TabularEditor3/issues/183).
- Se corrigió un cierre inesperado al hacer clic con el botón derecho en varias zonas de un Pivot Grid; consulta [#175](https://github.com/TabularEditor/TabularEditor3/issues/175).
- Se corrigió un problema por el que trazas externas de cambios que interferían provocaban actualizaciones TOM no deseadas.
- Se corrigió un problema en el modo del área de trabajo que mostraba una advertencia sobre "Cambios sin guardar"
- Se eliminó la propiedad memberId al desplegar roles; consulta [TE2 #906](https://github.com/TabularEditor/TabularEditor/issues/906).
- Se corrigió un problema con las operaciones de copiar/pegar del Explorador TOM (posiblemente relacionado con [#184](https://github.com/TabularEditor/TabularEditor/issues/184)).

## Vienes de Tabular Editor 2.x?

Mira [este vídeo](https://www.youtube.com/watch?v=pt3DdcjfImY) para hacerte una idea de las nuevas funcionalidades de Tabular Editor 3.

**Resumen de las principales funcionalidades de Tabular Editor 3:**

- IDE totalmente personalizable, con compatibilidad con varios monitores, Hi-DPI y temas
- Nuevo y potente editor de código DAX con autocompletado, comprobación de sintaxis, plegado de código y mucho, mucho más
- \*Modo del área de trabajo, que te permite guardar los cambios en disco y sincronizar los metadatos del modelo con Analysis Services simultáneamente
- \*Previsualiza los datos de las tablas con desplazamiento infinito, crea PivotGrids o escribe consultas DAX para explorar el modelo o probar la lógica de cálculo
- \*Programar actualizaciones de datos
- Actualizar los esquemas de las tablas tanto en los Data source del proveedor como en los orígenes de datos estructurados (sí, ¡incluso para consultas M!)
- Crear diagramas del Data model
- Crear scripts DAX que te permitan editar varias medidas u otros objetos calculados en un solo documento
- Grabar C# Scripts y guardarlos como macros (antes conocidas como "Custom Actions")
- Integración con el Analizador VertiPaq

\*=Solo mientras estés conectado a una instancia de Analysis Services o Power BI
