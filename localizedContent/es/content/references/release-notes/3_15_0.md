### Requisitos previos

Tabular Editor 3 usa .NET 6. Si estás usando la versión portátil, es posible que tengas que instalar manualmente el [.NET 6,0 Runtime for Desktop](https://dotnet.microsoft.com/en-us/download/dotnet/6.0/runtime) antes de poder iniciar Tabular Editor 3.

## Novedades en 3.15.0

Consulta nuestra [entrada del blog de la versión](https://blog.tabulareditor.com/2024/04/24/tabular-editor-3-april-2024-release/) para obtener un breve resumen de las actualizaciones más importantes de esta versión.

- Este mes incorporamos una nueva función en **Vista previa**: la **integración con el Optimizador de DAX**. Si tienes una cuenta del Optimizador de DAX, puedes conectarte a tus Workspaces, cargar archivos VPAX y ver los resultados de [Optimizador de DAX](https://daxoptimizer.com) directamente en Tabular Editor. [Más información](https://docs.tabulareditor.com/features/dax-optimizer-integration.html).
- Ahora admitimos agregar [expresiones de definición de cobertura de datos](https://learn.microsoft.com/en-us/analysis-services/tom/table-partitions?view=asallproducts-allversions#define-the-data-coverage-of-the-directquery-partition) a particiones en modo DirectQuery.
- Tabular Editor ahora agregará una anotación `__TEdtr` a tu modelo cuando los metadatos se guarden en disco (ya sea como Model.bim, Database.json o TMDL). Microsoft utiliza esta anotación para recopilar datos analíticos sobre el uso de Tabular Editor en modelos semánticos publicados en un Workspace de Power BI o Fabric. Esta anotación también puede ser necesaria para que ciertas funciones funcionen correctamente en Power BI / Fabric cuando un modelo se editó con Tabular Editor.
- El cuadro de diálogo **Nuevo modelo** ahora incluye una casilla **Direct Lake**\*, que configurará el modelo para usar el nuevo modo de almacenamiento Direct Lake en Fabric. [Más información](https://docs.tabulareditor.com/common/Datasets/direct-lake-dataset.html). Además, ahora puedes elegir explícitamente el _Modo de compatibilidad_ al introducir manualmente el _Nivel de compatibilidad_.
- Ahora puedes exportar [archivos VPAX ofuscados](https://www.sqlbi.com/blog/marco/2024/03/15/vpax-obfuscator-a-library-to-obfuscate-vpax-files/) con Tabular Editor. Esto puede ser útil cuando tus metadatos (nombres de objetos y expresiones DAX) son confidenciales, pero quieres compartir el archivo VPAX con una persona o un servicio para fines de optimización. Busca esta opción en la vista **Analizador VertiPaq**, en el menú desplegable situado junto al botón **Exportar**.

## Mejoras en 3.15.0

- Hemos actualizado [AMO/TOM a 19.79.1.1](https://www.nuget.org/packages/Microsoft.AnalysisServices.NetCore.retail.amd64). En esta actualización, se corrigió un error que en ocasiones podía hacer que se generaran particiones con rangos de política superpuestos al usar la opción "Aplicar política de actualización".
- El _nivel de compatibilidad_ predeterminado para los nuevos modelos de Power BI / Fabric se ha establecido en 1605.
- Cuando se implementa por primera vez un modelo Direct Lake, ahora realizamos una actualización automática, que es un requisito previo para evitar el retroceso a DirectQuery. Además, hay una nueva opción (habilitada de forma predeterminada) para realizar una actualización automática cada vez que se guarda un modelo Direct Lake, cuando se han agregado nuevas tablas. Esta preferencia se encuentra en **Herramientas > Preferencias > Implementación de modelos > Actualización de datos**.
- Se agrega una nueva opción de ordenación al generar scripts de elementos DAX: al habilitar "Ordenar los elementos del script alfabéticamente", todos los elementos DAX del script se ordenarán alfabéticamente por nombre.
- Se agregan seis nuevos comandos del editor y atajos de teclado predeterminados para trabajar con regiones de código plegables en los editores de DAX:
  - `Editor.Dax.CollapseAllFolds`: contraer/cerrar todas las regiones plegables del documento. Atajo predeterminado: **Ctrl+Alt+[**
  - `Editor.Dax.ExpandAllFolds`: expandir/abrir todas las regiones plegables del documento. Atajo predeterminado: **Ctrl+Alt+]**
  - `Editor.Dax.ToggleAllFolds`: alternar el estado de todas las regiones plegables entre expandido y contraído; esto garantiza que todas las regiones queden en el mismo estado. Atajo predeterminado: **Ctrl+Alt+;**
  - `Editor.Dax.CollapseFold`: contraer/cerrar la región plegable en la que se encuentra actualmente el cursor. Atajo predeterminado: **Ctrl+Shift+[**
  - `Editor.Dax.ExpandFold`: expandir/abrir la región plegable en la que se encuentra actualmente el cursor. Atajo predeterminado: **Ctrl+Shift+]**
  - `Editor.Dax.ToggleFold`: alternar el estado de la región plegable en la que se encuentra el cursor entre los estados abierto y cerrado. Atajo predeterminado: **Ctrl+Shift+;**
- El editor de DAX ahora permite eliminar palabras individuales dentro de referencias de tabla/columna/medida al usar **Ctrl+Retroceso** o **Ctrl+Supr**. Para eliminar la referencia completa, coloque el cursor al principio o al final de la referencia y use los mismos atajos.
- Los archivos "database.tmdl" ahora también se muestran en el cuadro de diálogo **Abrir modelo desde archivo**, además de los archivos "model.tmdl". No importa qué archivo se abra: Tabular Editor carga en cualquier caso los metadatos completos del modelo desde ambos archivos.
- Al recopilar estadísticas del Analizador VertiPaq para un modelo Direct Lake, de forma predeterminada solo se consultan las columnas residentes. Esto reduce considerablemente el tiempo necesario para recopilar estadísticas en modelos grandes. Aún puedes optar por incluir columnas no residentes cambiando la preferencia **Estadísticas para modelos Direct Lake** en **Herramientas > Preferencias > Analizador VertiPaq**.
- Se agregan preferencias para controlar el formato de moneda en **'Herramientas' > 'Preferencias' > 'Explorador TOM' > 'Localización'**: antes, la moneda estaba codificada de forma fija para usar el símbolo de dólar estadounidense ($) al inicio de la cadena de formato. Ahora puedes seleccionar un estilo de formato estándar en una lista desplegable con buscador, o definir tu propio símbolo y posicionamiento personalizados.

## Correcciones de errores en 3.15.0

- Se corrigió un problema por el que los atajos de teclado personalizados para "Editor.Dax.FormatLong" y "Editor.Dax.FormatShort" no funcionaban.
- Se corrigió un error por el que las macros no se cargaban por completo, a menos que las definiciones de macro en el archivo MacroActions.json estuvieran ordenadas por ID de macro. Consulta el [#1151](https://github.com/TabularEditor/TabularEditor3/issues/1151).
- Si falla una vista previa de tabla en el Asistente para importar tablas, ahora debería ser posible seguir previsualizando otras tablas.
- Las vistas previas de tabla del Asistente para importar tablas ahora deberían funcionar correctamente con los endpoints SQL de Lakehouse de Fabric.
- Al guardar un modelo como TMDL, la codificación predeterminada ahora es UTF8 sin marcas de orden de bytes (BOM), lo que debería ser compatible con Power BI Desktop.
- Se corrigió un error por el que los tooltips de URL de la página **Novedades** se quedaban en pantalla.
- El formateador DAX integrado ya no debería reemplazar referencias a KPI por la referencia de su medida base.
- Se corrigió un error por el que no se guardaban correctamente los ajustes de serialización "Ignorar particiones de actualización incremental" e "Ignorar etiquetas de linaje".

\*=Los modelos Direct Lake requieren Workspaces de Fabric. Por lo tanto, solo están disponibles para los clientes de Tabular Editor 3 Edición Enterprise.

## Vienes de Tabular Editor 2.x?

Mira [este vídeo](https://www.youtube.com/watch?v=pt3DdcjfImY) para hacerte una idea de las nuevas funcionalidades de Tabular Editor 3. Además, asegúrate de consultar nuestra [guía de incorporación](https://docs.tabulareditor.com/onboarding/index.html).

**Resumen de las principales características de Tabular Editor 3:**

- IDE totalmente personalizable, con compatibilidad con varios monitores, Hi-DPI y temas
- Nuevo y potente editor de código DAX con autocompletado, comprobación de sintaxis, plegado de código y mucho, mucho más
- Modo del área de trabajo, que te permite guardar tus cambios en disco y sincronizar los metadatos del modelo con Analysis Services al mismo tiempo
- Previsualiza datos de tablas con desplazamiento infinito, crea PivotGrids o escribe consultas DAX para explorar el modelo o probar la lógica de cálculo
- Programa actualizaciones de datos
- Actualiza los esquemas de las tablas tanto en Data source de proveedor como en orígenes de datos estructurados (sí, incluso para consultas M!)
- Crea diagramas de Data model
- Crea scripts DAX que te permitan editar varias medidas u otros objetos calculados en un solo documento
- Graba C# Scripts y guárdalos como macros (antes conocidas como "Acciones personalizadas")
- Integración con el Analizador VertiPaq
- Depurador de DAX
