## Novedades en 3.24

Consulta nuestro [blog de la versión](https://tabulareditor.com/blog/tabular-editor-3-october-2025-release) para ver un resumen rápido de las actualizaciones más importantes de esta versión.

Consulta también las [notas de la versión 3.24.0](xref:release-3-24-0) para ver todas las nuevas funciones y mejoras incluidas en la anterior versión principal.

## Correcciones de errores en la 3.24.2

- Se corrigió un error que provocaba una excepción no controlada cuando, durante una operación de actualización, la instancia de AS conectada devolvía varios Reports de errores/advertencias.

## Mejoras en la 3.24.1

- Se añadió compatibilidad en el Analizador semántico para las nuevas palabras clave de tipo de datos `LOGICAL`, `INT64`, `DECIMAL` y `TEXT` (todas son alias de palabras clave existentes), que pueden usarse como argumentos de funciones como [`CONVERT`](https://dax.guide/convert) y [`DATATABLE`](https://dax.guide/datatable).
- Se añadió compatibilidad en el Analizador semántico para nuevas funciones: [`ISINTEGER`](https://dax.guide/isinteger), [`ISCURRENCY`](https://dax.guide/iscurrency) y la función de cálculo Visual [`LOOKUPWITHTOTALS`](https://dax.guide/lookupwithtotals).
- Se añadió la opción de menú "Administrar paquetes de DAX..." al hacer clic con el botón derecho en los nodos **Modelo** o **Funciones** del Explorador TOM.

## Correcciones de errores en 3.24.1

- Se corrigió un error que provocaba una excepción no controlada al iniciar Tabular Editor 3 en un monitor con baja resolución de píxeles
- Se corrigió un error que impedía abrir un modelo desde una estructura de carpetas PBIP al usar la opción de menú **Archivo > Abrir > Modelo desde archivo...**; consulta [#1507](https://github.com/TabularEditor/TabularEditor3/issues/1507).
- Se corrigieron problemas de escalado de DPI relacionados con la vista del Administrador de paquetes de DAX.
- Se corrigió un error del Analizador semántico por el que no se reconocían las funciones de cálculo Visual [`FIRST`](https://dax.guide/first) y [`LAST`](https://dax.guide/last).
- Se corrigió un error del Analizador semántico que mostraba una advertencia falsa al llamar a una UDF como primer argumento de [`CALCULATE`](https://dax.guide/calculate); consulta [#1509](https://github.com/TabularEditor/TabularEditor3/issues/1509).
- El Administrador de paquetes de DAX ya no muestra una advertencia debido a diferencias en los saltos de línea de la expresión DAX (PBI Desktop puede cambiar `\n` a `\r\n` al guardar un modelo).
- Se corrigió el orden de las UDF entre los espacios de nombres en el Explorador TOM

## Vienes de Tabular Editor 2.x?

Mira [este vídeo](https://youtu.be/O4ATwdzCvWc) para ver un recorrido rápido por las principales funciones de Tabular Editor 3. Además, no dejes de consultar nuestra [guía de incorporación](https://docs.tabulareditor.com/onboarding/index.html).

**Descripción general de las principales características de Tabular Editor 3:**

- IDE totalmente personalizable, con compatibilidad con varios monitores, Hi-DPI y temas
- Nuevo y potente editor de código DAX con autocompletado, comprobación de sintaxis, plegado de código y mucho, mucho más
- \*Modo del área de trabajo, que te permite guardar los cambios en el disco y sincronizar a la vez los metadatos del modelo con Analysis Services
- \*Previsualiza los datos de las tablas con desplazamiento infinito, crea PivotGrids o escribe consultas DAX para explorar el modelo o probar la lógica de cálculo
- \*Programa actualizaciones de datos
- Actualiza los esquemas de tablas tanto en los Data sources de proveedor como en los orígenes de datos estructurados (sí, ¡incluso para consultas M!)
- Crea diagramas del Data model
- Crea scripts DAX que te permitan editar varias medidas u otros objetos calculados en un único documento
- Graba C# Scripts y guárdalos como macros (antes conocidas como "Custom Actions")
- Integración del Analizador VertiPaq
- Depurador de DAX
- Integración del Optimizador de DAX
- Acciones de código para refactorizar fácilmente tu código DAX.

\*=Solo mientras esté conectado a una instancia de Analysis Services o Power BI
