### Requisitos previos

Tabular Editor 3.3.0 (y versiones posteriores) usa .NET 6. Es posible que tengas que instalar el [.NET 6.0 Runtime for Desktop](https://dotnet.microsoft.com/en-us/download/dotnet/6.0/runtime) antes de poder iniciar Tabular Editor 3.3.0 (o una versión posterior). A partir de la versión 3.3.1, el instalador de Tabular Editor 3 descargará e instalará estos requisitos previos si faltan.

## Novedades de 3.7.0

Consulta nuestro [blog de lanzamiento](https://blog.tabulareditor.com/?p=2146) para ver un resumen de las actualizaciones más importantes de esta versión.

- Esta versión incorpora el [Editor de traducciones de metadatos](xref:metadata-translation-editor), que ofrece una forma rápida y sencilla de ver, comparar y editar todas las traducciones de metadatos aplicadas a los objetos del modelo.
- Tabular Editor 3 ahora admite el nuevo [Tabular Model Definition Language (TMDL)](https://powerbi.microsoft.com/en-us/blog/announcing-public-preview-of-the-tabular-model-definition-language-tmdl/) como formato para guardar/cargar los metadatos del modelo. Puedes habilitar el modo de serialización TMDL (versión preliminar) en **Preferencias > Herramientas > Guardar en carpeta**.

> [!NOTE]
> TMDL sigue en versión preliminar y, por tanto, esta funcionalidad también debe considerarse una característica en versión preliminar de Tabular Editor 3. Asegúrate de conservar una copia de seguridad de Model.bim / Database.json de los metadatos del modelo para evitar perder trabajo.\*\*.

## Mejoras menores en 3.7.0

- Ahora usamos la versión más reciente del [Analizador VertiPaq](https://github.com/sql-bi/VertiPaq-Analyzer). Esto corrige, entre otras cosas, un problema al recopilar estadísticas en modelos con cadenas de formato dinámicas.
- Ahora mostramos mensajes de error más útiles cuando una operación de pegado falla debido a metadatos de modelo incompatibles.
- Los eventos de seguimiento de Analysis Services que se reciben después de completarse un trabajo de actualización de datos ahora se asignan correctamente a ese trabajo, lo que permite la propagación correcta de mensajes de error/advertencia, así como de los recuentos de progreso. Consulta [#735](https://github.com/TabularEditor/TabularEditor3/issues/735).
- La ventana de vista previa de datos ahora puede usar la función DAX [`OFFSET`](https://dax.guide/offset), lo que permite que la vista previa muestre todas las filas de las tablas que no son DirectQuery, incluso cuando un modelo está en modo DirectQuery (por ejemplo, cuando hay tablas híbridas en el modelo). Consulta [#500](https://github.com/TabularEditor/TabularEditor3/issues/500) y [#837](https://github.com/TabularEditor/TabularEditor3/issues/837).
- Al desplegar un modelo como una nueva base de datos, ahora añadimos una partición ficticia a las tablas que usan una política de actualización incremental cuando no hay particiones en los metadatos del modelo, para garantizar que el despliegue se complete correctamente.
- La vista previa de datos conserva los filtros aplicados cuando se actualiza o se procesa una tabla. Consulta [#780](https://github.com/TabularEditor/TabularEditor3/issues/780).
- Cuando una tabla no tiene columnas en estado no consultable, la vista previa de datos ahora usa directamente el nombre de la tabla en la consulta DAX generada, en vez de encapsularlo en una llamada a [`SELECTCOLUMNS`](https://dax.guide/selectcolumns). Esto debería acelerar la vista previa de datos en versiones antiguas de Analysis Services. Consulta el [#126](https://github.com/TabularEditor/TabularEditor3/issues/126).
- TE3 ahora usa la versión más reciente de [Microsoft.Identity.Client](https://www.nuget.org/packages/Microsoft.Identity.Client).
- La detección de esquemas sin conexión ahora admite el uso de expresiones compartidas (M) que definen funciones personalizadas.

## Correcciones de errores en 3.7.0

- Había un problema en la versión más reciente de AdomdClient utilizada en 3.6.0, que hacía que un Pivot Grid fallara con un mensaje de error XML al conectarse al punto de conexión XMLA de Power BI. Hemos revertido AdomdClient a la versión [19.55.3.1](https://www.nuget.org/packages/Microsoft.AnalysisServices.AdomdClient.NetCore.retail.amd64/19.55.3.1), lo que por ahora soluciona este problema.
- Se corrigió un problema por el que las instancias locales de Power BI Desktop no se mostraban en la lista desplegable "Instancias locales".
- Se corrigió un problema por el que TE3 no permitía cargar un modelo que no tuviera especificado un nombre de base de datos.
- Los cuadros de diálogo "Please Wait" ya no deberían interferir con las ventanas de inicio de sesión de Azure AD.
- Mejoras de estabilidad.
- Se corrigió un problema por el que la aplicación se bloqueaba al intentar copiar/pegar medidas con cadenas de formato dinámicas; consulta el [#1099](https://github.com/TabularEditor/TabularEditor/issues/1099).
- La creación de scripts DAX para medidas con cadenas de formato o cadenas de formato dinámicas ahora funciona como se pretendía.
- Las columnas con ámbito de consulta ahora se controlan correctamente en el analizador semántico de DAX y en la función de autocompletado. Consulta el [#890](https://github.com/TabularEditor/TabularEditor3/issues/890).
- Los cuadros de diálogo "Abrir modelo/archivo" ahora filtran correctamente los archivos "Database.json", en lugar de mostrar todos los archivos .json de una carpeta.
- Se corrigieron algunos problemas Visuales con los botones en varios cuadros de diálogo de aviso.
- Se corrigió un problema por el que la selección de base de datos era incorrecta cuando la lista estaba filtrada. Consulta el [#886](https://github.com/TabularEditor/TabularEditor3/issues/886).
- Se corrigió un problema por el que la acción "Guardar modelo" (Ctrl+Alt+S) no estaba habilitada, aunque la barra de mensajes indicaba que el modelo tenía cambios sin guardar.
- Se corrigió un problema con errores semánticos "fantasma" en el Editor de DAX; consulta el [#792](https://github.com/TabularEditor/TabularEditor3/issues/792).
- La vista previa de datos ahora debería respetar la cadena de formato asignada a una columna; consulta el [#820](https://github.com/TabularEditor/TabularEditor3/issues/820).
- Los resultados de las consultas DAX ahora deberían mostrar la parte de la hora de los valores DateTime cuando no sea 00:00:00.

## Vienes de Tabular Editor 2.x?

Mira [este vídeo](https://www.youtube.com/watch?v=pt3DdcjfImY) para hacerte una idea de las nuevas funcionalidades de Tabular Editor 3. Además, asegúrate de consultar nuestra [guía de inicio](https://docs.tabulareditor.com/onboarding/index.html).

**Resumen de las principales características de Tabular Editor 3:**

- IDE totalmente personalizable, con compatibilidad con varios monitores, Hi-DPI y temas
- Nuevo y potente editor de código DAX con autocompletado, comprobación de sintaxis, plegado de código y mucho, mucho más
- \*Modo del área de trabajo, que te permite guardar los cambios en el disco y sincronizar simultáneamente los metadatos del modelo con Analysis Services
- \*Previsualiza los datos de las tablas con desplazamiento infinito, crea PivotGrids o escribe consultas DAX para explorar el modelo o probar la lógica de cálculo
- \*Programa actualizaciones de datos
- Actualiza los esquemas de las tablas tanto en Data sources de proveedor como en orígenes de datos estructurados (sí, ¡incluso para consultas M!)
- Crea diagramas del Data model
- Crea scripts DAX que te permiten editar varias medidas u otros objetos calculados en un solo documento
- Graba C# Scripts y guárdalos como macros (antes conocidos como "Custom Actions")
- Integración con el Analizador VertiPaq
- Depurador de DAX

\*=Solo cuando estés conectado a una instancia de Analysis Services o Power BI
