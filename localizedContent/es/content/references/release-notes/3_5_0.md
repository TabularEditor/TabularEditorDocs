### Requisitos previos

Tabular Editor 3.3.0 (y versiones posteriores) usa .NET 6. Es posible que tengas que instalar el [.NET 6.0 Runtime for Desktop](https://dotnet.microsoft.com/en-us/download/dotnet/6.0/runtime) antes de poder ejecutar Tabular Editor 3.3.0 (o una versión posterior). A partir de la versión 3.3.1, el instalador de Tabular Editor 3 descargará e instalará estos requisitos previos si faltan.

## Mejoras en 3.5.0

Consulta nuestra [entrada del blog de lanzamiento](https://blog.tabulareditor.com/2023/03/15/tabular-editor-3-march-2023-release/) para ver un resumen de las actualizaciones más importantes de esta versión.

- TOM actualizado a [19.57.2.4](https://www.nuget.org/packages/Microsoft.AnalysisServices.retail.amd64/).
- Esta versión incorpora los **grupos de tablas**, que te permiten organizar las tablas en "carpetas" en el Explorador TOM. [Más información](xref:table-groups).
- El acceso directo predeterminado para quitar comentarios del código ahora es Ctrl+U
- Ahora Tabular Editor intentará reconectarse automáticamente a Analysis Services cuando haya expirado el tiempo de espera de la sesión. Esto significa que ya no debería ser necesario cerrar y volver a abrir las Vistas previas de tabla, las ventanas de consulta DAX y los Pivot Grid. Consulta [#650](https://github.com/TabularEditor/TabularEditor3/issues/650) y [#803](https://github.com/TabularEditor/TabularEditor3/issues/803).
- Se ha añadido un nuevo conjunto de métodos sobrecargados de C# Script `Macro(...)`. Son un alias de `CustomAction(...)`. La documentación de la API se ha actualizado en consecuencia.
- Al usar Tabular Editor en **modo del área de trabajo**, las particiones de actualización incremental no se reemplazan cuando se actualiza la base de datos del Workspace. Además, ahora hay disponible una nueva opción de serialización, **Ignorar particiones de actualización incremental**, que viene habilitada de forma predeterminada para los modelos nuevos. Esta opción garantiza que las particiones controladas por la actualización incremental no se serialicen al guardar un modelo como archivo .bim o como una estructura de carpetas. Esto facilita la integración de modelos con actualización incremental en el control de versiones.
- Se ha agregado un botón en la barra de herramientas para activar o desactivar las carpetas de visualización en la **vista del Editor de perspectivas**. Al activar o desactivar una carpeta de visualización en el Editor de perspectivas, se activarán o desactivarán todos los objetos dentro de esa carpeta. El Editor de perspectivas ahora también ofrece una indicación Visual si una tabla o carpeta contiene tanto elementos incluidos en la perspectiva como elementos que no lo están.
- Tabular Editor ahora ofrece compatibilidad completa con Data sources de Oracle. Debes tener instalados los [controladores administrados de ODP.NET](https://www.oracle.com/database/technologies/odac-downloads.html), lo cual también es un requisito si quieres importar datos desde Oracle en Power BI Desktop.

## Correcciones de errores en 3.5.0

- El autoformato ya no debería borrar el código cuando el Editor de DAX está configurado para usar tabulaciones
- El botón "Abrir macro" de la vista **Macros** ahora estará habilitado siempre que se seleccione una macro
- Ya no debería ser necesario restablecer el diseño de la ventana para que aparezca la vista **Editor de perspectiva**
- La barra del menú principal ya no se restablecerá cuando se agregue una macro como nuevo botón
- Se corrigió un error que a veces impedía que Tabular Editor abriera un archivo .bim o una estructura de carpetas .json que contenía propiedades TOM específicas de Power BI.
- `ResolveObjectPath` ahora admite el formato de ruta "antiguo", similar a TE2. Consulta [#1077](https://github.com/TabularEditor/TabularEditor/issues/1077).
- Las reglas de BPA que dependen de la propiedad `Expression` en las tablas ya no deberían fallar.
- Ahora las macros se pueden guardar/compilar aunque su nombre contenga comillas dobles.
- Se corrigieron varios problemas con el método de C# Script `CustomAction(...)`.
- El Editor de perspectiva ahora gestiona correctamente las tablas ocultas que contienen medidas visibles.
- Se corrigió un problema que impedía establecer la propiedad **medida predeterminada** en el objeto Model desde la **vista de propiedades**.
- La opción del menú contextual **Tabla calculada** ya no se mostrará al trabajar sin conexión.
- Se corrigió un problema que hacía que la lista desplegable de autocompletado de C# "se tragara" una pulsación de la tecla de punto.
- Se corrigió un problema por el que el analizador semántico generaba un Report de un uso incorrecto de parámetros en la función DAX [XIRR](https://dax.guide/xirr).
- El Editor de relaciones ahora permite crear relaciones entre columnas de distintos tipos de datos, siempre que sean compatibles.
- Se corrigió el error "The key didn't match any rows" al intentar realizar una actualización del esquema en una tabla que usa una partición M.
- Las tablas que usan una política de actualización incremental ahora funcionan con la funcionalidad **Actualizar esquema de tabla**.

## ¿Vienes de Tabular Editor 2.x?

Mira [este vídeo](https://www.youtube.com/watch?v=pt3DdcjfImY) para hacerte una idea de las nuevas funcionalidades de Tabular Editor 3. Además, no dejes de consultar nuestra [guía de incorporación](https://docs.tabulareditor.com/onboarding/index.html).

**Resumen de las principales funcionalidades de Tabular Editor 3:**

- IDE totalmente personalizable, con compatibilidad con varios monitores, soporte Hi-DPI y temas
- Nuevo y potente editor de código DAX con autocompletado, comprobación de sintaxis, plegado de código y mucho, mucho más
- \*Modo del área de trabajo, que te permite guardar tus cambios en disco y sincronizar los metadatos del modelo con Analysis Services al mismo tiempo
- \*Previsualiza los datos de las tablas con desplazamiento infinito, crea PivotGrids o escribe consultas DAX para explorar el modelo o probar la lógica de cálculo
- \*Programa actualizaciones de datos
- Actualiza los esquemas de las tablas tanto en el proveedor como en los orígenes de datos estructurados (sí, incluso para consultas M!)
- Crea diagramas del Data model
- Crea scripts DAX que te permiten editar más de una medida u otros objetos calculados en un único documento
- Graba C# Scripts y guárdalos como macros (antes conocidas como "Custom Actions")
- Integración con el Analizador VertiPaq
- Depurador de DAX

\*=Solo cuando estés conectado a una instancia de Analysis Services o Power BI
