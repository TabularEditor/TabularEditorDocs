### Requisitos previos

Tabular Editor 3.3.0 (y versiones posteriores) usa .NET 6. Es posible que tengas que instalar el [.NET Runtime for Desktop](https://dotnet.microsoft.com/en-us/download/dotnet/6.0/runtime) 6.0 antes de poder iniciar Tabular Editor 3.3.0 (o una versión más reciente). A partir de la versión 3.3.1, el instalador de Tabular Editor 3 descargará e instalará estos requisitos previos si faltan.

## Mejoras en 3.8.0

Consulta nuestro [blog de versiones](https://blog.tabulareditor.com/2023/06/26/tabular-editor-3-june-2023-release/) para ver un resumen de las actualizaciones más importantes de esta versión.

- Tabular Editor 3 ahora te permite abrir [proyectos de Power BI Desktop (.pbip)](https://learn.microsoft.com/en-us/power-bi/developer/projects/projects-overview) o carpetas que contengan archivos .pbip. Cuando una carpeta de este tipo contiene varios Dataset, se muestra una lista que te permite seleccionar cuál quieres abrir.
- La [actualización de junio de 2023 de Power BI Desktop](https://powerbi.microsoft.com/en-us/blog/power-bi-june-2023-feature-summary/) ahora admite una gama mucho más amplia de operaciones de modelado que se pueden realizar mediante herramientas externas ("hardening"). En otras palabras, cuando conectes Tabular Editor a Power BI Desktop (junio de 2023 o posterior), ahora podrás agregar/editar columnas calculadas, tablas calculadas, relaciones, jerarquías y mucho más.
- Ahora usamos la versión más reciente de TOM/AMO ([19.65.4](https://www.nuget.org/packages/Microsoft.AnalysisServices.NetCore.retail.amd64)).

> [!NOTE]
> TMDL y los proyectos de Power BI Desktop (.pbip) siguen en versión preliminar y, por lo tanto, estas funciones también deben considerarse funciones en versión preliminar de Tabular Editor 3. Asegúrate de guardar una copia de seguridad de Model.bim / Database.json de los metadatos de tu modelo para evitar perder tu trabajo.\*\*.

## Correcciones de errores en 3.8.0

- La vista de dependencias ahora ofrece un rendimiento mucho mejor en modelos con gráficos de dependencias grandes y complejos.
- Algunos usuarios informaron de que editar ciertas propiedades desde la **vista de propiedades** hacía que la aplicación se bloqueara. Esto ya no debería ocurrir.
- Semantic Analyzer ya no debería mostrar errores cuando una consulta DAX contiene tanto columnas con ámbito de consulta como tablas con ámbito de consulta.
- Semantic Analyzer ahora gestiona correctamente expresiones DAX que contienen caracteres UTF-32 (emojis), aunque el editor DAX sigue teniendo problemas para trabajar con ellos. En general, recomendamos evitar el uso de emojis en expresiones DAX.
- Se corrigió un error por el que, al pulsar repetidamente Ctrl+Z (Deshacer), se borraba el Editor de expresiones e incluso se insertaban expresiones no relacionadas.
- El Editor de perspectivas ahora se actualiza correctamente para reflejar cuando se añaden o se eliminan objetos del modelo.
- El Analizador VertiPaq ya no debería cerrarse inesperadamente cuando un modelo contiene cadenas de formato dinámicas.
- Se corrigió una regresión al importar tablas desde Snowflake, que provocaba que se asignaran tipos de datos incorrectos a las columnas.
- Se corrigió una regresión al actualizar el esquema o importar tablas desde Snowflake, que obligaba a volver a introducir las propiedades implícitas del Data source en el cuadro de diálogo de conexión.
- Se corrigió un problema por el que los botones de la barra de herramientas relacionados con el Editor de expresiones desaparecían si el Editor de expresiones estaba acoplado como una ventana de herramientas (es decir, no como una ventana de documento).
- Los editores de DAX ahora realizan un nuevo análisis semántico cuando se realizan cambios en el modelo que pueden afectar a la semántica de DAX. Esto debería garantizar que los mensajes de error que se muestran en el editor de DAX sean siempre válidos.
- Se corrigió un problema por el que el Depurador de DAX no estaba disponible, incluso para consultas DAX aparentemente sencillas.

## Vienes de Tabular Editor 2.x?

Mira [este vídeo](https://www.youtube.com/watch?v=pt3DdcjfImY) para hacerte una idea de las nuevas características de Tabular Editor 3. Además, asegúrate de consultar nuestra [guía de incorporación](https://docs.tabulareditor.com/onboarding/index.html).

**Descripción general de las principales funcionalidades de Tabular Editor 3:**

- IDE totalmente personalizable, con compatibilidad con varios monitores, Hi-DPI y temas
- Nuevo y potente editor de código DAX con autocompletado, comprobación de sintaxis, plegado de código y mucho, mucho más
- \*Modo del área de trabajo, que te permite guardar los cambios en disco y sincronizar simultáneamente los metadatos del modelo con Analysis Services
- \*Previsualiza los datos de tablas con desplazamiento infinito, crea PivotGrids o escribe consultas DAX para explorar el modelo o probar la lógica de cálculo
- \*Programa actualizaciones de datos
- Actualiza los esquemas de tablas tanto en Data sources de proveedor como en los orígenes de datos estructurados (sí, ¡incluso para consultas M!)
- Crear diagramas de Data model
- Crear scripts DAX que te permitan editar varias medidas u otros objetos calculados en un único documento
- Grabar C# Scripts y guardarlos como macros (antes conocidas como "Acciones personalizadas")
- Integración con el Analizador VertiPaq
- Depurador de DAX

\*=Solo mientras estés conectado a una instancia de Analysis Services o Power BI
