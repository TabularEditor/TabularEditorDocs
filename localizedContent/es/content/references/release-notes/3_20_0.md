## Novedades en 3.20.0

Consulta nuestra [entrada del blog de lanzamiento](https://blog.tabulareditor.com/2025/02/21/tabular-editor-3-february-2025-release/) para ver un breve resumen de las actualizaciones más importantes de esta versión.

## Mejoras en 3.20.0

- Todas las [acciones de código](xref:code-actions) ahora muestran una descripción emergente en el menú contextual al pasar el cursor sobre el elemento del menú, con una breve explicación de la acción. La descripción emergente también incluye un enlace a un artículo de la base de conocimientos (KB) con más información sobre la acción.
- También hemos añadido algunas acciones de código nuevas en esta versión, debido a la demanda popular. Por ejemplo, ahora te sugeriremos que uses el operador `IN` en lugar de usar una expresión compuesta con `OR` / `||`. Consulta [aquí la lista completa de acciones de código disponibles](xref:code-actions#list-of-code-actions).
- Se ha añadido una nueva preferencia de la interfaz de usuario llamada "Usar rutas de archivo compactas" para alternar entre mostrar rutas de archivo compactas (abreviadas) y rutas de archivo completas en los menús Archivos recientes y Modelos recientes.
- Al editar un modelo cargado en Power BI Desktop (enero de 2025 o posterior), Tabular Editor ahora te permitirá editar cualquier objeto y propiedad de forma predeterminada, excepto tablas y particiones, en línea con las operaciones de modelado admitidas por la nueva [vista TMDL](https://learn.microsoft.com/en-us/power-bi/transform-model/desktop-tmdl-view) de Power BI Desktop.
- Al cambiar la expresión DAX de una tabla calculada, Tabular Editor 3 ahora intentará inferir las columnas resultantes de la tabla, incluso si la expresión contiene un error.
- Se ha añadido una nueva preferencia de Consulta DAX llamada "Mantener la ordenación y el filtrado existentes en la cuadrícula de resultados" a los menús y la barra de herramientas de Consulta DAX, lo que permite controlar cómo se conservan la ordenación y el filtrado en las cuadrículas de resultados entre ejecuciones de consulta.
- Hemos realizado varias mejoras en nuestra **Vista de diagrama**. Lo más importante es que ahora se representa mucho más rápido, especialmente cuando hay muchas tablas y relaciones. Pero también hay varios ajustes en los Visuales.
- Hemos mejorado la función de autocompletado de C#, de modo que ahora solo muestra los elementos relevantes para el objetivo actual. Por ejemplo, si escribes `Selected.Column.DataType = `, el cuadro de lista ahora solo mostrará los valores válidos de la enumeración [`DataType`](https://learn.microsoft.com/en-us/dotnet/api/microsoft.analysisservices.tabular.datatype?view=analysisservices-dotnet#fields). Además, si el mismo tipo existe en distintos espacios de nombres, ahora mostramos el espacio de nombres entre paréntesis para desambiguar los elementos de la lista.
- Se ha actualizado AMO/TOM a la versión más reciente [19.88.3](https://www.nuget.org/packages/Microsoft.AnalysisServices/).
- Hemos añadido compatibilidad de autocompletado para las nuevas funciones de inspección escalar de DAX, [`ISINT64`](https://dax.guide/isint64), [`ISBOOLEAN`](https://dax.guide/isboolean), [`ISDOUBLE`](https://dax.guide/isboolean), [`ISDECIMAL`](https://dax.guide/isdecimal), [`ISDATETIME`](https://dax.guide/isdatetime), [`ISSTRING`](https://dax.guide/isstring) y [`ISNUMERIC`](https://dax.guide/isnumeric).

## Correcciones de errores en 3.20.0

- La acción de código DAX **Reescribir usando ISBLANK** ya no se activará al usar comparaciones de igualdad no estrictas, p. ej. `x = BLANK()`, ya que esto es semánticamente diferente de `ISBLANK(x)`.
- Varias correcciones de errores relacionadas con excepciones no controladas en el Editor de DAX.
- El analizador semántico ahora muestra correctamente un mensaje de error en Mensajes si se usa una columna de agrupación en una llamada a [`SUMMARIZE`](https://dax.guide/summarize) sin relación con la tabla base.
- Al crear archivos VPAX desde la vista **Optimizador de DAX**, ahora usaremos el modo de extracción **Full** para modelos Direct Lake, generando un archivo VPAX válido para el análisis del Optimizador de DAX.
- Varias correcciones de errores y mejoras de estabilidad en Pivot Grid.
- Se corrigió un error por el cual desasignar/restablecer un acceso directo del Editor no surtía efecto hasta reiniciar la aplicación.
- Se corrigió un error por el que ciertos atajos del editor (como "Duplicar línea (Ctrl+D)") activaban una acción en el Explorador TOM con la misma combinación de teclas, incluso cuando el editor tenía el foco.
- Se corrigió un error por el que, al agregar un nivel a una jerarquía en un modelo de Power BI Desktop, no se incluía la referencia a la columna, lo que provocaba un error al intentar guardar el cambio.
- Se corrigió un error que provocaba un cierre inesperado al ejecutar un script DAX que contenía cierta sintaxis o errores.
- Crear y editar scripts DAX ya no debería hacer que la interfaz de usuario se bloquee, algo que podía ocurrir cuando el script contenía errores sintácticos o semánticos.
- Se corrigió un error que a veces hacía que la vista **Actualización de datos** se bloqueara o se cerrara de forma inesperada mientras una actualización estaba en curso.
- Se corrigió un error que provocaba un cierre inesperado al cargar un modelo y luego navegar a una partición con una [Definición de cobertura de datos](https://learn.microsoft.com/en-us/analysis-services/tom/table-partitions?view=asallproducts-allversions#define-the-data-coverage-of-the-directquery-partition).
- La función de autocompletado de los C# Script ahora debería funcionar correctamente al realizar varias inserciones en el script, como se describe en [#1327](https://github.com/TabularEditor/TabularEditor3/issues/1327).
- Al pulsar F12 (Ir a definición) en una referencia a una expresión de cadena de formato dinámica de una medida, ahora se navegará correctamente a la propiedad de la expresión de cadena de formato dinámica de la medida referenciada, en lugar de a la propiedad de la expresión base.
- Se corrigió un problema en nuestro Analizador semántico que no detectaba correctamente cuándo una relación se activaba mediante la función [`USERELATIONSHIP`](https://dax.guide/userelationship).
- Se corrigió un problema por el que aparecían errores falsos de dependencia circular mientras se editaba el DAX de columnas calculadas.
- Se corrigió un error que impedía a los titulares de una licencia **TE3 Edición Business** conectarse a los Workspace de Power BI Premium-Per-User (PPU) en algunas regiones.

> [!IMPORTANT]
> Debido a limitaciones técnicas, las versiones anteriores de TE3 a veces podían permitir que los titulares de una licencia de **Edición Business** se conectaran a los Workspace de Power BI Premium / Fabric Capacity, aunque [esto no está permitido según los términos de licencia de la Edición Business](xref:editions). Si tienes problemas para conectarte al punto de conexión XMLA de Power BI después de actualizar a 3.20.0, ponte en contacto con [licensing@tabulareditor.com](mailto:licensing@tabulareditor.com) para que podamos ayudarte a encontrar una solución adecuada de cara al futuro.

## Problemas conocidos en 3.20.0

- El Analizador semántico no detecta correctamente las activaciones de relaciones con `USERELATIONSHIP` cuando solo hay relaciones inactivas entre dos tablas.
- El Analizador semántico no siempre infiere el tipo de datos correcto para la columna devuelta por la función [`GENERATESERIES`](https://dax.guide/generateseries).
- La nueva **Vista de diagrama** no indica correctamente las relaciones bidireccionales.

## ¿Vienes de Tabular Editor 2.x?

Mira [este vídeo](https://youtu.be/O4ATwdzCvWc) para obtener un recorrido rápido por las principales funcionalidades de Tabular Editor 3. Además, asegúrate de consultar nuestra [guía de incorporación](https://docs.tabulareditor.com/onboarding/index.html).

**Descripción general de las principales funcionalidades de Tabular Editor 3:**

- IDE totalmente personalizable, con compatibilidad con varios monitores, pantallas Hi-DPI y temas
- Nuevo y potente editor de código DAX con autocompletado, comprobación de sintaxis, plegado de código y mucho, mucho más
- \*Modo del área de trabajo, Workspace, que te permite guardar los cambios en disco y sincronizar simultáneamente los metadatos del modelo con Analysis Services
- \*Previsualiza los datos de las tablas con desplazamiento infinito, crea PivotGrids o escribe una Consulta DAX para explorar el modelo o probar la lógica de cálculo
- \*Programar actualizaciones de datos
- Actualizar los esquemas de las tablas tanto en el Data source del proveedor como en el origen de datos estructurado (sí, incluso para consultas M)
- Crear diagramas del Data model
- Crear scripts DAX que te permitan editar varias medidas u otros objetos calculados en un solo documento
- Grabar scripts de C# y guardarlos como macros (antes conocidos como "Custom Actions")
- Integración del Analizador VertiPaq
- Depurador de DAX
- Integración del Optimizador de DAX
- Acciones de código para refactorizar fácilmente tu código DAX.

\*=Solo cuando estés conectado a una instancia de Analysis Services o Power BI
