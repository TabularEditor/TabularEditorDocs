## Novedades en 3.20.0

Consulta nuestro [blog de lanzamiento](https://blog.tabulareditor.com/2025/02/21/tabular-editor-3-february-2025-release/) para ver un resumen breve de las actualizaciones más importantes de esta versión.

## Mejoras en 3.20.0

- Ahora, todas las [acciones de código](xref:code-actions) muestran una información sobre herramientas en el menú contextual al pasar el cursor por encima del elemento del menú, con una breve descripción de la acción. La información sobre herramientas también incluye un enlace a un artículo de la base de conocimientos (KB) con más información sobre la acción.
- También hemos añadido algunas acciones de código nuevas en esta versión, a petición popular. Por ejemplo, ahora te sugeriremos que uses el operador `IN` en lugar de una expresión compuesta con `OR` / `||`. Consulta aquí la [lista completa de acciones de código disponibles](xref:code-actions#list-of-code-actions).
- Hemos incorporado una nueva preferencia de interfaz de usuario denominada "Usar rutas de archivo compactas" para alternar entre mostrar rutas de archivo compactas (abreviadas) y completas en los menús Archivos recientes y Modelos recientes.
- Al editar un modelo cargado en Power BI Desktop (enero de 2025 o posterior), Tabular Editor ahora te permitirá editar cualquier objeto y propiedad de forma predeterminada, excepto tablas y particiones, en línea con las operaciones de modelado compatibles con la nueva [Vista TMDL](https://learn.microsoft.com/en-us/power-bi/transform-model/desktop-tmdl-view) de Power BI Desktop.
- Al cambiar la expresión DAX de una tabla calculada, Tabular Editor 3 ahora intentará inferir las columnas resultantes de la tabla, incluso si la expresión contiene un error.
- Se ha añadido una nueva preferencia de Consulta DAX denominada "Mantener la ordenación y el filtrado existentes en la cuadrícula de resultados" a los menús y a la barra de herramientas de Consulta DAX, lo que permite a los usuarios controlar cómo se conservan la ordenación y el filtrado en las cuadrículas de resultados entre ejecuciones de consultas.
- Hemos realizado varias mejoras en nuestra **Vista de diagrama**. Lo más importante: ahora se representa mucho más rápido, especialmente cuando hay muchas tablas y relaciones. Pero también hay varios ajustes en Visual.
- Hemos mejorado la función de autocompletado de C#, de modo que ahora solo muestra los elementos relevantes para el objetivo actual. Por ejemplo, si escribes `Selected.Column.DataType = `, el cuadro de lista ahora solo mostrará los [valores válidos del enum `DataType`](https://learn.microsoft.com/en-us/dotnet/api/microsoft.analysisservices.tabular.datatype?view=analysisservices-dotnet#fields). Además, si existe el mismo tipo en distintos espacios de nombres, ahora mostramos el espacio de nombres entre paréntesis para desambiguar los elementos de la lista.
- Hemos actualizado AMO/TOM a la versión más reciente: [19.88.3](https://www.nuget.org/packages/Microsoft.AnalysisServices/).
- Hemos añadido compatibilidad de autocompletado para las nuevas funciones de inspección escalar de DAX: [`ISINT64`](https://dax.guide/isint64), [`ISBOOLEAN`](https://dax.guide/isboolean), [`ISDOUBLE`](https://dax.guide/isboolean), [`ISDECIMAL`](https://dax.guide/isdecimal), [`ISDATETIME`](https://dax.guide/isdatetime), [`ISSTRING`](https://dax.guide/isstring) y [`ISNUMERIC`](https://dax.guide/isnumeric).

## Correcciones en 3.20.0

- La acción de código DAX **Reescribir usando ISBLANK** ya no se activará al usar comparaciones de igualdad no estrictas, p. ej., `x = BLANK()`, ya que esto es semánticamente distinto de `ISBLANK(x)`.
- Varias correcciones de errores relacionadas con excepciones no controladas en el Editor de DAX.
- El analizador semántico ahora muestra correctamente mensajes de error si usas una columna de agrupación dentro de una llamada a [`SUMMARIZE`](https://dax.guide/summarize) que no tiene relación con la tabla base.
- Al crear archivos VPAX desde la vista **Optimizador de DAX**, ahora usaremos el modo de extracción **Full** para los modelos Direct Lake, lo que generará un archivo VPAX válido para su análisis en el Optimizador de DAX.
- Varias correcciones de errores y mejoras de estabilidad en Pivot Grid.
- Se corrigió un error por el que, al quitar/restablecer un acceso directo del editor, el cambio no surtía efecto hasta reiniciar la aplicación.
- Se corrigió un error por el que ciertos accesos directos del editor (como "Duplicar línea (Ctrl+D)") desencadenaban una acción en el Explorador TOM con la misma asignación de teclas, incluso cuando el editor tenía el foco.
- Se corrigió un error por el que, al agregar un nivel a una jerarquía en un modelo de Power BI Desktop, no se incluía la referencia a la columna, lo que provocaba un error al intentar guardar el cambio.
- Se corrigió un error que provocaba un cierre inesperado al ejecutar un script DAX que contenía cierta sintaxis o errores.
- Crear y editar scripts DAX ya no debería bloquear la interfaz de usuario, algo que podía ocurrir cuando el script contenía errores de sintaxis o semánticos.
- Se corrigió un error que a veces provocaba que la vista **Actualización de datos** se quedara bloqueada o se cerrara inesperadamente mientras había una actualización en curso.
- Se corrigió un error que provocaba un cierre inesperado al cargar un modelo y, a continuación, navegar a una partición con una [Definición de cobertura de datos](https://learn.microsoft.com/en-us/analysis-services/tom/table-partitions?view=asallproducts-allversions#define-the-data-coverage-of-the-directquery-partition).
- La función de autocompletado de los C# Scripts ahora debería funcionar correctamente al hacer varias inserciones en el C# Script, como en [#1327](https://github.com/TabularEditor/TabularEditor3/issues/1327).
- Al pulsar F12 (Ir a definición) en una referencia a la expresión de cadena de formato dinámica de una medida, ahora se navegará correctamente a la propiedad de esa expresión de cadena de formato dinámica de la medida referenciada, en lugar de a la propiedad de la expresión base.
- Se corrigió un problema con nuestro analizador semántico, que no detectaba correctamente cuándo una relación se activaba mediante la función [`USERELATIONSHIP`](https://dax.guide/userelationship).
- Se corrigió un problema por el que aparecían errores falsos de dependencia circular mientras se editaba el DAX de columnas calculadas.
- Se corrigió un error que impedía a los titulares de una licencia de **TE3 Edición Business** conectarse a Workspaces de Power BI Premium-Per-User (PPU) en algunas regiones.

> [!IMPORTANT]
> Debido a limitaciones técnicas, es posible que las versiones anteriores de TE3 a veces permitieran que los titulares de una licencia de **Edición Business** se conectaran a Workspaces de Power BI Premium / Fabric Capacity, aunque [esto no está permitido según los términos de licencia de la Edición Business](xref:editions). Si tienes problemas para conectarte al punto de conexión XMLA de Power BI después de actualizar a 3.20.0, ponte en contacto con [licensing@tabulareditor.com](mailto:licensing@tabulareditor.com) para que podamos ayudarte a encontrar una solución adecuada de cara al futuro.

## ¿Vienes de Tabular Editor 2.x?

Mira [este vídeo](https://youtu.be/O4ATwdzCvWc) para hacer un recorrido rápido por las principales funciones de Tabular Editor 3. Además, asegúrate de consultar nuestra [guía de incorporación](https://docs.tabulareditor.com/onboarding/index.html).

**Descripción general de las principales características de Tabular Editor 3:**

- IDE totalmente personalizable, con compatibilidad con varios monitores, Hi-DPI y temas
- Nuevo y potente editor de código DAX con autocompletado, comprobación de sintaxis, plegado de código y mucho, mucho más
- \*Modo del área de trabajo, que te permite guardar los cambios en el disco y sincronizar simultáneamente los metadatos del modelo con Analysis Services
- \*Previsualiza los datos de las tablas con desplazamiento infinito, crea PivotGrids o escribe Consultas DAX para explorar el modelo o probar la lógica de cálculo
- \*Programa actualizaciones de datos
- Actualiza los esquemas de las tablas tanto en el Data source del proveedor como en los orígenes de datos estructurados (sí, incluso para consultas M)
- Crea diagramas para el Data model
- Crea scripts de DAX que te permitan editar varias medidas u otros objetos calculados en un solo documento
- Graba C# Scripts y guárdalos como macros (antes conocidas como "Acciones personalizadas")
- Integración con el Analizador VertiPaq
- Depurador de DAX
- Integración con el Optimizador de DAX
- Acciones de código para refactorizar tu DAX fácilmente.

\*=Solo cuando estés conectado a una instancia de Analysis Services o Power BI
