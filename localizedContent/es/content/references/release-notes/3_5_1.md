### Requisitos previos

Tabular Editor 3.3.0 (y versiones posteriores) usa .NET 6. Es posible que tengas que instalar el [.NET 6.0 Runtime for Desktop](https://dotnet.microsoft.com/en-us/download/dotnet/6.0/runtime) antes de poder iniciar Tabular Editor 3.3.0 (o posterior). A partir de la versión 3.3.1, el instalador de Tabular Editor 3 descargará e instalará estos requisitos previos si faltan.

## Correcciones y mejoras en 3.5.1

- Se actualizó TOM a [19.60.0](https://www.nuget.org/packages/Microsoft.AnalysisServices.retail.amd64/).
- Se corrigió un problema en el cuadro de diálogo Editar relación que mostraba una advertencia de tipo de datos incompatible, aunque los tipos de datos de ambas columnas fueran idénticos. Consulta [#855](https://github.com/TabularEditor/TabularEditor3/issues/855). Además, cuando se conecta a una instancia de Analysis Services, el cuadro de diálogo ahora muestra advertencias de infracción de integridad referencial RI y de valores duplicados.
- Se corrigió un problema que a veces hacía que la aplicación se quedara bloqueada al abrir un archivo o una estructura de carpetas desde el disco.
- Se corrigió un problema que impedía editar expresiones de RLS cuando un rol contenía un "." en su nombre.
- Cuando un objeto se mostraba en varias carpetas de visualización, el Explorador TOM no actualizaba correctamente sus propiedades en todos los lugares. Esto se ha corregido.
- "Model" ahora se trata como una palabra reservada en el Analizador semántico. Consulta [#852](https://github.com/TabularEditor/TabularEditor3/issues/852).
- El archivo .pbitool (utilizado para la integración como herramienta externa con Power BI Desktop) ahora usa el icono actualizado de TE3.
- El cuadro de diálogo Crear base de datos del Workspace ahora valida el nombre de la base de datos, lo que evita caracteres especiales que no son compatibles con Analysis Services.
- Se corrigió un problema por el que la sangría automática sobrescribía caracteres en el editor de DAX. Consulta [#857](https://github.com/TabularEditor/TabularEditor3/issues/857).
- Ahora puedes crear o añadir tablas/grupos de cálculo directamente a un grupo de tablas desde el menú contextual del grupo de tablas con clic derecho.
- Se corrigió una regresión en 3.5.0: al importar columnas o realizar una actualización de esquema en una tabla desde un Data source de tipo MS SQL, se asignaba a la columna un tipo de datos incorrecto.
- Se corrigió un problema por el que, al cerrar un corchete o un paréntesis, el cursor a veces "saltaba" a una posición incorrecta en el editor de DAX.

## ¿Vienes de Tabular Editor 2.x?

Mira [este vídeo](https://www.youtube.com/watch?v=pt3DdcjfImY) para hacerte una idea de las nuevas funcionalidades de Tabular Editor 3. Además, asegúrate de consultar nuestra [guía de inicio](https://docs.tabulareditor.com/onboarding/index.html).

**Descripción general de las principales características de Tabular Editor 3:**

- IDE totalmente personalizable, con compatibilidad con múltiples monitores, soporte HiDPI y temas
- Nuevo y potente editor de código DAX con autocompletado, comprobación de sintaxis, plegado de código y mucho, mucho más
- \*Modo del área de trabajo, que permite guardar los cambios en disco y sincronizar simultáneamente los metadatos del modelo con Analysis Services
- \*Previsualizar datos de tablas con desplazamiento infinito, crear PivotGrids o escribir consultas DAX para explorar el modelo o probar la lógica de cálculo
- \*Programar actualizaciones de datos
- Actualizar los esquemas de las tablas tanto en Data sources de proveedor como en orígenes de datos estructurados (sí, ¡incluso para consultas M!)
- Crear diagramas del Data model
- Crear scripts DAX que permiten editar varias medidas u otros objetos calculados en un solo documento
- Grabar C# Scripts y guardarlos como macros (antes conocidas como "Custom Actions")
- Integración con el Analizador VertiPaq
- Depurador de DAX

\*=Solo disponible cuando se está conectado a una instancia de Analysis Services o Power BI
