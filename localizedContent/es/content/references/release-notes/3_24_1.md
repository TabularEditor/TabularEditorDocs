## Nuevo en 3.24

Consulta nuestro [blog de lanzamiento](https://tabulareditor.com/blog/tabular-editor-3-october-2025-release) para obtener una breve panorámica de las actualizaciones más importantes de esta versión.

Consulta también las [notas de la versión 3.24.0](xref:release-3-24-0) para ver todas las nuevas características y mejoras que se añadieron en la versión principal anterior.

## Mejoras en la 3.24.1

- Se añadió compatibilidad en el analizador semántico con nuevas palabras clave de tipo de datos: `LOGICAL`, `INT64`, `DECIMAL` y `TEXT` (todas ellas son alias de palabras clave existentes), que pueden usarse como argumentos de funciones como [`CONVERT`](https://dax.guide/convert) y [`DATATABLE`](https://dax.guide/datatable).
- Se añadió compatibilidad en el analizador semántico con nuevas funciones: [`ISINTEGER`](https://dax.guide/isinteger), [`ISCURRENCY`](https://dax.guide/iscurrency) y la función de cálculo Visual [`LOOKUPWITHTOTALS`](https://dax.guide/lookupwithtotals).
- Se añadió la opción de menú "Administrar paquetes DAX..." al hacer clic con el botón derecho sobre el nodo **Modelo** o **Funciones** en el Explorador TOM.

## Correcciones de errores en la versión 3.24.1

- Se corrigió un error que provocaba una excepción no controlada al iniciar Tabular Editor 3 en un monitor de baja resolución de píxeles
- Se corrigió un error que impedía abrir un modelo desde una estructura de carpetas PBIP al usar la opción de menú **Archivo > Abrir > Modelo desde archivo...**; consulta [#1507](https://github.com/TabularEditor/TabularEditor3/issues/1507).
- Se corrigieron problemas de escalado de DPI relacionados con la vista del Administrador de paquetes de DAX.
- Se corrigió un error del Analizador semántico por el que no se reconocían las funciones de cálculo Visual [`FIRST`](https://dax.guide/first) y [`LAST`](https://dax.guide/last).
- Se corrigió un error del Analizador semántico que mostraba una advertencia falsa al llamar a una UDF como primer argumento de [`CALCULATE`](https://dax.guide/calculate); consulta [#1509](https://github.com/TabularEditor/TabularEditor3/issues/1509).
- El Administrador de paquetes de DAX ya no muestra una advertencia debido a diferencias en los saltos de línea de la expresión DAX (PBI Desktop puede cambiar `\n` por `\r\n` al guardar un modelo).
- Se corrigió el orden de las UDF en los espacios de nombres del Explorador TOM

## Vienes de Tabular Editor 2.x?

Mira [este vídeo](https://youtu.be/O4ATwdzCvWc) para hacer un recorrido rápido por las principales características de Tabular Editor 3. Además, asegúrate de consultar nuestra [guía de incorporación](https://docs.tabulareditor.com/onboarding/index.html).

**Descripción general de las principales características de Tabular Editor 3:**

- IDE totalmente personalizable, con compatibilidad con varios monitores, pantallas Hi-DPI y temas
- Nuevo y potente editor de código DAX con autocompletado, comprobación de sintaxis, plegado de código y mucho, mucho más
- \*Modo del área de trabajo, que te permite guardar los cambios en el disco y sincronizar los metadatos del modelo con Analysis Services al mismo tiempo
- \*Previsualiza los datos de tabla con desplazamiento infinito, crea PivotGrids o escribe consultas DAX para explorar el modelo o probar la lógica de cálculo
- \*Programa actualizaciones de datos
- Actualiza los esquemas de las tablas tanto en Data sources del proveedor como en orígenes de datos estructurados (sí, ¡incluso para consultas M!)
- Crea diagramas del Data model
- Crea scripts de DAX que te permitan editar varias medidas u otros objetos calculados en un solo documento
- Graba C# Scripts y guárdalos como macros (antes conocidas como "Custom Actions")
- Integración con el Analizador VertiPaq
- Depurador de DAX
- Integración con el Optimizador de DAX
- Acciones de código para refactorizar fácilmente tu código DAX.

\*=Solo mientras esté conectado a una instancia de Analysis Services o Power BI
