## Novedades en 3.23.0

Consulta nuestro [blog de lanzamientos](https://blog.tabulareditor.com/) para obtener un breve resumen de las actualizaciones más importantes de esta versión.

- Tabular Editor 3 ahora admite las **funciones DAX definidas por el usuario (UDFs)**. Es una funcionalidad importante y de gran calado, y no podemos hacerle justicia en unas pocas viñetas. Consulta la [sección de documentación dedicada](xref:udfs) para saber más.
- Por si una gran funcionalidad no fuera suficiente, en esta versión también hemos añadido compatibilidad con los **Calendarios**. De nuevo, consulta la [sección de documentación dedicada](xref:calendars) para saber más.

> [!NOTE]
> [Tabular Editor 3.23.1](xref:release-3-23-1) incluye varias correcciones importantes de errores relacionadas con las UDF y los calendarios; por lo tanto, si tienes previsto usar estas funciones, recomendamos actualizar a la 3.23.1.

- El **Asistente para importar tablas** ahora admite **bases de datos SQL de Fabric** y **bases de datos reflejadas de Fabric** como **Data source**, tanto para modelos en modo Importación como en modo Direct Lake, de acuerdo con el [anuncio de la actualización de julio de 2025 de Power BI Desktop](https://powerbi.microsoft.com/en-us/blog/power-bi-july-2025-feature-summary/#post-30545-_Toc203388697).
- Compatibilidad nativa con [ARM64](https://learn.microsoft.com/en-us/windows/arm/overview).
- Preparativos para próximas funcionalidades de Power BI (que aún no podemos revelar).

## Mejoras en 3.23.0

- La ventana principal ahora recuerda su posición y tamaño. Se puede desactivar en Preferencia > Tabular Editor > Interfaz de usuario > Diseño.
- La vista de mensajes ahora copia la celda seleccionada en lugar de toda la fila de forma predeterminada; puedes usar Ctrl+Mayús+C para copiar toda la fila como antes. También hay un nuevo menú contextual al hacer clic con el botón derecho, con estas opciones (Copiar celda, Copiar fila).
- Dependencias de terceros actualizadas, incluidas AMO/TOM, a la [v. 19.103.2](https://www.nuget.org/packages/Microsoft.AnalysisServices/).
- Una nueva opción de ajuste de línea ahora está disponible en la barra de herramientas, el menú (Editar > Ajuste de línea) y la ventana de preferencia (Editores de texto > General). Los indicadores de ajuste de línea también se pueden habilitar en la preferencia.
- El autocompletado del editor de C# Script ahora ofrece una búsqueda más flexible (por ejemplo, buscar coincidencias parciales o por letras mayúsculas).
- Ahora puedes usar la acción "Aplicar" o "Aplicar selección" en consultas DAX que contengan instrucciones [`DEFINE COLUMN`](https://www.sqlbi.com/articles/introducing-define-column-in-dax-queries/) o [`DEFINE TABLE`](https://www.sqlbi.com/articles/introducing-define-table-in-dax-queries/), además de instrucciones `DEFINE MEASURE`. Esto creará las correspondientes columnas calculadas o tablas calculadas en el modelo, o actualizará sus expresiones DAX si ya existen.
- La ventana Ver la definición ahora permite desplazarse por el editor principal de DAX, en lugar de capturar el desplazamiento de la rueda del ratón cuando el puntero está sobre ella.
- La altura de la ventana Ver la definición ahora se ajusta de forma más precisa.
- El Analizador semántico ya no muestra el error "Una columna múltiple no se puede convertir en un valor escalar" cuando hay otros mensajes de error más indicativos.

## Correcciones de errores en 3.23.0

- Se corrigió un error por el que había que habilitar **Permitir operaciones de modelado no compatibles** para poder hacer cualquier cambio en un modelo en Power BI Desktop, aunque las operaciones de modelado eran compatibles.
- Se corrigió un error por el que la ventana principal superaba las dimensiones de la pantalla según la resolución y el escalado de DPI.
- Se corrigió un error por el que la configuración de personalización de la barra de herramientas, como "Iconos grandes", "Mostrar información sobre herramientas en las barras de herramientas", etc., no se conservaba tras reiniciar TE3.
- Se corrigió un fallo Visual por el que los controles desplegables de perspectiva e Idioma del Explorador TOM tenían botones duplicados.
- Se corrigieron varios problemas con la función "Ver la definición":
  - Ver la definición se cierra al deshacer o rehacer
  - Ver la definición no es visible cuando se usa en la última línea del editor de DAX
  - La vista "Ver la definición" se vuelve invisible cuando se elimina el salto de línea al final de la referencia previsualizada
  - La vista "Ver la definición" también se cierra al intentar, en su lugar, cerrar con ESC la ventana emergente de autocompletado
  - La vista "Ver la definición" también se cierra al intentar, en su lugar, borrar la selección rectangular con ESC
  - Cuando se añade una nueva línea a la izquierda de la referencia previsualizada, la vista "Ver la definición" se vuelve invisible y aparece incorrectamente un espacio de relleno vacío encima de la referencia previsualizada
  - Se corrigió el espacio de relleno vacío que aparecía debajo de una línea incorrecta si el cursor estaba en una línea distinta a la referencia previsualizada cuando se abre Ver la definición
- Los calltips del editor de código C# ahora tienen el tamaño correcto, lo que garantiza que todo el texto sea visible.
- Se corrigió un error por el que los Almacenes de Fabric no se enumeraban correctamente en el cuadro de diálogo de conexión de Fabric (en su lugar, se mostraban como tipo "Desconocido").
- El autocompletado de DAX ya no debería sugerir `DAY` para el parámetro &lt;Interval&gt; de la función [`PARALLELPERIOD`](https://dax.guide/parallelperiod), ya que no es un valor válido para ese parámetro.
- Los calltips ahora se actualizarán correctamente al alternar entre distintas alternativas de sintaxis con las flechas arriba/abajo.
- Se corrigió un error por el que las acciones de código no se actualizaban correctamente al cambiar entre objetos.
- Se corrigió el recorte de los números de línea en los editores de código al ampliar el zoom.

## Vienes de Tabular Editor 2.x?

Mira [este vídeo](https://youtu.be/O4ATwdzCvWc) para hacer un recorrido rápido por las principales características de Tabular Editor 3. Además, no dejes de consultar nuestra [guía de incorporación](https://docs.tabulareditor.com/onboarding/index.html).

**Descripción general de las principales características de Tabular Editor 3:**

- IDE totalmente personalizable, con soporte para varios monitores, Hi-DPI y temas
- Nuevo y potente editor de código DAX con autocompletado, comprobación de sintaxis, plegado de código y mucho, mucho más
- \*Modo del área de trabajo, que te permite guardar los cambios en disco y sincronizar simultáneamente los metadatos del modelo con Analysis Services
- \*Previsualizar los datos de tabla con desplazamiento infinito, crear PivotGrids o escribir consultas DAX para explorar el modelo o probar la lógica de cálculo
- \*Programar actualizaciones de datos
- Actualizar los esquemas de tabla tanto en el proveedor como en los orígenes de datos estructurados (sí, ¡incluso para consultas M!)
- Crear diagramas del Data model
- Crear scripts DAX que te permitan editar más de una medida u otros objetos calculados en un único documento
- Grabar C# Scripts y guardarlos como macros (antes conocidas como "Custom Actions")
- Integración con el Analizador VertiPaq
- Depurador de DAX
- Integración con el Optimizador de DAX
- Acciones de código para refactorizar fácilmente tu código DAX.

\*=Solo al conectarse a una instancia de Analysis Services o Power BI
