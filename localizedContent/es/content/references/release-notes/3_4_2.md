### Requisitos previos

Tabular Editor 3.3.0 (y versiones posteriores) usa .NET 6. Es posible que tengas que instalar el [.NET 6.0 Runtime for Desktop](https://dotnet.microsoft.com/en-us/download/dotnet/6.0/runtime) antes de poder iniciar Tabular Editor 3.3.0 (o posterior). A partir de la versión 3.3.1, el instalador de Tabular Editor 3 descargará e instalará estos requisitos previos si faltan.

## Correcciones de errores en 3.4.2

- Se corrigió un problema por el que el Analizador semántico informaba de errores de "dependencia circular" cuando no existían. Consulta [#811](https://github.com/TabularEditor/TabularEditor3/issues/811).
- Buscar/Reemplazar ya no debería reemplazar todas las coincidencias en el documento al establecer "Buscar en" en "Selección"; consulta [#293](https://github.com/TabularEditor/TabularEditor3/issues/293).
- Copiar y pegar configuraciones regionales entre modelos ya no debería hacer que la aplicación se bloquee; consulta el problema [#798](https://github.com/TabularEditor/TabularEditor3/issues/798). Ten en cuenta que se eliminan las ObjectTranslations que hacen referencia a objetos que no existen en el modelo de destino.
- El Depurador de DAX ahora debería permitir inspeccionar expresiones de seguimiento de varias filas en todos los casos.
- El Depurador de DAX ya no debería bloquearse al intentar depurar el resultado de una consulta como `EVALUATE { [medida que usa ALLSELECTED] }`

## ¿Vienes de Tabular Editor 2.x?

Mira [este vídeo](https://www.youtube.com/watch?v=pt3DdcjfImY) para hacerte una idea de las nuevas funciones de Tabular Editor 3. Además, asegúrate de consultar nuestra [guía de inicio](https://docs.tabulareditor.com/onboarding/index.html).

**Descripción general de las principales características de Tabular Editor 3:**

- IDE totalmente personalizable, con compatibilidad con varios monitores, pantallas Hi-DPI y temas
- Nuevo y potente editor de código DAX con autocompletado, comprobación de sintaxis, plegado de código y mucho, mucho más
- \*Modo del área de trabajo, que te permite guardar los cambios en el disco y sincronizar simultáneamente los metadatos del modelo con Analysis Services
- \*Previsualizar los datos de tablas con desplazamiento infinito, crear PivotGrids o escribir consultas DAX para explorar el modelo o probar la lógica de cálculo
- \*Programar actualizaciones de datos
- Actualizar los esquemas de las tablas tanto en Data sources de proveedor como en orígenes de datos estructurados (sí, incluso para consultas M)
- Crear diagramas de Data model
- Crear scripts DAX que te permitan editar varias medidas u otros objetos calculados en un único documento
- Graba C# Scripts y guárdalos como macros (antes conocidas como "Custom Actions")
- Integración con el Analizador VertiPaq
- \*Depurador de DAX

\*=Solo mientras se esté conectado a una instancia de Analysis Services o Power BI
