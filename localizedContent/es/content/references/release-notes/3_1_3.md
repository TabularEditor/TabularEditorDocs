## Correcciones de errores en 3.1.3

- Se corrigió una fuga de memoria que se producía cuando la aplicación estaba inactiva. Este problema existía desde la versión 3.1.1.

## Correcciones de errores en 3.1.2

- Se corrigió un problema por el que **Actualizar esquema de tabla** no funcionaba con Data source de SQL DW / Synapse SQL Pool; consulta [#231](https://github.com/TabularEditor/TabularEditor3/issues/231).
- Se corrigió un problema por el que el Best Practice Analyzer seguía teniendo en cuenta reglas ignoradas; consulta [#203](https://github.com/TabularEditor/TabularEditor3/issues/203).
- Se corrigió un problema por el que no se podían guardar modelos en el disco (archivo o carpeta) debido a un error de serialización JSON; consulta [#232](https://github.com/TabularEditor/TabularEditor3/issues/232).
- Se corrigió un problema que provocaba un cierre inesperado al inspeccionar los valores de propiedad de un origen de datos estructurado.

## Mejoras en 3.1.1

- Las expresiones compartidas (expresiones de Power Query / M) ahora pueden serializarse como archivos individuales. Puedes controlar la serialización en **Herramientas > Preferencias > Guardar en carpeta** o, para un modelo existente, en **Modelo > Opciones de serialización...** ([#218](https://github.com/TabularEditor/TabularEditor3/issues/218))
- Nuevas opciones para controlar cómo se aplica la pertenencia a perspectivas a los nuevos objetos en **Herramientas > Preferencias > Operaciones de modelado** (consulta la incidencia [#216](https://github.com/TabularEditor/TabularEditor3/issues/216)).
- La propiedad `PrivacySetting` ahora puede excluirse de la serialización para evitar problemas con Microsoft.AnalysisServices.Deployment. Esto se controla en **Modelo > Opciones de serialización...** (consulta la incidencia [#210](https://github.com/TabularEditor/TabularEditor3/issues/210)).
- Ahora puedes definir enums, structs y clases directamente en un C# Script, lo que facilita estructurar y reutilizar el código C#.
- El Analizador VertiPaq ahora incluye una opción para recopilar estadísticas de los datos (**Herramientas > Preferencias > Analizador VertiPaq**), lo que mejora la calidad de las estadísticas; por ejemplo, cuando las columnas tienen IsAvailableInMDX = false. Consulta la incidencia [#190](https://github.com/TabularEditor/TabularEditor3/issues/290).
- Cuando se crea un nuevo modelo, Tabular Editor 3 ahora pregunta, la primera vez que se guarda, si debe guardarse como archivo o como carpeta. También puedes configurar el modo predeterminado en **Tools > Preferencias > General > Default save format for new models**. Consulta la incidencia [#50](https://github.com/TabularEditor/TabularEditor3/issues/50).
- Mejor rendimiento de inicio (algunos usuarios han reportado mejoras adicionales del tiempo de arranque al añadir el conmutador `-multicorejit` al acceso directo de TabularEditor3.exe).
- Se reestructuró el menú "Model" para que opciones como "Nuevo grupo de cálculo" estén más a mano.
- Se actualizaron las bibliotecas cliente de AMO/TOM a la versión [19.26.1.7](https://www.nuget.org/packages/Microsoft.AnalysisServices.retail.amd64/).
- Ahora es posible iterar los miembros del rol del modelo de forma individual en el Best Practice Analyzer
- La opción "Workspace" del menú "Window" se ha cambiado de nombre a "Layout" para evitar confusiones con [Modo del área de trabajo](https://docs.tabulareditor.com/te3/tutorials/workspace-mode.html).
- Se añadió Ctrl+Shift+D como método abreviado para "Deploy"
- Compatibilidad con nuevas funciones DAX ([`HASH`](https://dax.guide/hash), [`NAMEOF`](https://dax.guide/nameof), [`COLUMNSTATISTICS`](https://dax.guide/columnstatistics), [`BITAND`](https://dax.guide/bitand), [`BITLSHIFT`](https://dax.guide/bitlshift), [`BITOR`](https://dax.guide/bitor), [`BITRSHIFT`](https://dax.guide/bitrshift), [`BITXOR`](https://dax.guide/bitxor)) y compatibilidad con el parámetro opcional de nombre de columna en [`SELECTCOLUMNS`](https://dax.guide/selectcolumns).

## Correcciones de errores en 3.1.1

- El Explorador TOM ya no debería bloquearse ni cerrarse inesperadamente al eliminar objetos (consulta las incidencias [#223](https://github.com/TabularEditor/TabularEditor3/issues/223), [#209](https://github.com/TabularEditor/TabularEditor3/issues/209)).
- Se corrigió un problema de colores con los temas oscuros en los editores ([#207](https://github.com/TabularEditor/TabularEditor3/issues/207)) y con los iconos ([#215](https://github.com/TabularEditor/TabularEditor3/issues/215))
- Las tablas ahora se ordenan alfabéticamente en el cuadro de diálogo "Crear/Editar relación" (consulta la incidencia [#214](https://github.com/TabularEditor/TabularEditor3/issues/214)
- Se corrigió un problema con los C# Script que impedía guardarlos como macros (consulta la incidencia [#208](https://github.com/TabularEditor/TabularEditor3/issues/208)).
- Se corrigió un problema por el que las contraseñas/credenciales del Data source no se aplicaban correctamente a la base de datos del Workspace (consulta la incidencia [#205](https://github.com/TabularEditor/TabularEditor3/issues/205)).
- La casilla "Select all columns" ya no debería provocar un bloqueo (consulta la incidencia [#204](https://github.com/TabularEditor/TabularEditor3/issues/204)).
- Tabular Editor ahora siempre busca en el directorio de instalación cuando un script hace referencia a DLL externas (mediante la directiva del precompilador `#r`); consulta la incidencia [#192](https://github.com/TabularEditor/TabularEditor3/issues/192).
- Se corrigió un error del esquema lingüístico que hacía que Q&A dejara de funcionar en algunos casos
- Filtrar por una columna de fecha en la vista previa de datos ya no debería provocar un bloqueo
- Al seleccionar todas las perspectivas en el desplegable de perspectivas del Explorador TOM, ya no se muestran los objetos que no están incluidos en ninguna perspectiva. También se corrigió un problema por el que el cuadro desplegable no actualizaba los nombres de las perspectivas.
- Ahora se permite usar ciertas palabras clave de DAX como variables (`WEEK`, `DAY`, `INTEGER`, etc.) sin mostrar un error.
- Se corrigió un problema por el que las opciones de serialización no se guardaban la primera vez que se guardaba un modelo.
- Se corrigió un problema con el analizador de DAX que provocaba un bloqueo al cargar un modelo (consulta el [#229](https://github.com/TabularEditor/TabularEditor3/issues/229)).

## ¿Vienes de Tabular Editor 2.x?

Mira [este vídeo](https://www.youtube.com/watch?v=pt3DdcjfImY) para hacerte una idea de las nuevas funciones de Tabular Editor 3.

**Descripción general de las principales características de Tabular Editor 3:**

- IDE totalmente personalizable, con soporte para varios monitores, Hi-DPI y temas
- Nuevo y potente editor de código DAX con autocompletado, comprobación de sintaxis, plegado de código y mucho, mucho más
- \*Modo del área de trabajo, que te permite guardar los cambios en disco y sincronizar simultáneamente los metadatos del modelo con Analysis Services
- \*Previsualiza los datos de las tablas con desplazamiento infinito, crea PivotGrids o escribe consultas DAX para explorar el modelo o probar la lógica de cálculo
- \*Programa actualizaciones de datos
- Actualiza los esquemas de las tablas tanto en el proveedor como en los orígenes de datos estructurados (sí, ¡incluso para consultas M!)
- Crea diagramas del Data model
- Crea scripts DAX que te permiten editar varias medidas u otros objetos calculados en un solo documento
- Graba C# Scripts y guárdalos como macros (antes se conocían como "Custom Actions")
- Integración con el Analizador VertiPaq

\*=Solo cuando estés conectado a una instancia de Analysis Services o Power BI
