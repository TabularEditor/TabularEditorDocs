## Mejoras de la versión 3.26.1

### Asistente de IA

- Se redujo el consumo de tokens del asistente de IA mediante un enfoque de divulgación progresiva mejorado para la detección de metadatos del modelo, lo que debería ayudar a evitar alcanzar los límites de velocidad de tokens de entrada.
- Se habilitó el uso del almacenamiento en caché para los modelos de Anthropic (antes no se almacenaban en caché)
- Herramientas más precisas y mejor orientadas que ayudan a los LLM a producir mejores resultados, incluida una herramienta de búsqueda de metadatos del modelo.
- Los proveedores de IA de OpenAI y Anthropic ahora admiten la sustitución opcional de la URL base, lo que permite usarlos con puntos de conexión regionales de OpenAI y con modelos implementados en Microsoft Foundry.
- Para OpenAI y Anthropic, el campo de nombre del modelo ahora se precompleta con modelos conocidos, sin dejar de permitir nombres de modelo personalizados.
- Al pasar el cursor sobre el contador de tokens del chat de IA, se muestra una descripción emergente con el desglose de tokens de entrada a precio completo, tokens de entrada en caché, escrituras en caché y tokens de salida. La barra deslizante sigue mostrando la presión sobre la ventana de contexto.

### Asistente para importar tablas

- El Asistente de importación de tablas ahora filtra las listas de catálogo y de esquema para las conexiones de Databricks según la configuración ya especificada en el Data source. Cuando ya hay un catálogo configurado, el asistente omite la consulta del catálogo. Las tablas y vistas se filtran por esquema cuando se especifica uno, lo que reduce el desorden en catálogos grandes.
- Las nuevas conexiones de Databricks ahora usan de forma predeterminada Implementation 2,0 en el cuadro de diálogo de conexión. Las conexiones existentes que no especifican explícitamente una versión de Implementation se mantienen sin cambios.
- Se agregó compatibilidad con la autenticación de service principal de Databricks mediante el flujo OAuth Machine-to-Machine (M2M). Una nueva opción "OAuth (M2M)" en el cuadro de diálogo de conexión de Databricks permite autenticarse con un Client ID y un Client Secret. El controlador ODBC se encarga automáticamente de la adquisición y la renovación de tokens. Esto funciona en todas las nubes de Databricks (Azure, AWS, GCP).

## Correcciones en 3.26.1

- Se corrigió un error por el que las instrucciones personalizadas de inyección permanente (p. ej., `core.md`) se incluían tanto en el prompt base del sistema como en la inyección de cada turno, lo que provocaba ~1.200 tokens de contenido duplicado en cada llamada a la API.
- Se corrigió un problema por el que los atajos de teclado Ctrl+Z (deshacer) y Ctrl+Y (rehacer) no funcionaban en el campo de entrada del chat del asistente de IA.
- Se corrigió un problema por el que el asistente de IA fallaba con un error HTTP 400 al usar modelos que no admiten valores de temperatura personalizados (por ejemplo, los modelos de razonamiento o-series de OpenAI). Ahora el asistente reintenta automáticamente sin el parámetro de temperatura cuando se detecta este error.
- Se corrigió el renderizado con poco contraste del código en línea (p. ej., `` `Tokenize()` ``) dentro de los mensajes de chat del usuario en el tema claro.
- Se corrigió un problema por el que, al hacer clic en "Permitir" (no en "Permitir para la sesión") en un aviso de consentimiento, se concedía silenciosamente el consentimiento a una segunda invocación de herramienta en el mismo turno sin volver a preguntar. Ahora cada invocación de herramienta requiere su propio consentimiento cuando se usa la aprobación de un solo uso.
- Se corrigió un problema por el que la búsqueda en la base de conocimientos del Asistente de IA fallaba cuando la consulta contenía caracteres especiales como `#` (por ejemplo, al buscar "compatibilidad con el lenguaje C#").
- Se corrigió un problema por el que el tooltip del endpoint del proveedor de IA no se actualizaba al cambiar de proveedor en el cuadro de diálogo de preferencias.
- Se corrigió un problema por el que el Asistente de IA fallaba al chatear con modelos hospedados en Databricks mediante un proveedor personalizado compatible con OpenAI.
- Se corrigió un problema por el que la invocación de herramientas del Asistente de IA fallaba silenciosamente al usar proveedores personalizados compatibles con OpenAI, como OpenRouter. El asistente recibía solicitudes del modelo para invocar herramientas, pero nunca las ejecutaba, lo que hacía que pareciera bloqueado o que respondiera solo con texto.
- Se corrigió un problema por el que el indicador en línea "Actualizando el contenido del documento" mostraba el ID del documento en lugar del contenido nuevo en el tooltip y en el botón Copiar.
- Se corrigió que la preferencia de ID de proyecto de OpenAI estuviera vinculada incorrectamente al ID de organización.
- Se corrigió un problema por el que el Asistente de importación de tablas reconstruía desde cero las opciones de la expresión M de Databricks en lugar de conservar las opciones originales del Data source, lo que podía provocar la pérdida de configuraciones no predeterminadas.
- Se corrigió un error que podía hacer que el Explorador TOM o la vista de propiedades se "bloquearan" después de que se produjera un fallo en el Asistente de importación de tablas, lo que obligaba a reiniciar la aplicación.
- Se corrigieron algunos problemas de localización por los que ciertas cadenas de la interfaz de usuario no se podían traducir.
- Se corrigió una `FileNotFoundException` no controlada al iniciar, cuando el runtime de Microsoft Edge WebView2 estaba instalado parcialmente o estaba dañado.
- Se corrigió un problema por el que los cuadros de diálogo modales de mensajes (por ejemplo, "No se detectaron cambios de esquema") podían aparecer en un monitor distinto al de la ventana principal de la aplicación, haciendo que Tabular Editor pareciera bloqueado. Ahora, los cuadros de diálogo siempre aparecen centrados en el monitor del formulario principal y, al hacer clic en el formulario principal deshabilitado, el cuadro de diálogo parpadea para llamar la atención.
- Se corrigió una NullReferenceException que podía producirse al importar tablas desde un Lakehouse de Fabric, si el identificador de Data source había cambiado desde la última vez que se guardaron los ajustes de importación de tablas (por ejemplo, después de actualizar el nivel de compatibilidad del modelo o de cambiar los tipos de partición).
- Se corrigió un problema por el que el cuadro de diálogo de autenticación de Windows Account Manager (WAM) podía aparecer detrás de la rueda de carga de "Espere" durante operaciones como "Actualizar esquema de tabla", haciendo que la aplicación pareciera bloqueada mientras esperaba la interacción del usuario en un cuadro de diálogo oculto.

## Vienes de Tabular Editor 2.x?

Mira [este vídeo](https://youtu.be/O4ATwdzCvWc) para hacer un recorrido rápido por las principales funcionalidades de Tabular Editor 3. Además, no dejes de consultar nuestra [guía de incorporación](https://docs.tabulareditor.com/onboarding/index.html).

**Descripción general de las principales características de Tabular Editor 3:**

- IDE totalmente personalizable, con soporte para varios monitores, Hi-DPI y temas
- Nuevo y potente editor de código DAX con autocompletado, comprobación de sintaxis, plegado de código y mucho, mucho más
- \*Modo del área de trabajo, que te permite guardar los cambios en disco y sincronizar simultáneamente los metadatos del modelo con Analysis Services
- \*Previsualizar datos de tablas con desplazamiento infinito, crear PivotGrids o escribir consultas DAX para explorar el modelo o probar la lógica de cálculo
- \*Programar actualizaciones de datos
- Actualizar los esquemas de tabla tanto en el Data source de tipo Provider como en los orígenes de datos estructurados (sí, ¡incluso para consultas M!)
- Crear diagramas del Data model
- Crear scripts DAX que te permitan editar varias medidas u otros objetos calculados en un único documento
- Grabar C# Scripts y guardarlos como macros (antes conocidas como "Acciones personalizadas")
- Integración del Analizador VertiPaq
- Depurador de DAX
- Integración del Optimizador de DAX
- Acciones de código para refactorizar fácilmente tu código DAX.

\*=Solo al estar conectado a una instancia de Analysis Services o Power BI
