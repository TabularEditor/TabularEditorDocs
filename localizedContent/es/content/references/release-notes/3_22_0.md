## Novedades en 3.22.0

Consulta nuestro [blog del lanzamiento](https://tabulareditor.com/blog/tabular-editor-3-june-2025-release) para obtener un breve resumen de las actualizaciones más importantes de esta versión.

(**Solo para la Edición Enterprise**) Tabular Editor 3 ahora es totalmente compatible con Direct Lake en OneLake y con Direct Lake en SQL. Esto significa:

- El Asistente de importación de tablas incluye nuevas opciones de importación para Warehouse y Lakehouse de Fabric. Ya no necesitas conocer el punto de conexión de análisis SQL del Warehouse/Lakehouse al que quieras conectarte. En su lugar, te mostramos una lista de todos los Warehouse/Lakehouse de tu Workspace para que puedas elegir el que quieras usar.
- En la última página del Asistente de importación de tablas, puedes elegir si usar Direct Lake en OneLake o en modo SQL, o si configurar la tabla en el clásico modo "import".
- Cuando el modelo contiene tablas en modo Direct Lake en OneLake, ahora también puedes agregar tablas en Import mode para crear modelos híbridos o de modo mixto.
- Al previsualizar datos en tablas Direct Lake, ya no limitamos el número de filas devueltas (por lo que puedes desplazarte libremente por todos los datos de la tabla, igual que en Import mode). Ten en cuenta que, cuando haces una vista previa de datos, todas las columnas se cargan en el modelo semántico en la capacidad de Fabric. Considera usar una Consulta DAX si te preocupa el consumo de memoria en tu capacidad de Fabric.

> [!NOTE]
> Ya no configuramos los modelos Direct Lake para usar una intercalación que distinga entre mayúsculas y minúsculas. Si tienes pensado usar Direct Lake en SQL con un Warehouse de Fabric que use una intercalación que distinga entre mayúsculas y minúsculas, debes completar manualmente la propiedad **Collation** de tu modelo semántico _antes_ de agregar tablas u otros objetos.

Para más información, consulta nuestro [artículo de recomendaciones de Direct Lake](xref:direct-lake-guidance).

## Mejoras en 3.22.0

- Al guardar un modelo como un único archivo, el cuadro de diálogo **Guardar archivo** ahora sugiere "model.bim" como nombre de archivo predeterminado.
- Se actualizaron varias dependencias a sus versiones más recientes, incluida [AMO/TOM a 19.98.0.3](https://www.nuget.org/packages/Microsoft.AnalysisServices/). Esta actualización corrige problemas de autenticación persistentes con cuentas profesionales o educativas y los avisos repetidos para iniciar sesión.
- Se cambió el nivel de compatibilidad predeterminado de los nuevos modelos semánticos de Power BI/Fabric a **1609**, lo que te permite controlar el comportamiento predeterminado de las [Selection Expressions](https://powerbi.microsoft.com/en-us/blog/deep-dive-into-selection-expressions-for-calculation-groups/) en tus grupos de cálculo, mediante la propiedad del modelo [SelectionExpressionBehavior](https://learn.microsoft.com/en-us/dotnet/api/microsoft.analysisservices.tabular.model.selectionexpressionbehavior?view=analysisservices-dotnet).
- Compatibilidad del editor de Consulta DAX con la palabra clave [`MPARAMETER`](https://dax.guide/st/mparameter/); consulta [#1467](https://github.com/TabularEditor/TabularEditor3/issues/1467).
- Nuestros editores DAX ahora admiten términos de búsqueda para el autocompletado **basados en palabras**. En otras palabras, si escribes `sales ytd` en el editor, el autocompletado ahora sugerirá medidas como `[Sales Margin YTD]`, `[Sales Revenue YTD]`, etc.
- Con la actualización de junio de 2025 de Power BI Desktop, [las herramientas externas ahora pueden realizar cualquier operación de escritura en el modelo semántico](https://powerbi.microsoft.com/en-us/blog/power-bi-june-2025-feature-summary/#post-30307-_Toc269410729). Por lo tanto, ya no restringimos las operaciones de modelado disponibles en Tabular Editor al conectarse a un modelo en esta o en versiones posteriores de Power BI Desktop.

## Correcciones de errores en 3.22.0

- Se agregó el control de excepciones al encontrarse con excepciones IOException durante las operaciones de guardado en carpeta (cuando a veces necesitamos eliminar carpetas de la estructura de carpetas, lo que puede verse bloqueado por bloqueos del sistema de archivos aplicados por sistemas de control de versiones).
- Se corrigió un error por el que ciertos atajos de teclado (tanto los predeterminados como los personalizados) no funcionaban cuando la página "Novedades" estaba enfocada.
- Al asignar un atajo de teclado personalizado que ya está asignado a otro comando, ahora mostramos un mensaje de advertencia y quitamos el atajo del otro comando, para evitar ambigüedades.
- Se corrigió un error por el que el Analizador semántico mostraba un mensaje de error falso al usar referencias de columna no calificadas en una función de ventana DAX. Consulta [#1460](https://github.com/TabularEditor/TabularEditor3/issues/1460).
- Se corrigió un error que a veces podía hacer que la vista **Actualización de datos** se quedara bloqueada, lo que obligaba a reiniciar la aplicación para solucionarlo. Consulta [#1461](https://github.com/TabularEditor/TabularEditor3/issues/1461).
- Se corrigieron varios problemas menores del editor DAX relacionados con el autocompletado, el formato automático y el resaltado de sintaxis de palabras clave.
- Ahora se pueden modificar los nombres de Query Group (es decir, las "carpetas de visualización" para expresiones M compartidas y particiones M). Además, si pegas un objeto con un Query Group en un modelo que no contenga dicho Query Group, se creará automáticamente para evitar el mensaje de error en posteriores operaciones de guardado o implementación del modelo.
- Si pegas un objeto que contiene una referencia a una Shared Expression en un modelo que no contiene esa Shared Expression, eliminamos la referencia para evitar errores de tipo "el objeto no existe" al guardar o implementar el modelo en Analysis Services / Power BI.
- Se corrigió un error por el que, a veces, no era posible deshacer cambios anteriores después de guardar los metadatos del modelo de nuevo en el servidor.

## Vienes de Tabular Editor 2.x?

Mira [este vídeo](https://youtu.be/O4ATwdzCvWc) para hacer un recorrido rápido por las principales funcionalidades de Tabular Editor 3. Además, no dejes de consultar nuestra [guía de incorporación](https://docs.tabulareditor.com/onboarding/index.html).

**Resumen de las principales funcionalidades de Tabular Editor 3:**

- IDE totalmente personalizable, con soporte para varios monitores, Hi-DPI y temas
- Nuevo y potente editor de código DAX con autocompletado, comprobación de sintaxis, plegado de código y mucho, mucho más
- \*Modo del área de trabajo, que te permite guardar los cambios en el disco y sincronizar simultáneamente los metadatos del modelo con Analysis Services
- \*Previsualiza los datos de las tablas con desplazamiento infinito, crea PivotGrids o escribe Consultas DAX para explorar el modelo o probar la lógica de cálculo
- \*Programa actualizaciones de datos
- Actualiza los esquemas de las tablas tanto en los Data sources del proveedor como en los orígenes de datos estructurados (sí, incluso para consultas M)
- Crea diagramas del Data model
- Crea scripts DAX que te permitan editar varias medidas u otros objetos calculados en un único documento
- Graba C# Scripts y guárdalos como macros (antes conocidas como "Acciones personalizadas")
- Integración con el Analizador VertiPaq
- Depurador de DAX
- Integración con el Optimizador de DAX
- Acciones de código para refactorizar fácilmente tu código DAX.

\*=Solo mientras estés conectado a una instancia de Analysis Services o Power BI
