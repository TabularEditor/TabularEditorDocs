### Requisitos previos

Tabular Editor 3.3.0 (y versiones posteriores) usa .NET 6. Puede que tengas que instalar el [.NET 6.0 Runtime para escritorio](https://dotnet.microsoft.com/en-us/download/dotnet/6.0/runtime) antes de poder iniciar Tabular Editor 3.3.0 (o una versión posterior). A partir de la versión 3.3.1, el instalador de Tabular Editor 3 descargará e instalará estos requisitos previos si faltan.

## Novedades de 3.10.0

Consulta nuestro [blog de la versión](https://blog.tabulareditor.com/2023/08/23/tabular-editor-3-august-2023-release/) para ver un resumen de las actualizaciones más importantes de esta versión.

- Tabular Editor ahora permite agregar tablas a [Datasets de Direct Lake](https://learn.microsoft.com/en-us/power-bi/enterprise/directlake-overview) mediante el Asistente para importar tablas. Además, puedes actualizar el esquema de las tablas existentes de la forma habitual. Tabular Editor utiliza el [punto de conexión SQL de tu Lakehouse de Fabric](https://learn.microsoft.com/en-us/fabric/data-engineering/lakehouse-sql-endpoint) para obtener los esquemas de las tablas. También hemos publicado una [entrada del blog sobre el Dataset de DirectLake, por si quieres saber más](https://blog.tabulareditor.com/2023/08/23/fabric-direct-lake-dataset/).

> [!NOTE]
> Esta versión de Tabular Editor te permitirá importar tablas o actualizar el esquema de las tablas en modelos que ya tengan una conexión a Direct Lake. El Asistente para importar tablas todavía no permite crear una nueva conexión de Direct Lake en un modelo vacío. Esto se añadirá en una versión futura.

> [!WARNING]
> Una vez que realices algún cambio en un Dataset de Direct Lake a través del punto de conexión XMLA, [ya no podrás abrir ese Dataset en la funcionalidad de modelado web de Fabric](https://learn.microsoft.com/en-us/power-bi/enterprise/directlake-overview#enable-xmla-read-write:~:text=Direct%20Lake%20datasets%20created%20or%20modified%20by%20using%20XMLA%2Dbased%20tools%20cannot%20be%20opened%20in%20the%20Web%20modelling%20feature.).

- Los C# Scripts ahora admiten [directivas del preprocesador](xref:csharp-scripts#compatibility) para determinar qué versión de Tabular Editor estás usando. Esto te permite escribir scripts compatibles tanto con Tabular Editor 2 como con Tabular Editor 3.

## Mejoras en 3.10.0

- Ahora usamos la versión más reciente de [AMO/TOM](https://www.nuget.org/packages/Microsoft.AnalysisServices.NetCore.retail.amd64/). Esta actualización añade compatibilidad con Binary XML, lo que debería ayudar a acelerar las operaciones XMLA. Más detalles en esta [entrada del blog de Microsoft](https://powerbi.microsoft.com/en-us/blog/improving-the-communication-performance-of-xmla-based-tools/).
- También actualizamos [TMDL a preview-3](https://www.nuget.org/packages/Microsoft.AnalysisServices.Tabular.Tmdl.NetCore.retail.amd64/19.65.12.3-TmdlPreview), que incluye varias correcciones de errores.

> [!NOTE]
> TMDL sigue en versión preliminar y, por lo tanto, esta función también debe considerarse una función en versión preliminar de Tabular Editor 3. Asegúrate de conservar una copia de seguridad de los metadatos de tu modelo en Model.bim / Database.json para evitar perder trabajo.

- Al clonar objetos, ahora añadimos el sufijo "-Copy" en lugar de " Copy" para reducir el riesgo de dejar espacios en blanco al final de los nombres de los objetos. Consulta [#940](https://github.com/TabularEditor/TabularEditor3/issues/940).
- La ventana emergente de autocompletado ahora ajusta automáticamente su ancho para adaptarse al elemento más ancho que contenga. Consulta [#965](https://github.com/TabularEditor/TabularEditor3/issues/965).
- Cuando se guarda un archivo .bim de copia de seguridad para un modelo de Power BI Desktop, ahora usamos el nombre del archivo .pbix en lugar del GUID de la base de datos; consulta [#958](https://github.com/TabularEditor/TabularEditor3/issues/958).
- Mejoras de rendimiento en la Vista previa de tabla.

## Correcciones de errores en 3.10.0

- Se corrigió un error que impedía que los sinónimos (metadatos lingüísticos) se guardaran al usar "Guardar en carpeta".
- Se corrigió un error por el que el archivo .bim de copia de seguridad tenía una marca de tiempo incorrecta; consulta [#967](https://github.com/TabularEditor/TabularEditor3/issues/967).
- Se corrigió un error que impedía al usuario cancelar la implementación cuando, por cualquier motivo, fallaba el guardado de la copia de seguridad.
- El panel Buscar de la Vista previa de tabla ahora debería generar expresiones de búsqueda DAX válidas en segundo plano; consulta [#955](https://github.com/TabularEditor/TabularEditor3/issues/955).
- Se corrigió un problema que hacía que la acción "Ir a definición (F12)" se deshabilitara cuando el cursor se colocaba sobre una función DAX; consulta [#962](https://github.com/TabularEditor/TabularEditor3/issues/962).
- Se corrigió un error que, en algunos casos, impedía cargar modelos desde archivos .vpax.
- La altura del panel de "recuento de resultados" en las consultas DAX ahora debería ajustarse automáticamente a su contenido.
- La cuadrícula de Propiedades ahora muestra correctamente expresiones de varias líneas que comienzan con uno o más saltos de línea, igual que al editar la expresión dentro de la cuadrícula.
- Se corrigió un error que podía hacer que Pivot Grid quedara atrapado en un bucle infinito de mensajes de error cuando se eliminaban del modelo remoto columnas que se habían agregado a Pivot Grid.
- Se corrigió un error por el que las macros añadidas a la barra de menús dejaban de funcionar tras reiniciar Tabular Editor.
- Se corrigió un error por el que, al intentar realizar una Actualización de esquema en varias tablas procedentes de la misma tabla de origen, el proceso fallaba en todas excepto en una.

## ¿Vienes de Tabular Editor 2.x?

Mira [este vídeo](https://www.youtube.com/watch?v=pt3DdcjfImY) para hacerte una idea de las nuevas características de Tabular Editor 3. Además, asegúrate de consultar nuestra [guía de inicio](https://docs.tabulareditor.com/getting-started/index.html).

**Descripción general de las principales características de Tabular Editor 3:**

- IDE totalmente personalizable, con compatibilidad con varios monitores, Hi-DPI y temas
- Nuevo y potente editor de código DAX con autocompletado, comprobación de sintaxis, plegado de código y mucho, mucho más
- \*Modo del área de trabajo Workspace, que te permite guardar tus cambios en disco y sincronizar los metadatos del modelo con Analysis Services simultáneamente
- \*Previsualiza los datos de tabla con desplazamiento infinito, crea PivotGrids o escribe consultas DAX para explorar el modelo o probar la lógica de cálculo
- \*Programa actualizaciones de datos
- Actualizar los esquemas de las tablas tanto en los Data sources del proveedor como en los orígenes de datos estructurados (sí, incluso para consultas M)
- Crear diagramas del Data model
- Crear scripts de DAX que permitan editar varias medidas u otros objetos calculados en un solo documento
- Grabar C# Scripts y guardarlos como macros (antes conocidas como "Acciones personalizadas")
- Integración con el Analizador VertiPaq
- Depurador de DAX

\*=Solo cuando se está conectado a una instancia de Analysis Services o de Power BI
