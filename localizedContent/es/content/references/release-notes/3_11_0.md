### Requisitos previos

Tabular Editor 3.3.0 (y versiones posteriores) usa .NET 6. Es posible que tengas que instalar el [Runtime de escritorio de .NET 6.0](https://dotnet.microsoft.com/en-us/download/dotnet/6.0/runtime) antes de poder iniciar Tabular Editor 3.3.0 (o una versión posterior). A partir de la versión 3.3.1, el instalador de Tabular Editor 3 descargará e instalará estos requisitos previos si faltan.

## Novedades de 3.11.0

Consulta nuestro [blog de versiones](https://blog.tabulareditor.com/2023/09/25/september-release-tabular-editor-3-11-0/) para ver un resumen de las actualizaciones más importantes de esta versión.

- Como se indicó en las [notas de la versión del mes pasado](xref:release-3-10-1), Tabular Editor no te permitía crear un [Dataset de Direct Lake](https://learn.microsoft.com/en-us/power-bi/enterprise/directlake-overview) desde cero. ¡Ahora ya es posible! El único requisito previo es que el modelo no tenga ninguna tabla ni ningún Data source explícito definido, y que el modo de compatibilidad esté configurado como PowerBI. Consulta nuestra [entrada del blog sobre Direct Lake](https://blog.tabulareditor.com/2023/08/23/fabric-direct-lake-dataset/) si quieres saber más.

> [!WARNING]
> Una vez que realices algún cambio en un Dataset de Direct Lake mediante el punto de conexión XMLA, [ya no podrás abrir ese Dataset en la función de modelado en la web de Fabric](https://learn.microsoft.com/en-us/power-bi/enterprise/directlake-overview#enable-xmla-read-write:~:text=Direct%20Lake%20datasets%20created%20or%20modified%20by%20using%20XMLA%2Dbased%20tools%20cannot%20be%20opened%20in%20the%20Web%20modelling%20feature.).

## Mejoras en 3.11.0

- Ahora usamos la versión más reciente de [AMO/TOM](https://www.nuget.org/packages/Microsoft.AnalysisServices.NetCore.retail.amd64/) y [TMDL](https://www.nuget.org/packages/Microsoft.AnalysisServices.Tabular.Tmdl.NetCore.retail.amd64), que incluye varias correcciones de errores y mejoras de estabilidad.

> [!NOTE]
> TMDL sigue en versión preliminar y, como tal, esta función también debe considerarse una función en versión preliminar de Tabular Editor 3. Asegúrate de conservar una copia de seguridad de los metadatos del modelo en Model.bim / Database.json para evitar perder trabajo.

- Nuestro analizador de DAX y el autocompletado ahora admiten el tercer parámetro recién añadido a [`MINX`](https://dax.guide/minx)/[`MAXX`](https://dax.guide/maxx). Además, ahora también ofrecemos sugerencias para la función [`MATCHBY`](https://dax.guide/matchby), que se puede usar como parámetro opcional de las funciones de ventana.
- La opción Native Query del Asistente de importación de tablas ahora está disponible al importar desde Databricks, siempre que se haya especificado un catálogo.

## Correcciones de errores en 3.11.0

- La lista de nombres de tabla reservados en DAX se ha actualizado usando el DMV `$system.discover_keywords`. Como consecuencia, nuestro formateador de DAX integrado ahora pone comillas correctamente a determinados nombres de tabla que también son palabras reservadas. Por ejemplo, una referencia a una tabla llamada `status` ahora siempre irá entre comillas. Consulta [#991](https://github.com/TabularEditor/TabularEditor3/issues/991).
- El autocompletado ahora sugiere las palabras clave adecuadas para el parámetro _&lt;Skip&gt;_ de las funciones de ventana de DAX, como [`RANK`](https://dax.guide), entre otras.
- El cambio de nombre por lotes ahora funciona también con carpetas; consulta [#797](https://github.com/TabularEditor/TabularEditor3/issues/797).
- Se corrigieron algunos problemas relacionados con los diseños personalizados. Consulta [#711](https://github.com/TabularEditor/TabularEditor3/issues/711). También se corrigió un problema por el que las personalizaciones de los menús integrados no se conservaban correctamente.
- Se corrigió un error en el Asistente de implementación cuando el modelo contenía una tabla de actualización incremental sin particiones. En este caso, la interfaz del Asistente de implementación no permitía al usuario omitir la implementación de particiones de esta tabla, lo que provocaba que se generara un TMSL no válido (sin particiones para la tabla en cuestión).
- Se corrigió un error por el que, al ejecutar una consulta DAX sin espacios en blanco entre [`EVALUATE`](https://dax.guide) y la expresión siguiente, se producía un error.
- Se corrigieron numerosos problemas con el Depurador de DAX. Consulta [#954](https://github.com/TabularEditor/TabularEditor3/issues/954), [#971](https://github.com/TabularEditor/TabularEditor3/issues/971) y [#984](https://github.com/TabularEditor/TabularEditor3/issues/984).

## ¿Vienes de Tabular Editor 2.x?

Mira [este vídeo](https://www.youtube.com/watch?v=pt3DdcjfImY) para hacerte una idea de las nuevas funcionalidades de Tabular Editor 3. Además, asegúrate de consultar nuestra [guía de inicio](https://docs.tabulareditor.com/getting-started/index.html).

**Descripción general de las principales funcionalidades de Tabular Editor 3:**

- IDE totalmente personalizable, con soporte para varios monitores, Hi-DPI y temas
- Nuevo y potente editor de código DAX con autocompletado, comprobación de sintaxis, plegado de código y mucho más
- \*Modo del área de trabajo, que te permite guardar los cambios en el disco y sincronizar simultáneamente los metadatos del modelo con Analysis Services
- \*Previsualiza los datos de las tablas con desplazamiento infinito; crea PivotGrids o escribe consultas DAX para explorar el modelo o probar la lógica de cálculo
- \*Programa actualizaciones de datos
- Actualiza los esquemas de las tablas tanto en el Data source del proveedor como en los orígenes de datos estructurados (sí, incluso para consultas M!)
- Crea diagramas de Data model
- Crea scripts DAX que te permitan editar varias medidas u otros objetos calculados en un único documento
- Graba C# Scripts y guárdalos como macros (antes se conocían como "Custom Actions")
- Integración con el Analizador VertiPaq
- Depurador de DAX

\*=Solo mientras estés conectado a una instancia de Analysis Services o de Power BI
