## Requisitos previos

Tabular Editor 3 usa .NET 6. Si usas la versión portátil, es posible que tengas que instalar manualmente el [.NET 6.0 Runtime for Desktop](https://dotnet.microsoft.com/en-us/download/dotnet/6.0/runtime) antes de poder iniciar Tabular Editor 3.

## Correcciones de errores en 3.16.1

- Se corrigió un error que impedía que los archivos de diagrama guardados (.te3diag) se abrieran correctamente; consulta [#1313](https://github.com/TabularEditor/TabularEditor3/issues/1313)

## Mejoras en 3.16.1

- Los resultados de las consultas DAX ahora usan el «formato de hora larga» para los valores de fecha y hora, en lugar del «formato de hora corta». De este modo, también se muestran los segundos en la cuadrícula de resultados.

## Novedades en 3.16.0

Visita nuestro [blog de versiones](https://blog.tabulareditor.com/) para ver un resumen rápido de las actualizaciones más importantes de esta versión.

En esta versión hemos renovado a fondo el funcionamiento de las **Pivot Grids**. Algunos puntos destacados:

- ¡Ahora puedes guardar y cargar diseños de Pivot Grid en/desde archivos! Se usa un nuevo formato de archivo, `.te3pivot`, para almacenar una representación JSON del diseño de Pivot Grid.
- Ahora se pueden usar objetos ocultos (medidas, columnas, jerarquías) en una Pivot Grid, y cambiar el nombre de los objetos ya no debería hacer que la Pivot Grid pierda su diseño ni borre ningún filtro.
- Una nueva y mejorada **lista de campos** de la Pivot Grid, que ahora se muestra de forma predeterminada y ofrece una experiencia más familiar al agregar campos a la Pivot Grid.

Consulta la [documentación de Pivot Grid](https://docs.tabulareditor.com/te3/features/pivot-grid.html) para obtener más información sobre estos cambios.

## Mejoras en 3.16.0

- Se redujo la cantidad de mensajes de error inútiles o redundantes que se muestran cuando las expresiones DAX contienen errores sintácticos o semánticos.
- Los Scripts DAX ahora admiten expresiones no especificadas o vacías. Esto te permite definir un Script DAX que no modifique las expresiones existentes en los objetos, o un Script DAX que establezca explícitamente una expresión como vacía.
- Ahora puedes exportar los resultados de consultas DAX a CSV/Excel. Busca las opciones **Exportar datos** y **Exportar datos (todos los resultados)** en el menú **Consulta**.
- Al exportar estadísticas del Analizador VertiPaq, ahora sugerimos un nombre predeterminado para el archivo VPAX en función del nombre de la base de datos.
- Cambiamos los atajos de teclado predeterminados para `Editor.Dax.CollapseAllFolds`, `Editor.Dax.ExpandAllFolds` y `Editor.Dax.ToggleAllFolds` de **Ctrl+Alt+(tecla)** a **Ctrl+Shift+(tecla)**. Esto es para evitar conflictos con determinados caracteres que requieren la tecla **AltGr** (equivalente a **Ctrl+Alt**) para poder escribirse en teclados no estadounidenses. Los atajos **Ctrl+Shift+(tecla)** se usaban anteriormente para `Editor.Dax.CollapseFold`, `Editor.Dax.ExpandFold` y `Editor.Dax.ToggleFold`. Sus combinaciones predeterminadas se han cambiado a **Ctrl+(tecla)**.
- Se actualizó [AMO/TOM a 19.80.0](https://www.nuget.org/packages/Microsoft.AnalysisServices.NetCore.retail.amd64).
- Los menús y las barras de herramientas ahora están bloqueados de forma predeterminada, lo que evita que se reposicionen por accidente. Para desbloquearlos, ve a **Herramientas > Personalizar... > Opciones** y desmarca la opción **Bloquear menús y barras de herramientas**.
- Nuestro analizador semántico de DAX ahora admite la sintaxis `WITH VISUAL SHAPE` utilizada en consultas DAX que contienen [cálculos de Visual](https://learn.microsoft.com/en-us/power-bi/transform-model/desktop-visual-calculations-overview). Ten en cuenta que todavía no se admite la depuración de este tipo de consultas (ya que solo se pueden depurar medidas y los cálculos de Visual no son medidas).
- Mostrar una advertencia al depurar el valor de una celda de Pivot Grid o una consulta DAX que incluya elementos de cálculo en el contexto de filtro (algo que el depurador de TE3 todavía no admite).

## Correcciones de errores en 3.16.0

- Al pulsar combinaciones con la tecla Ctrl que no tengan ningún atajo asignado, ya no deberían insertarse caracteres de control especiales en los editores de código.
- Las macros que se hayan modificado y guardado ya no deberían ejecutarse varias veces cuando se invocan desde la interfaz de usuario.
- El método `.Output()` ahora debería mostrar el cuadro de diálogo de salida en ejecuciones posteriores de la macro, incluso si en una ejecución anterior se marcó la opción **No mostrar más salidas**.
- Las referencias a nombres de tabla sin comillas que además coincidan con palabras clave de DAX (como `Currency`) ya no se tratarán como palabras clave durante el formateo automático (es decir, no se pondrán en mayúsculas). Además, ciertas propiedades de objetos en los Scripts DAX ahora también pueden estar vacías. Consulta la [documentación actualizada de Scripts DAX](https://docs.tabulareditor.com/te3/features/dax-scripts.html) para obtener más información.
- Los errores durante la ejecución de Scripts DAX ahora se gestionan correctamente, y los cambios realizados por el script antes de que se produjera el error se revierten automáticamente.
- Se corrigió un error por el cual los botones de macro asignados a barras de herramientas personalizadas no se conservaban tras reiniciar la aplicación.
- Se corrigió un error que provocaba que Tabular Editor se bloquease al usar el método de extensión `ForEach<T>(IEnumerable<T>, Action<T>)` en un C# Script.
- Se corrigieron errores relacionados con la eliminación/restablecimiento de atajos de teclado, que no funcionaban correctamente para algunos comandos.
- Se corrigió un error por el cual el botón Guardar permanecía deshabilitado tras finalizar una operación de actualización.
- Se corrigió un error por el cual determinadas acciones podían hacer que el **Explorador TOM** se quedara bloqueado y no se comportara correctamente hasta reiniciar la aplicación.
- Se corrigió un error por el que, al eliminar un objeto en el **Explorador TOM**, la selección saltaba al nodo **Model**, en lugar de al nodo padre más cercano del objeto eliminado.
- Se corrigieron varios errores relacionados con las Pivot Grid.
- El analizador semántico ya no debería generar errores falsos al comparar con `BLANK()` en expresiones como `"string value" IN { BLANK() }`.
- Se corrigió un error por el que los cambios en la configuración de serialización no se conservaban cuando un modelo se cargaba desde una carpeta TMDL. Consulta el [#1285](https://github.com/TabularEditor/TabularEditor3/issues/1285).
- Al ordenar una columna en la Vista previa de tabla, cuando la columna tiene `IsAvailableInMDX = false` (es decir, no se han creado jerarquías de atributos para la columna), no recurriremos a consultas `TOPN`, ya que `TOPNSKIP` no admite ordenar por columnas sin jerarquías de atributos.

## ¿Vienes de Tabular Editor 2.x?

Mira [este vídeo](https://www.youtube.com/watch?v=pt3DdcjfImY) para hacerte una idea de las nuevas funcionalidades de Tabular Editor 3. Además, asegúrate de consultar nuestra [guía de inicio](https://docs.tabulareditor.com/onboarding/index.html).

**Resumen de las principales características de Tabular Editor 3:**

- IDE totalmente personalizable, con compatibilidad con varios monitores, Hi-DPI y temas
- Nuevo y potente editor de código DAX con autocompletado, comprobación de sintaxis, plegado de código y mucho, mucho más
- \*Modo del área de trabajo, que te permite guardar los cambios en el disco y, al mismo tiempo, sincronizar los metadatos del modelo con Analysis Services
- \*Previsualiza los datos de la tabla con desplazamiento infinito, crea Pivot Grids o escribe consultas DAX para explorar el modelo o probar la lógica de cálculo
- \*Programa actualizaciones de datos
- Actualiza los esquemas de las tablas tanto en Data sources de proveedor como en orígenes de datos estructurados (sí, ¡incluso para consultas M!)
- Crea diagramas del Data model
- Crea Scripts DAX que te permitan editar varias medidas u otros objetos calculados en un único documento
- Graba C# Scripts y guárdalos como macros (antes conocidos como "Custom Actions")
- Integración con el Analizador VertiPaq
- Depurador de DAX

\*=Solo cuando estés conectado a una instancia de Analysis Services o Power BI
