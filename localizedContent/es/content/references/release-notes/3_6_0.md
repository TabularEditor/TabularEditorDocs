### Requisitos previos

Tabular Editor 3.3.0 (y versiones posteriores) usa .NET 6. Es posible que tengas que instalar el runtime de escritorio de .NET 6.0 (v6.0) antes de poder iniciar Tabular Editor 3.3.0 (o una versión posterior): [.NET 6.0 Runtime for Desktop](https://dotnet.microsoft.com/en-us/download/dotnet/6.0/runtime). A partir de la versión 3.3.1, el instalador de Tabular Editor 3 descargará e instalará estos requisitos previos si faltan.

## Novedades en 3.6.0

Consulta nuestro [blog de lanzamiento](https://blog.tabulareditor.com/2023/04/19/tabular-editor-3-april-2023-release/) para obtener un resumen de las actualizaciones más importantes de esta versión.

- Tabular Editor 3 ahora ofrece compatibilidad nativa con **Databricks**. Esto significa que ahora puedes conectarte a un punto de conexión de Databricks SQL al importar tablas con el Asistente para importar tablas. Para más información, consulta nuestra [entrada del blog de lanzamiento](https://blog.tabulareditor.com/2023/04/19/tabular-editor-3-april-2023-release).
- Añadimos compatibilidad con las nuevas funciones DAX [`RANK`](https://dax.guide/rank) y [`ROWNUMBER`](https://dax.guide/rownumber) introducidas en la [actualización de abril de 2023 de Power BI Desktop](https://powerbi.microsoft.com/en-us/blog/power-bi-april-2023-feature-summary/). Además, nuestro autocompletado de código ahora muestra todas las palabras clave disponibles para los argumentos &lt;Blanks&gt; y &lt;Order&gt; de las funciones de ventana y [`ORDERBY`](https://dax.guide/orderby).

## Mejoras menores en 3.6.0

- Ya no realizamos una comprobación de conflictos (potencialmente lenta) antes de guardar los cambios en la base de datos del Workspace. La idea es que cada desarrollador use su propia base de datos personal del Workspace, así que no deberían producirse conflictos.
- C# Script: La interfaz `IAnnotationObject` se ha ampliado con el método `ClearAnnotations()`.
- Al agregar una expresión de cadena de formato dinámica a una medida, ahora borramos la propiedad de cadena de formato estática (ya que no es posible que las medidas tengan asignadas a la vez cadenas de formato estáticas y dinámicas).
- TOM actualizado a [19.61.1.4](https://www.nuget.org/packages/Microsoft.AnalysisServices.NetCore.retail.amd64).
- El nivel de compatibilidad predeterminado para los nuevos modelos de Power BI creados con Tabular Editor ahora es 1601. Además, el cuadro de diálogo "Crear nuevo modelo" ahora te permite especificar cualquier nivel de compatibilidad.

## Correcciones de errores en 3.6.0

- Se actualizaron las instrucciones de descarga de Oracle para indicar a los usuarios que utilicen los controladores ODAC (no ODP.NET).
- Se eliminaron las restricciones sobre caracteres especiales en los nombres de las bases de datos al abrir o crear una base de datos nueva.
- Se corrigió un problema que provocaba que la propiedad del nombre de la base de datos dentro del archivo .bim / Database.json se actualizara con el nombre de la base de datos del Workspace.
- Mejoras en la estabilidad del Depurador de DAX.
- Los archivos ".tmuo" ya no se guardan cuando la opción "Crear archivo de opciones de usuario (.tmuo)" está desmarcada.
- Se corrigió un problema por el que el autocompletado de código a veces "absorbía" los corchetes de cierre.
- El analizador semántico ya no informa de un error al usar [`SELECTEDMEASURE`](https://dax.guide/selectedmeasure) (y funciones relacionadas) en el contexto de las expresiones de medida y las expresiones de cadena de formato dinámicas.
- Se corrigió un problema por el que la consulta M generada para nuevas tablas importadas desde un Data source implícito (SQL) existente no funcionaba correctamente al intentar actualizar la tabla.

## ¿Vienes de Tabular Editor 2.x?

Mira [este vídeo](https://www.youtube.com/watch?v=pt3DdcjfImY) para hacerte una idea de las nuevas características de Tabular Editor 3. Además, no dejes de consultar nuestra [guía de introducción](https://docs.tabulareditor.com/onboarding/index.html).

**Resumen de las principales características de Tabular Editor 3:**

- IDE totalmente personalizable, con compatibilidad con varios monitores, Hi-DPI y temas
- Nuevo y potente editor de código DAX con autocompletado, comprobación de sintaxis, plegado de código y mucho, mucho más
- \*Modo del área de trabajo, que te permite guardar los cambios en disco y sincronizar los metadatos del modelo con Analysis Services al mismo tiempo
- \*Vista previa de los datos de las tablas con desplazamiento infinito, y posibilidad de crear PivotGrids o escribir consultas DAX para explorar el modelo o probar la lógica de cálculo
- \*Programación de actualizaciones de datos
- Actualización de los esquemas de tablas tanto en Data sources de proveedor como en orígenes de datos estructurados (sí, incluso para consultas M)
- Crear diagramas del Data model
- Crear scripts DAX que te permitan editar, en un único documento, varias medidas u otros objetos calculados
- Grabar C# Scripts y guardarlos como macros (antes conocidos como "Custom Actions")
- Integración del Analizador VertiPaq
- Depurador de DAX

\*=Solo cuando se esté conectado a una instancia de Analysis Services o Power BI
