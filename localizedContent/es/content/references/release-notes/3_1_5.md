## Mejoras en 3.1.5

- Ahora puedes **suplantar la identidad** de usuarios o roles en consultas DAX, Pivot Grid y vistas previas de tabla. Esto es útil para probar la configuración de RLS y OLS. [Más información](https://docs.tabulareditor.com/onboarding/refresh-preview-query.html#impersonation).
- Los scripts DAX ahora admiten grupos de cálculo y elementos de cálculo. Esto te permite ver toda la lógica del grupo de cálculo (incluidas las expresiones de cadena de formato dinámicas) en un único documento. [Ejemplo](https://docs.tabulareditor.com/onboarding/dax-script-introduction.html#example-3-calculation-group).
- Al hacer clic con el botón derecho en una referencia de medida en una consulta DAX o un script DAX, ahora tienes la opción de añadir la definición de esa medida, [incluidas todas las dependencias](https://docs.tabulareditor.com/onboarding/dax-script-introduction.html#define-measures).
- Ahora puedes desplazar los diagramas arrastrando un área vacía (consulta [#275](https://github.com/TabularEditor/TabularEditor3/issues/275)).

## Correcciones de errores en 3.1.5

- Se corrigió un problema con el actualizador de esquema integrado cuando se encuentra un valor `null` en una expresión M (consulta [#282](https://github.com/TabularEditor/TabularEditor3/issues/282)).
- El analizador semántico ya no reporta un error al usar la función `RELATED` como argumento de `SELECTCOLUMNS` (cuando no se especifica el argumento de nombre).
- Ahora se pueden editar/cambiar el nombre de las macros sin que desaparezcan de las personalizaciones de la barra de herramientas (consulta [#266](https://github.com/TabularEditor/TabularEditor3/issues/266)).
- Ahora puedes cambiar el nombre o eliminar todas las barras de herramientas personalizadas después de reiniciar la aplicación (consulta [#260](https://github.com/TabularEditor/TabularEditor3/issues/260)).
- Se corrigieron la selección y el menú contextual al seleccionar una carpeta que contiene otras carpetas (consulta [#248](https://github.com/TabularEditor/TabularEditor3/issues/248)).
- Se evitó un cierre inesperado al seleccionar simultáneamente tablas/objetos de tabla.
- Se corrigieron varios problemas/cierres inesperados al importar tablas (ODBC NullReferenceException y otros).

## ¿Vienes de Tabular Editor 2.x?

Mira [este vídeo](https://www.youtube.com/watch?v=pt3DdcjfImY) para hacerte una idea de las nuevas características de Tabular Editor 3.

**Descripción general de las principales características de Tabular Editor 3:**

- IDE totalmente personalizable, con compatibilidad para varios monitores y Hi-DPI, y con temas
- Nuevo y potente editor de código DAX con autocompletado, comprobación de sintaxis, plegado de código y mucho, mucho más
- \*Modo del área de trabajo, que te permite guardar los cambios en disco y sincronizar simultáneamente los metadatos del modelo con Analysis Services
- \*Previsualiza los datos de las tablas con desplazamiento infinito, crea PivotGrids o escribe consultas DAX para explorar el modelo o probar la lógica de cálculo
- \*Programa actualizaciones de datos
- Actualiza los esquemas de tablas tanto en el proveedor como en los Data sources de tipo origen de datos estructurado (sí, ¡incluso para consultas M!)
- Crea diagramas del Data model
- Crea scripts DAX que te permiten editar más de una medida u otros objetos calculados en un único documento
- Graba C# Scripts y guárdalos como macros (antes conocidas como "Custom Actions")
- Integración del Analizador VertiPaq

\*=Solo mientras estés conectado a una instancia de Analysis Services o Power BI
