**NOTA:** Si instalaste anteriormente una de las versiones BETA, tendrás que desinstalarla manualmente antes de instalar esta nueva versión.

## Problemas conocidos

- El Asistente para importar tablas no se incluyó en esta versión debido a algunos errores de alta prioridad que había que corregir. ¡Estará disponible lo antes posible! Como solución alternativa, simplemente crea una nueva tabla y especifica la consulta de partición (SQL o M). Después, haz clic con el botón derecho en la tabla y elige "Actualizar esquema de tabla..." para importar columnas.
- Consulta la [hoja de ruta](https://github.com/TabularEditor/TabularEditor3/issues/12) para obtener más información.

## Correcciones de errores en 3.0.9

- Se incluyó una DLL que faltaba y que, de lo contrario, provocaría un bloqueo al intentar agregar miembros del rol.

## Mejoras en 3.0.8

- Se agregó una opción para ejecutar solo las instrucciones EVALUATE seleccionadas en consultas DAX (atajo Mayús+F5); consulta el problema [#47](https://github.com/TabularEditor/TabularEditor3/issues/47).
- Se ha agregado la opción de usar el atajo Ctrl+/ para comentar o descomentar código; consulta el problema [#46](https://github.com/TabularEditor/TabularEditor3/issues/46). También puedes cambiar el estilo de comentario (-- o //) en Herramientas > Preferencias > Editor de DAX > General.
- Script DAX ahora también generará scripts para los KPI de medidas, las columnas calculadas y las tablas calculadas de la selección actual. También agregamos comentarios para que el código sea más legible (esto se puede desactivar en Herramientas > Preferencias > Script DAX).
- Ahora hay una opción para agregar automáticamente un carácter de salto de línea al inicio de todas las expresiones DAX cuando se trabaja con un modelo de Power BI. Esto hace que el código se vea mejor en la barra de fórmulas de Power BI Desktop. Esto se puede desactivar en Herramientas > Preferencias >. Consulta el issue [#26](https://github.com/TabularEditor/TabularEditor3/issues/26).
- Al agregar miembros de roles de Active Directory de Windows, ahora aparece el cuadro de diálogo Selector de objetos del directorio; consulta el issue [#152](https://github.com/TabularEditor/TabularEditor3/issues/152).
- La opción "Agregar metadatos de implementación" ahora está disponible en Herramientas > Preferencias > Implementación del modelo.
- Mejoras en Pivot Grid:
  - Ahora puedes arrastrar objetos del Explorador TOM a todas las áreas de Pivot Grid; consulta [#49](https://github.com/TabularEditor/TabularEditor3/issues/49).
  - Se agregó una opción en el menú contextual del Explorador TOM para agregar medidas, columnas y jerarquías directamente a Pivot Grid.
  - Se agregó el botón "Mostrar campo" en Pivot Grid para mostrar todos los campos disponibles en la base de datos conectada.
  - Se agregó la opción "Ir al objeto" en el menú contextual de Pivot Grid.
- Errores de relación (rutas ambiguas, estado no válido, etc.) ahora se detectan y se indican en el Explorador TOM

## Correcciones de errores en 3.0.8

- Se cambió el color del signo de exclamación de "Ejecutar consulta" de rojo a verde.
- Se corrigió un bloqueo al invertir relaciones desde la Vista de diagrama
- Se corrigió un problema en el Editor de relaciones por el que los cambios no se guardaban correctamente
- Se corrigió un bloqueo que podía producirse al iniciar TE3 desde PBI Desktop mientras se ejecutaba una instancia de SSAS en el equipo local.
- Se corrigió un error que impedía crear modelos nuevos con una base de datos de un Workspace en el punto de conexión XMLA de Power BI.
- Se corrigió un error por el que la selección de código se borraba después de usar la acción "Comentar".
- El Asistente de implementación ahora recuerda el último servidor usado.
- Las expresiones de columnas calculadas en Script DAX ya no deberían mostrar mensajes de error semántico falsos sobre la falta de contexto de fila.
- Se corrigió un error que impedía a los titulares de licencias de la Edición Business usar el servidor de Workspace Integrada de Visual Studio.
- Se corrigió un error tipográfico en "successful".
- Se corrigió un problema por el que el formateador automático a veces duplicaba código al escribir un paréntesis de cierre.
- Se agregó la opción "Guardar copia de seguridad de Model.bim" en la interfaz de usuario; consulta el issue [#37](https://github.com/TabularEditor/TabularEditor3/issues/37).
- La selección de perspectiva ya no debería quedarse "atascada"; consulta el issue [#52](https://github.com/TabularEditor/TabularEditor3/issues/52).
- La cuadrícula de propiedades ahora muestra correctamente ejemplos de formato de fecha; consulta el issue [#86](https://github.com/TabularEditor/TabularEditor3/issues/86).
- Se corrigió el color de resaltado al buscar; consulta la incidencia [#98](https://github.com/TabularEditor/TabularEditor3/issues/98).
- "Buscar todo" ahora funciona como se esperaba; consulta la incidencia [#107](https://github.com/TabularEditor/TabularEditor3/issues/107).
- Ahora se detecta correctamente el elemento de cálculo que hace que los tipos de datos de la medida se conviertan en Variant; consulta la incidencia [#111](https://github.com/TabularEditor/TabularEditor3/issues/111).
- Se corrigió un falso error semántico al usar EARLIER en la función FILTER; consulta la incidencia [#118](https://github.com/TabularEditor/TabularEditor3/issues/118).
- Se corrigió un falso error semántico al usar KEEPFILTERS con FILTER; consulta la incidencia [#122](https://github.com/TabularEditor/TabularEditor3/issues/122).
- Una expresión de filas de detalle en blanco (o solo con comentarios) ya no genera un indicador de error; consulta la incidencia [#127](https://github.com/TabularEditor/TabularEditor3/issues/127).
- Se corrigieron múltiples solicitudes de credenciales; consulta la incidencia [#131](https://github.com/TabularEditor/TabularEditor3/issues/131).
- Se corrigió un problema de importación de vpax; consulta la incidencia [#133](https://github.com/TabularEditor/TabularEditor3/issues/133).
- Se corrigió el comportamiento de la cuadrícula de propiedades al seleccionar una carpeta de visualización; consulta la incidencia [#134](https://github.com/TabularEditor/TabularEditor3/issues/134).
- El diagrama ya no elimina la tabla al pulsar "Eliminar" (en su lugar, la quita del diagrama); consulta la incidencia [#139 ](https://github.com/TabularEditor/TabularEditor3/issues/139).
- Pivot Grid ahora debería gestionar correctamente los campos de fecha; consulta la incidencia [#142](https://github.com/TabularEditor/TabularEditor3/issues/142).
- Copiar un grupo de cálculo ya no creará medidas con nombres duplicados, y el número de precedencia también se modificará para garantizar que sea único; consulta las incidencias [#155](https://github.com/TabularEditor/TabularEditor3/issues/155) y [#156](https://github.com/TabularEditor/TabularEditor3/issues/156).
- El botón "Descomentar código" ahora funciona también en líneas con sangría; consulta [#159](https://github.com/TabularEditor/TabularEditor3/issues/159).
