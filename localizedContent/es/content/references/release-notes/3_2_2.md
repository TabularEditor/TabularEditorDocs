## Mejoras en 3.2.2

- Se agregó compatibilidad con Query Group (también conocido como "carpetas" de Power Query) para particiones y Named Expression
- Ahora se pregunta por el tipo de la primera partición. Consulta [#406](https://github.com/TabularEditor/TabularEditor3/issues/406)
- Se alineó la presentación de los nombres de las relaciones para que sea similar a cómo el Analizador VertiPaq en DAX Studio muestra los nombres.
- Ahora se permiten comas de depuración en el Editor de expresiones; consulta [#407](https://github.com/TabularEditor/TabularEditor3/issues/407)
- Ahora puedes hacer doble clic en cualquier objeto de expresión en el Explorador TOM (como las particiones) para mostrar el editor rápido
- Se cambió la etiqueta "Mixed" por "Hybrid" en tablas que contienen varios tipos de particiones
- Pivot Grid:
  - Pivot Grid ahora muestra un total general al segmentar por un campo que solo tiene un valor (cuando el campo se usa en el área de filas/columnas)
  - Los campos agregados a Pivot Grid a través del menú contextual del Explorador TOM ahora se agregan "más a la derecha". - La celda de Pivot Grid recibe el foco al hacer clic con el botón derecho
- Mejoras en el Depurador de DAX:
  - Ahora puedes ver y habilitar/deshabilitar filtros individuales desde el contexto de filtro externo
  - Se agregaron descripciones emergentes para los tipos de valores de Watch/Locals.
  - Mejor determinación y ordenación de los elementos locales de la llamada actual (incluye subexpresiones dentro del mismo contexto de evaluación, etc.)
  - Ya no se muestra una transición de contexto en la pila del contexto de evaluación al depurar una referencia a una medida (entra en la referencia de la medida para ver la transición de contexto)
  - Ya no se usa TREATAS con SSAS 2016
  - Se mejoró F10 (paso por encima) al trabajar con medidas
  - Se mejoró el autocompletado en el Editor de expresiones de seguimiento (ahora puede sugerir columnas del contexto de fila, variables, etc.)
- TOM actualizado a [19.34.1](https://www.nuget.org/packages/Microsoft.AnalysisServices.retail.amd64/)

## Correcciones de errores en 3.2.2

- Se agregaron más propiedades de TOM, p. ej.: Modo de política de actualización (se usa para configurar la política de actualización de tablas híbridas)
- Se corrigió un bloqueo del asistente de activación; ver [#419](https://github.com/TabularEditor/TabularEditor3/issues/419)
- Se garantiza que las barras de herramientas personalizadas sigan pudiendo arrastrarse/acoplarse y personalizarse después de la primera restauración del diseño. Ver la incidencia [#413](https://github.com/TabularEditor/TabularEditor3/issues/413)
- Corregido un problema por el que el panel de propiedades no se mostraba al invocarlo desde el menú contextual, ver [#375](https://github.com/TabularEditor/TabularEditor3/issues/375)
- Se evita un bloqueo al ir a Actualización de datos desde la barra de estado; ver la incidencia [#370](https://github.com/TabularEditor/TabularEditor3/issues/370)
- Corregido un problema por el que la aplicación se bloqueaba al cerrarse mientras se ejecutaba una actualización en segundo plano.
- Se evita que el Editor de expresiones aparezca al hacer clic en el diagrama. Ver la incidencia [#43](https://github.com/TabularEditor/TabularEditor3/issues/43). También mejora el comportamiento al seleccionar y arrastrar columnas en la Vista de diagrama.
- Se agregaron campos faltantes a VeritPaq Analyzer, ver [#372](https://github.com/TabularEditor/TabularEditor3/issues/372)
- Se mejoró el comportamiento del Pivot Grid con las columnas del grupo de cálculo. Ver [#412](https://github.com/TabularEditor/TabularEditor3/issues/412)
- Se garantiza que la Vista de dependencias se actualice cada vez que finalice un análisis semántico (por ejemplo, cuando se modifica un modelo a través de Power BI); ver [#401](https://github.com/TabularEditor/TabularEditor3/issues/401)
- No se sugieren funciones del grupo de cálculo en el autocompletado, a menos que se esté escribiendo una expresión de elemento de cálculo; ver la incidencia [#387](https://github.com/TabularEditor/TabularEditor3/issues/387)
- Corregido un problema por el que, tras descomentar una selección, "deshacer" ya no se comportaba correctamente. Ver [#301](https://github.com/TabularEditor/TabularEditor3/issues/301)
- Se agregaron atajos para comentar (Ctrl+Shift+K) y descomentar (Ctrl+Shift+U). Ver [#371](https://github.com/TabularEditor/TabularEditor3/issues/371)
- Se agregaron atajos (Alt+flecha arriba/abajo) para mover hacia arriba/abajo las líneas seleccionadas en los editores de texto. Ver [#371](https://github.com/TabularEditor/TabularEditor3/issues/371)
- Corregir la excepción StackOverflowException al arrastrar ventanas de determinadas maneras. Consulta [#354](https://github.com/TabularEditor/TabularEditor3/issues/354)
- Corregir un error por el que el grabador de macros no se detenía al cerrar una vista de C# Script, lo que provocaba un bloqueo posterior al cambiar un objeto o una propiedad del modelo.
- El grabador de macros ahora también genera código para cambios en expresiones de elementos de cálculo, así como para cambios en el nivel de jerarquía y en el ordinal del elemento de cálculo. Consulta [#230](https://github.com/TabularEditor/TabularEditor3/issues/230)
- Depurador: corregir la generación de DAX para variables en un contexto de filtro externo; consulta el problema [#418](https://github.com/TabularEditor/TabularEditor3/issues/418)
- Comprobar y eliminar TablePermissions con referencias de tabla no válidas al cargar el modelo (antes, TE no permitía cargar un modelo con referencias no válidas de este tipo)
- Limpiar el filtro cuando se quita/oculta un campo de un Pivot Grid; consulta el problema [#427](https://github.com/TabularEditor/TabularEditor3/issues/427)

## Vienes de Tabular Editor 2.x?

Mira [este vídeo](https://www.youtube.com/watch?v=pt3DdcjfImY) para hacerte una idea de las nuevas funcionalidades de Tabular Editor 3. Además, asegúrate de consultar nuestra [guía de inicio](https://docs.tabulareditor.com/onboarding/index.html).

**Descripción general de las principales funcionalidades de Tabular Editor 3:**

- IDE totalmente personalizable, con compatibilidad con varios monitores, Hi-DPI y temas
- Nuevo y potente editor de código DAX con autocompletado, comprobación de sintaxis, plegado de código y mucho, mucho más
- \*Modo del área de trabajo, que te permite guardar los cambios en el disco y sincronizar los metadatos del modelo con Analysis Services al mismo tiempo
- \*Previsualizar datos de tabla con desplazamiento infinito, crear PivotGrids o escribir consultas DAX para explorar el modelo o probar la lógica de cálculo
- \*Programar actualizaciones de datos
- Actualizar los esquemas de tabla tanto en el proveedor como en los orígenes de datos estructurados (Data sources; sí, ¡incluso para consultas M!)
- Crear diagramas del Data model
- Crear scripts DAX que te permitan editar varias medidas u otros objetos calculados en un único documento
- Grabar C# Script y guardarlos como macros (antes conocidos como "Custom Actions")
- Integración del Analizador VertiPaq
- \*Depurador de DAX

\*=Solo cuando estés conectado a una instancia de Analysis Services o Power BI
