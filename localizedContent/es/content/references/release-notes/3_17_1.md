## Correcciones de errores en 3.17.1

- Se corrigió un error relacionado con la configuración regional del sistema operativo, que provocaba un bloqueo al abrir el cuadro de diálogo de **Preferencias** por primera vez.

## Novedades de 3.17.0

Echa un vistazo a nuestro [blog de lanzamiento](https://blog.tabulareditor.com/2024/08/14/tabular-editor-3-august-2024-release) para obtener un breve resumen de las actualizaciones más importantes de esta versión.

- Nos complace informarte de que Tabular Editor 3 ya está disponible para [.NET 8](https://learn.microsoft.com/en-us/dotnet/core/whats-new/dotnet-8/overview). Seguiremos ofreciendo soporte para .NET 6 hasta finales de 2024; sin embargo, recomendamos cambiar a la versión de .NET 8 lo antes posible. En nuestras pruebas, la versión de .NET 8 se inicia aprox. un 20 % más rápido y, además, consume menos memoria.
- Las versiones portátiles de Tabular Editor 3 ahora incluyen el runtime de .NET correspondiente, por lo que ya no necesitas instalar por separado el Windows Desktop Runtime de .NET 6/8.

> [!IMPORTANT]
> .NET 6 quedará [fuera de soporte en noviembre de 2024](https://dotnet.microsoft.com/en-us/platform/support/policy/dotnet-core). Si ahora no puedes instalar el [runtime de escritorio de .NET 8](https://dotnet.microsoft.com/en-us/download/dotnet/8.0/runtime), ponte en contacto con tu departamento de TI para planificarlo con antelación, ya que dejaremos de proporcionar compilaciones de Tabular Editor 3 para .NET 6 después de diciembre de 2024. Como alternativa, usa nuestras versiones portátiles, que incluyen el runtime de .NET necesario.

## Mejoras en 3.17.0

- Los resultados de la Consulta DAX y las Vistas previas de tabla ahora muestran el "formato de hora larga" para los valores de fecha y hora, en lugar del "formato de hora corta". Así, los segundos también se muestran en la cuadrícula de resultados.
- Las expresiones compartidas ahora pueden excluirse del despliegue mediante el Asistente de implementación. Esta mejora te permite conservar tal cual las expresiones compartidas existentes en el servidor de destino al desplegar un modelo, incluso si en el modelo que se despliega se han creado/eliminado/modificado expresiones compartidas.
- La ventana **Locals** del **Depurador de DAX** ahora te permite "inspeccionar" los resultados, de forma similar a la ventana **Watch**. De forma predeterminada, los resultados del inspector se muestran en un cuadro de diálogo emergente en lugar de como una nueva Consulta DAX, pero puedes controlar este comportamiento en **Herramientas > Preferencias > Depurador de DAX**. Haz clic en el icono de la lupa junto al valor en la vista **Locals** o **Watch** para inspeccionarlo. Consulta la [documentación actualizada del Depurador de DAX](xref:dax-debugger#locals) para obtener más información.
- Al depurar un predicado escalar en un argumento de filtro de [`CALCULATE`](https://dax.guide/calculate) o [`CALCULATETABLE`](https://dax.guide/calculatetable), la vista **Locals** ahora muestra detalles adicionales sobre el predicado escalar y la expresión de filtro resultante. Consulta la [documentación actualizada del Depurador de DAX](xref:dax-debugger#scalar-predicates) para obtener más información.
- Los Scripts DAX ahora pueden aplicarse aunque contengan errores semánticos. Se mostrará una advertencia si hay errores semánticos, para que el usuario final decida si quiere continuar con los errores actuales o cancelar y corregirlos. Los errores de sintaxis, en cambio, seguirán impidiendo que el script se aplique.
- Se añadió compatibilidad con autocompletado para algunas funciones DAX `INFO.*` más.
- La función Integración del Optimizador de DAX (versión preliminar) ahora te permite cambiar entre cuentas personales o de grupo en escenarios de licencias de grupo. Puedes elegir qué cuenta quieres usar desde el menú desplegable **Options**. Consulta también [#1288](https://github.com/TabularEditor/TabularEditor3/issues/1288). Además, ahora puedes cambiar el estado _Fixed_ o _Ignored_ de un problema directamente desde la vista de resultados del Optimizador de DAX.
- El Asistente de importación de tablas y la función Actualizar esquema de tabla ahora admiten **Databricks Multicloud**, además de **Azure Databricks**. Para obtener más información, consulta el debate en [#1347](https://github.com/TabularEditor/TabularEditor3/discussions/1347).
- Las preferencias del Analizador VertiPaq se ampliaron para configurar el tamaño del lote de columnas. Esta preferencia controla el número de columnas que se procesan a la vez durante el análisis de estadísticas de columnas y se encuentra en **Herramientas > Preferencias > Analizador VertiPaq**. El valor predeterminado de esta preferencia es 50.
- Formato personalizado aplicado a los Pivot Grids (iconos, barras de datos, celdas resaltadas, etc.) ahora se conserva cuando el diseño del Pivot Grid se guarda en un archivo y posteriormente se vuelve a cargar.
- Los Pivot Grids ahora también conservan cualquier ordenación aplicada a los campos cuando se actualiza la cuadrícula o cuando el diseño del Pivot Grid se guarda en un archivo y después se vuelve a cargar.

## Correcciones de errores en 3.17.0

- Se corrigió un error introducido en 3.16.0 que provocaba que la aplicación se congelara durante varios segundos después de cargar un archivo de diagrama (.te3diag)
- Se corrigió un problema que impedía guardar entre sesiones las opciones del cuadro de diálogo "Buscar/Reemplazar" al usar el botón "Buscar todo". Ahora, las últimas preferencias de búsqueda se conservarán en la siguiente sesión.
- Se corrigió un problema que impedía implementar la eliminación de Data sources, roles y miembros de rol en un modelo existente. Anteriormente, las casillas correspondientes del Asistente de implementación estaban deshabilitadas cuando no había Data sources ni roles en el modelo. Ahora, estas casillas permanecen habilitadas al implementar cambios en un modelo existente, lo que permite implementar la eliminación de estos objetos.
- Se corrigió un problema que impedía que los archivos con la extensión .ovpax se abrieran como archivo o como modelo. Ahora se admite abrir archivos y modelos .ovpax.
- Se corrigió un error por el que el cuadro de diálogo de archivos podía sugerir un tipo de archivo incorrecto al exportar archivos ofuscados desde el Analizador VertiPaq. Ahora, al exportar un archivo ofuscado, el cuadro de diálogo de guardado sugerirá correctamente usar la extensión ".ovpax".
- Se corrigió un error por el que los botones de la barra de herramientas se quedaban deshabilitados cuando el Editor de expresiones se acoplaba como una ventana de herramientas en lugar de como una pestaña de documento.
- Se corrigió un error que impedía abrir modelos semánticos guardados como ".tmdl" con [archivos de Proyecto de Power BI (PBIP)](https://learn.microsoft.com/en-us/power-bi/developer/projects/projects-overview).
- Se corrigió un error que hacía que la aplicación se quedara colgada durante la activación manual.
- El grabador de macros ya no debería generar código duplicado. Además, ahora el grabador también debería generar código cuando se eliminen objetos.
- La configuración de **formato automático** para los **editores de DAX**, incluidas Consulta DAX, Script DAX y las expresiones DAX en el Editor de expresiones, ahora se aplica al instante, sin necesidad de reiniciar la aplicación.
- Se corrigió un problema por el cual, al desmarcar la opción **Encrypt connection**, no se desactivaba el cifrado en la cadena de conexión de los Data sources de Microsoft SQL Server.
- Se corrigió un error por el que el nombre de usuario y la contraseña de ODBC no se añadían a la cadena de conexión al importar tablas o actualizar el esquema de tabla. Este problema impedía a los usuarios importar tablas o actualizar el esquema de la tabla al usar conexiones ODBC con algunos Data sources, como Snowflake u Oracle.
- Se corrigió un error por el que no era posible enumerar los DSN de ODBC al importar tablas si en el equipo no había DSN de usuario, es decir, User DSNs.
- Al consultar la información de esquema desde un origen ODBC, ahora utilizamos varios mecanismos de respaldo, lo que debería mejorar la compatibilidad con más controladores ODBC, como "Oracle in OraClient12Home1".
- Se corrigió un error por el que los Data sources a los que se hacía referencia en consultas M aparecían como Data sources implícitos en el Asistente para importar tablas.
- Se corrigió un error por el que algunos botones de la barra de herramientas de Pivot Grid permanecían deshabilitados después de volver a cargar un Pivot Grid desde un archivo.
- Se corrigió un error por el que las constantes en expresiones M (como `Order.Ascending`) provocaban un error de **Identificador desconocido** al usar la función **Actualizar esquema de tabla**. Consulta el [#1362](https://github.com/TabularEditor/TabularEditor3/issues/1362).
- Al usar la función DAX `DIVIDE` con un tercer argumento no numérico, el analizador de DAX ahora determina correctamente que el tipo de datos resultante es Variant.
- Se corrigió un error que impedía a Tabular Editor cargar los metadatos del modelo desde una carpeta TMDL sin el archivo database.tmdl.

> [!NOTE]
> Microsoft ha anunciado que [TMDL está en GA](https://powerbi.microsoft.com/en-us/blog/announcing-general-availability-of-tabular-model-definition-language-tmdl/) a partir del August 2nd, 2024. Ten en cuenta que Tabular Editor 3 utiliza una versión de la [biblioteca cliente AMO/TOM](https://www.nuget.org/packages/Microsoft.AnalysisServices.NetCore.retail.amd64/19.79.1.1), que se publicó antes de que se anunciara que TMDL estaba en GA. Esta versión debería seguir siendo totalmente compatible con TMDL; aun así, recomendamos mantener una copia de seguridad de tus archivos TMSL (.bim), por si acaso.

## Vienes de Tabular Editor 2.x?

Mira [este vídeo](https://www.youtube.com/watch?v=pt3DdcjfImY) para hacerte una idea de las nuevas funcionalidades de Tabular Editor 3. Además, no dejes de consultar nuestra [guía de incorporación](https://docs.tabulareditor.com/onboarding/index.html).

**Descripción general de las principales características de Tabular Editor 3:**

- IDE totalmente personalizable, con compatibilidad con varios monitores, Hi-DPI y temas
- Nuevo y potente editor de código DAX con autocompletado, comprobación de sintaxis, plegado de código y mucho, mucho más
- \*Modo del área de trabajo, que te permite guardar los cambios en disco y sincronizar simultáneamente los metadatos del modelo con Analysis Services
- \*Previsualiza los datos de las tablas con desplazamiento infinito, crea PivotGrids o escribe consultas DAX para explorar el modelo o probar la lógica de cálculo
- \*Programa actualizaciones de datos
- Actualiza los esquemas de tabla tanto en orígenes de datos de proveedor como en orígenes de datos estructurados (sí, ¡incluso para consultas M!)
- Crea diagramas del Data model
- Crea scripts DAX que te permitan editar varias medidas u otros objetos calculados en un único documento
- Graba C# Scripts y guárdalos como macros (antes conocidas como "Acciones personalizadas")
- Integración con el Analizador VertiPaq
- Depurador de DAX

\*=Solo cuando estés conectado a una instancia de Analysis Services o Power BI
