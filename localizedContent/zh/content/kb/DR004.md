代码操作 `DR004`（可读性） **为变量添加前缀**

## 说明

建议为变量使用一致的前缀，以便更容易将它们与表引用区分开来。 建议为变量使用一致的前缀，以便更容易将它们与表引用区分开来。 默认前缀为 `_`，但你可以在 **工具 > 偏好 > DAX编辑器 > 代码操作** 中配置要使用的前缀，以匹配你偏好的风格。

### 示例

修改前：

```dax
VAR sales = SUM('Internet Sales'[Sales Amount])
RETURN sales * 1.25;
```

修改后：

```dax
VAR _sales = SUM('Internet Sales'[Sales Amount])
RETURN _sales * 1.25;
```

## 为什么 Tabular Editor 会建议这样做？

此代码操作旨在提升 DAX 代码的可读性。 通过为变量采用一致的命名约定，你可以更容易理解每个变量的用途，并区分变量与表。 通过为变量采用一致的命名约定，你可以更容易理解每个变量的用途，并区分变量与表。

此外，为变量使用特殊字符（例如下划线）作为前缀，有助于避免与表名发生命名冲突——以防未来在模型中添加了与变量同名的表。
