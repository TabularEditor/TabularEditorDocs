### 先决条件

Tabular Editor 3.3.0（及更高版本）使用 .NET 6。 在启动 Tabular Editor 3.3.0（或更高版本）之前，你可能需要先安装 [.NET 6.0 桌面运行时](https://dotnet.microsoft.com/en-us/download/dotnet/6.0/runtime)。 从 3.3.1 开始，如果缺少这些先决条件，Tabular Editor 3 安装程序会下载并安装它们。

## 3.5.0 的改进

查看我们的 [发布博客](https://blog.tabulareditor.com/2023/03/15/tabular-editor-3-march-2023-release/)，快速了解本次发布中最重要的更新。

- 将 TOM 更新到 [19.57.2.4](https://www.nuget.org/packages/Microsoft.AnalysisServices.retail.amd64/)。
- 本版本引入了**表格组**，让你可以在 TOM Explorer 中将表格整理到“文件夹”中。 [了解更多](xref:table-groups)。
- 取消注释代码的默认快捷键现已设为 Ctrl+U
- 当会话因超时失效后，Tabular Editor 现在会尝试自动重新连接到 Analysis Services。 这意味着不再需要关闭并重新打开表格预览、DAX 查询窗口和 Pivot Grid。 参见 [#650](https://github.com/TabularEditor/TabularEditor3/issues/650)、[#803](https://github.com/TabularEditor/TabularEditor3/issues/803)。
- 新增了一组重载的 C# Script 方法 `Macro(...)`。 它们是 `CustomAction(...)` 的别名。 API 文档也已相应更新。
- 在**工作区模式**下使用 Tabular Editor 时，Workspace 数据库更新不会覆盖增量刷新的分区。 此外，新增了一个序列化选项 **忽略增量刷新分区**，并且对新模型默认启用。 启用该选项后，将模型保存为 .bim 或文件夹结构时，不会序列化由增量刷新管理的分区。 这让使用增量刷新的模型更容易接入版本控制。
- 在**透视编辑器视图**中新增工具栏按钮，用于切换是否显示“显示文件夹”。 在透视编辑器中切换某个显示文件夹时，会同时切换该文件夹内的所有对象。 透视编辑器现在还会提供 Visual 提示：当某个表或文件夹同时包含已纳入透视和未纳入透视的项目时，会显示相应标识。
- Tabular Editor 现已全面支持 Oracle 数据源。 你必须安装 [ODP.NET 托管驱动程序](https://www.oracle.com/database/technologies/odac-downloads.html)；如果你想在 Power BI Desktop 中从 Oracle 导入数据，这也是必需条件。

## 3.5.0 缺陷修复

- 当 DAX编辑器配置为使用制表符时，自动格式化将不再清空代码
- **宏** 视图中的“打开宏”按钮现在会在选中某个宏时启用
- 现在不再需要重置窗口布局，**透视编辑器** 视图也能正常显示
- 当将宏添加为新按钮时，主菜单栏将不再重置
- 修复了一个问题：在某些情况下，Tabular Editor 无法打开包含 Power BI 专用 TOM 属性的 .bim 文件或 .json 文件夹结构。
- `ResolveObjectPath` 现在支持与 TE2 类似的“旧”路径格式。 参见 [#1077](https://github.com/TabularEditor/TabularEditor/issues/1077)。
- 依赖表上 `Expression` 属性的 BPA 规则将不再失败。
- 即使宏的名称包含双引号，现在也可以保存/编译宏。
- 修复了 C# Script 方法 `CustomAction(...)` 的若干问题。
- 透视编辑器现在能正确处理包含可见度量值的隐藏表。
- 修复了一个问题：无法通过 **属性视图** 在模型对象上设置 **默认度量值** 属性。
- 离线工作时，上下文菜单中将不再显示 **计算表格** 选项。
- 修复了一个问题：C# 自动完成下拉列表会“吞掉”按下句点键的操作。
- 修复了语义分析器错误地报告 DAX [XIRR](https://dax.guide/xirr) 函数参数用法不正确的问题。
- 关系编辑器现在允许在不同但兼容的数据类型列之间创建关系。
- 修复了在使用 M 分区的表上执行架构更新时出现 “The key didn't match any rows” 错误的问题。
- 应用增量刷新的刷新策略的表现在也能使用“更新表架构”功能了。

## 从 Tabular Editor 2.x 迁移过来的吗？

观看 [这个视频](https://www.youtube.com/watch?v=pt3DdcjfImY)，了解 Tabular Editor 3 的新功能。 另外，别忘了查看我们的 [新手入门指南](https://docs.tabulareditor.com/onboarding/index.html)。

**Tabular Editor 3 主要功能概览：**

- 完全可自定义的 IDE，支持多显示器、Hi-DPI 和主题
- 全新强大的 DAX 代码编辑器，提供自动完成、语法检查、代码折叠等更多功能
- \*工作区模式：可将更改保存到磁盘，并同时将模型元数据同步到 Analysis Services
- \*通过无限滚动预览表数据，创建 PivotGrid，或编写 DAX 查询来浏览模型或测试计算逻辑
- \*安排数据刷新
- 在 Provider 数据源和 Structured数据源上更新表架构（是的，连 M 查询也支持！）
- 创建 Data model 图表
- 创建 DAX 脚本，让你在一个文档中同时编辑多个度量值或其他计算对象
- 录制 C# Script 并保存为宏（以前称为“Custom Actions”）
- 集成 VertiPaq分析器
- DAX调试器

\*=仅在连接到 Analysis Services 或 Power BI 实例时可用
