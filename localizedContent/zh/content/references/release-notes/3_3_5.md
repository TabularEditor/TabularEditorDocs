## 重要！

**注意**：Tabular Editor 3.3.0（及更高版本）使用 .NET 6。 在启动 Tabular Editor 3.3.0（或更高版本）之前，您可能需要先安装 [.NET 6.0 桌面运行时](https://dotnet.microsoft.com/en-us/download/dotnet/6.0/runtime)。 从 3.3.1 开始，如果缺少这些先决条件，Tabular Editor 3 安装程序会自动下载并安装。

## 3.3.5 的改进

- 新增启动画面和图标，与我们新版网站的品牌形象保持一致。
- 将 TOM 更新至 [19.48.0](https://www.nuget.org/packages/Microsoft.AnalysisServices.NetCore.retail.amd64)。
- Tabular Editor 3 现提供一组可配置的[策略](https://docs.tabulareditor.com/common/policies.html)，便于 IT 组织对 Tabular Editor 3 的部分功能进行管控。
- 在“信息”“宏”和“BPA”视图中，现在可以锁定列宽。
- 改进了宏视图：新增右键菜单和工具栏按钮，用于打开/编辑宏。
- 为 [`EVALUATEANDLOG`](https://dax.guide/evaluateandlog) 和 [`SAMPLEAXISWITHLOCALMINMAX`](https://dax.guide/sampleaxiswithlocalminmax) 这两个 DAX 函数新增语义分析器与自动补全支持。
- 当模型从磁盘加载时，“文件”菜单中的“Revert”选项现在也可用。
- DAX调试器现在支持更多类型的 DAX 查询。具体来说，现在当查询使用 [`ADDCOLUMNS`](https://dax.guide/addcolumns)、[`CALCULATETABLE`](https://dax.guide/calculatetable)、[`SAMPLE`](https://dax.guide/sample)、[`SAMPLEAXISWITHLOCALMINMAX`](https://dax.guide/sampleaxiswithlocalminmax)、[`SUBSTITUTEWITHINDEX`](https://dax.guide/substitutewithindex) 和 [`SUMMARIZE`](https://dax.guide/summarize) 时，也可以调试单元格值。 这应能确保 PBI Desktop 生成的大多数查询都可以在 TE3 调试器中使用。 注意：只能调试来源于显式度量值的单元格值。
- 在 DAX调试器中新增“Step into selection”选项，方便导航。 使用快捷键（Ctrl+B），或在代码中的任意位置右键并选择“Step into selection”进行导航。
- 在 TOM Explorer 中更清晰地显示带有错误/警告的对象。 改进了“数据刷新”选项卡中的图标。

## 3.3.5 的错误修复

- 更可靠地检测 Azure AS 开发人员层级（D1）；此前有时会导致商业版客户无法访问 AAS 实例。
- 允许在 PBI Desktop 受限模式下更改角色名称，参见 [#624](https://github.com/TabularEditor/TabularEditor3/issues/624)。
- 修复无法设置 SortByColumn 的问题，参见 [#625](https://github.com/TabularEditor/TabularEditor3/issues/625)。
- 修复了一个问题：当在连接字符串中通过“Extended Properties”指定连接字符串属性时，会导致 Tabular Editor 无法连接到 ODBC 数据源。
- 默认情况下，Snowflake ODBC 使用 TOP 行数限制子句。
- 修复作为脚本/宏执行时，等待加载指示器遮挡“SelectItem”对话框的问题。 参见 [#622](https://github.com/TabularEditor/TabularEditor3/issues/622)。
- 修复调试器在筛选器中处理（空白）值和二进制值时的方式问题。
- 如果连接因无效的 Binary XML 失败，则自动回退到 `Protocol Format=xml`。 参见 [#637](https://github.com/TabularEditor/TabularEditor3/issues/637)。
- 修复“批量重命名”在 Power BI Desktop 模型中不可用的问题。
- 为 M 解析器添加对特殊字符的支持，参见 [#638](https://github.com/TabularEditor/TabularEditor3/issues/638)。
- 将 'version' 视为 DAX 保留字（即始终加引号；若未加引号使用或将其用作变量名，则会报错）。

## 从 Tabular Editor 2.x 升级过来？

观看 [此视频](https://www.youtube.com/watch?v=pt3DdcjfImY)，快速了解 Tabular Editor 3 的新功能。 另外，别忘了查看我们的[入门指南](https://docs.tabulareditor.com/onboarding/index.html)。

**Tabular Editor 3 主要功能概览：**

- 完全可自定义的 IDE，支持多显示器、Hi-DPI 和主题
- 全新强大的 DAX 代码编辑器，支持自动补全、语法检查、代码折叠等更多功能
- \*工作区模式，可将更改保存到磁盘，同时将模型元数据同步到 Analysis Services
- \*使用无限滚动预览表数据，创建 PivotGrid，或编写 DAX 查询来浏览模型或测试计算逻辑
- \*计划数据刷新
- 在 Provider 和 Structured数据源中更新表架构（对，M 查询也支持！）
- 创建 Data model 图
- 创建 DAX 脚本，让你在一个文档中编辑多个度量值或其他计算对象
- 录制 C# Script 并保存为宏（以前称为“自定义操作”）
- VertiPaq分析器集成
- \*DAX调试器

\*=仅在连接到 Analysis Services 或 Power BI 实例时可用
