## 3.1.1 中的改进

- 共享表达式（Power Query / M 表达式）现在可以序列化为单独的文件。 你可以在 **工具 > 偏好 > 保存到文件夹** 中控制序列化行为；对于现有模型，则可在 **模型 > 序列化选项...** 中设置 ([#218](https://github.com/TabularEditor/TabularEditor3/issues/218))
- 新增选项：可在 **工具 > 偏好 > 建模操作** 中控制将透视成员关系应用到新对象的方式（见问题 [#216](https://github.com/TabularEditor/TabularEditor3/issues/216)）。
- 现在可以在序列化时排除 `PrivacySetting` 属性，以避免 Microsoft.AnalysisServices.Deployment 的相关问题。 可在 **模型 > 序列化选项...** 中进行控制（见问题 [#210](https://github.com/TabularEditor/TabularEditor3/issues/210)）。
- 你现在可以直接在 C# Script 中定义枚举、结构体和类，更便于组织与复用 C# 代码。
- VertiPaq分析器现在提供从数据中收集统计信息的选项（**工具 > 偏好 > VertiPaq分析器**），从而提升统计信息质量，例如当列的 IsAvailableInMDX = false 时。 查看问题 [#190](https://github.com/TabularEditor/TabularEditor3/issues/290)。
- 创建新模型后，Tabular Editor 3 现在会在首次保存时提示你选择将模型保存为文件还是文件夹。 你也可以在 **工具 > 偏好 > 常规 > 新建模型的默认保存格式** 中设置默认模式。 查看问题 [#50](https://github.com/TabularEditor/TabularEditor3/issues/50)。
- 改进了启动性能 (有用户报告称，在 TabularEditor3.exe 快捷方式中添加 `-multicorejit` 开关后，启动时间还能进一步缩短)。
- 重构了“模型”菜单，让“新建计算组”等选项更容易找到。
- 将 AMO/TOM 客户端库更新至 [19.26.1.7](https://www.nuget.org/packages/Microsoft.AnalysisServices.retail.amd64/)。
- 现在可以在 Best Practice Analyzer 中逐一遍历模型角色成员
- 为避免与[工作区模式](https://docs.tabulareditor.com/te3/tutorials/workspace-mode.html)混淆，“窗口”菜单中的“Workspace”已更名为“布局”。
- 将 Ctrl+Shift+D 添加为“部署”的快捷键
- 新增对 DAX 函数（[`HASH`](https://dax.guide/hash)、[`NAMEOF`](https://dax.guide/nameof)、[`COLUMNSTATISTICS`](https://dax.guide/columnstatistics)、[`BITAND`](https://dax.guide/bitand)、[`BITLSHIFT`](https://dax.guide/bitlshift)、[`BITOR`](https://dax.guide/bitor)、[`BITRSHIFT`](https://dax.guide/bitrshift)、[`BITXOR`](https://dax.guide/bitxor)）的支持，并在 [`SELECTCOLUMNS`](https://dax.guide/selectcolumns) 中支持可选的列名参数。

## 3.1.1 中的错误修复

- 在删除对象时，TOM Explorer 不应再出现卡死或崩溃（参见问题 [#223](https://github.com/TabularEditor/TabularEditor3/issues/223)、[#209](https://github.com/TabularEditor/TabularEditor3/issues/209)）。
- 修复了编辑器（[#207](https://github.com/TabularEditor/TabularEditor3/issues/207)）和图标（[#215](https://github.com/TabularEditor/TabularEditor3/issues/215)）在深色主题下的颜色问题
- 表格现在会在 "创建/编辑关系" 对话框中按字母顺序排序 (参见问题 [#214](https://github.com/TabularEditor/TabularEditor3/issues/214)
- 修复了一个与 C# Script 相关的问题，这会导致无法将脚本保存为宏（参见问题 [#208](https://github.com/TabularEditor/TabularEditor3/issues/208)）。
- 修复了一个问题：数据源密码/凭据未能正确应用到 Workspace 数据库（参见问题 [#205](https://github.com/TabularEditor/TabularEditor3/issues/205)）。
- “选择所有列”复选框不应再导致崩溃（参见问题 [#204](https://github.com/TabularEditor/TabularEditor3/issues/204)）。
- 当脚本通过 `#r` 预编译器指令引用外部 DLL 时，Tabular Editor 现在始终会在安装目录中查找（参见问题 [#192](https://github.com/TabularEditor/TabularEditor3/issues/192)）。
- 修复了语言架构的一个问题，导致 Q&A 在某些情况下无法使用
- 在“数据预览”中对日期列进行筛选不应再导致崩溃
- 在 TOM Explorer 的“透视”下拉列表中选择“所有透视”时，不再显示未包含在任何透视中的对象。 另外还修复了下拉框未能更新透视名称的问题。
- 允许将某些 DAX 关键字作为变量使用（`WEEK`、`DAY`、`INTEGER` 等） 且不会报错。
- 修复了一个问题：首次保存模型时，序列化选项不会被保存。
- 修复了 DAX 解析器的一个问题，该问题会导致加载模型时崩溃（参见 [#229](https://github.com/TabularEditor/TabularEditor3/issues/229)）。

## 从 Tabular Editor 2.x 迁移过来？

观看 [此视频](https://www.youtube.com/watch?v=pt3DdcjfImY)，了解 Tabular Editor 3 的新功能。

**Tabular Editor 3 主要功能概览：**

- 完全可自定义的 IDE，支持多显示器、Hi-DPI 和主题
- 全新强大的 DAX 代码编辑器，具备自动完成、语法检查、代码折叠等更多功能
- \*工作区模式，可让你将更改保存到磁盘，并同时将模型元数据同步到 Analysis Services
- \*通过无限滚动预览表数据，创建 PivotGrid，或编写 DAX 查询来浏览模型或验证计算逻辑
- \*计划数据刷新
- 更新 Provider 数据源和 Structured数据源的表架构（是的，连 M 查询也适用！）
- 创建 Data model 图表
- 创建 DAX 脚本，以便在一个文档中编辑多个度量值或其他计算对象
- 录制 C# Script 并保存为宏（此前称为“Custom Actions”）
- VertiPaq分析器集成

\*=仅在连接到 Analysis Services 或 Power BI 实例时才可用
