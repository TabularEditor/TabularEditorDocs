## 3.1.5 中的改进

- 你现在可以在 DAX 查询、Pivot Grid 和表格预览中**模拟**用户或角色。 这对于测试 RLS 和 OLS 设置很有用。 [更多信息](https://docs.tabulareditor.com/onboarding/refresh-preview-query.html#impersonation)。
- DAX 脚本现在支持计算组和计算项。 这使你可以在一个文档中查看所有计算组逻辑（包括格式字符串表达式）。 [示例](https://docs.tabulareditor.com/onboarding/dax-script-introduction.html#example-3-calculation-group)。
- 在 DAX 查询或 DAX 脚本中右键点击某个度量值引用时，现在可以选择添加该度量值的定义，[包括所有依赖项](https://docs.tabulareditor.com/onboarding/dax-script-introduction.html#define-measures)。
- 现在你可以通过拖拽空白区域来平移关系图（见 [#275](https://github.com/TabularEditor/TabularEditor3/issues/275)）。

## 3.1.5 中的错误修复

- 修复内置架构更新器在 M 表达式中遇到 `null` 值时的问题（见 [#282](https://github.com/TabularEditor/TabularEditor3/issues/282)）。
- 在 `SELECTCOLUMNS` 的参数中使用 `RELATED` 函数时（未指定名称参数），语义分析器不再 Report 错误。
- 现在可以编辑或重命名宏，而不会从工具栏自定义设置中消失（见 [#266](https://github.com/TabularEditor/TabularEditor3/issues/266)）。
- 现在在重启应用后，可以重命名/删除所有自定义工具栏（见 [#260](https://github.com/TabularEditor/TabularEditor3/issues/260)）。
- 修复在选择包含子文件夹的文件夹时，选择和上下文菜单存在的问题（见 [#248](https://github.com/TabularEditor/TabularEditor3/issues/248)）。
- 防止同时选择表/表对象时崩溃。
- 修复导入表时的多项问题和崩溃（ODBC NullReferenceException 等）。

## 从 Tabular Editor 2.x 迁移过来？

观看[这个视频](https://www.youtube.com/watch?v=pt3DdcjfImY)，快速了解 Tabular Editor 3 的新功能。

**Tabular Editor 3 主要功能概览：**

- 完全可自定义的 IDE，支持多显示器、Hi-DPI 和主题
- 全新强大的 DAX 代码编辑器，提供自动完成、语法检查、代码折叠等更多功能
- \*工作区模式，可将更改保存到磁盘，并同时将模型元数据同步到 Analysis Services
- \*支持无限滚动预览表数据，创建 PivotGrid，或编写 DAX 查询来浏览模型并测试计算逻辑
- \*安排数据刷新
- 在 Provider 数据源和 Structured数据源中更新表架构（没错，连 M 查询也支持！）
- 创建 Data model 图表
- 创建 DAX 脚本，在一个文档中编辑多个度量值或其他计算对象
- 录制 C# Script 并保存为宏（以前称为“自定义操作”）
- VertiPaq分析器集成

\*=仅在连接到 Analysis Services 或 Power BI 实例时才可用
