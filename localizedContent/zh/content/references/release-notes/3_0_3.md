## 已知问题

- 由于需要先修复一些高优先级 Bug，表导入向导未能赶上此版本。 我们会尽快推出！ 作为临时变通方案，只需新建一个表并指定分区查询（SQL 或 M）。 然后右键单击该表，选择“更新表架构...”，即可导入列。
- 更多信息请参阅[路线图](https://github.com/TabularEditor/TabularEditor3/issues/12)。

## 3.0.3 中的新功能：

- 在处理视图中右键单击某个项，现在可以将其重新加入队列；如果该项代表单个表，还会打开该表的数据预览。

## 3.0.3 中修复的问题：

- 当语义分析器在计算列/计算表格上检测到错误时，将不再修改该表中的列（此前在某些情况下会导致关系被删除）
- 修复了查找/替换对话框中某些控件启用状态不正确的问题
- 根据遥测数据和错误 Report 修复了多个问题，例如：
  - 现在可以正常复制/粘贴带有 Lineage tag 的列
  - 在表已重命名时，将表粘贴覆盖到现有表不再导致崩溃
  - “还原”按钮现在会在刷新本地 TOM 之前先重新连接到 AS，从而避免崩溃
  - 在属性网格为空时右键单击不再导致崩溃
  - 以及许多其他修复……
- 修复“Server”下拉框的最近使用项 MRU 未能正确保存的问题
- 修复工作区模式无法从“用户选项”文件正确加载数据源覆盖值等多个问题。 参见 [#28](https://github.com/TabularEditor/TabularEditor3/issues/28)。
- 修复并改进多项“虚假”DAX 语义错误（支持 ROLLUPADDISSUBTOTAL、带 0 个参数的 COUNTROWS 等）
- 改进多项 DAX 函数的自动补全/参数提示（TOPN、TOPNPERLEVEL、DATEADD/DATESINPERIOD/PARALLELPERIOD 的 interval 参数等），并改进 RETURN 关键字后的提示。

## 从 Tabular Editor 2.x 迁移过来？

观看 [此视频](https://www.youtube.com/watch?v=pt3DdcjfImY)，快速了解 Tabular Editor 3 的新功能。

**Tabular Editor 3 主要功能概览：**

- 完全可自定义的 IDE，支持多显示器、Hi-DPI 和主题
- 全新强大的 DAX 代码编辑器，支持自动完成、语法检查、代码折叠等更多功能
- \*工作区模式：可将更改保存到磁盘，同时将模型元数据同步到 Analysis Services
- \*支持无限滚动预览表数据，创建 PivotGrid，或编写 DAX 查询来浏览模型或测试计算逻辑
- \*更新 Provider 和 Structured数据源上的表架构（是的，连 M 查询也支持！）
- \*安排数据刷新计划
- 创建 Data model 示意图
- 创建 DAX 脚本，让你在一个文档中编辑多个度量值或其他计算对象
- 录制 C# Script 并保存为宏（旧称“Custom Actions”）
- 集成 VertiPaq分析器

\*=仅在连接到 Analysis Services 或 Power BI 实例时可用
