## 已知问题

- 由于需要先修复一些高优先级 Bug，“表导入向导”未能包含在此版本中。 我们会尽快提供！ 作为一种变通方案，只需新建一个表，并指定分区查询（SQL 或 M）。 然后右键单击该表，选择“更新表架构...”，就能导入列。
- 更多信息见[路线图](https://github.com/TabularEditor/TabularEditor3/issues/12)。

## 3.0.5 中的改进：

- 图表视图现在看起来好多了（浅色和深色主题均适用），并且在画布上拖动表时，关系连线会实时更新。

## 3.0.5 中的错误修复：

- 表达式编辑器在隐藏状态下，选择图表项或 TOM Explorer 中的项时不会自动弹出。 表达式编辑器在隐藏状态下，选择图表项或 TOM Explorer 中的项时不会自动弹出。 改为在 TOM Explorer 中双击该项，或通过“视图”菜单将其重新显示 (详见问题 [#43](https://github.com/TabularEditor/TabularEditor3/issues/43)。
- DAX 解析器在实际上不存在循环依赖时，将不再报告循环依赖(详见问题 [#63](https://github.com/TabularEditor/TabularEditor3/issues/63) 和 [#64](https://github.com/TabularEditor/TabularEditor3/issues/64))
- 修复了 DAX 解析器在计算表列（CalculatedTableColumns）上确定 SourceColumn 值时的问题；该问题可能导致在加载模型时删除列，进而删除关系。
- 如果将某些关键字用作 DAX 变量（这是不允许的），DAX 解析器现在会显示错误。 详见问题 [#58](https://github.com/TabularEditor/TabularEditor3/issues/58)。
- 在开始搜索时按下“:”，现在会立即切换到“整个模型”和 Dynamic LINQ 模式。
- Dynamic LINQ 现在也会搜索分区和计算组项，详见问题 [#70](https://github.com/TabularEditor/TabularEditor3/issues/70)。

## 从 Tabular Editor 2.x 迁移过来？

观看[此视频](https://www.youtube.com/watch?v=pt3DdcjfImY)，快速了解 Tabular Editor 3 的新增功能。

**Tabular Editor 3 主要功能概览：**

- 完全可自定义的 IDE，支持多显示器、高 DPI 和主题
- 全新强大的 DAX 代码编辑器，提供自动补全、语法检查、代码折叠等众多功能
- \*工作区模式：在将更改保存到磁盘的同时，同步模型元数据到 Analysis Services
- \*通过无限滚动预览表数据，创建 PivotGrid，或编写 DAX 查询，以浏览模型或测试计算逻辑
- \*更新 Provider 数据源和 Structured数据源上的表架构（是的，连 M 查询也支持！）
- \*安排数据刷新计划
- 创建 Data model 图
- 创建 DAX 脚本，让你在同一文档中批量编辑多个度量值或其他计算对象
- 录制 C# Script 并保存为宏（以前称为“自定义操作”）
- 集成 VertiPaq分析器

\*=仅在连接到 Analysis Services 或 Power BI 实例时才可用
