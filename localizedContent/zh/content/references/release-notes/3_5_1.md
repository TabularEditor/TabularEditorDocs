### 先决条件

Tabular Editor 3.3.0（及更高版本）使用 .NET 6。 在启动 Tabular Editor 3.3.0（或更高版本）之前，你可能需要先安装 [.NET 6.0 桌面运行时](https://dotnet.microsoft.com/en-us/download/dotnet/6.0/runtime)。 从 3.3.1 开始，如果缺少这些先决条件，Tabular Editor 3 安装程序将自动下载并安装它们。

## 3.5.1 中的修复与改进

- 将 TOM 更新至 [19.60.0](https://www.nuget.org/packages/Microsoft.AnalysisServices.retail.amd64/)。
- 修复了“编辑关系”对话框会显示不兼容数据类型警告的问题，即使两列的数据类型完全相同也会出现该警告。参见 [#855](https://github.com/TabularEditor/TabularEditor3/issues/855)。 此外，当连接到 Analysis Services 实例时，该对话框现在会显示 RI 违规和重复值警告。
- 修复了从磁盘打开文件或文件夹结构时，应用有时会卡死的问题。
- 修复了当角色名称中包含“.”时无法编辑 RLS 表达式的问题。
- 当某个对象显示在多个显示文件夹中时，TOM Explorer 没有在所有位置正确刷新它的属性。 该问题已修复。
- “Model”现已在语义分析器中被视为保留字，参见 [#852](https://github.com/TabularEditor/TabularEditor3/issues/852)。
- The .pbitool 文件 (用于与 PBI Desktop 进行外部工具集成) 现在使用更新后的 TE3 图标。
- “创建 Workspace 数据库”对话框现在会验证数据库名称，避免使用 Analysis Services 不支持的特殊字符。
- 修复了 DAX 编辑器中自动缩进会覆盖字符的问题。 参见 [#857](https://github.com/TabularEditor/TabularEditor3/issues/857)。
- 现在，你可以通过表格组的右键上下文菜单，直接在表格组中创建并添加表/计算组。
- 修复了 3.5.0 中的一个回归问题：从 MS SQL 风格的数据源导入列，或对表执行架构更新时，可能会为列分配错误的数据类型。
- 修复了在 DAX 编辑器中闭合方括号或圆括号时，光标有时会“跳”到错误位置的问题。

## 从 Tabular Editor 2.x 迁移过来？

观看[这个视频](https://www.youtube.com/watch?v=pt3DdcjfImY)，快速了解 Tabular Editor 3 的新功能。 另外，别忘了查看我们的[上手指南](https://docs.tabulareditor.com/onboarding/index.html)。

**Tabular Editor 3 主要功能概览：**

- 完全可自定义的 IDE，支持多显示器、Hi-DPI 和主题切换
- 全新的强大 DAX 代码编辑器，具备自动完成、语法检查、代码折叠等众多功能
- \*工作区模式，可将更改保存到磁盘，并同时将模型元数据同步到 Analysis Services
- \*通过无限滚动预览表数据，创建 PivotGrid，或编写 DAX 查询来浏览模型或测试计算逻辑
- \*设置数据定时刷新
- 同时更新 Provider 和 Structured数据源的表架构（没错，连 M 查询也支持！）
- 创建 Data model 图
- 创建 DAX 脚本，使您能够在单个文档中编辑多个度量值或其他计算对象
- 录制 C# Script 并保存为宏（以前称为“自定义操作”）
- VertiPaq分析器集成
- DAX调试器

\*=仅在连接到 Analysis Services 或 Power BI 实例时可用
