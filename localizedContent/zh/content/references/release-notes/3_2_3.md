## 3.2.3 中的改进

- 将 TOM 更新至 [19.36.0](https://www.nuget.org/packages/Microsoft.AnalysisServices.retail.amd64/)
- 新建 PBI 模型使用兼容级别 1569
- 允许使用 ExportProperties 对 CalculationGroups 和 CalculationItems 进行序列化。 参见 [#976](https://github.com/TabularEditor/TabularEditor/issues/976)
- 允许在 .AddExternalMember 和 .AddWindowsMember 上指定 MemberID，参见 [#484](https://github.com/TabularEditor/TabularEditor3/issues/484) 和 [#483](https://github.com/TabularEditor/TabularEditor3/issues/483)
- 允许在调用 ExecuteCommand 时指定原始 XMLA，如 [#495](https://github.com/TabularEditor/TabularEditor3/issues/495) 中所讨论
- 允许在 Pivot Grid 中任意位置右键，将 Pivot Grid MDX 查询复制到剪贴板，参见 [#494](https://github.com/TabularEditor/TabularEditor3/issues/494)。
- 改进 BPA 视图（显示严重性、类别）以及工具提示。 参见 [#489](https://github.com/TabularEditor/TabularEditor3/issues/489)。
- 将 VertiPaq分析器中“Read statistics from data”的默认设置改为“false”，以与 DAX Studio / Bravo 保持一致。
- 新增选项，使自动补全弹窗不那么频繁（仅在输入字母后才弹出），参见 [#459](https://github.com/TabularEditor/TabularEditor3/issues/459/)。 可在 **Tools > 偏好 > DAX编辑器 > Code Assist** 中找到此设置。

## 3.2.3 的错误修复

- 修复表格权限未能正确反序列化的问题。 参见 [#446](https://github.com/TabularEditor/TabularEditor3/issues/446)。
- 修复在特定布局下启动时崩溃的问题，参见 [#455](https://github.com/TabularEditor/TabularEditor3/issues/455)。
- 修复了一个问题：当度量值或其他对象名称包含花括号时，可能导致应用程序崩溃。
- 当“Include sensitive”设置为 false 时，移除带星号标记的属性（密钥、密码）。 参见 [#479](https://github.com/TabularEditor/TabularEditor3/issues/479)。
- 修复在将无效的文件/文件夹名称用作对象名称，或对象名称末尾包含空格时，“保存到文件夹”的序列化问题。 类似于 [#952](https://github.com/TabularEditor/TabularEditor/issues/952) 和 [#978](https://github.com/TabularEditor/TabularEditor/issues/978)。
- 至少选中 1 个非计算表格时，显示所有刷新选项，参见问题 [#487](https://github.com/TabularEditor/TabularEditor3/issues/487)
- 修复语义分析器在使用 UNION 时的列沿袭问题，参见 [#482](https://github.com/TabularEditor/TabularEditor3/issues/482)。
- 修复快捷键在浮动窗口中不生效的问题，参见 [#480](https://github.com/TabularEditor/TabularEditor3/issues/480)。
- 不再为计算表格、计算组或 DirectQuery 表显示“EnableRefreshPolicy”属性，参见 [#470](https://github.com/TabularEditor/TabularEditor3/issues/470)。
- 修复在某些情况下（例如 Oracle 数据源），通过“导入表格向导”创建新的 Structured数据源时出现的问题。
- 修复通过导入向导添加 Structured ODBC 数据源时可能导致崩溃的问题。
- 修复在关系图中点击另一张表的列时选择异常的问题。
- 修复分区模式更改后，TOM Explorer 未能更新以显示该分区正确的对象类型的问题
- 修复 Pivot Grid 中显示的度量值格式，参见 [#492](https://github.com/TabularEditor/TabularEditor3/issues/492)
- 如果光标处已经有左括号，自动完成不应再插入括号，参见 [#459](https://github.com/TabularEditor/TabularEditor3/issues/459/)
- 修复 CONVERT 的第二个参数（数据类型）的自动完成功能，参见 [#474](https://github.com/TabularEditor/TabularEditor3/issues/474)。
- 启动新的重命名操作时，应先完成任何已有的重命名操作，参见 [#461](https://github.com/TabularEditor/TabularEditor3/issues/461/)。
- 修复仅执行所选内容时，多个 EVALUATE 语句未遵循 ORDER BY 的问题。 同时修复了一个问题：当表名同时也是 DAX 函数名时，ORDER BY 不生效。 参见 [#472](https://github.com/TabularEditor/TabularEditor3/issues/472)

## 从 Tabular Editor 2.x 迁移过来？

观看 [此视频](https://www.youtube.com/watch?v=pt3DdcjfImY)，快速了解 Tabular Editor 3 的新功能。 另外，别忘了查看我们的 [入门指南](https://docs.tabulareditor.com/onboarding/index.html)。

**Tabular Editor 3 主要功能概览：**

- 完全可自定义的 IDE，支持多显示器、Hi-DPI 和主题
- 全新强大的 DAX 代码编辑器，支持自动补全、语法检查、代码折叠等，还有更多功能
- \*工作区模式：可将更改保存到磁盘，并同时将模型元数据同步到 Analysis Services
- \*可通过无限滚动预览表数据，创建 PivotGrid，或编写 DAX 查询来浏览模型并测试计算逻辑
- \*计划数据刷新
- 支持更新提供程序数据源和 Structured数据源的表架构（没错，连 M 查询也支持！）
- 创建 Data model 图表
- 创建 DAX 脚本，让你在一个文档中编辑多个度量值或其他计算对象
- 录制 C# Script，并保存为宏（以前称为“Custom Actions”）
- 集成 VertiPaq分析器
- \*DAX调试器

\*=仅在连接到 Analysis Services 或 Power BI 实例时可用
