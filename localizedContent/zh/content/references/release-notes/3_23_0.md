## 3.23.0 新增功能

请查看我们的[发布博客](https://blog.tabulareditor.com/)，快速了解本次发布中最重要的更新。

- Tabular Editor 3 现已支持 **DAX 用户自定义函数 (UDFs)**。 这是一项重要的大型功能，仅用几条要点难以充分说明。 如需了解更多，请参阅[专门的文档章节](xref:udfs)。
- 仿佛一个重大功能还不够，本次发布还新增了对 **Calendars** 日历功能的支持。 同样，如需了解更多，请参阅[专门的文档章节](xref:calendars)。

> [!NOTE]
> [Tabular Editor 3.23.1](xref:release-3-23-1) 包含多项与 UDF 和日历相关的重要 Bug 修复。如果你计划使用这些功能，建议更新到 3.23.1。

- **表导入向导**现已支持以 **Fabric SQL Databases** 和 **Fabric Mirrored Databases** 作为数据源，适用于导入模式和 Direct Lake 模式的模型，与 [Power BI Desktop 2025 年七月更新公告](https://powerbi.microsoft.com/en-us/blog/power-bi-july-2025-feature-summary/#post-30545-_Toc203388697) 一致。
- 原生支持 [ARM64](https://learn.microsoft.com/en-us/windows/arm/overview)。
- 为即将推出的 Power BI 功能做准备（目前还不能透露）。

## 3.23.0 的改进

- 主窗口现在会记住其位置和大小。 可在 偏好 > Tabular Editor > User Interface > Layout 中禁用。
- 信息视图现在默认复制所选单元格，而不是整行；仍可使用 Ctrl+Shift+C 像以前一样复制整行。 此外，还新增了右键快捷菜单，同样包含（复制单元格、复制行）等选项。
- 已将包括 AMO/TOM 在内的第三方依赖项更新至 [v. 19.103.2](https://www.nuget.org/packages/Microsoft.AnalysisServices/)。
- 新增“自动换行”选项现已可在工具栏、菜单（Edit > Word Wrap）以及偏好窗口（Text Editors > General）中使用。 也可以在偏好中启用换行标记。
- C# Script 编辑器的自动补全现支持更灵活的搜索（例如可搜索部分匹配，也可按大写字母/驼峰命名进行搜索）。
- 现在，除了 `DEFINE MEASURE` 语句外，你还可以对包含 [`DEFINE COLUMN`](https://www.sqlbi.com/articles/introducing-define-column-in-dax-queries/) 或 [`DEFINE TABLE`](https://www.sqlbi.com/articles/introducing-define-table-in-dax-queries/) 语句的 DAX 查询使用“Apply”或“Apply selection”命令。 这将会在模型中创建相应的计算列或计算表格；如果它们已存在，则会更新其 DAX 表达式。
- “窥视定义”窗口现在允许你滚动主 DAX 编辑器；当鼠标悬停在该窗口上时，将不再拦截滚轮滚动输入。
- “窥视定义”窗口高度现在调整得更紧凑。
- 当存在其他更明确的错误信息时，语义分析器将不再显示“Multiple column cannot be converted to a scalar value”错误。

## 3.23.0 的 Bug 修复

- 修复了一个问题：尽管建模操作是受支持的，但在 Power BI Desktop 中对模型进行任何更改仍必须先启用 **Allow unsupported modelling operations**。
- 修复了一个问题：在某些分辨率和 DPI 缩放设置下，主窗口会超出屏幕尺寸。
- 修复了“较大图标”、“在工具栏上显示屏幕提示”等工具栏自定义设置在重启 TE3 后不会保留的问题。
- 修复了一个界面显示异常：TOM Explorer 的透视和 Language 下拉控件出现重复按钮。
- 修复了“窥视定义”功能中的多项问题：
  - 在撤销/重做操作时，“窥视定义”会被关闭
  - 在 DAX 编辑器的最后一行使用“窥视定义”时将不可见
  - 删除被窥视引用所在行的行尾字符后，“窥视定义”会消失
  - 按 ESC 关闭自动补全弹窗时，“窥视定义”也会随之关闭
  - 按下 ESC 试图清除矩形选择时，“窥视定义”也会被关闭
  - 在被窥视引用左侧新增一行后，“窥视定义”会消失，并且会在被窥视引用上方错误出现空白填充区域
  - 修复了一个问题：触发“窥视定义”时，如果插入点不在被窥视引用所在行，会在错误的行下方出现空白填充区域
- C# 代码编辑器中的调用提示现在大小正确，确保所有调用提示文本都可见。
- 修复了一个问题：Fabric 连接对话框中未能正确列出 Fabric Warehouses（而是显示为“Unknown”类型）。
- DAX 自动完成不再会为 [`PARALLELPERIOD`](https://dax.guide/parallelperiod) 函数的 &lt;Interval&gt; 参数建议 `DAY`，因为这不是该参数的有效值。
- 使用上下方向键在不同语法替代项之间切换时，调用提示现在会正确更新。
- 修复了在对象之间切换时代码操作未能正确更新的问题。
- 修复了一个问题：放大时代码编辑器中的行号会被截断。

## 从 Tabular Editor 2.x 迁移过来？

观看[此视频](https://youtu.be/O4ATwdzCvWc)，快速了解 Tabular Editor 3 的主要功能。 另外，别忘了查看我们的[上手指南](https://docs.tabulareditor.com/onboarding/index.html)。

**Tabular Editor 3 主要功能概览：**

- 完全可自定义的 IDE，支持多显示器、Hi-DPI 和主题
- 全新强大的 DAX 代码编辑器，具备自动完成、语法检查、代码折叠等功能，远不止于此
- \*工作区模式：在将更改保存到磁盘的同时，将模型元数据同步到 Analysis Services
- \*支持无限滚动预览表数据，创建 PivotGrids，或编写 DAX 查询来浏览模型或测试计算逻辑
- \*安排数据刷新
- 同时更新 Provider 与 Structured数据源的表架构（没错，连 M 查询也支持！）
- 创建 Data model 图
- 创建 DAX 脚本，让你在单个文档中编辑多个度量值或其他计算对象
- 录制 C# Script 并保存为宏（以前称为“自定义操作”）
- 集成 VertiPaq分析器
- DAX调试器
- DAX优化器集成
- 使用“代码操作”轻松重构你的DAX。

\*=仅在连接到 Analysis Services 或 Power BI 实例时才可用
