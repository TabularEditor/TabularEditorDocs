## 3.4.2 中的错误修复

- 修复了语义分析器在并不存在时仍会 Report “循环依赖”错误的问题。 参见 [#811](https://github.com/TabularEditor/TabularEditor3/issues/811)。
- 当“查找范围”设置为“所选内容”时，“查找/替换”将不再替换文档中的所有匹配项，参见 [#293](https://github.com/TabularEditor/TabularEditor3/issues/293)。
- 在模型之间复制/粘贴区域设置不再导致崩溃，参见问题 [#798](https://github.com/TabularEditor/TabularEditor3/issues/798)。 注意：引用目标模型中不存在对象的 ObjectTranslations 会被删除。 注意：引用目标模型中不存在对象的 ObjectTranslations 会被删除。
- DAX调试器现在应能在所有情况下检查多行观察表达式。
- 在尝试调试类似 `EVALUATE { [Measure that uses ALLSELECTED] }` 这类查询结果时，DAX调试器不再崩溃

**Tabular Editor 3 主要功能概览：**

- 完全可自定义的 IDE，支持多显示器与 Hi-DPI，并提供主题
- 全新的强大 DAX 代码编辑器，提供自动完成、语法检查、代码折叠等更多功能
- \*工作区模式，可将更改保存到磁盘，并同时将模型元数据同步到 Analysis Services
- \*通过无限滚动预览表数据，创建 PivotGrid，或编写 DAX 查询来浏览模型或测试计算逻辑
- \*计划数据刷新
- 在 Provider 和 Structured数据源上更新表架构（是的，连 M 查询也支持！）
- 创建 Data model 图
- 创建 DAX 脚本，让你在一个文档中编辑多个度量值或其他计算对象
- 录制 C# Script 并将其保存为宏（原称“自定义操作”）
- VertiPaq分析器集成
- \*DAX调试器

\*=仅在连接到 Analysis Services 或 Power BI 实例时可用
