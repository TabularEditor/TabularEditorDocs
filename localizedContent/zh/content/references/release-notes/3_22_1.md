## 3.22.0 新增功能

请查看我们的[发布博客](https://blog.tabulareditor.com/)，快速了解此版本中最重要的更新内容。

(**仅限企业版**) Tabular Editor 3 现已全面支持 Direct Lake over OneLake 和 Direct Lake over SQL。 这意味着：

- 表导入向导新增了适用于 Fabric Warehouse 和 Lakehouse 的导入选项。 连接到您想要连接的 Warehouse/Lakehouse 时，您不再需要知道其 SQL analytics endpoint。 我们将改为在你的 Workspace 中显示所有 Warehouse/Lakehouse 的列表，方便你选择要使用的项目。
- 在“表导入向导”的最后一页中，你可以选择使用 OneLake 中的 Direct Lake 或 SQL 模式，或者将表配置为传统的“导入”模式。
- 当模型在 Direct Lake over OneLake 模式下包含表时，你现在也可以添加导入模式的表，以创建混合模式/混合模型。
- 在预览 Direct Lake 表中的数据时，我们不再限制返回的行数（你可以像在导入模式中一样，自由滚动浏览表中的所有数据）。 注意：执行数据预览时，所有列都会被加载到 Fabric capacity 中的语义模型。 如果你担心 Fabric capacity 上的内存占用，建议改用 DAX 查询。

> [!NOTE]
> 我们不再将 Direct Lake 模型配置为使用大小写 _敏感_ 的排序规则。 如果你计划通过 SQL 使用 Direct Lake，并连接到使用区分大小写排序规则的 Fabric Warehouse，那么在添加任何表或其他对象之前，必须手动填写语义模型的 **Collation** 属性。

## 3.22.0 中的改进

- 将模型保存为单个文件时，**保存文件** 对话框现在会建议将 "model.bim" 作为默认文件名。
- 已将多项依赖项更新到最新版本，包括将 [AMO/TOM 更新到 19.98.0.3](https://www.nuget.org/packages/Microsoft.AnalysisServices/)。 此次更新修复了长期存在的工作或学校账户身份验证问题，以及反复弹出的登录提示。
- DAX 查询编辑器现已支持 [`MPARAMETER`](https://dax.guide/st/mparameter/) 关键字，详见 [#1467](https://github.com/TabularEditor/TabularEditor3/issues/1467)。
- 我们的 DAX 编辑器现在支持 **基于单词** 的自动补全检索。 换句话说，如果你在编辑器中输入 `sales ytd`，自动补全现在会建议诸如 `[Sales Margin YTD]`、`[Sales Revenue YTD]` 等度量值。
- 随着 Power BI Desktop 2025 年六月更新，[外部工具现在可以对语义模型执行任何写入操作](https://powerbi.microsoft.com/en-us/blog/power-bi-june-2025-feature-summary/#post-30307-_Toc269410729)。 因此，当连接到该版本或更高版本的 Power BI Desktop 中的模型时，我们不再限制 Tabular Editor 中可用的建模操作。

## 3.22.1 中的错误修复

- 修复了一个问题：**数据刷新** 视图会变成空白，且在重启应用之前不会显示后续的刷新操作。 见 [#1461](https://github.com/TabularEditor/TabularEditor3/issues/1461)。

## 3.22.0 中的错误修复

- 在执行“保存到文件夹”操作时，遇到 IOException 异常时现已加入异常处理（我们有时需要删除文件夹结构中的某些文件夹，但这可能会被版本控制系统施加的文件系统锁阻止）。
- 修复了一个问题：当“新增功能”页面获得焦点时，某些键盘快捷键（默认和自定义）无法使用。
- 当你为某个命令分配的自定义快捷键已被其他命令占用时，我们现在会显示警告信息，并从其他命令中移除该快捷键，以避免歧义。
- 修复了一个问题：在 DAX 窗口函数中使用未限定的列引用时，语义分析器会错误地显示一条错误信息。 见 [#1460](https://github.com/TabularEditor/TabularEditor3/issues/1460)。
- 修复了一个可能会导致 **数据刷新** 视图偶尔卡死的错误，需要重启应用才能恢复。 查看 [#1461](https://github.com/TabularEditor/TabularEditor3/issues/1461)。
- 修复了若干与自动完成、自动格式化以及关键字语法高亮相关的 DAX 编辑器小问题。
- 现在可以修改 Query Group 名称（即共享 M 表达式和 M 分区的“显示文件夹”）。 此外，如果将带有 Query Group 的对象粘贴到不包含该 Query Group 的模型中，系统会自动创建该 Query Group，以避免在后续保存或部署模型时出现错误信息。
- 修复了一个问题：将模型元数据保存回服务器后，有时无法撤销之前的更改。

## 从 Tabular Editor 2.x 升级而来？

观看 [此视频](https://youtu.be/O4ATwdzCvWc)，快速了解 Tabular Editor 3 的主要功能。 另外，别忘了查看我们的 [入门指南](https://docs.tabulareditor.com/onboarding/index.html)。

**Tabular Editor 3 主要功能概览：**

- 完全可自定义的 IDE，支持多显示器、Hi-DPI 和主题
- 全新的强大 DAX 代码编辑器，支持自动完成、语法检查、代码折叠等更多功能
- \*工作区模式：允许你将更改保存到磁盘，并同时将模型元数据同步到 Analysis Services
- \*通过无限滚动预览表数据，创建 PivotGrid，或编写 DAX 查询来浏览模型或测试计算逻辑
- \*计划数据刷新任务
- 在 Provider 和 Structured数据源中更新表架构（是的，连 M 查询也支持！）
- 创建 Data model 图表
- 创建 DAX 脚本，让你在一个文档中编辑多个度量值或其他计算对象
- 录制 C# Script 并保存为宏（以前称为“Custom Actions”）
- VertiPaq分析器集成
- DAX调试器
- DAX优化器集成
- 使用 Code Actions 轻松重构 DAX 代码。

\*=仅在连接到 Analysis Services 或 Power BI 实例时可用
