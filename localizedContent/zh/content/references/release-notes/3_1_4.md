## 3.1.4 中的改进

- 将部分工具栏快捷命令添加到了菜单中（例如，现在可以在 **工具 > 管理 BPA 规则...** 下管理 BPA 规则）
- 现在可以在 TOM Explorer 中选中计算列和计算表格并右键，创建对应的 DAX 脚本
- 改进了 Pivot Grid 中“信息”里的错误消息行为
- 在安装程序中新增“启动 Tabular Editor 3”按钮，参见 [#202](https://github.com/TabularEditor/TabularEditor3/issues/202)

## 3.1.4 中的错误修复

- 名称包含方括号的列/度量值，以及名称包含单引号的表格，现在不会再产生误报的语义错误提示，参见 [#239](https://github.com/TabularEditor/TabularEditor3/issues/239)。
- 修复了一个 bug：将某些单词用作变量名时会出现错误的语义错误提示。
- 修复了一个 bug：在 DAX 中定义变量时有时会导致崩溃。
- 修复了一个 bug：在使用带特定参数的 TOPNSKIP 时会出现错误的语义错误提示。
- 当用户点击“取消”按钮时，`SelectObject<T>` 方法（以及封装它的方法）现在会返回 null，参见 [#247](https://github.com/TabularEditor/TabularEditor3/issues/247)。
- 修复了 `ExportProperties` / `ImportProperties` 方法对对象集合不生效的问题，参见 [#245](https://github.com/TabularEditor/TabularEditor3/issues/245)。
- 修复了一个导致无法粘贴计算项的问题，参见 [#237](https://github.com/TabularEditor/TabularEditor3/issues/237)
- 修复了一个会在更新包含名称相近列（例如 “Customer Key” 和 “CustomerKey”）的表的架构时出现的“Duplicate key”错误。

## 从 Tabular Editor 2.x 升级过来？

观看[此视频](https://www.youtube.com/watch?v=pt3DdcjfImY)，了解 Tabular Editor 3 的新功能。

**Tabular Editor 3 主要功能概览：**

- 完全可自定义的 IDE，支持多显示器、高 DPI 和主题
- 全新强大的 DAX 代码编辑器，提供自动补全、语法检查、代码折叠等更多功能
- - 工作区模式，可将更改保存到磁盘，并同时将模型元数据同步到 Analysis Services
- - 通过无限滚动预览表数据，创建 PivotGrid，或编写 DAX 查询来浏览模型或测试计算逻辑
- - 定时刷新数据
- 更新 Provider 和 Structured数据源的表架构（没错，连 M 查询也支持！）
- 创建 Data model 图
- 创建 DAX 脚本，在一个文档中批量编辑多个度量值或其他计算对象
- 录制 C# Script 并保存为宏（原称“自定义操作”）
- 集成 VertiPaq分析器

\*=仅在连接到 Analysis Services 或 Power BI 实例时可用
