## 3.15.0 改进内容

- 我们已将 [AMO/TOM 更新到 19.79.1.1](https://www.nuget.org/packages/Microsoft.AnalysisServices.NetCore.retail.amd64)。 在此更新中，我们修复了一个问题：在调用“应用刷新策略”选项时，某些情况下可能会生成范围重叠的策略分区。
- 新建 Power BI / Fabric 模型的默认 _兼容级别_ 设为 1605。
- Direct Lake 模型初次部署时，我们现在会执行一次自动刷新，这是避免回退到 DirectQuery 的先决条件。 此外，还新增了一个（默认启用的）选项：当保存 Direct Lake 模型且新增了表时，也会自动刷新。 此设置位于 **工具 > 偏好 > 模型部署 > 数据刷新** 中。 此外，还新增了一个（默认启用的）选项：当保存 Direct Lake 模型且新增了表时，也会自动刷新。 此设置位于 **工具 > 偏好 > 模型部署 > 数据刷新** 中。

## 3.15.0 问题修复

- 修复了为“Editor.Dax.FormatLong”和“Editor.Dax.FormatShort”自定义的键盘快捷键无法生效的问题。
- 修复了一个问题：除非 MacroActions.json 文件中的宏定义按宏 ID 排序，否则宏不会被完整加载。详见 [#1151](https://github.com/TabularEditor/TabularEditor3/issues/1151)。
- 如果在“导入表向导”中某个表格预览失败，现在也可以继续预览其他表。
- “导入表向导”中的表格预览现在可以在 Fabric Lakehouse SQL 端点上正常工作。
- 将模型保存为 TMDL 时，默认编码现为不带字节顺序标记（BOM）的 UTF-8，应与 Power BI Desktop 兼容。
- 修复了 **新增功能** 页面上的 URL 工具提示会一直停留在屏幕上的问题。
- 内置的 DAX 格式化器不再将 KPI 引用替换为其基础度量值的引用。
- 修复了“忽略增量刷新分区”和“忽略 Lineage tag”这两项序列化设置未能正确保存的问题。

观看[此视频](https://www.youtube.com/watch?v=pt3DdcjfImY)，快速了解 Tabular Editor 3 的新功能。 另外，也请务必查看我们的[入门指南](https://docs.tabulareditor.com/onboarding/index.html)。 另外，也请务必查看我们的[入门指南](https://docs.tabulareditor.com/onboarding/index.html)。

**Tabular Editor 3 主要功能概览：**

- 完全可自定义的 IDE，支持多显示器、Hi-DPI 和主题切换
- 全新强大的 DAX 代码编辑器，支持自动补全、语法检查、代码折叠等更多功能
- 工作区模式：可将更改保存到磁盘，同时将模型元数据同步到 Analysis Services
- 支持无限滚动的表数据预览；创建 PivotGrid，或编写 DAX 查询来浏览模型并测试计算逻辑
- 计划数据刷新
- 同时更新提供程序 Provider 和 Structured数据源上的表架构（是的，甚至适用于 M 查询！）
- 创建 Data model 图
- 创建 DAX 脚本，使您可以在单个文档中编辑多个度量值或其他计算对象
- 录制 C# Script 并保存为宏（以前称为“自定义操作”）
- 集成 VertiPaq分析器
- DAX调试器
