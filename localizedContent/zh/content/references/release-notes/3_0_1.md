## 从 Tabular Editor 2.x 迁移过来？

观看[此视频](https://www.youtube.com/watch?v=pt3DdcjfImY)，快速了解 Tabular Editor 3 的新功能。

**Tabular Editor 3 主要功能概览：**

- 完全可自定义的 IDE，支持多显示器、Hi-DPI 和主题
- 全新强大的 DAX 代码编辑器，具备自动补全、语法检查、代码折叠等丰富功能
- \*工作区模式：可将更改保存到磁盘，并同时将模型元数据同步到 Analysis Services
- \*支持无限滚动预览表数据，可创建 PivotGrid，或编写 DAX 查询来浏览模型或验证计算逻辑
- \*可在 Provider 数据源和 Structured数据源上更新表架构（是的，甚至包括 M 查询！）
- \*支持计划数据刷新
- 创建 Data model 图表
- 创建 DAX 脚本：在同一文档中批量编辑多个度量值或其他计算对象
- 录制 C# Script，并保存为宏（此前称为“自定义操作”）
- VertiPaq分析器集成

\*=仅在连接到 Analysis Services 或 Power BI 实例时可用

## 已知问题

- 表导入向导未包含在此版本中。 它将在即将发布的 3.0.3 中提供。 作为一种变通方案，只需创建新表并指定分区查询（SQL 或 M）。 然后，右键单击该表，选择“更新表架构...”，以导入列。
- 更多信息请参见 [roadmap](https://github.com/TabularEditor/TabularEditor3/issues/12)。

## 3.0.1 中的新功能：

（下方列表对比了 Tabular Editor 3.0.1 与上一版 Beta 版）

- 更新了 [EULA](https://tabulareditor.com/tabular-editor-software-license-agreement_ver-1-0/)，并将 [ThirdPartyNotices.txt](https://github.com/TabularEditor/TabularEditor3/blob/master/ThirdPartyNotices.txt) 添加到安装程序中。
- Schema Compare 现也适用于 Provider（旧版）数据源和查询分区
- 首次使用“Format DAX”按钮时，Tabular Editor 会询问您是否同意将 DAX 代码发送至 www.daxformatter.com Web 服务（Tabular Editor 3 的后续版本将支持离线 DAX 代码格式化，因此届时无需再征得同意）。
- 改进了激活向导中的部分文本，并在当前许可证过期时允许输入新的许可证密钥。
- 在属性网格中新增连接字符串编辑器
- 在“工具 > 偏好 > 架构比较”中新增选项，可全局忽略某些类型的架构差异。
