## 3.1.3 的错误修复

- 修复了应用在空闲时发生的内存泄漏问题。 该问题自 v. 3.1.1 起一直存在。 该问题自 v. 3.1.1 起一直存在。

## 3.1.2 的错误修复

- 修复了“更新表架构”在 SQL DW / Synapse SQL Pool 数据源上无法使用的问题，参见 [#231](https://github.com/TabularEditor/TabularEditor3/issues/231)。
- 修复了已忽略的规则仍会被 Best Practice Analyzer 计入的问题，参见 [#203](https://github.com/TabularEditor/TabularEditor3/issues/203)。
- 修复了因 JSON 序列化错误导致模型无法保存到磁盘（文件或文件夹）的问题，参见 [#232](https://github.com/TabularEditor/TabularEditor3/issues/232)。
- 修复了在检查 Structured数据源 的属性值时导致崩溃的问题。

## 3.1.1 中的改进

- 共享表达式（Power Query / M 表达式）现在可以序列化为单独的文件。 现在可以将 `PrivacySetting` 属性从序列化中排除，以避免 Microsoft.AnalysisServices.Deployment 相关问题。 该设置可在 **模型 > 序列化选项...** 中配置（参见问题 [#210](https://github.com/TabularEditor/TabularEditor3/issues/210)）。
- 在 **工具 > 偏好 > 建模操作** 下新增选项，用于控制将透视成员关系应用到新对象的方式（参见问题 [#216](https://github.com/TabularEditor/TabularEditor3/issues/216)）。
- 现在可以将 `PrivacySetting` 属性从序列化中排除，以避免 Microsoft.AnalysisServices.Deployment 相关问题。 该设置可在 **模型 > 序列化选项...** 中配置（参见问题 [#210](https://github.com/TabularEditor/TabularEditor3/issues/210)）。
- 你现在可以直接在 C# Script 中定义枚举、结构体和类，更便于组织与复用 C# 代码。
- VertiPaq分析器现在提供从数据中收集统计信息的选项（**工具 > 偏好 > VertiPaq分析器**），可提升统计质量，例如当列的 IsAvailableInMDX = false 时。 参见问题 [#190](https://github.com/TabularEditor/TabularEditor3/issues/290)。 参见问题 [#190](https://github.com/TabularEditor/TabularEditor3/issues/290)。
- 创建新模型后，Tabular Editor 3 现在会在首次保存时提示你：要将模型保存为文件还是文件夹。 你也可以在 **Tools > 偏好 > General > Default save format for new models** 里设置新模型的默认保存格式。 参见问题 [#50](https://github.com/TabularEditor/TabularEditor3/issues/50)。 你也可以在 **Tools > 偏好 > General > Default save format for new models** 里设置新模型的默认保存格式。 参见问题 [#50](https://github.com/TabularEditor/TabularEditor3/issues/50)。
- 改进了启动性能（有用户反馈，在 TabularEditor3.exe 的快捷方式中添加 `-multicorejit` 参数后，启动时间还能进一步缩短）。
- 重构了“模型”菜单，让“新建计算组”等选项更易找到。
- 将 AMO/TOM 客户端库更新至 [19.26.1.7](https://www.nuget.org/packages/Microsoft.AnalysisServices.retail.amd64/)。
- 现在可在 Best Practice Analyzer 中对模型角色成员逐一迭代
- 为避免与 [工作区模式](https://docs.tabulareditor.com/te3/tutorials/workspace-mode.html) 混淆，“窗口”菜单中的“Workspace”已更名为“布局”。
- 新增 Ctrl+Shift+D 作为“部署”的快捷键
- 支持新的 DAX 函数（[`HASH`](https://dax.guide/hash)、[`NAMEOF`](https://dax.guide/nameof)、[`COLUMNSTATISTICS`](https://dax.guide/columnstatistics)、[`BITAND`](https://dax.guide/bitand)、[`BITLSHIFT`](https://dax.guide/bitlshift)、[`BITOR`](https://dax.guide/bitor)、[`BITRSHIFT`](https://dax.guide/bitrshift)、[`BITXOR`](https://dax.guide/bitxor)），并支持在 [`SELECTCOLUMNS`](https://dax.guide/selectcolumns) 中使用可选的列名参数。

## 3.1.1 中的错误修复

- 在删除对象时，TOM Explorer 不再卡死或崩溃（参见问题 [#223](https://github.com/TabularEditor/TabularEditor3/issues/223)、[#209](https://github.com/TabularEditor/TabularEditor3/issues/209)）。
- 修复了编辑器在深色主题下（[#207](https://github.com/TabularEditor/TabularEditor3/issues/207)）以及图标（[#215](https://github.com/TabularEditor/TabularEditor3/issues/215)）的颜色问题
- “创建/编辑关系”对话框中的表现在按字母顺序排序（参见问题 [#214](https://github.com/TabularEditor/TabularEditor3/issues/214)
- 修复了 C# Script 无法保存为宏的问题（参见问题 [#208](https://github.com/TabularEditor/TabularEditor3/issues/208)）。
- 修复了数据源密码/凭据未能正确应用到 Workspace 数据库的问题（参见问题 [#205](https://github.com/TabularEditor/TabularEditor3/issues/205)）。
- “选择所有列”复选框不再导致崩溃（参见问题 [#204](https://github.com/TabularEditor/TabularEditor3/issues/204)）。
- 当脚本引用外部 DLL（使用 `#r` 预编译器指令）时，Tabular Editor 现在始终会在安装目录中查找它们，参见问题 [#192](https://github.com/TabularEditor/TabularEditor3/issues/192)。
- 修复了语言架构导致 Q&A 在某些情况下停止工作的问题
- 在“数据预览”中按日期列进行筛选不再导致崩溃
- 在 TOM Explorer 的透视下拉列表中选择“所有透视”时，未包含在任何透视中的对象将不再显示。 同时修复了下拉框无法更新透视名称的问题。 同时修复了下拉框无法更新透视名称的问题。
- 允许将某些 DAX 关键字用作变量（`WEEK`、`DAY`、`INTEGER` 等）， 且不再显示错误。 且不再显示错误。
- 修复了首次保存模型时未能存储序列化选项的问题。
- 修复了 DAX 解析器的一个问题：在加载模型时可能导致崩溃（参见 [#229](https://github.com/TabularEditor/TabularEditor3/issues/229)）。

## 从 Tabular Editor 2.x 升级而来？

观看[此视频](https://www.youtube.com/watch?v=pt3DdcjfImY)，快速了解 Tabular Editor 3 的新功能。

**Tabular Editor 3 主要功能概览：**

- 完全可自定义的 IDE，支持多显示器、Hi-DPI 和主题切换
- 全新强大的 DAX 代码编辑器，支持自动补全、语法检查、代码折叠等更多功能
- \*工作区模式：可将更改保存到磁盘，同时将模型元数据同步到 Analysis Services
- \*预览支持无限滚动的表数据，创建 PivotGrids，或编写 DAX 查询来浏览模型或测试计算逻辑
- \*计划数据刷新
- 在 Provider 和 Structured数据源中更新表架构（是的，连 M 查询也支持！）
- 创建 Data model 图
- 创建 DAX 脚本，可在一个文档中编辑多个度量值或其他计算对象
- 录制 C# Script 并保存为宏（原名“自定义操作”）
- 集成 VertiPaq分析器

\*=仅当连接到 Analysis Services 或 Power BI 实例时可用
