## 3.16.0 新功能

查看我们的 [发布博客](https://blog.tabulareditor.com/2024/06/13/tabular-editor-3-july-2024-release/)，快速概览本次发布中最重要的更新。

在本次版本中，我们对 **Pivot Grid** 的工作机制进行了大幅改造。 亮点包括： 亮点包括：

- 现在可以将 Pivot Grid 布局保存到文件，并从文件加载！ 现在可以将 Pivot Grid 布局保存到文件，并从文件加载！ 新增文件格式 `.te3pivot`，用于存储 Pivot Grid 布局的 JSON 表示。
- 现在 Pivot Grid 可以使用隐藏对象（度量值、列、层次结构）；同时，对象重命名也不再会导致 Pivot Grid 丢失布局或清除任何筛选器。

如需了解这些更改的更多信息，请参阅 [Pivot Grid 文档](https://docs.tabulareditor.com/features/pivot-grid.html)。

## 3.16.0 中的错误修复

- 按下没有分配任何快捷键绑定的 Ctrl 组合键时，现在不会再把特殊控制字符插入代码编辑器。
- 已修改并保存的宏通过 UI 调用时，现在不会再执行多次。
- 即使之前一次执行中勾选了 **不再显示更多输出** 选项，后续执行宏时，`.Output()` 方法也会继续显示输出对话框。
- 未加引号的表名引用如果恰好也是 DAX 关键字（例如 `Currency`），自动格式化时不会再把它当作关键字处理（也就是不会把它转成大写）。 此外，DAX脚本中的某些对象属性现在也可以为空。 想了解更多信息，可以看看更新后的 [DAX脚本文档](https://docs.tabulareditor.com/features/dax-scripts.html)。 此外，DAX脚本中的某些对象属性现在也可以为空。 想了解更多信息，可以看看更新后的 [DAX脚本文档](https://docs.tabulareditor.com/features/dax-scripts.html)。
- DAX脚本执行期间的错误现在会被更稳妥地处理；在遇到错误之前脚本所做的更改会自动回滚。
- 修复了一个问题：分配到自定义工具栏的宏按钮在重启应用后不会保留。
- 修复了一个问题：在 C# Script 中使用 `ForEach<T>(IEnumerable<T>, Action<T>)` 扩展方法时，可能导致 Tabular Editor 崩溃。
- 修复了移除/重置快捷键在某些命令上不起作用的问题。
- 修复了一个问题：刷新操作完成后，“保存”按钮仍会保持禁用状态。
- 修复了一个问题：某些操作可能导致 **TOM Explorer** 无响应，并在重启应用前一直无法正常工作。
- 修复了一个问题：在 **TOM Explorer** 中删除对象时，选中项会跳到 **Model** 节点，而不是跳到被删除对象最近的父节点。
- 修复了多项与 Pivot Grid 相关的错误。
- 语义分析器在针对 `BLANK()` 进行比较时不应再产生误报错误，例如 `"string value" IN { BLANK() }` 这类表达式。
- 修复了一个问题：当从 TMDL 文件夹加载模型时，更改序列化设置不会被保留。 参见 [#1285](https://github.com/TabularEditor/TabularEditor3/issues/1285)。 参见 [#1285](https://github.com/TabularEditor/TabularEditor3/issues/1285)。
- 在“表格预览”中对列进行排序时，如果该列的 `IsAvailableInMDX = false`（即该列尚未创建任何属性层次结构），我们现在会改用 `TOPN` 查询来预览数据，因为 `TOPNSKIP` 不支持按没有属性层次结构的列进行排序。
