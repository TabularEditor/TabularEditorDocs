## 3.8.0 中的错误修复

- 对于依赖关系图庞大且复杂的模型，依赖关系视图现在性能提升明显。
- 有用户反馈，在**属性视图**中编辑某些属性会导致应用程序崩溃。 现在应该不会再出现这种情况。 现在应该不会再出现这种情况。
- 当 DAX 查询同时包含查询作用域列和查询作用域表时，Semantic Analyzer 现在应该不会再报告错误。
- Semantic Analyzer 现在可以正确处理包含 UTF-32 字符（表情符号）的 DAX 表达式，不过 DAX 编辑器在处理这些字符时仍有问题。 总体而言，我们建议避免在 DAX 表达式中使用表情符号。 总体而言，我们建议避免在 DAX 表达式中使用表情符号。
- 修复了一个问题：反复按 Ctrl+Z（撤销）会清空表达式编辑器，甚至引入不相关的表达式。
- 当模型中新增/移除对象时，透视编辑器现在会正确更新以反映这些变化。
- 当模型包含动态格式字符串时，VertiPaq分析器现在应该不会再崩溃。
- 修复从 Snowflake 导入表时的回归问题，该问题会导致列被分配错误的数据类型。
- 修复在更新架构/从 Snowflake 导入表时的回归问题：连接对话框中必须重新输入隐式数据源属性。
- 修复当表达式编辑器以工具窗口（即非文档窗口）停靠时，与表达式编辑器相关的工具栏按钮会消失的问题。
- 当对模型进行了可能影响 DAX 语义的更改时，DAX 编辑器现在会执行新的语义分析。 这应能确保 DAX 编辑器中显示的错误信息始终有效。 这应能确保 DAX 编辑器中显示的错误信息始终有效。
- 修复即使是看似简单的 DAX 查询，DAX调试器也会不可用的问题。

**Tabular Editor 3 主要功能概览：**

- 完全可自定义的 IDE，支持多显示器、高 DPI 和主题
- 全新强大的 DAX 代码编辑器，具备自动补全、语法检查、代码折叠等众多功能
- \*工作区模式：可将更改保存到磁盘，并同时将模型元数据同步到 Analysis Services
- \*支持无限滚动预览表数据，可创建 PivotGrid，或编写 DAX 查询来浏览模型或测试计算逻辑
- \*支持计划数据刷新
- 在 Provider 数据源和 Structured数据源上更新表架构（没错，连 M 查询也支持！）
- 创建 Data model 关系图
- 创建 DAX 脚本，在单个文档中编辑多个度量值或其他计算对象
- 录制 C# Script 并保存为宏（此前称为“Custom Actions”）
- 集成 VertiPaq分析器
- DAX调试器

\*=仅在连接到 Analysis Services 或 Power BI 的实例时可用
