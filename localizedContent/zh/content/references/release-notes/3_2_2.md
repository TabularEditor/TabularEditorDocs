## 3.2.2 中的改进

- 为分区和 Named Expression 新增对 Query Group（也就是 Power Query“文件夹”）的支持
- 首次创建分区时提示选择分区类型。 参见 [#406](https://github.com/TabularEditor/TabularEditor3/issues/406)
- 调整关系名称的显示对齐方式，使其更接近 DAX Studio 中 VertiPaq分析器的显示方式。
- 现已允许在表达式编辑器中使用调试逗号，参见 [#407](https://github.com/TabularEditor/TabularEditor3/issues/407)
- 现在可以在 TOM Explorer 中双击任意表达式对象（如分区），即可显示快速编辑器
- 对于包含多种分区类型的表，将标签“Mixed”更改为“Hybrid”
- Pivot Grid：
  - 当对仅包含单一值的字段进行切片时（且该字段位于行/列区域），Pivot Grid 现在会显示总计
  - 通过 TOM Explorer 上下文菜单添加到 Pivot Grid 的字段现在会添加到“最右侧”。右键单击时会将焦点定位到 Pivot Grid 单元格
- DAX调试器改进：
  - 现在可以在外层筛选语境中查看并启用/禁用单个筛选器
  - 为监视/局部变量的值类型添加了工具提示。
  - 更准确地从当前调用中识别并排序局部项（包括同一评估语境中的子表达式等）
  - 在调试度量值引用时，不再在评估语境堆栈中显示语境转换（深入到该度量值引用即可看到语境转换）
  - 针对 SSAS 2016 时不再使用 TREATAS
  - 改进了在度量值中使用 F10（单步跳过）的体验
  - 改进了“观察表达式”编辑器的自动补全功能（现在可提示行语境中的列、变量等）
- 已将 TOM 更新到 [19.34.1](https://www.nuget.org/packages/Microsoft.AnalysisServices.retail.amd64/)

## 3.2.2 问题修复

- 新增了更多 TOM 属性，例如：刷新策略模式（用于配置混合表格的刷新策略）
- 修复激活向导卡死问题，参见 [#419](https://github.com/TabularEditor/TabularEditor3/issues/419)
- 确保自定义工具栏在首次还原布局后仍可拖动/停靠并进行自定义。 参见问题 [#413](https://github.com/TabularEditor/TabularEditor3/issues/413)
- 修复通过右键菜单调用时属性面板不显示的问题，参见 [#375](https://github.com/TabularEditor/TabularEditor3/issues/375)
- 防止通过状态栏进入“数据刷新”时发生崩溃，参见问题 [#370](https://github.com/TabularEditor/TabularEditor3/issues/370)
- 修复在后台刷新运行时关闭应用会导致崩溃的问题。
- 防止在关系图中点击时弹出表达式编辑器。 参见问题 [#43](https://github.com/TabularEditor/TabularEditor3/issues/43)。 同时改进了在图表视图中选择和拖动列的行为。
- 为 VertiPaq Analyzer 补充缺失字段，参见 [#372](https://github.com/TabularEditor/TabularEditor3/issues/372)
- 改进了计算组列在 Pivot Grid 中的行为。 参见 [#412](https://github.com/TabularEditor/TabularEditor3/issues/412)
- 确保每次语义分析完成时都更新依赖关系视图（例如通过 Power BI 修改模型时），参见 [#401](https://github.com/TabularEditor/TabularEditor3/issues/401)
- 除非正在编写计算项表达式，否则自动完成中不再建议计算组函数，参见问题 [#387](https://github.com/TabularEditor/TabularEditor3/issues/387)
- 修复取消注释某个选区后，“撤销”不再正常工作的问题。 参见 [#301](https://github.com/TabularEditor/TabularEditor3/issues/301)
- 新增注释（Ctrl+Shift+K）/取消注释（Ctrl+Shift+U）快捷键。 参见 [#371](https://github.com/TabularEditor/TabularEditor3/issues/371)
- 新增在文本编辑器中将所选行上移/下移的快捷键（Alt+上/下方向键）。 参见 [#371](https://github.com/TabularEditor/TabularEditor3/issues/371)
- 修复在以特定方式拖动窗口时触发 StackOverflowException 的问题。 参见 [#354](https://github.com/TabularEditor/TabularEditor3/issues/354)
- 修复一个问题：关闭 C# Script 视图时宏录制器不会停止，导致在更改模型对象或属性时发生后续崩溃。
- 宏录制器现在也会为计算项表达式更改，以及层级级别和计算项序号更改生成代码。 参见 [#230](https://github.com/TabularEditor/TabularEditor3/issues/230)
- 调试器：修复在外部筛选语境中为变量生成 DAX 的问题，参见议题 [#418](https://github.com/TabularEditor/TabularEditor3/issues/418)
- 在模型加载时检查并删除表引用无效的 TablePermissions（此前，TE 将不允许加载包含此类无效引用的模型）
- 当字段从 Pivot Grid 中移除或隐藏时清除筛选，参见问题 [#427](https://github.com/TabularEditor/TabularEditor3/issues/427)

## 从 Tabular Editor 2.x 迁移过来？

观看 [此视频](https://www.youtube.com/watch?v=pt3DdcjfImY)，快速了解 Tabular Editor 3 的新功能。 另外，别忘了查看我们的 [上手指南](https://docs.tabulareditor.com/onboarding/index.html)。

**Tabular Editor 3 主要功能概览：**

- 完全可自定义的 IDE，支持多显示器、Hi-DPI 和主题
- 全新强大的 DAX 代码编辑器，支持自动完成、语法检查、代码折叠等更多功能
- \*工作区模式，使你可以将更改保存到磁盘，并同时将模型元数据同步到 Analysis Services
- \*以无限滚动方式预览表数据，创建 PivotGrids 或编写 DAX 查询，以浏览模型或测试计算逻辑
- \*计划数据刷新
- 在 Provider 和 Structured数据源上更新表架构（没错，连 M 查询也支持！）
- 创建 Data model 示意图
- 创建 DAX 脚本，让你在单个文档中编辑多个度量值或其他计算对象
- 录制 C# Script 并保存为宏（以前称为“Custom Actions”）
- VertiPaq分析器集成
- \*DAX调试器

\*=仅在连接到 Analysis Services 或 Power BI 实例时可用
