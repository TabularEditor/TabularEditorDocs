## 3.3.6 中的改进

- 将 TOM 更新到 [19.52.2.2](https://www.nuget.org/packages/Microsoft.AnalysisServices.NetCore.retail.amd64)。
- 新增对度量值“格式字符串表达式”的支持（兼容级别 1601 或更高）。
- 现在可以通过主表达式编辑器编辑表的 `SourceExpression` 和 `PollingExpression` M 查询。
- 在表达式属性（例如 `SourceExpression`、`PollingExpression`、`FormatStringExpression` 等）上右键单击 在属性网格中会提供一个快捷方式，可在主表达式编辑器中编辑该值。 在属性网格中会提供一个快捷方式，可在主表达式编辑器中编辑该值。
- 数据列的 `SourceColumn` 属性值现在会显示在 TOM Explorer 的“Expression”列中
- 为 TOM Explorer 和 VertiPaq分析器 的列标题新增工具提示。
- 改进 M 解析器，并在执行离线架构更新时支持更多 M 函数，包括对 CommandTimeout 及其他连接选项的支持。
- 当后台数据刷新操作完成时，Tabular Editor 现在会显示一条 Toast 弹出式通知。 可在 **Tools > 偏好 > Features > Notifications** 下禁用此功能。
- 新增对 DAX 新函数的支持（[OFFSET](https://dax.guide/offset)、[TOJSON](https://dax.guide/tojson)、[TOCSV](https://dax.guide/tocsv) 等）。
- 宏现在会在后台线程中编译，从而提升应用启动速度，同时也修复了 [#708](https://github.com/TabularEditor/TabularEditor3/issues/708)。
- 更新为使用最新版本的 [VertiPaq分析器](https://github.com/sql-bi/VertiPaq-Analyzer)。
- 将兼容级别 1600 设为 SQL Server 2022 模型的新默认值。

## 3.3.6 中的错误修复

- DAX调试器现在会在调试时正确保留原始查询表达式，从而保留原始的影子筛选语境，以及 auto-exist 对表筛选器所做的任何修改。 除此之外，这也意味着包含 `ALLSELECTED` 的表达式现在可以正确调试，参见 [#669](https://github.com/TabularEditor/TabularEditor3/issues/669)。 除此之外，这也意味着包含 `ALLSELECTED` 的表达式现在可以正确调试，参见 [#669](https://github.com/TabularEditor/TabularEditor3/issues/669)。
- 改进了表导入向导 / 架构更新期间的异步连接。 改进了表导入向导 / 架构更新期间的异步连接。 同时还改进了错误处理——例如当 SQL 连接因缺少证书而失败时。
- 修复了在宏视图中“删除”按钮无法使用的问题。
- 修复了当字段名包含空格，如 `[Order Date]`时，M 解析器不支持记录的问题。 这样一来，只要某个 M 表达式包含这类记录引用，“架构更新”功能就能在离线模式下正常工作。 这样一来，只要某个 M 表达式包含这类记录引用，“架构更新”功能就能在离线模式下正常工作。

- 修复了一个错误：现在可以搜索所有表达式属性了，详见 [#663](https://github.com/TabularEditor/TabularEditor3/issues/663)。
- 修复了关闭模型时调试器脚本未关闭的问题。
- 不再允许向计算表格添加分区。
- 修复了使用查找/替换对话框时应用有时会崩溃的问题。
- 修复了“新建 DAX脚本”按钮的工具提示错误，详见 [#678](https://github.com/TabularEditor/TabularEditor3/issues/678)。
- 修复了关闭浮动文档时不会提示保存的问题，详见 [#673](https://github.com/TabularEditor/TabularEditor3/issues/673)。
- 修复了在“数据预览”中切换列筛选器的 (All) 或 (Blank) 值时，应用有时会崩溃的问题。
- 修复了在使用“架构更新”或“表导入向导”时 Snowflake 数据类型推断的问题（NUMBER 列会被导入为 Double，而它们应被导入为 Int64 或 Decimal）。
- 当在 [SUMMARIZE](https://dax.guide/summarize) 中使用扩展列时，语义分析器不应再报告错误。
- 当未加引号将“ID”用作表引用时，语义分析器现在会报告错误。

**Tabular Editor 3 主要功能概览：**

- 完全可自定义的 IDE，支持多显示器、Hi-DPI 和主题
- 全新强大的 DAX 代码编辑器，支持自动完成、语法检查、代码折叠等更多功能
- \*工作区模式，可将更改保存到磁盘，同时将模型元数据同步到 Analysis Services
- \*通过无限滚动预览表数据，创建 PivotGrid，或编写 DAX 查询来浏览模型或测试计算逻辑

- 更新 Provider 数据源和 Structured数据源的表架构（没错，连 M 查询也支持！）
- 创建 Data model 图表
- 创建 DAX脚本，以便在单个文档中编辑多个度量值或其他计算对象
- 录制 C# Script 并将其保存为宏（原称“自定义操作”）
- VertiPaq分析器集成
- \*DAX调试器
