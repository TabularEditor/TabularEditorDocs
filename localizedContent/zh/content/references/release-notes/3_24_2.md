## 3.24 新增内容

查看我们的 [发布博客](https://tabulareditor.com/blog/tabular-editor-3-october-2025-release)，快速了解本次发布中最重要的更新。

也请参阅 [3.24.0 版本的发行说明](xref:release-3-24-0)，查看上一主要版本中新增的所有功能和改进。

## 3.24.2 的 Bug 修复

- 修复了一个 Bug：在刷新操作期间，如果连接的 AS 实例报告了多个错误/警告，可能会导致未处理的异常。

## 3.24.1 中的改进

- Semantic Analyzer 新增对数据类型关键字 `LOGICAL`、`INT64`、`DECIMAL` 和 `TEXT` 的支持（它们都是现有关键字的别名），可作为 [`CONVERT`](https://dax.guide/convert) 和 [`DATATABLE`](https://dax.guide/datatable) 等函数的参数使用。
- Semantic Analyzer 新增对函数 [`ISINTEGER`](https://dax.guide/isinteger)、[`ISCURRENCY`](https://dax.guide/iscurrency) 以及 Visual 计算函数 [`LOOKUPWITHTOTALS`](https://dax.guide/lookupwithtotals) 的支持。
- 在 TOM Explorer 中右键单击 **Model** 或 **Functions** 节点时，新增“管理 DAX 组件...”菜单选项。

## 3.24.1 版本 Bug 修复

- 修复了在低分辨率显示器上启动 Tabular Editor 3 时会触发未处理异常的问题
- 修复了一个 bug：使用 **文件 > 打开 > 从文件打开模型...** 菜单选项时，无法从 PBIP 文件夹结构打开模型的问题，参见 [#1507](https://github.com/TabularEditor/TabularEditor3/issues/1507)。
- 修复了与 DAX 组件管理器视图相关的 DPI 缩放问题。
- 修复了语义分析器的 bug：未能识别 Visual 计算函数 [`FIRST`](https://dax.guide/first) 和 [`LAST`](https://dax.guide/last)。
- 修复了语义分析器的 bug：将 UDF 作为 [`CALCULATE`](https://dax.guide/calculate) 的第一个参数调用时，会误报警告，参见 [#1509](https://github.com/TabularEditor/TabularEditor3/issues/1509)。
- DAX 组件管理器不再因为 DAX 表达式中的行结束符差异而显示警告（PBI Desktop 在保存模型时可能会将 `\n` 替换为 `\r\n`）。
- 修复 TOM Explorer 中各命名空间下 UDF 的排序问题

## 从 Tabular Editor 2.x 迁移过来？

观看 [此视频](https://youtu.be/O4ATwdzCvWc)，快速了解 Tabular Editor 3 的主要功能。 另外，也别忘了查看我们的 [入门指南](https://docs.tabulareditor.com/onboarding/index.html)。 另外，也别忘了查看我们的 [入门指南](https://docs.tabulareditor.com/onboarding/index.html)。

**Tabular Editor 3 主要功能概览：**

- 完全可自定义的 IDE，支持多显示器、Hi-DPI 以及主题
- 全新强大的 DAX 代码编辑器，提供自动完成、语法检查、代码折叠等更多功能
- 工作区模式，让你在把更改保存到磁盘的同时，将模型元数据同步到 Analysis Services
- 通过无限滚动预览表数据，创建 PivotGrid，或编写 DAX 查询来浏览模型或测试计算逻辑
- - 设置数据刷新计划
- 在提供程序和 Structured数据源上更新表架构（没错，连 M 查询也支持！）
- 创建 Data model 图
- 创建 DAX 脚本，让你在一个文档中编辑多个度量值或其他计算对象
- 录制 C# Script 并保存为宏（以前称为“自定义操作”）
- VertiPaq分析器集成
- DAX调试器
- DAX优化器集成
- 通过代码操作轻松重构你的 DAX。

\*=仅在连接到 Analysis Services 或 Power BI 实例时适用
