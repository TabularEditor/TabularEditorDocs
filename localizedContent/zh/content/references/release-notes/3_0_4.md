## 已知问题

- 由于需要先修复一些高优先级的 Bug，“表导入向导”未能包含在此版本中。 会尽快提供！ 作为临时解决方案，只需新建一个表，并指定分区查询（SQL 或 M）。 然后右键单击该表，选择“更新表架构...”，以导入列。
- 更多信息请参阅[路线图](https://github.com/TabularEditor/TabularEditor3/issues/12)。

## 3.0.4 中的 Bug 修复：

- 刷新 Best Practice Analyzer 不再导致崩溃

## 3.0.3 中的新功能：

- 在处理视图中右键单击某个项，现在可以将该项重新加入队列；或者，如果该项代表单个表，则会打开该表的数据预览。

## 3.0.3 中的 Bug 修复：

- 当语义分析器在计算列/计算表格中检测到错误时，将不再修改该表中的列（此前在某些情况下会导致关系被删除）
- 修复了“查找/替换”对话框中某些控件启用状态的问题
- 基于遥测数据和错误 Report 的多项 Bug 修复，例如：
  - 现已可正常复制/粘贴带有 Lineage tag 的列
  - 当表被重命名时，将表粘贴覆盖到现有表不再导致崩溃
  - “还原”按钮现在会在刷新本地 TOM 之前重新连接到 AS，不再导致崩溃
  - 当属性网格为空时，在其中右键单击不再导致崩溃
  - 以及许许多多其他内容……
- 修复“Server”下拉列表的 MRU 未能正确保存的问题
- 修复了工作区模式无法从“用户选项”文件正确加载数据源覆盖值等多个问题。 查看 [#28](https://github.com/TabularEditor/TabularEditor3/issues/28)。
- 修复并改进了多处“幽灵” DAX 语义错误（支持 ROLLUPADDISSUBTOTAL、带 0 个参数的 COUNTROWS 等）
- 改进了多个 DAX 函数的自动完成/参数提示（TOPN、TOPNPERLEVEL、DATEADD/DATESINPERIOD/PARALLELPERIOD 的 interval 参数等），以及在 RETURN 关键字之后等场景。

## 从 Tabular Editor 2.x 升级过来？

观看 [此视频](https://www.youtube.com/watch?v=pt3DdcjfImY)，快速了解 Tabular Editor 3 的新功能。

**Tabular Editor 3 主要功能概览：**

- 完全可自定义的 IDE，支持多显示器、Hi-DPI 和主题
- 全新的强大 DAX 代码编辑器，具备自动完成、语法检查、代码折叠等更多功能
- \*工作区模式，可在将更改保存到磁盘的同时，把模型元数据同步到 Analysis Services
- \*支持无限滚动预览表数据、创建 PivotGrids，或编写 DAX 查询来浏览模型或测试计算逻辑
- \*在 Provider 和 Structured数据源上更新表架构（是的，甚至支持 M 查询！）
- \*计划数据刷新
- 创建 Data model 图
- 创建 DAX 脚本，让你在单个文档中编辑多个度量值或其他计算对象
- 录制 C# Script 并保存为宏（以前称为“Custom Actions”）
- VertiPaq分析器集成

\*=仅在连接到 Analysis Services 或 Power BI 实例时可用
