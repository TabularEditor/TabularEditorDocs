### 先决条件

Tabular Editor 3.3.0（及更高版本）使用 .NET 6。 在启动 Tabular Editor 3.3.0（或更高版本）之前，你可能需要先安装 [.NET 6.0 桌面运行时](https://dotnet.microsoft.com/en-us/download/dotnet/6.0/runtime)。 从 3.3.1 开始，如果缺少这些先决条件，Tabular Editor 3 安装程序会自动下载并安装。

## 3.9.0 新增功能

查看我们的[发布博客](https://blog.tabulareditor.com/2023/07/21/tabular-editor-3-july-2023-release/)，快速了解本次发布的重点更新。

- Azure for Government 云上的客户现在可以指定 Tabular Editor 3 在连接 Power BI REST API 时应使用的基础 URL，例如在从 Power BI Dataflow 导入表时。 这个设置在 **工具 > 偏好 > Power BI > Power BI 服务** 里。
- 在 DAX 查询结果或 Pivot Grid 中选择多个单元格时，我们现在会在应用程序状态栏中显示所选值的汇总信息（总和、最小值、最大值、平均值等）。 按住 Shift/Ctrl，即可像平常一样选择范围。 Shift+单击列标题会选中该列中的所有单元格。 见 [#475](https://github.com/TabularEditor/TabularEditor3/issues/475)。

## 3.9.0 改进

- 我们重新调整了“偏好”对话框中的部分页面和区段，使导航更便捷。
- 我们现在使用最新版本的 TOM/AMO（[19.65.7.2](https://www.nuget.org/packages/Microsoft.AnalysisServices.NetCore.retail.amd64)）以及 [TMDL preview-3](https://www.nuget.org/packages/Microsoft.AnalysisServices.Tabular.Tmdl.NetCore.retail.amd64/19.65.7.2-TmdlPreview)。
- 我们还将 AdomdClient 更新到了 [19.65.7.2](https://www.nuget.org/packages/Microsoft.AnalysisServices.AdomdClient.NetCore.retail.amd64/)，这将提升 Pivot Grid 组件的性能。
- VertiPaq分析器已更新至版本 [1.2.18](https://www.nuget.org/packages/Dax.Metadata)。
- 当计算表格的表达式留空时，我们现在会立即在 TOM Explorer 中显示错误图标。
- 现在，即使 Pivot Grid 或关系图未处于焦点（且此类文档最多只打开一个），也可以向其中添加对象。 此外，当你只是想验证某个度量值时，现在可以右键单击该度量值直接创建 Pivot Grid，减少几次点击。
- 如果尝试将会生成 'Variant' 类型值的表达式赋给计算列（这不被允许），我们现在会显示错误信息。
- 我们新增了一个单独选项，可在 **Tools > Preferences > Model Deployment** 下保存压缩的 model.bim 备份。 你现在可以选择在每次部署时保存备份、在每次保存时保存备份（连接到 Analysis Services 实例时），或同时启用两者。 注意，此功能在 TE3 桌面版中不可用。 这也修复了问题 [#796](https://github.com/TabularEditor/TabularEditor3/issues/796)。
- 我们为 VertiPaq分析器 增加了更多信息列，以与 [DAX Studio](https://daxstudio.org) 中提供的列保持一致。

> [!NOTE]
> TMDL 仍处于预览阶段，因此该功能也应视为 Tabular Editor 3 的预览功能。 记得保留一份模型元数据的 Model.bim / Database.json 备份，避免丢失工作成果。

## 3.9.0 中的错误修复

- 修复在 DAX编辑器以及 TOM Explorer / 信息视图中遇到的“循环依赖”误报。 见问题 [#920](https://tabulareditor.zendesk.com/agent/tickets/920) 和 [#931](https://github.com/TabularEditor/TabularEditor3/issues/931)。
- 修复了一个问题：导致“查找和替换”对话框无法在浮动（未停靠）的文档窗口中搜索。
- 多项稳定性改进（除其他修复外，还应能解决 [#937](https://github.com/TabularEditor/TabularEditor3/issues/937)）。
- 修复了在尝试格式化包含窗口函数的 DAX 代码时发生的崩溃。
- 修复了一个问题：在 **工具 > 偏好设置 > 代理设置** 下将代理类型选择为“无”时，Tabular Editor 仍会使用系统代理，反之亦然。
- 语义分析器现在能正确处理 [`DETAILROWS`](https://dax.guide/detailrows) 函数，见 [#938](https://github.com/TabularEditor/TabularEditor3/issues/938)。
- 修复了一个问题：更改宏名称后，宏的文件夹结构不会更新，必须重启应用程序才会生效。
- 修复了一个问题：连接到 Power BI Desktop 时，某些列和层级属性会变为只读，导致用户无法正确设置计算表格、关系等。 见 [#929](https://github.com/TabularEditor/TabularEditor3/issues/929) 和 [#930](https://github.com/TabularEditor/TabularEditor3/issues/930)。
- 修复了 DAX编辑器在使用带有某些参数的 [`WINDOW`](https://dax.guide/window) 函数时发生的崩溃。
- 修复了 M 解析器未能正确解释 `[]` 的问题，该问题会导致离线架构比较器无法从 `PowerPlatform.Dataflows([])` 这类 M 表达式中推断元数据。 见 [#936](https://github.com/TabularEditor/TabularEditor3/issues/936)。
- 修复了一个问题：如果为某个度量值指定了格式错误的格式字符串，PivotGrid 会崩溃。
- 修复了一个问题：当使用 [`SWITCH`](https://dax.guide/switch) 且一个或多个分支返回 [`BLANK`](https://dax.guide/blank) 时，语义分析器会将结果表达式的数据类型错误地指定为“Variant”。
- 修复了几个与表格预览中的列筛选相关的问题。 在获取某一列的唯一值列表时，我们现在会读取前 5000 个值（按升序）。 此前只会读取 500 个值，且未指定排序方式，导致行为不一致。 见 [#924](https://github.com/TabularEditor/TabularEditor3/issues/924)。

## 从 Tabular Editor 2.x 升级过来？

看[这个视频](https://www.youtube.com/watch?v=pt3DdcjfImY)，快速了解 Tabular Editor 3 的新功能。 另外，别忘了查看我们的[入门指南](https://docs.tabulareditor.com/onboarding/index.html)。

**Tabular Editor 3 主要功能概览：**

- 完全可自定义的 IDE，支持多显示器、Hi-DPI 和主题
- 全新强大的 DAX 代码编辑器，提供自动完成、语法检查、代码折叠等更多功能
- \*工作区模式，可将更改保存到磁盘，并同时将模型元数据同步到 Analysis Services
- \*通过无限滚动预览表数据，创建 PivotGrids，或编写 DAX 查询来浏览模型或测试计算逻辑
- \*计划数据刷新
- 更新 Provider 和 Structured数据源的表架构（没错，连 M 查询也支持！）
- 创建 Data model 图
- 创建 DAX 脚本，让你在一个文档中编辑多个度量值或其他计算对象
- 录制 C# Script 并将其保存为宏（以前称为“自定义操作”）
- 集成 VertiPaq分析器
- DAX调试器

\*=仅在连接到 Analysis Services 或 Power BI 实例时可用
