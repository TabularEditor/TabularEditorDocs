## 3.21.0 新增功能

请查看我们的[发布博客](https://tabulareditor.com/blog/tabular-editor-3-april-2025-release)，快速了解本次发布中最重要的更新。

## 3.21.0 改进内容

- 在此版本中，我们对 **图表视图** 进行了更多 Visual 微调与改进：
  - 图标以 Visual 方式指示列的数据类型。
  - 关系启用双向交叉筛选时显示双向箭头
  - 表格右上角的 chevron 按钮可让你快速在以下列显示方式之间切换：显示所有列、仅显示关键列，或不显示任何列。
  - 多项字体大小调整及其他细微优化
- 已将 AMO/TOM 更新到最新版本 [19.94.1.1](https://www.nuget.org/packages/Microsoft.AnalysisServices/19.94.1.1)。
- 我们在 `Model` 对象上新增了一个属性：`MetadataSource`，其中包含模型元数据从何处加载的信息。 这对 C# Script 很有用，例如脚本需要遍历模型元数据加载位置所在的文件夹结构中的文件时。 更多信息请参阅 [API 文档](https://docs.tabulareditor.com/api/TabularEditor.TOMWrapper.Model.html#TabularEditor_TOMWrapper_Model_MetadataSource)。
- 通过我们的 **DAX优化器集成**，DAX优化器用户现在也可以查看 RLS 和计算项表达式的结果。
- 你现在可以通过 **TOM Explorer** 将单个对象的脚本导出为 [TMDL (Tabular Model Definition Language)](https://learn.microsoft.com/en-us/analysis-services/tmdl/tmdl-overview?view=asallproducts-allversions)。 当你想把 TMDL 脚本从 Tabular Editor 复制到 Power BI Desktop 时，这会很有帮助。 右键单击对象，然后选择 **导出脚本 > TMDL**。 你可以选择将脚本导出到文件，或直接导出到剪贴板。

## 3.21.0 中的错误修复

- 在 **数据刷新** 视图中按“进度”或“持续时间”排序现在可以正常工作。 此外，对于某些对象，“持续时间”在刷新操作结束时不再被重置。
- 与 **数据刷新** 视图中进度事件的显示相关的多项错误修复。 例如，即使之前的会话已过期（比如 Tabular Editor 空闲了 60 分钟或更久），现在也应该能收到进度事件。
- 当分区部署被禁用，或虽已启用但跳过增量刷新的分区时，我们现在也会保持表上的刷新策略设置不变（即 **源表达式**、**轮询表达式** 等）
- 当表仅通过非活动关系相连，并使用 [`USERELATIONSHIP`](https://dax.guide/userelationship/) 函数激活关系时，语义分析器现在不会再 Report 错误。
- 修复了语义分析器在使用 [`GENERATESERIES`](https://dax.guide/generateseries/) 函数时，会判断出错误的结果数据类型的问题。
- 修复了语义分析器在 DAX 脚本/查询中通过 [`DEFINE`](https://dax.guide/DEFINE) 定义并引用表和列时，误 Report “未找到”错误的问题。
- 现在无法再通过 **TOM Explorer** 为对象指定空名称。
- 某些此前会被当作保留关键字处理的词，在作为未加引号的表/变量引用时，将不会再产生语义错误。
- 修复了在使用 IF.EAGER 或仅包含 2 个参数的 IF 时，DAX调试器未能正确将非活动分支置灰的问题。
- 当使用 Workspace 连接打开模型时（即。 Tabular Editor 3 的工作区模式功能），如果当初建立与 Workspace 数据库的连接时使用的身份验证方式是 Microsoft Entra MFA，我们现在会正确显示 Microsoft Entra 的 MFA 提示。
- 我们的代码编辑器现已全面支持宽字符 / Unicode 表情符号 (注意：这并不是鼓励你在 DAX 中使用表情符号，但现在即使用了也不会破坏编辑器 :))。
- 如果 .tmuo 文件包含无效的 JSON，Tabular Editor 现在会显示一条错误信息，并继续加载模型，但不会应用该文件中指定的选项，而不是导致未处理的异常。
- 修复了一个问题：当从磁盘加载模型时，如果该表在加载的元数据中未指定任何分区，则其刷新策略选项不会显示在 **Properties** 网格中。
- 由于与其他编辑器快捷键冲突，`Ctrl+U`（取消注释代码）和 `Ctrl+/`（切换注释）快捷键无法正常工作。 我们已从发生冲突的命令中移除了默认快捷键。 如果你需要为 _Editor.Lowercase_ 和 _Editor.Uppercase_ 命令（之前的快捷键分别为 `Ctrl+U` 和 `Ctrl+Shift+U`）设置键盘绑定，可通过 **Tools > 偏好 > Keyboard** 进行配置。

## 从 Tabular Editor 2.x 迁移过来？

观看 [这个视频](https://youtu.be/O4ATwdzCvWc)，快速了解 Tabular Editor 3 的主要功能。 另外，也别忘了查看我们的 [上手指南](https://docs.tabulareditor.com/onboarding/index.html)。

**Tabular Editor 3 主要功能概览：**

- 完全可自定义的 IDE，支持多显示器、Hi-DPI 和主题
- 全新的强大 DAX 代码编辑器，提供自动完成、语法检查、代码折叠等更多功能
- \*工作区模式，让你可以将更改保存到磁盘，并同时将模型元数据同步到 Analysis Services
- \*通过无限滚动预览表数据，创建 PivotGrids，或编写 DAX 查询来浏览模型或测试计算逻辑
- \*计划数据刷新
- 在 Provider 数据源和 Structured数据源上更新表架构（没错，连 M 查询也支持！）
- 创建 Data model 关系图
- 创建 DAX 脚本，让你在一个文档中编辑多个度量值或其他计算对象
- 录制 C# Script 并保存为宏（以前称为“Custom Actions”）
- VertiPaq分析器集成
- DAX调试器
- DAX优化器集成
- 通过 Code Actions 轻松重构 DAX。

\*=仅在连接到 Analysis Services 或 Power BI 的实例时才可用
