## 3.25.0 新增功能

查看我们的[发布博客](https://tabulareditor.com/blog/tabular-editor-3-january-2026-release)，快速了解本次发布中最重要的更新。

- Tabular Editor 现在支持 [**中文和西班牙语（预览）**](xref:references-application-language)。 所有翻译最初由 AI 完成，并经过人工校对。 日语、德语和法语也作为实验性语言提供，但尚未审校。 所有翻译最初由 AI 完成，并经过人工校对。 日语、德语和法语也作为实验性语言提供，但尚未审校。
- Tabular Editor 现包含一组 [**内置 Best Practice Analyzer (BPA) 规则**](xref:built-in-bpa-rules)，涵盖格式设置、元数据、模型布局、DAX 表达式和翻译等方面的关键最佳实践。
- Tabular Editor 现包含 **Semantic Bridge** 功能，让你可以从 Databricks [Metric Views](https://docs.databricks.com/aws/en/metric-views/) 轻松创建语义模型（_仅限 Tabular Editor 3 企业版_）。
- 新增一个选项，可在保存语义模型时同时生成 [**用于 Fabric 的支持文件**](xref:save-with-supporting-files)。 新增一个选项，可在保存语义模型时同时生成 [**用于 Fabric 的支持文件**](xref:save-with-supporting-files)。 启用后，Tabular Editor 会在模型元数据旁创建并更新 .platform 和 definition.pbism 文件，以匹配 Fabric repository 结构。
- 此版本还新增了 **日历编辑器**，让你可以更轻松地管理用于日历型时间智能的日历对象。
- 最后同样重要的是，我们新增了一个 [**高级刷新**](xref:advanced-refresh) 对话框，你可以在其中配置并行度、增量刷新的生效日期等高级刷新选项。 你还可以指定 [刷新覆盖配置文件](xref:refresh-overrides)（_仅 Tabular Editor 3 商业版和企业版_）。 你还可以指定 [刷新覆盖配置文件](xref:refresh-overrides)（_仅 Tabular Editor 3 商业版和企业版_）。

## 3.25.0 的改进

- 现在支持在“模拟身份”对话框中直接设置连接字符串的 CustomData 属性。
- DAX 组件管理器视图现在会下载并显示 DaxLib 组件图标。
- DAX 组件管理器现在会缓存组件元数据，因此即使没有网络连接也能显示组件信息。
- 我们已将 AS 客户端库更新到最新版本的 Microsoft.AnalysisServices 组件 [19.108.5](https://www.nuget.org/packages/Microsoft.AnalysisServices/19.108.5)。 我们已将 AS 客户端库更新到最新版本的 Microsoft.AnalysisServices 组件 [19.108.5](https://www.nuget.org/packages/Microsoft.AnalysisServices/19.108.5)。 此更新让你可以控制 Database 对象上的新属性 [UnicodeCharacterBehavior](https://learn.microsoft.com/en-us/dotnet/api/microsoft.analysisservices.tabular.database.unicodecharacterbehavior?view=analysisservices-dotnet#microsoft-analysisservices-tabular-database-unicodecharacterbehavior)。
- 新增 Power BI 专用的列数据类别：经度、纬度和地点。
- 如果你尝试将模型的兼容级别设置为服务器不支持的值，我们现在会显示错误信息。 这样可以避免后续在保存对模型的更改时出现意外错误。 如果你尝试将模型的兼容级别设置为服务器不支持的值，我们现在会显示错误信息。 这样可以避免后续在保存对模型的更改时出现意外错误。 如果你出于迁移目的需要设置不受支持的兼容级别，请将模型元数据保存到磁盘（.bim 或 TMDL 格式），然后在不附加任何 Workspace 数据库的情况下打开该元数据。
- 当你直接从服务器 / XMLA 端点打开模型时，现在也会保存用户选项（.tmuo）文件。 这些文件存储在你的 %LocalAppData%\TabularEditor3\UserOptions 文件夹中。 这些文件存储在你的 %LocalAppData%\TabularEditor3\UserOptions 文件夹中。
- 在数据刷新上下文菜单中新增“完全刷新（忽略刷新策略）”选项，这样当表受增量刷新的刷新策略管控时，你就可以轻松将数据刷新到现有分区中，而不会在服务器上触发分区的添加/删除/合并操作。
- 重新整理了数据刷新上下文菜单中的选项，将最常用的选项置顶。
- 在刷新窗格中新增“开始时间”，从而可以按刷新开始时间排序，例如将最新的刷新显示在最前面。
- 在“偏好设置”对话框中新增搜索窗格，可快速搜索各项偏好设置。
- 在最佳做法规则编辑器列表中新增“用户定义函数”和“日历”。
- 为列新增 `UsedInCalendars` 属性，让你可以在 C# Script 中轻松检查某列是否被任何日历对象使用。
- “文件 > 关闭”菜单命令已重命名为“关闭文档”，并且现在可以关闭主文档区域中的任何面板或文档。 **Ctrl + W** 被设为默认快捷键。 **Ctrl + W** 被设为默认快捷键。
- 语义分析器新增支持：[`DATEADD`](https://dax.guide/DATEADD) <extension> 参数中的新关键字 `ENDALIGNED`。 [更多信息](https://learn.microsoft.com/en-us/dax/dateadd-function-dax#extension-parameter-for-small--large-period-shifts)。 [更多信息](https://learn.microsoft.com/en-us/dax/dateadd-function-dax#extension-parameter-for-small--large-period-shifts)。
- 当以 model.bim 或 TMDL 文件夹结构保存，并启用“忽略增量刷新分区”序列化选项时，我们现在会添加一个虚拟分区来替代 PolicyRange 分区。 当以 model.bim 或 TMDL 文件夹结构保存，并启用“忽略增量刷新分区”序列化选项时，我们现在会添加一个虚拟分区来替代 PolicyRange 分区。 这样可以确保模型仍可在 Power BI Desktop 中加载，因为不会保存任何不含分区的表。

## 3.25.0 中的 Bug 修复

- 修复了一个关键问题：这个问题会导致部分用户在使用 Azure AD 身份验证时无法连接到 Power BI REST API 或 Databricks。
- 修复了在删除同时被日历对象使用的列时会发生的未处理异常。
- 修复了“关系编辑器”中的列下拉列表，使其会自动显示完整的列名。
- 当复制包含一个或多个日历的表时，现在会自动重命名日历，以确保在整个模型中名称唯一。
- 在数据刷新过程中遇到多个错误/警告时，将不再触发未处理的异常。
- 修复了执行“全部更新”后，DAX 组件列表仍显示旧信息的问题。
- 修复了实体分区的 `SchemaName` 属性无法撤销的问题。
- 当发生 [表扩展](https://www.sqlbi.com/articles/expanded-tables-in-dax/) 时，不再建议使用代码操作 [DI005 - 将表筛选重写为标量谓词](https://docs.tabulareditor.com/kb/DI005.html)，因为将此类筛选重写为列筛选不一定会产生相同的结果。
- 当连接字符串指定了当前系统未安装的 OLE DB 提供程序时，点击“连接字符串”属性旁的省略号“…”按钮将不再导致未处理的异常。
- 当启用相关偏好时，TMDL 导出现在会正确排除增量刷新的分区，与 JSON 行为一致。
- 修复了在尝试从 Dataflows 更新或导入时点击“登录...”按钮会显示以下错误信息的问题：`此页面通常不会显示，可能表明存在网络钓鱼尝试。`
