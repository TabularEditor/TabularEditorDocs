## 已知问题

- 由于需要先修复一些高优先级 Bug，“表导入向导”未能包含在此版本中。 我们会尽快提供！ 作为替代方案，你可以直接新建一个表，然后指定分区查询（SQL 或 M）。 然后右键单击该表，选择 "Update Table Schema..." 以导入列。
- 更多信息见[路线图](https://github.com/TabularEditor/TabularEditor3/issues/12)。

## 3.0.9 中的错误修复

- 补齐了一个缺失的 DLL，否则在尝试添加角色成员时会导致崩溃。

## 3.0.8 中的改进

- 新增选项：在 DAX 查询中仅执行所选的 EVALUATE 语句（快捷键 Shift+F5），参见问题 [#47](https://github.com/TabularEditor/TabularEditor3/issues/47)。
- 允许使用快捷键 Ctrl+/ 注释/取消注释代码，参见问题 [#46](https://github.com/TabularEditor/TabularEditor3/issues/46)。 你也可以在“工具 > 偏好 > DAX编辑器 > 常规”中更改注释风格（-- 或 //）。
- DAX脚本现在也会为当前所选内容中的 KPI 度量值、计算列和计算表格生成脚本。 我们还会添加注释，让代码更易读（可在“工具 > 偏好 > DAX脚本编写”中禁用）。
- 现在提供一个选项：在针对 Power BI 模型工作时，自动在所有 DAX 表达式开头添加换行符。 这样代码在 Power BI Desktop 的公式栏中看起来更美观。 可在“工具 > 偏好 >”下将其禁用，参见问题 [#26](https://github.com/TabularEditor/TabularEditor3/issues/26)。
- 添加 Windows AD 角色成员时，现在会弹出“目录对象选择器”对话框，参见问题 [#152](https://github.com/TabularEditor/TabularEditor3/issues/152)。
- “添加部署元数据”选项现在可以在“工具 > 偏好 > 模型部署”下找到。
- Pivot Grid 改进：
  - 你现在可以把 TOM Explorer 对象拖到 Pivot Grid 的所有区域中，参见问题 [#49](https://github.com/TabularEditor/TabularEditor3/issues/49)。
  - 在 TOM Explorer 的右键菜单中新增选项，可将度量值、列和层次结构直接添加到 Pivot Grid。
  - 在 Pivot Grid 中新增“显示字段”按钮，用于显示所连接数据库中的所有可用字段。
  - 在 Pivot Grid 中新增右键菜单选项“转到对象”。
- 关系错误（路径不明确、状态无效等） 现在会在 TOM Explorer 中检测并标示出来

## 3.0.8 中的错误修复

- 将“执行查询”的感叹号颜色从红色改为绿色。
- 修复了在图表视图中反转关系时导致的崩溃
- 修复了关系编辑器中更改无法正确保存的问题
- 修复了在本机运行 SSAS 实例时，从 PBI Desktop 启动 TE3 会导致崩溃的问题。
- 修复了在 Power BI XMLA endpoint 上使用 Workspace 数据库时无法创建新模型的问题。
- 修复了使用“注释”操作后代码选区被清空的问题。
- Deployment Wizard 现在会记住上次使用的服务器。
- DAX脚本的计算列表达式不应再错误地提示“缺少行语境”的语义错误信息。
- 修复了商业版许可证用户无法使用 Visual Studio 集成 Workspace 服务器的问题。
- 修复了“successful”的拼写错误。
- 修复了在输入右括号时，自动格式化器有时会导致代码重复的问题。
- 在 UI 中新增“保存 Model.bim 备份”选项，参见问题 [#37](https://github.com/TabularEditor/TabularEditor3/issues/37)。
- 透视选择不应再“卡住”，参见问题 [#52](https://github.com/TabularEditor/TabularEditor3/issues/52)。
- 属性网格现在可以正确显示日期格式示例，参见问题 [#86](https://github.com/TabularEditor/TabularEditor3/issues/86)。
- 修复搜索时高亮颜色不正确的问题，参见 issue [#98](https://github.com/TabularEditor/TabularEditor3/issues/98)。
- “查找全部”现在按预期工作，参见 issue [#107](https://github.com/TabularEditor/TabularEditor3/issues/107)。
- 现在可正确识别会导致度量值数据类型降级为 Variant 的计算项，参见 issue [#111](https://github.com/TabularEditor/TabularEditor3/issues/111)。
- 修复在 FILTER 中使用 EARLIER 时误报语义错误的问题，参见 issue [#118](https://github.com/TabularEditor/TabularEditor3/issues/118)。
- 修复在 FILTER 中使用 KEEPFILTERS 时误报语义错误的问题，参见 issue [#122](https://github.com/TabularEditor/TabularEditor3/issues/122)。
- 空白（或仅含注释）的 Detail Rows Expression 不再触发错误提示，参见 issue [#127](https://github.com/TabularEditor/TabularEditor3/issues/127)。
- 修复重复弹出凭据提示的问题，参见 issue [#131](https://github.com/TabularEditor/TabularEditor3/issues/131)。
- 修复 vpax 导入问题，参见 issue [#133](https://github.com/TabularEditor/TabularEditor3/issues/133)。
- 修复选择显示文件夹时属性网格的异常行为，参见 issue [#134](https://github.com/TabularEditor/TabularEditor3/issues/134)。
- 在关系图中按“Delete”不再删除表（改为仅从关系图中移除），参见 issue [#139 ](https://github.com/TabularEditor/TabularEditor3/issues/139)。
- Pivot Grid 现在可以正确处理日期字段，参见 issue [#142](https://github.com/TabularEditor/TabularEditor3/issues/142)。
- 复制计算组时不再创建名称重复的度量值；同时会调整优先级编号以确保唯一性，参见 issue [#155](https://github.com/TabularEditor/TabularEditor3/issues/155) 和 [#156](https://github.com/TabularEditor/TabularEditor3/issues/156)。
- “取消注释代码”按钮现在也适用于带缩进的行，参见 [#159](https://github.com/TabularEditor/TabularEditor3/issues/159)。
