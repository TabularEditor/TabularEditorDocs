## 从 Tabular Editor 2.x 升级过来？

观看[这个视频](https://www.youtube.com/watch?v=pt3DdcjfImY)，快速了解 Tabular Editor 3 的新功能。

**Tabular Editor 3 主要功能概览：**

- 完全可自定义的 IDE，支持多显示器、Hi-DPI 以及主题
- 全新的强大 DAX 代码编辑器，支持自动完成、语法检查、代码折叠等更多功能
- \*工作区模式，可将更改保存到磁盘，并同时将模型元数据同步到 Analysis Services
- \*支持无限滚动预览表数据，创建 PivotGrids，或编写 DAX 查询来浏览模型或验证计算逻辑
- \*可同时更新 Provider 与 Structured数据源的表架构（没错，连 M 查询也支持！）
- \*安排数据刷新
- 创建 Data model 图
- 创建 DAX 脚本，让你在一个文档中编辑多个度量值或其他计算对象
- 录制 C# Script 并保存为宏（以前称为“Custom Actions”）
- 集成 VertiPaq分析器

\*=仅在连接到 Analysis Services 或 Power BI 实例时可用

## 已知问题

- 表导入向导未包含在此版本中。 它将在即将发布的 3.0.3 版本中提供。 作为变通办法，只需新建一个表，并指定分区查询（SQL 或 M）。 然后，右键单击该表并选择“更新表架构...”，以导入列。
- 想了解更多信息，可以看看[路线图](https://github.com/TabularEditor/TabularEditor3/issues/12)。

## 3.0.2 中的新功能：

- 你现在可以在“帮助 > 关于 Tabular Editor”对话框中更改许可证密钥（仅在当前未加载任何模型时才会显示该选项）。
- 在主工具栏和“文件”菜单中新增了“Revert”按钮（你可能需要还原到“Default Workspace”才能看到它）。 单击后，该按钮会从已连接的源（Power BI 模型或 Analysis Services 数据库）重新加载 TOM。 如果在“工具 > 偏好设置”中禁用了自动选项，可使用它在两者之间手动同步更改。

## 3.0.2 中的缺陷修复：

- DAX编辑器现已支持 PATHITEM / PATHITEMREVERSE DAX 函数中的 TEXT / INTEGER 关键字。
- DAX编辑器的偏好设置现在会在各次会话之间保留（参见 [#23](https://github.com/TabularEditor/TabularEditor3/issues/23)）
- 对 DAX编辑器的自动完成/参数提示做了多项改进。
- 修复问题 [#14](https://github.com/TabularEditor/TabularEditor3/issues/14)（对象错误未显示在 TOM Explorer 中）
- 支持 KMeansClustering DAX 扩展函数。
- 修复了一个缺陷：DAX Parser 在 SWITCH 函数中将某个值与 BLANK() 比较时会报告错误。
- 一些细微的 Visual 更新
- 修复了一个崩溃问题：当尝试执行会编辑/添加 Power BI Desktop 文件不支持的对象的 DAX 脚本时会崩溃（例如，[我们目前仍无法通过外部工具添加计算表格或计算列](https://docs.microsoft.com/en-us/power-bi/transform-model/desktop-external-tools)）
- 连接到 Power BI Desktop 文件时，现在可以在度量值上编辑 DetailRowsExpressions。
- 修复了 SciLexer.dll 的一个缺陷，该缺陷在某些情况下会导致应用在启动后立即崩溃。
- DAX 查询中的列名现在按原样显示（不会在大写字母前添加空格
