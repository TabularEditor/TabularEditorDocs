## 3.1.6 问题修复

- “偏好”、“SelectItems”和“保存/编辑宏”对话框现已支持调整大小，参见 [#51](https://github.com/TabularEditor/TabularEditor3/issues/51)。
- 当针对使用原生查询导入数据的 Power BI Desktop 模型执行“更新表架构”操作时，我们会将 Power BI Desktop 实例置于前台，因为 Desktop 会弹出需要确认的对话框，确认后“更新表架构”操作才能完成。参见问题 [#55](https://github.com/TabularEditor/TabularEditor3/issues/55)。
- 升级后任务栏图标不再出现异常。 参见 [#163](https://github.com/TabularEditor/TabularEditor3/issues/163)。
- 更新表架构时，现在会显示一个连接对话框，可在其中更改身份验证模式。 参见 [#262](https://github.com/TabularEditor/TabularEditor3/issues/262)。
- 现在仅在将模型部署到 Azure AS 或 Power BI 时，才会自动移除角色成员属性 `MemberId`。 参见 [#263](https://github.com/TabularEditor/TabularEditor3/issues/263)。 参见 [#263](https://github.com/TabularEditor/TabularEditor3/issues/263)。
- 修复了在排队多个刷新操作时 UI 冻结的问题，参见 [#281](https://github.com/TabularEditor/TabularEditor3/issues/281)。
- AAD 对象选择器现在会生成格式正确的对象名称。 参见 [#286](https://github.com/TabularEditor/TabularEditor3/issues/286)。 参见 [#286](https://github.com/TabularEditor/TabularEditor3/issues/286)。
- 执行用于创建计算组的 DAX 脚本时，现在会正确分配序号，参见 [#296](https://github.com/TabularEditor/TabularEditor3/issues/296)。
- 现在可以使用 CTRL + 方向键展开/折叠 TOM Explorer 中的所有节点，类似 TE 2.x，参见 [#297](https://github.com/TabularEditor/TabularEditor3/issues/297)。
- 执行创建计算组的 DAX 脚本时，列名现在会被正确赋值，参见 [#298](https://github.com/TabularEditor/TabularEditor3/issues/298)。
- 修复了在美国/非美国模式之间切换时，DAX 脚本区域设置未能正确更新的问题。
- 修复了尝试连接到包含未设置兼容级别的数据库的 AS 服务器时可能发生的崩溃（当连接到与 Analysis Services 多维实例共享其数据目录的 Analysis Services 表格实例时会出现）。
- 修复了一个有时会导致“更新表架构”功能 Report 所有列均在源中缺失的问题。
- 修复了在使用模拟身份时，表格预览功能可能导致网格列配置不正确的问题。
- 修复了保存模型时语义更改通知栏不消失的问题。
- 修复了保存模型时属性网格中的编辑不会自动提交的问题。

## 从 Tabular Editor 2.x 升级而来？

观看 [此视频](https://www.youtube.com/watch?v=pt3DdcjfImY)，快速了解 Tabular Editor 3 的新功能。

**Tabular Editor 3 主要功能概览：**

- 完全可自定义的 IDE，支持多显示器、Hi-DPI 和主题
- 全新的强大 DAX 代码编辑器，支持自动完成、语法检查、代码折叠等更多功能
- \*工作区模式，可在将更改保存到磁盘的同时，将模型元数据同步到 Analysis Services
- \*支持无限滚动预览表数据，创建 PivotGrid，或编写 DAX 查询来浏览模型或测试计算逻辑
- \*计划数据刷新
- 在提供程序和 Structured数据源上更新表架构（没错，连 M 查询也支持！）
- 创建 Data model 图表
- 创建 DAX 脚本，以便在一个文档中编辑多个度量值或其他计算对象
- 录制 C# Script 并保存为宏（以前称为“Custom Actions”）
- VertiPaq分析器集成

\*=仅在连接到 Analysis Services 或 Power BI 实例时可用
