## 已知问题

- 由于需要优先修复的一些高优先级 Bug，“表导入向导”未能包含在此版本中。 我们会尽快提供！ 作为临时解决方法，你只需新建一个表，并指定分区查询（SQL 或 M）。 然后右键单击该表，选择“更新表架构...”，即可导入列。
- 更多信息请参阅[路线图](https://github.com/TabularEditor/TabularEditor3/issues/12)。

## 3.0.6 中的改进：

- 支持咨询许可证。 [更多详细信息](https://tabulareditor.com/license-terms)。
- DAX 查询现在会显示行数和耗时（参见问题 [#10](https://github.com/TabularEditor/TabularEditor3/issues/10)）
- DAX 查询现在默认限制为 1000 行，以防止超时/内存问题。 你可以在“工具 > 偏好 > 数据浏览 > DAX 查询”中更改此限制。
- DAX 查询结果中的列现在可以排序和筛选（仅限本地排序/筛选）。 参见问题 [#60](https://github.com/TabularEditor/TabularEditor3/issues/60)
- 网格（DAX 查询结果、表格预览等）中的数据 现在可以复制到剪贴板
- 复制 DAX 表达式时，语法高亮现在也会一并保留到剪贴板中。 参见问题 [#80](https://github.com/TabularEditor/TabularEditor3/issues/80)。
- 通过 C# Script 表达式添加计算表格时，语义分析器现在会根据 DAX 表达式立即将计算表格的列添加到表中（而不是等脚本执行后才添加）。
- 已将 TOM 客户端库更新至 v. 19.22.0.1
- 现已提供便携版（.zip）。 请注意：如果使用便携版，则必须 [手动将 Tabular Editor 3 注册到 Power BI Desktop](https://docs.microsoft.com/en-us/power-bi/transform-model/desktop-external-tools#registering-external-tools)。

## 3.0.6 中的错误修复：

- 现在可以在 C# Script 中使用 `EvaluateDax`、`ExecuteCommand`、`SelectTable`、`SelectMeasure` 等方法（类似 Tabular Editor 2.x），参见 [#44](https://github.com/TabularEditor/TabularEditor3/issues/44)。
- `CustomAction` C# Script 方法现在可正常工作，参见 [#79](https://github.com/TabularEditor/TabularEditor3/issues/79)。
- 改进了与 AS 的重新连接（应减少“connection lost”导致的崩溃）
- 现在可以通过 C# Script 实例化 Best Practice Analyzer：`new TabularEditor.Shared.BPA.Analyzer();`——这与 Tabular Editor 2.x 不同，因为当时 Analyzer 类位于另一个命名空间。 参见问题 [#45](https://github.com/TabularEditor/TabularEditor3/issues/45)
- 选择文件夹时现已支持复制操作，参见 [#84](https://github.com/TabularEditor/TabularEditor3/issues/84)
- 改进了表格预览中日期列的筛选，参见 [#77](https://github.com/TabularEditor/TabularEditor3/issues/77)
- 修复了语义引擎功能覆盖未生效的问题，参见 [#62](https://github.com/TabularEditor/TabularEditor3/issues/62)。
