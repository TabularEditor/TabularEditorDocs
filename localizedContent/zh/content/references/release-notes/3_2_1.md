## 3.2.1 中的改进

- 调试器已在多个方面得到改进。 例如，现在可以通过右键上下文菜单更轻松地管理观察表达式。 你还可以在编辑器中右键单击，快速将度量值、变量引用或子表达式添加到观察表达式视图中。 更多信息请参阅[这篇文章](https://docs.tabulareditor.com/te3/features/dax-debugger.html)。
- 新增一个序列化选项，可确保将角色/透视的 JSON 数组保存到磁盘时按名称排序，详见 [#341](https://github.com/TabularEditor/TabularEditor3/issues/341)。 你可以在 **工具 > 偏好 > 文件格式 > 按名称排序数组** 下找到这个新选项（如果要更改现有模型的设置，可以去 **模型 > 序列化选项...**）。

## 3.2.1 中的缺陷修复

- 调试器现在支持使用 ROLLUPGROUP 函数的 DAX 查询。
- 修复了一个问题：当从 Pivot Grid 启动并对包含两个唯一值的列应用筛选器时，会向 DAX调试器发送反向筛选。
- DAX调试器稳定性整体提升，详见 [#369]((https://github.com/TabularEditor/TabularEditor3/issues/369)。
- DAX调试器中的“单步跳过”(F10) 和“单步进入”(F11) 快捷键现在用起来更直观。
- 修复了在 TOM Explorer 尚未初始化时加载模型会崩溃的问题，详见 [#357](https://github.com/TabularEditor/TabularEditor3/issues/357)。
- 修复了在应用程序重启或在布局之间切换时，Workspace 布局无法正确还原的问题。
- 确保在自动隐藏面板展开时加载其内容
- 当省略 [SEARCH](https://dax.guide/search) 或 [FIND](https://dax.guide/find) 函数的第三个参数时，语义分析器不再报告错误。
- 使用 [ROLLUPGROUP](https://dax.guide/rollupgroup) 和 [ROLLUP](https://dax.guide/rollup) DAX 函数时，语义分析器不再报告错误 (有时会出现在 PBI Desktop 生成的查询中)。
- 对离线 DAX 格式化程序进行多项改进，使其与 https://daxformatter.com 更加一致。
- 修复在 DAX 脚本中将注释框添加到上一条表达式的问题，参见 [#331](https://github.com/TabularEditor/TabularEditor3/issues/331)。
- 修复应用在启动时崩溃的问题，参见 [#328](https://github.com/TabularEditor/TabularEditor3/issues/328)。
- 支持一次为多个对象设置翻译属性，参见 [#359](https://github.com/TabularEditor/TabularEditor3/issues/359)。
- 修复刷新进度指示器“卡住”的问题，参见 [#360](https://github.com/TabularEditor/TabularEditor3/issues/360)。
- 修复关系图字体缩放问题，参见 [#351](https://github.com/TabularEditor/TabularEditor3/issues/351)。
- 修复在高 dpi 下调用树渲染的问题，参见 [#355](https://github.com/TabularEditor/TabularEditor3/issues/355)。
- 确保从文件加载 DAX 脚本时会立即进行解析，参见 [#352](https://github.com/TabularEditor/TabularEditor3/issues/352)。
- 修复一个错误：在首次更改后，TE3 会停止从 AS 同步元数据，参见 [#353](https://github.com/TabularEditor/TabularEditor3/issues/353)。
- 修复在某些情况下依赖关系树无法正确重建的问题，参见 [#364](https://github.com/TabularEditor/TabularEditor3/issues/364)。
- 替换现有表时不再导致崩溃，参见 [#368](https://github.com/TabularEditor/TabularEditor3/issues/368)。
- 修复 BPA 必须先打开 BPA 视图才会运行的问题，参见 [#367](https://github.com/TabularEditor/TabularEditor3/issues/367)。
- 连接到 AS 标准层（不允许包含多个分区的表）时，使用单个分区进行 AS 变更检测。 参见 [#336](https://github.com/TabularEditor/TabularEditor3/discussions/336)。
- 通过架构比较添加的列将继承其透视成员关系。 参见 [#342](https://github.com/TabularEditor/TabularEditor3/discussions/342)。
- 对 Web 代理使用默认网络凭据（在通过代理连接位于代理后的 AS 时，应可解决 407 错误）。
- 当变量引用被加上引号时，语义分析器不应再 Report 错误，参见 [#302](https://github.com/TabularEditor/TabularEditor3/discussions/302)。
- 改进了来自 Pivot Grid 的错误信息显示（现在会显示在 **信息视图** 中）。

## 从 Tabular Editor 2.x 迁移过来？

观看[此视频](https://www.youtube.com/watch?v=pt3DdcjfImY)，快速了解 Tabular Editor 3 的新功能。 另外，也别忘了查看我们的[入门指南](https://docs.tabulareditor.com/onboarding/index.html)。

**Tabular Editor 3 主要功能概览：**

- 完全可自定义的 IDE，支持多显示器、高 DPI 和主题
- 全新强大的 DAX 代码编辑器，支持自动完成、语法检查、代码折叠等更多功能
- \*工作区模式，允许你将更改保存到磁盘，同时将模型元数据同步到 Analysis Services
- \*使用无限滚动预览表格数据，创建 PivotGrid，或编写 DAX 查询来浏览模型或测试计算逻辑
- \*安排数据刷新
- 在提供程序数据源和 Structured数据源上同时更新表架构（没错，连 M 查询也支持！）
- 创建 Data model 图
- 创建 DAX 脚本，让你在一个文档中编辑多个度量值或其他计算对象
- 录制 C# Script 并保存为宏（以前称为“Custom Actions”）
- VertiPaq分析器集成
- \*DAX调试器

\*=只在连接到 Analysis Services 或 Power BI 实例时可用
