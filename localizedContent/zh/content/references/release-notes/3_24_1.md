## 3.24 新增

查看我们的[发布博客](https://tabulareditor.com/blog/tabular-editor-3-october-2025-release)，快速概览本次发布中最重要的更新。

另请参阅[3.24.0 版本的发行说明](xref:release-3-24-0)，了解上一个主要版本中新增的全部功能与改进。

## 3.24.1 改进

- 语义分析器新增对数据类型关键字 `LOGICAL`、`INT64`、`DECIMAL` 和 `TEXT` 的支持（它们都是现有关键字的别名），可作为 [`CONVERT`](https://dax.guide/convert) 和 [`DATATABLE`](https://dax.guide/datatable) 等函数的参数使用。
- 语义分析器新增对新函数 [`ISINTEGER`](https://dax.guide/isinteger)、[`ISCURRENCY`](https://dax.guide/iscurrency) 以及 Visual 计算函数 [`LOOKUPWITHTOTALS`](https://dax.guide/lookupwithtotals) 的支持。
- 在 TOM Explorer 中右键单击 **Model** 或 **Functions** 节点时，新增“Manage DAX 组件...”菜单项。

## 3.24.1 Bug 修复

- 修复了一个 Bug：在低分辨率显示器上启动 Tabular Editor 3 时会引发未处理的异常
- 修复了一个 Bug：使用 **文件 > 打开 > 从文件打开模型...** 菜单选项时，无法从 PBIP 文件夹结构中打开模型。参见 [#1507](https://github.com/TabularEditor/TabularEditor3/issues/1507)。
- 修复与 DAX 组件管理器视图相关的 DPI 缩放问题。
- 修复语义分析器 Bug：无法识别 Visual 计算函数 [`FIRST`](https://dax.guide/first) 和 [`LAST`](https://dax.guide/last)。
- 修复语义分析器 Bug：在调用 [`CALCULATE`](https://dax.guide/calculate) 时，将 UDF 作为第一个参数会错误地显示警告。参见 [#1509](https://github.com/TabularEditor/TabularEditor3/issues/1509)。
- DAX 组件管理器不再因为 DAX 表达式中的行尾差异而显示警告（PBI Desktop 在保存模型时可能会将 `\n` 改为 `\r\n`）。
- 修复 TOM Explorer 中不同命名空间下 UDF 的排序问题

## 从 Tabular Editor 2.x 迁移过来？

观看 [此视频](https://youtu.be/O4ATwdzCvWc)，快速了解 Tabular Editor 3 的主要功能。 另外，也别忘了查看我们的 [上手指南](https://docs.tabulareditor.com/onboarding/index.html)。

**Tabular Editor 3 主要功能概览：**

- 完全可自定义的 IDE，支持多显示器、Hi-DPI 和主题
- 全新且强大的 DAX 代码编辑器，支持自动完成、语法检查、代码折叠等更多功能
- \*Workspace 工作区模式：在将更改保存到磁盘的同时，将模型元数据同步到 Analysis Services
- \*以无限滚动方式预览表数据，创建 PivotGrid，或编写 DAX 查询来浏览模型或测试计算逻辑
- \*计划数据刷新
- 在 Provider 数据源和 Structured数据源上更新表架构（是的，连 M 查询也支持！）
- 创建 Data model 图
- 创建 DAX 脚本，让你在一个文档中编辑多个度量值或其他计算对象
- 录制 C# Script 并将其保存为宏（以前称为“自定义操作”）
- VertiPaq分析器集成
- DAX调试器
- DAX优化器集成
- Code Actions，轻松重构 DAX。

\*=仅在连接到 Analysis Services 或 Power BI 的实例时
