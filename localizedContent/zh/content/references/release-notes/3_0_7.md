## 已知问题

- 由于需要先修复一些高优先级 Bug，“表导入向导”未能包含在此版本中。 我们会尽快提供！ 临时替代方案：新建一张表，并指定分区查询（SQL 或 M）。 然后，右键单击该表，选择“更新表架构...”，以导入列。

## 3.0.7 中的改进

- 允许 C# Script 使用自定义编译器，参见问题 [#97](https://github.com/TabularEditor/TabularEditor3/issues/97)

## 3.0.7 中的 Bug 修复

- “Standard Edition”已更名为“商业版”
- 修复了语义分析器中的一个问题：该问题有时会导致堆栈溢出异常并使工具崩溃（参见问题 [#95](https://github.com/TabularEditor/TabularEditor3/issues/95)）。
- 修复了语义分析器中的一个问题：该问题有时会报告不存在的循环依赖错误。
- 语义分析器现已支持 SUBSTITUTEWITHINDEX DAX 函数
- 修复了部分 JOIN DAX 函数（NATURALINNERJOIN 等）的问题 修复了部分 JOIN DAX 函数（NATURALINNERJOIN 等）的问题 这有时会返回错误的列集（参见问题 [#103](https://github.com/TabularEditor/TabularEditor3/issues/103)）
- 自动补全现在只会在 RELATED 和 RELATEDTABLE 中显示相关的列/表（参见问题 [#112](https://github.com/TabularEditor/TabularEditor3/issues/112)）
- 自动补全现在也支持 ORDER BY（参见问题 [#53](https://github.com/TabularEditor/TabularEditor3/issues/53)）
- 编辑器在创建文件夹、复制/粘贴单个对象等操作时不再消失（参见问题 [#90](https://github.com/TabularEditor/TabularEditor3/issues/90)）
- 计算组现在会显示所有相关的右键菜单项（包括刷新选项），参见问题 [#100](https://github.com/TabularEditor/TabularEditor3/issues/100)
- 修复了在查询中添加 ORDER BY 语句后，DAX 查询不返回任何结果的问题，参见问题  [#108](https://github.com/TabularEditor/TabularEditor3/issues/108)
