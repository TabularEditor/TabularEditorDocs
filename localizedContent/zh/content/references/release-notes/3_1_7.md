## 3.1.7 中的改进

- 当 DAX 查询导致 Analysis Services 返回错误信息时，现在会在原先的结果网格位置直接显示该信息。
- 网格现在可以缩放（Ctrl+鼠标滚轮）！
- 自动完成与自动缩进的多项改进。
- 新增右键菜单，可轻松为角色添加表格权限（以前需要通过分配 RLS 表达式或指定 OLS 设置来添加表格权限）。 参见问题 [#257](https://github.com/TabularEditor/TabularEditor3/issues/257)。 你可以一次在多个角色中添加多个表格权限。
- “选择数据库”对话框现在会记住列的排序顺序。 参见 [#321](https://github.com/TabularEditor/TabularEditor3/issues/321)。

## 3.1.7 中的错误修复

- 修复 SaveModelMetadataBackup 在数据库 ID 上不起作用的问题（当 ID 与名称不一致时）。
- 修复重置自动格式设置偏好时发生崩溃的问题
- 修复 VPAX 中 TOM 导出缺失的问题。 修复 VPAX 包含 TOM 时信息对话框显示的问题。
- 改进 DAX（离线）格式化，详见问题 [#306](https://github.com/TabularEditor/TabularEditor3/issues/306)。
- 在使用默认（离线）格式化器时，修复 `FormatDax(...)` 和 `CallDaxFormatter()` 辅助函数的问题，详见 [#307](https://github.com/TabularEditor/TabularEditor3/issues/307)。
- 修复自动完成有时会“卡住”的问题。
- 在更改 Windows 显示分辨率或缩放比例后，出现显示缩放问题的窗口/对话框应会减少。 参见 [#308](https://github.com/TabularEditor/TabularEditor3/issues/308) 和 [#81](https://github.com/TabularEditor/TabularEditor3/issues/81)。
- 自动格式化程序不应再重复多行注释块。 参见 [#319](https://github.com/TabularEditor/TabularEditor3/issues/319]。
- 修复了后台语义分析错误地报告无效 DAX 错误的问题，参见 [#311](https://github.com/TabularEditor/TabularEditor3/issues/311)。
- 通过菜单栏打开 BPA 管理器不应再导致崩溃，参见 [#312](https://github.com/TabularEditor/TabularEditor3/issues/312)。
- 在工作区模式下保存从 Database.json 文件加载的模型时，不应再出现“Save failed!” 错误，参见问题 [#322](https://github.com/TabularEditor/TabularEditor3/issues/322)。
- 在应用 DAX 脚本时，现在会正确保留注释，参见问题 [#323](https://github.com/TabularEditor/TabularEditor3/issues/323)。

## 从 Tabular Editor 2.x 迁移过来？

观看 [此视频](https://www.youtube.com/watch?v=pt3DdcjfImY)，快速了解 Tabular Editor 3 的新功能。

**Tabular Editor 3 主要功能概览：**

- 完全可自定义的 IDE，支持多显示器、Hi-DPI 和主题
- 全新且强大的 DAX 代码编辑器，提供自动完成、语法检查、代码折叠等丰富功能
- \*工作区模式，让你可以同时将更改保存到磁盘，并将模型元数据同步到 Analysis Services
- \*支持无限滚动预览表数据，创建 PivotGrid，或编写 DAX 查询来浏览模型或测试计算逻辑
- \*安排数据刷新
- 同时更新 Provider 和 Structured数据源上的表架构（是的，甚至适用于 M 查询！）
- 创建 Data model 图
- 创建 DAX 脚本，让你在一个文档中编辑多个度量值或其他计算对象
- 录制 C# Script，并保存为宏（以前称为“Custom Actions”）
- 集成 VertiPaq分析器

\*=仅在连接到 Analysis Services 或 Power BI 实例时可用
