### 先决条件

Tabular Editor 3.3.0（及更高版本）使用 .NET 6。 在启动 Tabular Editor 3.3.0（或更高版本）之前，你可能需要先安装 [.NET 6.0 Runtime for Desktop](https://dotnet.microsoft.com/en-us/download/dotnet/6.0/runtime)。 从 3.3.1 开始，如果缺少这些先决条件，Tabular Editor 3 安装程序会自动下载并安装。

## 3.10.0 新增功能

阅读我们的[发布博文](https://blog.tabulareditor.com/2023/08/23/tabular-editor-3-august-2023-release/)，快速了解本次发布中最重要的更新。

- Tabular Editor 现已支持通过“导入表向导”向 [Direct Lake Dataset](https://learn.microsoft.com/en-us/power-bi/enterprise/directlake-overview) 添加表。 此外，你也可以用常规方式更新现有表的架构。 Tabular Editor 会使用你的 Fabric Lakehouse 的 [SQL 终结点](https://learn.microsoft.com/en-us/fabric/data-engineering/lakehouse-sql-endpoint) 来获取表架构。 如果你想了解更多，我们也发布了一篇关于 DirectLake 的[博客文章](https://blog.tabulareditor.com/2023/08/23/fabric-direct-lake-dataset/)。

> [!NOTE]
> 此版本的 Tabular Editor 允许你在已具有 Direct Lake 连接的模型中导入表或更新表架构。 “导入表向导”目前仍不支持在空模型上创建新的 Direct Lake 连接。 该功能将在未来的版本中加入。

> [!WARNING]
> 一旦你通过 XMLA endpoint 对 Direct Lake Dataset 进行任何更改，你将[无法再在 Fabric 的 Web 建模功能中打开该 Dataset](https://learn.microsoft.com/en-us/power-bi/enterprise/directlake-overview#enable-xmla-read-write:~:text=Direct%20Lake%20datasets%20created%20or%20modified%20by%20using%20XMLA%2Dbased%20tools%20cannot%20be%20opened%20in%20the%20Web%20modelling%20feature.)。

- C# Script 现已支持[预处理器指令](xref:csharp-scripts#compatibility)，用于判断你正在使用的 Tabular Editor 版本。 这使你能够编写同时兼容 Tabular Editor 2 和 Tabular Editor 3 的脚本。

## 3.10.0 改进内容

- 我们现在使用最新版本的 [AMO/TOM](https://www.nuget.org/packages/Microsoft.AnalysisServices.NetCore.retail.amd64/)。 此更新新增了对二进制 XML 的支持，有助于提升 XMLA 操作的速度。 更多详细信息见这篇 [Microsoft 博客](https://powerbi.microsoft.com/en-us/blog/improving-the-communication-performance-of-xmla-based-tools/)。 此更新新增了对二进制 XML 的支持，有助于提升 XMLA 操作的速度。 更多详细信息见这篇 [Microsoft 博客](https://powerbi.microsoft.com/en-us/blog/improving-the-communication-performance-of-xmla-based-tools/)。
- 我们还将 [TMDL 更新到 preview-3](https://www.nuget.org/packages/Microsoft.AnalysisServices.Tabular.Tmdl.NetCore.retail.amd64/19.65.12.3-TmdlPreview)，其中包含多项错误修复。

> [!NOTE]
> TMDL 仍处于预览阶段，因此该功能也应视为 Tabular Editor 3 的预览功能。 务必保留模型元数据的 Model.bim / Database.json 备份，以免丢失工作成果。

- 在克隆对象时，我们现在会添加后缀“-Copy”，而不是“ Copy”，以降低对象名称末尾留下多余空格的风险，详见 [#940](https://github.com/TabularEditor/TabularEditor3/issues/940)。
- 自动完成弹出窗口现在会自动调整宽度，以适配其中最宽的条目。 参见 [#965](https://github.com/TabularEditor/TabularEditor3/issues/965)。
- 当为 Power BI Desktop 模型保存备份 .bim 文件时，我们现在会使用 .pbix 文件名，而不是数据库 GUID。参见 [#958](https://github.com/TabularEditor/TabularEditor3/issues/958)。
- 表格预览性能改进。

## 3.10.0 中的错误修复

- 修复了使用“保存到文件夹”时，同义词（语言学元数据）无法保存的问题。
- 修复了备份 .bim 文件时间戳不正确的问题，参见 [#967](https://github.com/TabularEditor/TabularEditor3/issues/967)。
- 修复了当备份保存因任何原因失败时，用户无法取消部署的问题。
- 表格预览中的“查找”面板现在会在后台生成有效的 DAX 搜索表达式，参见 [#955](https://github.com/TabularEditor/TabularEditor3/issues/955)。
- 修复了当光标放在 DAX 函数上时，“转到定义（F12）”操作会被禁用的问题，参见 [#962](https://github.com/TabularEditor/TabularEditor3/issues/962)。
- 修复了在某些情况下无法从 .vpax 文件加载模型的问题。
- DAX 查询中的“结果计数”面板高度现在会自动调整以适配其内容。
- 属性网格现在能正确显示以一个或多个换行符开头的多行表达式，效果与在网格内编辑表达式时一致。
- 修复了一个问题：当从远程模型中删除已添加到 Pivot Grid 的列时，Pivot Grid 可能会陷入无限错误消息循环。
- 修复了一个问题：添加到菜单栏的宏在重启 Tabular Editor 后会停止工作。
- 修复了一个问题：对源自同一源表的多张表执行架构更新时，除其中一张外其余都会失败。

## 从 Tabular Editor 2.x 迁移而来？

观看 [此视频](https://www.youtube.com/watch?v=pt3DdcjfImY)，了解 Tabular Editor 3 的新功能。 另外，也别忘了查看我们的 [上手指南](https://docs.tabulareditor.com/getting-started/index.html)。

**Tabular Editor 3 主要功能概览：**

- 完全可自定义的 IDE，支持多显示器、Hi-DPI 和主题
- 全新的强大 DAX 代码编辑器，支持自动完成、语法检查、代码折叠等更多功能
- \*工作区模式，让你既能将更改保存到磁盘，又能同时将模型元数据同步到 Analysis Services
- \*通过无限滚动预览表数据，创建 Pivot Grid，或编写 DAX 查询来浏览模型或测试计算逻辑
- \*定时刷新数据
- 同时更新 Provider 数据源和 Structured数据源上的表架构（是的，连 M 查询也可以！）
- 创建 Data model 图
- 创建 DAX 脚本，让你在一个文档中编辑多个度量值或其他计算对象
- 录制 C# Script 并保存为宏（以前称为“Custom Actions”）
- VertiPaq分析器集成
- DAX调试器

\*=仅在连接到 Analysis Services 或 Power BI 实例时可用
