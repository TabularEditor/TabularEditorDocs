查看我们的 [发布博客](https://blog.tabulareditor.com/?p=2146)，快速了解此版本最重要的更新。

> [!NOTE]
> TMDL 仍处于预览阶段，因此该功能也必须视为 Tabular Editor 3 的预览功能。 请务必为模型元数据保留一份 Model.bim / Database.json 备份，以免丢失工作成果。\*\*。 请务必为模型元数据保留一份 Model.bim / Database.json 备份，以免丢失工作成果。\*\*。

## 3.7.0 中的小幅改进

- 现已使用最新版本的 [VertiPaq分析器](https://github.com/sql-bi/VertiPaq-Analyzer)。 这也修复了在包含动态格式字符串的模型上收集统计信息等方面的问题。 这也修复了在包含动态格式字符串的模型上收集统计信息等方面的问题。
- 当由于模型元数据不兼容导致粘贴操作失败时，我们现在会显示更有意义的错误信息。
- 数据刷新作业完成后收到的 Analysis Services 跟踪事件，现在会正确归属到该作业，从而确保错误/警告信息以及进度计数能够正确传递。 参见 [#735](https://github.com/TabularEditor/TabularEditor3/issues/735)。
- 数据预览窗口现在可以使用 DAX [`OFFSET`](https://dax.guide/offset) 函数，使预览能够显示所有非 DirectQuery 表的全部行——即使模型处于 DirectQuery 模式（例如模型中存在混合表格时）也是如此。参见 [#500](https://github.com/TabularEditor/TabularEditor3/issues/500)、[#837](https://github.com/TabularEditor/TabularEditor3/issues/837)。
- 将模型部署为新数据库时，如果模型元数据中不存在任何分区，我们现在会为使用增量刷新策略的表添加一个虚拟分区，以确保部署成功。
- 当表被刷新/处理时，数据预览会保留已应用的所有筛选器。 参见 [#780](https://github.com/TabularEditor/TabularEditor3/issues/780)。
- 当某个表不包含任何处于不可查询状态的列时，数据预览现在会在生成的 DAX 查询中直接使用表名，而不是将其包裹在对 [`SELECTCOLUMNS`](https://dax.guide/selectcolumns) 的调用中。 这应该能加快旧版本 Analysis Services 上的数据预览速度。 参见 [#126](https://github.com/TabularEditor/TabularEditor3/issues/126)。 这应该能加快旧版本 Analysis Services 上的数据预览速度。 参见 [#126](https://github.com/TabularEditor/TabularEditor3/issues/126)。
- TE3 现已使用最新版本的 [Microsoft.Identity.Client](https://www.nuget.org/packages/Microsoft.Identity.Client)。
- 离线架构检测现已支持使用用于定义自定义函数的共享 (M) 表达式。

- 修复了一个问题：Power BI Desktop 的本地实例未显示在“本地实例”下拉列表中。
- 修复了一个问题：当模型未指定数据库名称时，TE3 不允许加载该模型。
- “请稍候”对话框现在不会再遮挡 Azure AD 登录窗口。
- 稳定性改进。
- 修复了一个问题：尝试复制/粘贴带有动态格式字符串的度量值时，应用会崩溃。参见 [#1099](https://github.com/TabularEditor/TabularEditor/issues/1099)。
- 对带有格式字符串/动态格式字符串的度量值进行 DAX 脚本化现在可以按预期工作了。
- 查询作用域列现在会被 DAX 语义分析器和自动完成功能正确处理。 参见 [#890](https://github.com/TabularEditor/TabularEditor3/issues/890)。 参见 [#890](https://github.com/TabularEditor/TabularEditor3/issues/890)。

- 修复了多个提示对话框中按钮的一些 Visual 显示问题。
- 修复了一个问题：当列表被筛选时，数据库选择会不正确。 修复了一个问题：当列表被筛选时，数据库选择会不正确。 参见 [#886](https://github.com/TabularEditor/TabularEditor3/issues/886)。

- 修复了 DAX编辑器中出现“幽灵”语义错误的问题。参见 [#792](https://github.com/TabularEditor/TabularEditor3/issues/792)。
- 数据预览现在会遵循分配给列的格式字符串。参见 [#820](https://github.com/TabularEditor/TabularEditor3/issues/820)。
- 当 DateTime 值的时间部分不为 00:00:00 时，DAX 查询结果现在会显示该时间部分。

**Tabular Editor 3 主要功能概述：**

- 完全可自定义的 IDE，支持多显示器、Hi-DPI 和主题
- 全新强大的 DAX 代码编辑器，提供自动补全、语法检查、代码折叠等更多功能
- \*工作区模式：可将更改保存到磁盘，并同时将模型元数据同步到 Analysis Services
- \*可无限滚动预览表数据，创建 PivotGrid 或编写 DAX 查询，以浏览模型或测试计算逻辑
- \*计划数据刷新任务
- 在 Provider 数据源和 Structured数据源上更新表架构（是的，甚至包括 M 查询！）
- 创建 Data model 关系图
- 创建 DAX 脚本，让你在一个文档中编辑多个度量值或其他计算对象
- 录制 C# Script 并将其保存为宏（以前称为“Custom Actions”）
- 集成 VertiPaq分析器
- DAX调试器

\*=仅在连接到 Analysis Services 或 Power BI 实例时可用
