## 已知问题

- 由于需要先修复一些高优先级 Bug，表导入向导未能包含在此版本中。 我们会尽快提供！ 临时替代方案：创建一张新表，并指定分区查询（SQL 或 M）。 然后右键单击该表，选择“更新表架构...”以导入列。
- 更多信息请参阅[路线图](https://github.com/TabularEditor/TabularEditor3/issues/12)。

## 3.0.10 的改进

- 将 AMO/TOM 更新到版本 [19.22.5](https://www.nuget.org/packages/Microsoft.AnalysisServices.retail.amd64/)。
- 允许在“选择项目”对话框中进行增量搜索，例如在向关系图添加表时，详见问题 [#167](https://github.com/TabularEditor/TabularEditor3/issues/167)。

## 3.0.10 的 Bug 修复

- 修复 TOM Explorer 在尝试将对象拖入文本编辑器时丢失当前选中项的问题
- 修复语义分析器导致误报“A single value for column …… 当模型中某处的 DAX 表达式使用 ALLEXCEPT 时，可能会出现“cannot be determined”错误，参见 [#140](https://github.com/TabularEditor/TabularEditor3/issues/140)。
- 宏录制器现可用于计算项表达式、格式字符串表达式、度量值详细信息行表达式等，参见 [#143](https://github.com/TabularEditor/TabularEditor3/issues/143)。
- 为 C# Script 新增了 `Model.AddCalculationGroupTable(<name>)` 方法。 同时也修复了问题 [#144](https://github.com/TabularEditor/TabularEditor3/issues/144)。
- DAX脚本不再添加额外缩进，参见 [#165](https://github.com/TabularEditor/TabularEditor3/issues/165)。
- 已修复 Pivot Grid 连接问题，参见 [#166](https://github.com/TabularEditor/TabularEditor3/issues/166)。
- 已修复 VertiPaq分析器的数据类型问题，参见 [#169](https://github.com/TabularEditor/TabularEditor3/issues/169)。

## 从 Tabular Editor 2.x 升级而来？

观看 [此视频](https://www.youtube.com/watch?v=pt3DdcjfImY)，快速了解 Tabular Editor 3 的新功能。

**Tabular Editor 3 主要功能概览：**

- 完全可自定义的 IDE，支持多显示器、Hi-DPI 以及主题
- 全新强大的 DAX 代码编辑器，支持自动完成、语法检查、代码折叠等更多功能
- \*工作区模式，可将更改保存到磁盘，并同时将模型元数据同步到 Analysis Services
- \*支持无限滚动预览表数据，可创建 Pivot Grid 或编写 DAX 查询来浏览模型或测试计算逻辑
- \*可在 Provider 数据源和 Structured数据源上更新表架构（是的，甚至支持 M 查询！）
- \*计划数据刷新
- 创建 Data model 图
- 创建 DAX脚本，以便在同一文档中编辑多个度量值或其他计算对象
- 录制 C# Script 并保存为宏（原称“Custom Actions”）
- VertiPaq分析器集成

\*=仅在连接到 Analysis Services 或 Power BI 实例时才可用
