## 重要！

**注意**：Tabular Editor 3.3.0（及更高版本）使用 .NET 6。 在启动 Tabular Editor 3.3.0（或更高版本）之前，你可能需要先安装[适用于桌面的.NET 6.0 运行时](https://dotnet.microsoft.com/en-us/download/dotnet/6.0/runtime)。 从 3.3.1 开始，如果缺少这些先决条件，Tabular Editor 3 安装程序会下载并安装它们。

## 3.4.0 中的新功能

- 将 TOM 更新到 [19.54.1](https://www.nuget.org/packages/Microsoft.AnalysisServices.NetCore.retail.amd64)。
- Tabular Editor 3 现已支持在 Power BI Desktop 2022 年十二月更新中宣布的 [新 DAX 窗口函数](https://pbidax.wordpress.com/2022/12/15/introducing-dax-window-functions-part-1/)。
- 现已支持自定义键盘快捷键！ 你是否希望无论在什么上下文中，都始终使用 CTRL+S 来保存模型？ 你是否想为模型刷新分配一个快捷键？ 或者为编辑器添加一个快捷键，用于[选择当前选中内容的所有出现位置](https://github.com/TabularEditor/TabularEditor3/issues/630)？ 在 **工具 > 偏好 > 键盘** 中可以找到此功能。 更多信息请参见[此处](xref:preferences#tabular-editor--keyboard)。
- 所有代码编辑器现在除支持矩形选择（拖动时按住 ALT，或使用 SHIFT + 方向键）外，还支持多重选择（即按住 CTRL 并用鼠标拖动，以创建新的选区）。

## 3.4.0 中的小幅改进

- Tabular Editor 3 现在会在 Power BI“外部工具”功能区中显示正确的图标。
- 可在属性网格中统一编辑列类型属性。 现在你可以在属性网格中通过输入列名来指定列。 当列数超过 7 个时，将显示“选择列”对话框，而不是下拉列表。 这也修复了 [#738](https://github.com/TabularEditor/TabularEditor3/issues/738)。
- 在“表格预览”中移除了列名中的方括号
- 现在可以在导入表向导中连接并选择其他 Power BI Workspace。
- Pivot Grid 字段列表现在可以作为可停靠窗口使用，参见 [#742](https://github.com/TabularEditor/TabularEditor3/issues/742)。
- 当源描述中包含 HTML 标记时，DAX 调用提示现在也能正确排版。
- 现在，所有代码编辑器在使用键盘进行选择时，矩形选区功能都有了大幅改进。 此外，在矩形选区下，复制/粘贴现在也更符合直觉。
- 在“工具 > 自定义”对话框（位于“Options”选项卡下）中新增了锁定菜单和工具栏的选项，以防止误将其拖动。
- Scintilla 的多重选择“添加下一个/逐个添加”现在会遵循“查找”对话框中设置的搜索选项。
- 允许 File.SaveModel 快捷键在任何上下文下都能生效，参见 [#762](https://github.com/TabularEditor/TabularEditor3/issues/762)。
- 现在可以取消耗时较长的保存操作，例如当数据库被刷新操作锁定时。 参见 [#730](https://github.com/TabularEditor/TabularEditor3/issues/730)。
- 创建/编辑关系对话框现在支持调整大小，参见 [#732](https://github.com/TabularEditor/TabularEditor3/issues/732)。
- 允许在所有树形列表上用方向键展开/折叠（包括 BPA、依赖关系视图和数据刷新视图），参见 [#775](https://github.com/TabularEditor/TabularEditor3/issues/775)。
- 我们已认真听取你的反馈！ “显示表的关系”已回归到依赖关系视图，参见 [#731](https://github.com/TabularEditor/TabularEditor3/issues/731)。

## 3.4.0 Bug 修复

- AAD 连接对话框现在在所有情况下都会以模态对话框显示。
- 批量编辑多个对象时，会创建撤销事务
- 现在可以通过右键点击并选择“重置”来清除对象属性（例如 SortByColumn）。
- 新增右键快捷方式，用于编辑表表达式属性（SourceExpression、PollingExpression 等） 现已提供，参见 [#721](https://github.com/TabularEditor/TabularEditor3/issues/721)。
- 将字段拖到 Pivot Grid 上时，Pivot Grid 不再冻结，参见 [#676](https://github.com/TabularEditor/TabularEditor3/issues/676)。
- 修复问题 [#723](https://github.com/TabularEditor/TabularEditor3/issues/723)。
- 修复了在从 .tmuo 文件中指定的现有 Power BI Dataflow 导入时，会导致表架构无法更新并使“导入表向导”崩溃的问题。
- 修复了 DAX调试器生成的观察表达式 DAX 查询无效的问题，参见 [#770](https://github.com/TabularEditor/TabularEditor3/issues/770)。
- 修复了一个问题：该问题有时会导致调试器窗口崩溃（白色背景上显示红色叉号）。
- 修复了一个问题：“导入表向导”有时会显示名称为空的 Dataflow。
- “还原”按钮现在应始终处于启用状态。 此外，只有在做出更改时才会显示确认提示。 参见 [#729](https://github.com/TabularEditor/TabularEditor3/issues/729)。
- 显示字段列表不再清空 Pivot Grid，参见 [#741](https://github.com/TabularEditor/TabularEditor3/issues/741)。
- 修复了一个问题：应用启动时，TOM Explorer 中的列有时会被“截断”。
- TOM Explorer 现仅允许在 Power BI 数据集中添加 EntityPartitions
- 修复依赖项视图崩溃问题，参见 [#758](https://github.com/TabularEditor/TabularEditor3/issues/758)
- 修复了一个问题：应用程序重启后，浮动窗口可能会出现在可见屏幕区域之外。参见 [#652](https://github.com/TabularEditor/TabularEditor3/issues/652)。
- 修复“最近使用的服务器”未保存的问题。
- 修复了与操作在“模型”和/或“上下文”菜单中的显示方式相关的几个问题。 例如，“模型”菜单中的“Script DAX”只会为当前选择中的对象生成脚本，而按设计应为模型中的所有对象生成脚本。
- 修复文本编辑器横向滚动条的行为。
- SQL 连接对话框现在会将对连接字符串所做的更改保存下来，参见 [#755](https://github.com/TabularEditor/TabularEditor3/issues/755)。
- 在“选择数据库”对话框中筛选列表后再选择数据库，现在可确保实际加载的是正确的数据库。 参见 [#761](https://github.com/TabularEditor/TabularEditor3/issues/761)。
- 修复在表格中滚动时的问题 (参见 [#733](https://github.com/TabularEditor/TabularEditor3/issues/733)，以及应用筛选条件时行数不更新的问题。

## 从 Tabular Editor 2.x 升级过来？

观看 [此视频](https://www.youtube.com/watch?v=pt3DdcjfImY)，快速了解 Tabular Editor 3 的新功能。 另外，也记得查看我们的 [快速上手指南](https://docs.tabulareditor.com/onboarding/index.html)。

**Tabular Editor 3 主要功能概览：**

- 完全可自定义的 IDE，支持多显示器、Hi-DPI 和主题
- 全新强大的 DAX 代码编辑器，支持自动补全、语法检查、代码折叠等更多功能
- \*工作区模式，可将更改保存到磁盘，并同时将模型元数据同步到 Analysis Services
- \*通过无限滚动预览表数据，创建 PivotGrids，或编写 DAX 查询来浏览模型或测试计算逻辑
- \*计划数据刷新
- 更新 Provider 和 Structured数据源的表架构（没错，连 M 查询也支持！）
- 创建 Data model 图
- 创建 DAX 脚本，让你在一个文档中编辑多个度量值或其他计算对象
- 录制 C# Script 并保存为宏（以前叫“自定义操作”）
- VertiPaq分析器集成
- \*DAX调试器

\*=仅在连接到 Analysis Services 或 Power BI 实例时可用
