## 3.25.0 新增功能

查看我们的[发布博客](https://tabulareditor.com/blog/tabular-editor-3-january-2026-release)，快速了解本次发布中最重要的更新。

- Tabular Editor 现已提供 [**中文和西班牙语（预览版）**](xref:references-application-language)。 所有翻译最初均由 AI 完成，并经过人工校对。 日语、德语和法语也作为实验性语言提供，但尚未经过审校。
- Tabular Editor 现已内置一组 [**Best Practice Analyzer (BPA) 规则**](xref:built-in-bpa-rules)，覆盖格式设置、元数据、模型布局、DAX 表达式和翻译等方面的关键最佳实践。
- Tabular Editor 现已包含 **Semantic Bridge** 功能，让你可以轻松地从 Databricks [Metric Views](https://docs.databricks.com/aws/en/metric-views/) 创建语义模型（仅限 Tabular Editor 3 企业版）。
- 新增一个选项，可在保存语义模型时包含 [**用于 Fabric 的支持文件**](xref:save-with-supporting-files)。 启用后，Tabular Editor 会在模型元数据的同时创建并更新 .platform 和 definition.pbism 文件，以符合 Fabric repository 结构。
- 此版本还新增了 **Calendar Editor**，让你更轻松地管理基于日历的时间智能所使用的日历对象。
- 最后但同样重要的是，我们新增了 [**高级刷新**](xref:advanced-refresh) 对话框，你可以在其中配置并行度、增量刷新的生效日期等高级刷新选项。 你还可以指定[刷新覆盖配置文件](xref:refresh-overrides)（_仅 Tabular Editor 3 商业版和企业版_）。

## 3.25.0 中的改进

- 新增：支持在“模拟”对话框中直接设置连接字符串的 CustomData 属性。
- DAX 组件管理器视图现在会下载并显示 DaxLib 组件图标。
- DAX 组件管理器现在会缓存组件元数据，因此即使没有互联网连接也能显示组件信息。
- 我们已将 AS 客户端库更新到最新版本 [19.108.5](https://www.nuget.org/packages/Microsoft.AnalysisServices/19.108.5)。 此更新让你可以控制 Database 对象上的新属性 [UnicodeCharacterBehavior](https://learn.microsoft.com/en-us/dotnet/api/microsoft.analysisservices.tabular.database.unicodecharacterbehavior?view=analysisservices-dotnet#microsoft-analysisservices-tabular-database-unicodecharacterbehavior)。
- 新增了 Power BI 专用的列数据类别：经度、纬度和地点。
- 如果你尝试将模型的兼容级别设置为服务器不支持的值，我们现在会显示错误信息。 这样可以避免稍后在保存对模型的更改时出现意外错误。 如果出于迁移目的需要设置不受支持的兼容级别，先把模型元数据保存到磁盘（.bim 或 TMDL 格式），然后在不附加 Workspace 数据库的情况下打开该元数据。
- 当你直接从服务器 / XMLA 端点打开模型时，也会保存用户选项（.tmuo）文件。 这些文件存储在你的 %LocalAppData%\TabularEditor3\UserOptions 文件夹中。
- 我们在数据刷新上下文菜单中新增了“完全刷新（忽略策略）”选项。这样，当表受增量刷新的刷新策略治理时，你就可以轻松将数据刷新到现有分区，而不会在服务器上触发分区新增/删除/合并操作。
- 我们重新整理了数据刷新上下文菜单中的选项，把最常用的选项置顶。
- 刷新窗格新增“开始时间”，可以按刷新开始时间排序，例如将最近一次刷新显示在最前。
- 我们在“偏好设置”对话框中新增了搜索窗格，便于快速查找单个偏好设置。
- 在最佳实践规则编辑器列表中新增了“用户定义函数”和“日历”。
- 为列新增了 `UsedInCalendars` 属性，方便你在 C# Script 中检查某列是否被任何日历对象使用。
- “文件 > 关闭”菜单操作已重命名为“关闭文档”，并且现在可以关闭主文档区域中的任何面板或文档。 **Ctrl + W** 已设为默认快捷键。
- 语义分析器新增支持 [`DATEADD`](https://dax.guide/DATEADD) <extension> 参数中的新关键字 `ENDALIGNED`。 [了解更多信息](https://learn.microsoft.com/en-us/dax/dateadd-function-dax#extension-parameter-for-small--large-period-shifts)。
- 当保存为 model.bim 或 TMDL 文件夹结构，并启用“忽略增量刷新分区”序列化选项时，我们现在会添加一个虚拟分区来替代 PolicyRange 分区。 这可确保模型仍可在 Power BI Desktop 中加载，因为不会保存任何未分区的表。

## 3.25.0 中的错误修复

- 修复了一个关键问题：该问题会导致部分用户在使用 Azure AD 身份验证时无法连接到 Power BI REST API 或 Databricks。
- 修复了一个未处理的异常：当删除同时被日历对象使用的列时会触发该异常。
- 修复了关系编辑器中的列下拉列表，使其会自动显示完整的列名。
- 当复制包含一个或多个日历的表时，系统现在会自动重命名这些日历，以确保整个模型中的名称唯一。
- 数据刷新过程中遇到多个错误/警告时，现在不会再触发未处理的异常。
- 修复了在执行“全部更新”后，DAX 组件列表仍显示过时信息的问题。
- 修复了一个问题：实体分区的 `SchemaName` 属性无法撤销。
- 当发生[表扩展](https://www.sqlbi.com/articles/expanded-tables-in-dax/)时，不再建议使用代码操作 [DI005 - 将表筛选器重写为标量谓词](https://docs.tabulareditor.com/kb/DI005.html)，因为将此类筛选器重写为列筛选器不一定会产生相同的结果。
- 当连接字符串指定了当前系统未安装的 OLE DB 提供程序时，点击“连接字符串”属性旁的省略号按钮现在不会再导致未处理的异常。
- 启用相关偏好设置后，TMDL 导出现在会正确排除增量刷新分区，与 JSON 的行为一致。
- 修复了一个问题：在尝试从数据流更新或导入时，点击“登录…”按钮会显示以下错误信息：`This page is not normally shown and could be a sign of a phishing attempt.`

## 3.25.1 中的错误修复

- 修复了部分字符串错误提示内置 BPA 规则仅适用于企业版的问题。

## 3.25.2 中的错误修复

- 从 .exe 安装程序中移除了对 PowerShell 的依赖（该依赖在某些情况下会导致 Tabular Editor 3 无法安装）

## 3.25.3 中的错误修复

- 修复了一个错误：DAX 查询中的空 EVALUATE 语句会导致未处理的异常。
- 修复了一个错误：在 DAX 表达式编辑器中，将鼠标悬停在各种对象引用上时会出现未处理的异常。

## 从 Tabular Editor 2.x 迁移过来？

观看[此视频](https://youtu.be/O4ATwdzCvWc)，快速了解 Tabular Editor 3 的主要功能。 此外，也别忘了查看我们的[入门指南](https://docs.tabulareditor.com/onboarding/index.html)。

**Tabular Editor 3 主要功能概览：**

- 完全可自定义的 IDE，支持多显示器、Hi-DPI 和主题
- 全新强大的 DAX 代码编辑器，具备自动补全、语法检查、代码折叠等丰富功能
- \*工作区模式，可将更改保存到磁盘，并同时将模型元数据同步到 Analysis Services
- \*支持无限滚动预览表数据，创建 PivotGrids，或编写 DAX 查询来浏览模型或验证计算逻辑
- \*设置定时数据刷新
- 同时更新 Provider 数据源和 Structured数据源上的表架构（没错，连 M 查询也支持！）
- 创建 Data model 图
- 创建 DAX 脚本，让你在一个文档中编辑多个度量值或其他计算对象
- 录制 C# Script 并保存为宏（以前称为“自定义操作”）
- 集成 VertiPaq分析器
- DAX调试器
- 集成 DAX优化器
- 通过代码操作轻松重构 DAX。

\*=仅在连接到 Analysis Services 或 Power BI 实例时可用
