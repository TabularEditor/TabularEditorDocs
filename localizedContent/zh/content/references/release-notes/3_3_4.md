## 重要！

**注意**：Tabular Editor 3.3.0（及更高版本）使用 .NET 6。 在启动 Tabular Editor 3.3.0（或更高版本）之前，你可能需要先安装 [.NET 6.0 桌面运行时](https://dotnet.microsoft.com/en-us/download/dotnet/6.0/runtime)。 从 3.3.1 开始，如果缺少这些先决条件，Tabular Editor 3 安装程序会自动下载并安装。

## 3.3.4 中的改进

- 将 TOM 更新到 [19.46.0](https://www.nuget.org/packages/Microsoft.AnalysisServices.NetCore.retail.amd64) 版。
- 将 Microsoft.Data.SqlClient 更新到 [5.0.0](https://www.nuget.org/packages/Microsoft.Data.SqlClient) 版。
- 将 SciLexer.dll 更新到 4.4.6 版，不受此 [CVE](https://cve.mitre.org/cgi-bin/cvename.cgi?name=2019-16294) 影响。
- 多项小修复与改进。
- “新建/编辑关系”对话框改进：列会按字母顺序排序，并按匹配类型排序。 我们现在也会根据“from-column”的名称自动建议“to-column”。
- 将新建 Power BI Dataset 的默认 CL 更改为 1571，并新增对 [ObjectTranslation.Altered](https://docs.microsoft.com/en-us/dotnet/api/microsoft.analysisservices.tabular.objecttranslation.altered?view=analysisservices-dotnet) 属性的支持。在基于 AS Dataset 创建 DirectQuery 时，应将该属性设置为 "true"，以防元数据翻译被源端覆盖。
- 新增快捷键 **Ctrl+Alt+S**，用于保存当前模型。 参见 [#595](https://github.com/TabularEditor/TabularEditor3/issues/595)、[#561](https://github.com/TabularEditor/TabularEditor3/issues/561)。
- 新增快捷键 **Ctrl+Shift+F6**，用于“Debug Comma”DAX 格式化。 参见 [#595](https://github.com/TabularEditor/TabularEditor3/issues/595)。
- 离线架构检测现已可正常工作，即使将 Server Name、Database Name 等参数存储在共享的 M 表达式中也是如此。

## 3.3.4 中的 Bug 修复

- 修复“更新表架构”选项会将所有列都 Report 为已删除的问题，参见 [#612](https://github.com/TabularEditor/TabularEditor3/issues/612)。
- 修复在使用嵌套筛选表达式时出现的误报语义分析器错误。
- 修复在对通过关系而非公共列联接的表使用 `NATURALINNERJOIN` / `NATURALLEFTOUTERJOIN` 时出现的语义分析器误报错误。
- 修复尝试显示 ODBC 连接器的连接对话框时的问题，参见 [#448](https://github.com/TabularEditor/TabularEditor3/issues/448)
- Microsoft.Data.SqlClient 的 AAD 身份验证现在会显示模态弹窗(而不是打开操作系统浏览器)
- 修复 SQL Server 连接对话框中的“高级按钮”有时会导致崩溃的问题。
- 表导入向导现在允许在 Azure Synapse 上预览数据(数据预览查询中不再添加 NOLOCK 提示)
- 32 位安装程序现在应能正确检测 32 位 .NET 桌面运行时
- DAX 自动完成现在也可以建议 Variant 类型的标量变量，参见 [#615](https://github.com/TabularEditor/TabularEditor3/issues/615)
- 修复 DAX调试器无法显示包含嵌套条件分支的本地/观察表达式的问题。
- 将 SciLexer.dll 放在应用程序文件夹中，而不是将其解压/加载到 %LocalAppData%\Temp 中。 这应能避免在组策略阻止从该位置加载 DLL 时偶尔出现的“Exception has been thrown by the target of an invocation” 错误。
- 修复属性网格在多选时无法设置嵌套属性的问题。 参见 [#614](https://github.com/TabularEditor/TabularEditor3/issues/614)。
- 修复属性网格未刷新以显示动态添加/删除属性的问题，例如切换“增量刷新策略”属性时。
- 修复一个问题：AAS Developer Tier 实例会被视为 Standard Tier 实例，导致无法连接到 TE3 商业版
- 修复 NAMEOF 自动完成在某些情况下无法显示所有列/度量值引用的问题，参见 [#620](https://github.com/TabularEditor/TabularEditor3/issues/620)。
- 修复某些 DAX 关键字（START、ORDER、RETURN 等） 在未加引号用作表引用时未被视为错误的问题。 该问题也会导致包含对这类表的带引号引用的代码无法正确格式化。
- 修复“4096 (0x1000) is an invalid culture identifier”这一“无效的区域设置标识符”错误，参见 [#576](https://github.com/TabularEditor/TabularEditor3/issues/576)。
- 修复查找/替换搜索时可能导致崩溃的问题，参见 [#611](https://github.com/TabularEditor/TabularEditor3/issues/611)。
- 在只读模式下显示所有属性，参见 [#596](https://github.com/TabularEditor/TabularEditor3/issues/596)。
- 已移除窗口固定/取消固定时的动画效果，详见 [#595](https://github.com/TabularEditor/TabularEditor3/issues/595)。
- 在 DAX 注释中不再自动补全括号，详见 [#597](https://github.com/TabularEditor/TabularEditor3/issues/597)。
- 修复在“首选项”对话框中将默认 CL 值设置为非标准值时导致崩溃的问题。

## 从 Tabular Editor 2.x 升级而来？

观看 [此视频](https://www.youtube.com/watch?v=pt3DdcjfImY)，快速了解 Tabular Editor 3 的新功能。 另外，也别忘了查看我们的 [入门指南](https://docs.tabulareditor.com/onboarding/index.html)。

**Tabular Editor 3 主要功能概览：**

- 完全可自定义的 IDE，支持多显示器、高 DPI 和主题
- 全新强大的 DAX 代码编辑器，具备自动完成、语法检查、代码折叠等更多功能
- \*工作区模式，可将更改保存到磁盘，并同时将模型元数据同步到 Analysis Services
- \*支持无限滚动预览表数据，创建 PivotGrid，或编写 DAX 查询来浏览模型或测试计算逻辑
- \*支持计划数据刷新
- 在提供程序和 Structured数据源上更新表架构（是的，甚至包括 M 查询！）
- 创建 Data model 图表
- 创建 DAX 脚本，让你在一个文档中编辑多个度量值或其他计算对象
- 录制 C# Script 并保存为宏（以前称为“自定义操作”）
- 集成 VertiPaq分析器
- \*DAX调试器

\*=仅在连接到 Analysis Services 或 Power BI 实例时才可用
